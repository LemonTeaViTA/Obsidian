## 🔄 核心工作流

### 流程一：Raw 文档整理（从原始文章提取知识）

```
inbox → 分类到 staged → 逐篇提取到 wiki → processed
```

**步骤**：
1. 将新文件放入 `raws/inbox/`
2. 按内容分类移动到 `staged/{category}/`
3. 逐篇阅读 staged 文件，提取有价值的知识点
4. 写入对应的 wiki 文件（如 `wiki/RAG/检索召回与优化.md`）
5. 完成后移动到 `processed/`，在 `优化记录.md` 记录

**原则**：
- 一次只处理一个文件，人工审核
- 只提取核心知识（跳过教程、安装命令）
- 保留数据来源和项目链接

---

### 流程二：面试题目同步（核心流程）

当面试题目.md 中有题目但 wiki 里没有对应答案时：

```
发现缺漏 → 补充 wiki 知识点 → 同步面试题目.md → 记录优化
```

**三步操作**：

1. **补充 wiki**：在对应的 wiki 文件中写题解
   - 格式：`###` 主题目 + `####` 子题目
   - 位置：放到该文件中逻辑上合适的章节

2. **同步面试题目**：在 `wiki/面试题目.md` 对应模块下添加链接
   - 格式：`- [ ] 题目 ➡️ 👉 [[文件名#标题|查看核心解析]]`
   - 确保锚点与实际标题完全一致

3. **记录变更**：在 `优化记录.md` 当天日期下追加日志

---

### 流程三：完整性检查（防遗漏）

定期执行，确保知识库没有缺漏：

**检查 A：面试题目 → wiki 方向**
- 遍历 `面试题目.md` 中所有条目
- 没有 `➡️ 👉 [[...]]` 链接的 = wiki 中缺少对应答案
- 按优先级补充

**检查 B：wiki → 面试题目方向**
- 遍历所有 wiki 文件中的 `###` 题解标题
- 检查是否都在 `面试题目.md` 中有对应条目
- 缺少的补充进去

**检查 C：断链检测**
- 检查所有 `[[...]]` 链接指向的文件和锚点是否存在
- 修复断链

```bash
# 快速查找没有链接的面试题目（即 wiki 中缺答案的）
grep -n '^\- \[ \]' wiki/面试题目.md | grep -v '➡️'
```

## ⚠️ 防误删机制

之前出现过清理重复内容时误删正常段落的问题。操作规范：

1. **编辑前先备份**：对大段删除操作，先确认删除范围的上下文
2. **编辑后立即验证**：
   - 检查文件结构（`grep -n "^## \|^### "` 确认标题层级完整）
   - 检查总行数是否合理
   - 检查是否有重复段落
3. **不要一次性删除超过 20 行**：分步操作，每步验证
4. **Python 脚本操作文件时**：操作完打印 diff 或关键行确认

```bash
# 编辑后快速验证
grep -n "^## \|^### " wiki/面试题目.md  # 检查结构
grep -c "^\- \[ \]" wiki/面试题目.md     # 检查题目总数
```
