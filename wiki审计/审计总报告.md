# wiki 知识库审计总报告

> 本报告由首席审计官聚合 2 份全局机械检查 + 21 份领域专家报告产出，覆盖 `wiki/` 下 130 个 `.md` 文件。只读审计，未修改任何 wiki 原文件。

## 一、全局结论

整库健康度 **🔴 红**。**内容质量普遍优秀**——领域知识准确、对齐 2026 现状、面试深度足够，LLM / JVM / RAG-评估 / 并发 / 文档解析 / 数据格式等单元达到"可直接面试自测"水准；但**工程治理层全面欠账**。头号根因是一次内容大重构（旧 `Agent核心概念` / `Prompt与Harness` / `AI编程工具` / `Agent 工程实践` 被拆分迁移）后**反向引用未同步**，导致链接大面积腐化。最该先修的是两件事：**① `面试题目.md` 学习中枢的 76 条死链**（违反"面试题目 100% 同步"硬规则，整个 LLM/Agent 板块"查看核心解析"点不动）；**② 数据完整性硬伤**（9 文件 UTF-8 乱码、2 处算法代码笔误无法运行、19 处 Java 代码中文弯引号无法编译、Baize LLM 选型自相矛盾）。其次才是结构红旗（7 个文件 >800 行）与 SSoT 去重。链接修复 ROI 最高：多数死锚是被复制的同一坏锚，定位一次即可批量修。

- **单元统计**：21 个领域单元 + 2 个全局机械检查。领域单元健康度 **🔴 12 / 🟡 8 / 🟢 1**。
- **红旗合计**：机械层 M1 16（7 超长 + 9 乱码）+ M2 98（41 死文件链 + 57 面试题死锚）；领域红旗集中在超长、SSoT、Diátaxis 混类、代码不可运行。
- **黄旗合计**：机械层 M1 159（含 71 缺语言标签 + 41 缺顶部摘要 + 24 H2 超限 + 14 超长 600–800 等）+ M2 123（31 slug 锚 + 90 漂移锚 + 孤儿 + MOC 遗漏）；领域黄旗合计约 100 条。
