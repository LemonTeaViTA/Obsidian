### P2 / 建议优化

**[P2-1] static 成员变量存储位置表述在 JDK 7+ 已过时**
- 文件：`面向对象.md`，L369 注释「静态成员变量：存在方法区（元空间）」、L385 callout「存储在方法区（JDK 8+ 的元空间）」
- 问题：HotSpot 自 JDK 7 起，static 字段实际随 Class 对象（类的镜像 mirror）存放在**堆**中，而非方法区/元空间。元空间存放的是类元数据，static 变量值不在其中。这是常见但技术上已过时的"教科书答案"。
- 修复建议：补一句说明「JDK 7+ static 字段实际存于堆中的 Class 对象，元空间仅存类元数据」，避免被资深面试官追问翻车。属求职深度优化点。

**[P2-2] Java 概述.md 关于 Java 11 LTS 的措辞**
- 文件：`Java 概述.md`，L99「第一个非 Oracle 免费商用的 LTS」
- 问题：表述略含糊。准确说法是 Java 11 起 Oracle JDK 商用收费、而 OpenJDK 构建（含各厂商发行版）可免费商用。
- 修复建议：改为「Oracle JDK 商用收费的起点，同时 OpenJDK 发行版可免费商用」之类更精确表述。低优先级。
