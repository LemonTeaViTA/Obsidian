## ③ 领域知识点准确性评价

**总体：准确度高、紧跟 2026 现状，面试深度足够。** 亮点与需注意点如下。

**准确且有深度的部分：**
- **向量检索算法**：KNN O(N×D) 推导、ANN 三大思想（减候选集/压缩向量/分层精排）、HNSW/IVF/PQ/RaBitQ 原理与压缩比均正确。PQ ~384x（768 维 FP32=3072B → 8 子段×1B=8B）、RaBitQ 32x 计算无误。
- **元数据过滤×向量索引**：Pre-/Post-/Hybrid filter 三策略 + 各家默认策略表，是很多资料缺失的工程深度，准确且实用。HNSW 因 `not in` 列表过长降级暴力扫描的机理与去重小节呼应得当。
- **Embedding 三代→四代演进**：第四代 LLM-backbone 主导 MTEB、与 BERT 系成本/维度权衡，对齐 2024-2026 现状。
- **分块四方法学（L1-L4）+ 正交的内容增强/在线扩展**：框架清晰，是本单元最大亮点；前沿四法（Contextual Retrieval / Late Chunking / 判别式分块 / 命题化切割）论文出处齐全，Anthropic 实测数据（35%/49%/67%）与公开口径一致；ColBERT（arXiv:2004.12832）、Dense X Retrieval（arXiv:2312.06648）引用正确。
- **检索策略**：BM25 公式、RRF（smooth_k=60）、Bi/Cross-encoder、HyDE/Multi-Query/Step-back 均正确；查询理解以 Function Calling/Agentic RAG 取代传统四分类，判断与 2026 生产实践一致，**贴合 Java+Agent 求职方向**。
- **Code RAG**：tree-sitter AST 切分、多粒度索引、调用图（LSP 主+静态分析兜底）、增量更新级联、Claude Code 无索引模式——内容贴合 Agent/Coding Agent 方向，质量高。

**需注意/核实：**
- LLM→Embedding"去 causal mask"作为通用步骤过度概括（见 P2）。
- KaLM-Embedding 12B 登顶 MTEB 参数量存疑（见 P2）。
- Code RAG.md 模型表中 `Qwen2.5-Coder-Embedding`、`CodeGemma-Embedding` 名称建议核实是否存在该专用 embedding 变体（Voyage-code-3、Jina Code Embeddings、CodeBERT/GraphCodeBERT 确实存在）。
- KNN 单次查询毫秒数（1万≈5ms…10亿≈500s）为单线程无 SIMD 的悲观估算，作为数量级示意可接受，但实际生产带 SIMD 会快很多，建议注明"未优化单线程估算"。
- VimRAG（arXiv:2602.12735）、判别式分块（arXiv:2602.23370）为 2026-02 极新论文，无法独立核验，记为待核实而非错误。
