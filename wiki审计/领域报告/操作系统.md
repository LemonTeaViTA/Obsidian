## 三、领域知识点准确性评价

### ✅ 准确的部分（与 2026 年现状对齐）
1. **多级页表与五级页表**：「Linux 5.x 甚至引入了五级页表」准确——5-level paging 自 Linux 4.14（2017）合入，Intel Ice Lake 起硬件支持，2026 年是 mainstream 描述。
2. **TLB 与进程切换**：「进程切换刷 TLB / 线程不刷」正确，且点出了 PCID（隐含）的优化背景虽未展开但不算缺漏。
3. **Linux CFS**：用红黑树 + vruntime 描述准确。**轻微缺漏**：可补一句「Linux 6.6（2023.10）已默认启用 EEVDF（Earliest Eligible Virtual Deadline First）调度器取代 CFS」——2026 年 Java + Agent 面试官若深入到内核版本可能追问。建议加一句 callout 注。
4. **死锁 4 条件**：准确，命名规范。
5. **inode/数据块/硬链接软链接**：准确。
6. **select/poll/epoll 对比表**：技术细节正确，红黑树 + 就绪链表 + 回调机制描述到位。

### ⚠️ 可优化点
1. **进程切换开销描述**：行 110 「切换页表、刷新 TLB」——在现代 x86_64 启用 PCID 后并非每次都全刷 TLB，可加一句「现代 CPU 借助 PCID/ASID 优化跨进程 TLB 复用」以体现深度。
2. **协程对比**：行 124-132 「栈大小默认 1-8MB」——Linux 上 pthread 默认栈是 8MB（Java HotSpot 默认 1MB），表述模糊；Go goroutine 初始栈 2KB（不是 2-8KB），可改成「2KB 起，按需自动扩展」更精确。
3. **缺页中断描述**：行 84-94 缺少对 **major page fault vs minor page fault** 的区分（minor 仅需建立映射不读盘），这是高频追问点。
4. **`<` epoll ET 模式**：行 230-232 ET 段未提「fd 必须设置 O_NONBLOCK」——这是面试紧追问的标准考点，缺失会失分。
5. **未覆盖的高频 OS 面试题**：
   - **零拷贝（mmap / sendfile / splice）**——Java + Agent 岗在 Netty/Kafka 场景必问，目前 wiki 全局零拷贝散落在 Redis/集合框架未集中。
   - **fork & copy-on-write**——Redis BGSAVE/AOF rewrite 必问的底层。
   - **用户态/内核态切换代价**——Java 锁 vs synchronized 优化、协程价值的根因。
   - **信号（SIGTERM/SIGKILL）与僵尸进程**——Linux 运维基础。
   - **大页/Huge Page**——JVM 调优场景常考。
6. **代码示例**：本文档无代码块，故无代码正确性问题。但 epoll 章节若未来加示例，必须配 C 代码（含 EAGAIN 处理）。

### 求职方向对齐评价
✅ 内容紧扣 Java 后端 + Agent 方向的 OS 基础（虚拟内存/线程/调度/IO 多路复用），未塞入纯系统编程实战（如内核驱动）或运维教程，对齐良好。但**缺零拷贝/COW/用户态切换代价**这 3 题，是 Java + Agent 岗 OS 题的核心短板，强烈建议补齐。
