# 生产环境检索延迟优化策略

> RAG 检索链路在生产环境的延迟优化手段——并行检索、语义缓存、两阶段检索（粗召回 + Reranker 精排）、动态 Top-K，配合超时降级 / 模型降级兜底。

## 延迟优化六策略

1. ==并行检索==：多知识源同时查询
2. ==语义缓存==：语义相似的查询共享结果，命中率 >30%
3. ==两阶段检索==：低成本模型粗召回 Top-100，高精度 Reranker 精排 Top-10
4. ==超时降级==：超时后返回无检索的 LLM 直接回答
5. ==动态 Top-K==：简单查询检索少，复杂查询检索多
6. ==模型降级==：高峰时使用更便宜的模型

其中并行检索 / 语义缓存 / 两阶段检索 / 动态 Top-K 是检索链路本身的优化；超时降级 / 模型降级是与「可观测 / 成本控制」交叉的兜底手段（Agent 侧的成本控制见 [[Agent 可观测性#五、Agent 成本控制]]）。
