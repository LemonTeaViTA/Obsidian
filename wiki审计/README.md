# wiki 审计

本目录是对 `wiki/` 知识库做**方法论合规性 + 内容质量**审计的工具集与产物归档。**只读审计，只出报告，不修改任何 wiki 原文件。**

## 审计依据

规则来自 `wiki-管理方法论.md` §十「机械检查清单」与 `QUALITY_CHECKLIST.md`，分两类：

- **客观可校验**：UTF-8 乱码、frontmatter 四字段、行数阈值（>600🟡/>800🔴）、代码块语言标签、callout 类型/嵌套/连用、标题混 `[[]]`、wikilink 坏链、anchor 精确匹配、面试题双向同步、孤儿页、MOC 覆盖。
- **需专家判断**：Diátaxis 类型混杂、SSoT 重复、内容越界/求职方向对齐（Java + Agent）、标题层级逻辑、>400 行渐进式摘要、知识点准确性/时效性/面试深度。

## 工具集

### 快速检查脚本 `check.sh`

**用途**: 日常机械检查,无需启动完整审计工作流。可用于 git pre-commit hook 或 CI/CD。

```bash
# 全量检查
./wiki审计/check.sh --all

# 分项检查
./wiki审计/check.sh --encoding   # UTF-8 乱码
./wiki审计/check.sh --links      # wikilink 坏链(抽样)
./wiki审计/check.sh --structure  # 行数/H1/层级
./wiki审计/check.sh --callouts   # callout 规范
```

**输出**: 人类可读的彩色文本,列出发现的问题。

### 审计工作流 `_audit_workflow.js`

**用途**: 全量审计(3-6 个月执行一次),生成完整的领域报告。

**产出**: 21 个领域报告 + 2 个机械检查报告 + 1 个总报告,写入 `历史审计/YYYY-MM/`。

### 单元配置 `units.json`

**用途**: 定义 21 个审计单元(领域)的文件清单、报告路径、P0 提示。

**复用**: 下次审计前根据 wiki 结构变化(新增域、文件拆分)更新此 JSON,工作流自动读取。

**增量审计**: 临时注释掉不审的单元,只审部分域。

## 多 agent 分工

- **全局机械检查**（2 agent）：`机械检查报告.md`（格式类）、`链接同步检查报告.md`（链接/同步类）。
- **领域专家审计**（21 agent,配置来自 `units.json`）：每个戴「知识库管理专家 + 领域知识点专家」两顶帽子。
- **汇总**（1 agent）：读全部报告产出 `审计总报告.md`（红黄绿记分卡 + P0/P1/P2 清单 + 修复顺序）。

## 报告怎么读

1. 先看 `_当前/审计总报告.md` 拿全局结论和优先级。
2. 看 `_当前/进度.md` 确认各单元是否跑完。
3. 钻取具体问题去 `_当前/领域报告/` 和机械检查报告。

## 下次审计前的准备

1. **更新 `units.json`** — 如果 wiki 结构有变(新域、文件拆分、改名),同步到配置
2. **跑一次 `check.sh --all`** — 快速自查当前健康度
3. **检查方法论** — `wiki-管理方法论.md` §十 机械检查清单是否有新规则
4. **启动审计工作流** — 新产物写入 `历史审计/YYYY-MM/`
5. **对比历史** — 用 `diff` 对比两次 `审计总报告.md`,追踪趋势
