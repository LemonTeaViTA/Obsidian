# Hermes Agent 框架

> Hermes Agent 是 Nous Research 开源的自进化 AI Agent 框架，核心理念：**用得越多越聪明**。不需要手写 Rules 规定工具调用顺序，框架会从实际任务中自动提炼经验。

---

## 一、自进化机制

Hermes 最核心的特点是内置学习循环——Agent 完成任务后会自动把解决过程沉淀成可复用的 Skill。

**触发条件**：

任务完成后，Hermes 会评估这次执行是否值得沉淀：
1. 任务复杂度超过阈值（步骤数 > N，或涉及多个工具组合）
2. 执行路径是新的（与已有 Skill 的相似度低于阈值）
3. 任务成功完成（失败的路径不沉淀）

**Skill 的格式**：

自动生成的 Skill 是 Markdown 文件，包含：
- 触发场景描述（什么情况下用这个 Skill）
- 执行步骤（从实际 Thought/Action 序列中提炼）
- 关键工具和参数模板
- 注意事项（从失败重试中总结的坑）

**"发现更好路径"的判断机制**：

Hermes 会持续追踪同类任务的执行效率（步骤数、耗时、成功率）。当新的执行路径比已有 Skill 的路径效率提升超过阈值时，自动更新 Skill。这个机制让 Skill 不是一次性写死的，而是随着使用不断优化。

**和手动管理 Skills 的区别**：

OpenClaw 需要开发者手动编写和维护 Skills，Hermes 的 Skills 是从真实任务中自动提炼的。好处是 Skill 更贴近实际使用场景，坏处是自动生成的 Skill 质量不如精心手写的稳定，需要定期人工审核。

## 二、三层记忆系统

Hermes 的记忆系统分三层，每层解决不同时间跨度的记忆问题：

| 层次 | 存储内容 | 技术实现 | 生命周期 |
|------|---------|---------|---------|
| 对话历史层 | 当前会话的完整 Thought/Action/Observation 序列 | SQLite（`conversations` 表） | 当前会话，可持久化到磁盘 |
| 检索层 | 历史对话的可检索索引 | FTS5 全文检索 | 跨会话，永久保存 |
| 摘要层 | 长对话的压缩摘要 | 大模型自动生成 Markdown | 跨会话，永久保存 |

**三层如何协作**：

1. 每轮对话实时写入 SQLite 对话历史层
2. 会话结束或接近 context 上限时，触发摘要层：大模型读取本次会话，提炼关键信息（用户偏好、重要决策、任务结论）写入 Markdown 摘要文件
3. 新会话开始时，FTS5 检索层根据当前任务语义，从历史摘要中召回相关片段注入上下文

**摘要触发时机**：
- 会话 token 数超过阈值（默认 80% context 窗口）
- 用户主动结束会话
- 任务完成后的自动归档

**与 OpenClaw Memory 系统的差异**：

OpenClaw 用 sqlite-vec 做向量检索，支持语义相似度召回；Hermes 用 FTS5 做关键词检索，配合大模型摘要做语义理解。Hermes 的方案更轻量（不需要向量模型），但语义召回精度略低于 OpenClaw。

## 三、安全机制

Hermes 的安全机制在框架层面强制执行，不依赖大模型自己判断风险。

**四层安全防护**：

**1. 用户授权**

每个工具调用都需要用户预先授权。Hermes 启动时展示工具权限清单，用户明确同意后才能使用。未授权的工具调用直接拒绝，不会传给 LLM 决策。

**2. 危险命令审批工作流**

对于不可逆操作（删除文件、执行 Shell 命令、发送消息、调用付费 API），Hermes 会：
1. 暂停执行，向用户展示即将执行的操作详情
2. 等待用户明确确认（输入 `yes` 或点击确认按钮）
3. 确认后执行，并记录审计日志
4. 超时未确认则取消操作

**3. 容器隔离**

代码执行类工具（Python 解释器、Shell 执行器）在 Docker 容器中运行：
- 文件系统隔离：只能访问挂载的工作目录，无法访问宿主机其他路径
- 网络隔离：默认禁止出站网络请求，需要网络访问的工具需要显式声明
- 资源限制：CPU、内存、执行时间均有上限，防止资源耗尽

**4. 上下文扫描**

每次工具返回结果注入上下文前，Hermes 会扫描内容，检测 Prompt 注入特征（如"忽略之前的指令"、"你现在是..."等模式）。检测到可疑内容时，对该内容打标记并告知 LLM，而不是直接过滤（避免误杀正常内容）。

---

## 四、广泛兼容性

**模型支持**：通过 LiteLLM 统一接口，支持 200+ 大模型，包括 OpenAI、Anthropic、Google、Mistral、本地 Ollama 模型等。切换模型只需改配置，不需要改代码。

**多端接入**：支持 Telegram、Discord、Slack、Web UI 等多种接入方式，同一个 Agent 可以同时在多个平台上运行。

---

## 五、和 OpenClaw 的区别与选型建议

| 维度 | OpenClaw | Hermes |
|------|---------|--------|
| 安全机制 | 依赖大模型自己判断风险 | 框架层面强制授权和审批，不依赖 LLM |
| 技能沉淀 | 手动编写和维护 Skills，质量可控 | 自动从任务中提炼 Skill，贴近实际但需人工审核 |
| 记忆检索 | SQLite + FTS5 + sqlite-vec 向量检索 | SQLite + FTS5 + 大模型摘要，无向量检索 |
| 模型支持 | 需要手动配置 LLM 接口 | 200+ 模型开箱即用（LiteLLM） |
| 多端接入 | 主要面向飞书/企微等企业 IM | Telegram/Discord/Slack 等消费级平台 |
| 适用场景 | 企业内部工具、需要精确控制行为的场景 | 个人助手、需要自适应学习的场景 |

**选型建议**：
- 企业内部部署、需要严格权限控制、Skills 需要精确定义 → OpenClaw
- 个人使用、希望 Agent 自动学习优化、需要接入多种消费级平台 → Hermes
- 两者都支持 MCP 协议，工具层可以复用
