## 1. Java 概述（6 题）

### 题 1：Java 语言有哪些主要特点？

**答案：**

1. **面向对象**：封装、继承、多态
2. **跨平台性**：一次编译，处处运行（JVM）
3. **自动内存管理**：GC 自动回收垃圾对象
4. **多线程支持**：内置多线程机制
5. **安全性**：字节码验证、沙箱机制
6. **健壮性**：强类型、异常处理、边界检查
7. **即时编译**：JIT 编译器
8. **丰富的类库**：Java API

**记忆口诀：** 面跨自多安健即类

---

### 题 2：JVM、JRE、JDK 三者有什么区别？

**答案：**

**包含关系：** JDK > JRE > JVM

| 名称 | 全称 | 作用 | 包含内容 |
|------|------|------|---------|
| JVM | Java Virtual Machine | 运行字节码 | 类加载器、运行时数据区、执行引擎 |
| JRE | Java Runtime Environment | 运行 Java 程序 | JVM + 核心类库 |
| JDK | Java Development Kit | 开发 Java 程序 | JRE + 开发工具（javac、java 等） |

---

### 题 3：Java 跨平台原理是什么？

**答案：**

.java 源文件 → javac 编译 → .class 字节码（平台无关）→ JVM 解释/编译 → 机器码（平台相关）

**关键：** 字节码平台无关，JVM 平台相关。每个平台有自己的 JVM 实现，将相同的字节码转换为对应的机器码。

---

### 题 4：Java 的发展历史？主要版本有哪些？

**答案：**

**重要版本：**
- Java 5（2004）：泛型、注解
- Java 7（2011）：try-with-resources
- **Java 8（2014）**：Lambda、Stream API（最经典）
- Java 11（2018）：LTS
- **Java 17（2021）**：LTS（当前主流）
- **Java 21（2023）**：LTS（虚拟线程）

**LTS 版本：** 8、11、17、21

---

### 题 5：Java 的主要应用领域有哪些？

**答案：**

1. **企业级后端开发**：Spring Boot、微服务
2. **大数据处理**：Hadoop、Spark、Flink
3. **移动开发**：Android
4. **金融系统**：银行、证券、支付
5. **云计算**：Kubernetes、Dubbo
6. **嵌入式系统**：IoT 设备
7. **游戏开发**：服务器端、Minecraft

---

### 题 6：如何配置 Java 开发环境？

**答案：**

1. 下载并安装 JDK（推荐 Java 17 或 21）
2. 配置环境变量：
   - JAVA_HOME = JDK 安装路径
   - PATH = %JAVA_HOME%\bin
3. 验证安装：`java -version`、`javac -version`
4. 安装 IDE（推荐 IntelliJ IDEA）
5. 配置 Maven（可选）

## 2. 基础语法（10 题）

### 题 7：Java 有哪些基本数据类型？各占用多少字节？

**答案：**

| 类型 | 关键字 | 字节 | 取值范围 |
|------|--------|------|---------|
| 字节型 | byte | 1 | -128 ~ 127 |
| 短整型 | short | 2 | -32768 ~ 32767 |
| 整型 | int | 4 | -2³¹ ~ 2³¹-1 |
| 长整型 | long | 8 | -2⁶³ ~ 2⁶³-1 |
| 单精度 | float | 4 | ±3.4×10³⁸ |
| 双精度 | double | 8 | ±1.7×10³⁰⁸ |
| 字符型 | char | 2 | \u0000 ~ \uffff |
| 布尔型 | boolean | 1/4* | true/false |

*boolean：数组中 1 字节，单独使用时未定义

---

### 题 8：& 和 && 的区别？

**答案：**

| 运算符 | 名称 | 特点 | 使用场景 |
|--------|------|------|---------|
| & | 逻辑与/按位与 | 两边都执行 | 位运算 |
| && | 短路与 | 左边 false 时右边不执行 | 常规判断（推荐） |

**示例：**
```java
// 短路保护
if (str != null && str.length() > 0) {
    // 安全：如果 str 为 null，不会执行 length()
}
```

---

### 题 9：final、finally、finalize 的区别？

**答案：**

| 关键字 | 作用 | 使用场景 |
|--------|------|---------|
| final | 修饰符，表示不可变 | 修饰类（不可继承）、方法（不可重写）、变量（不可修改） |
| finally | 异常处理的一部分 | try-catch-finally，最终会执行的代码块 |
| finalize() | Object 类的方法 | GC 回收前调用（已废弃） |

---

### 题 10：== 和 equals 的区别？

**答案：**

| 对比项 | == | equals() |
|--------|----|---------|
| 本质 | 运算符 | Object 类的方法 |
| 基本数据类型 | 比较值 | 不能使用 |
| 引用数据类型 | 比较地址 | 默认比较地址，可重写比较值 |

**示例：**
```java
String s1 = new String("abc");
String s2 = new String("abc");
s1 == s2;      // false（地址不同）
s1.equals(s2); // true（值相同）
```

---

### 题 11：switch 语句能否用在 byte/long/String 类型上？

**答案：**

- Java 5 之前：`switch` 只支持 `byte/short/char/int`。
- Java 5 起：支持 `enum`。
- Java 7 起：支持 `String`。
- `long` 在所有版本中都不支持用于 `switch` 表达式。

---

### 题 12：break、continue、return 的区别及作用？

**答案：**

- `break`：跳出当前整个循环或 `switch`。
- `continue`：结束本次循环，进入下一次循环。
- `return`：结束当前方法并返回。

---

### 题 13：用效率最高的方法计算 2 乘以 8？

**答案：**

使用左移运算：`2 << 3`。

左移 3 位等价于乘以 $2^3$，结果为 16。

---

### 题 14：说说自增自减运算？

**答案：**

- 前缀：`++a` / `--a`，先自增(减)再参与表达式。
- 后缀：`a++` / `a--`，先参与表达式再自增(减)。

示例：
```java
int a = 1;
int b = ++a; // a=2, b=2
int c = a++; // a=3, c=2
```

经典坑：
```java
int i = 1;
i = i++;
System.out.println(i); // 1
```
原因是后缀自增会先返回旧值参与赋值，最终把旧值写回。

---

### 题 15：float 是怎么表示小数的？（补充）

**答案：**

`float` 按 IEEE 754 单精度浮点数表示，32 位由三部分组成：

- 符号位（Sign）
- 指数位（Exponent）
- 尾数位（Mantissa）

通用形式是：
$$
V = (-1)^S \times M \times 2^E
$$

因为二进制浮点是近似表示，小数运算会有精度误差。

---

### 题 16：讲一下数据准确性高是怎么保证的？（补充）

**答案：**

金融计算不要用 `float/double`，常见两种方案：

1. 使用 `BigDecimal`（推荐）。
2. 把金额转成最小货币单位（如分）用整数计算。

示例：
```java
BigDecimal a = new BigDecimal("0.1");
BigDecimal b = new BigDecimal("0.2");
System.out.println(a.add(b)); // 0.3
```

## 3. 面向对象（16 题）

### 题 17：面向对象和面向过程的区别？

**答案：**

- 面向过程：以步骤/函数为中心，关注“怎么做”。
- 面向对象：以对象/类为中心，关注“谁来做”。

面向对象更利于封装、复用和扩展。

---

### 题 18：面向对象编程有哪些特性？

**答案：**

三大特性：封装、继承、多态。

- 封装：隐藏内部实现，仅暴露必要接口。
- 继承：子类复用并扩展父类能力。
- 多态：同一调用在不同对象上表现出不同行为。

---

### 题 19：多态解决了什么问题？（补充）

**答案：**

多态让“同一接口，不同实现”成为可能，提升扩展性与可替换性。

- 父类引用可指向不同子类对象。
- 方法具体执行在运行时动态绑定。

这让业务代码依赖抽象而不是具体实现，减少分支判断和耦合。

---

### 题 20：重载和重写的区别？

**答案：**

- 重载（Overload）：同一个类中，方法名相同、参数列表不同。
- 重写（Override）：子类对父类同签名方法提供新实现。

补充要点：

- 重载看“参数”；重写看“继承关系 + 方法签名一致”。
- 重写要遵循里氏替换原则，子类应可替代父类。

---

### 题 21：访问修饰符 public、private、protected、默认 的区别？

**答案：**

Java 的 4 种访问级别：

- `public`：任意位置可见。
- `protected`：同包可见，不同包仅子类可见。
- 默认（不写）：仅同包可见。
- `private`：仅当前类内部可见。

补充：`private/protected` 不能修饰外部顶级类。

---

### 题 22：this 关键字有什么作用？

**答案：**

`this` 代表当前对象引用，常见用途：

1. 访问当前对象成员：`this.field`。
2. 解决形参与成员变量同名冲突。
3. 调用当前类的其他构造方法：`this(...)`。

---

### 题 23：抽象类和接口有什么区别？

**答案：**

- 抽象类：描述“is-a”，可有状态、构造器、抽象/非抽象方法。
- 接口：描述能力约束，更偏“can-do / has-a”，面向规范。
- 一个类只能继承一个抽象类，但可实现多个接口。

常见补充：

- 抽象类可以定义构造方法。
- 接口不能定义构造方法。
- Java 类不支持多继承；接口支持多继承。

---

### 题 24：成员变量与局部变量的区别有哪些？

**答案：**

- 定义位置：成员变量在类中，局部变量在方法/代码块中。
- 生命周期：成员变量随对象存在，局部变量随方法调用结束而销毁。
- 默认值：成员变量有默认值，局部变量必须先赋值再使用。
- 修饰符：成员变量可用访问控制符和 `static`，局部变量不行。

---

### 题 25：static 关键字了解吗？

**答案：**

`static` 可修饰变量、方法、代码块、内部类：

- 静态变量：类级共享，一份数据。
- 静态方法：类级调用，不能直接访问实例成员。
- 静态代码块：类加载时执行一次。
- 静态内部类：独立于外部类实例。

---

### 题 26：final 关键字有什么作用？

**答案：**

- 修饰类：类不能被继承。
- 修饰方法：方法不能被重写。
- 修饰变量：变量只能赋值一次。

注意：`final` 引用不能改指向，但引用对象内部状态可变。

---

### 题 27：final、finally、finalize 的区别？

**答案：**

- `final`：语言修饰符（类/方法/变量约束）。
- `finally`：异常处理结构，通常用于资源释放。
- `finalize`：`Object` 的方法，GC 前可能调用，现代开发不推荐依赖。

---

### 题 28：== 和 equals 的区别？

**答案：**

- `==`：
    - 基本类型比较值。
    - 引用类型比较是否同一对象地址。
- `equals`：比较内容相等性（默认与 `==` 一样，很多类如 `String` 已重写）。

---

### 题 29：为什么重写 equals 时必须重写 hashCode？

**答案：**

哈希容器（如 `HashMap/HashSet`）先按 `hashCode` 定位桶，再用 `equals` 判等。

如果 `equals` 相等但 `hashCode` 不同，会导致：

- 逻辑相等对象分散到不同桶。
- 查询、去重、覆盖行为异常。

契约要求：`equals` 相等的两个对象，`hashCode` 必须相等。

---

### 题 30：Java 是值传递还是引用传递？

**答案：**

Java 只有值传递。

- 基本类型：传递值副本。
- 引用类型：传递“引用值（地址）的副本”。

所以方法内可以修改对象内容，但不能改变调用方引用变量本身的指向。

---

### 题 31：说说深拷贝和浅拷贝的区别？

**答案：**

- 浅拷贝：复制对象本身，引用类型字段只复制地址，可能共享子对象。
- 深拷贝：递归复制引用对象，新旧对象完全独立。

常见实现：

- 浅拷贝：`clone()`（默认语义）。
- 深拷贝：手动拷贝/序列化反序列化。

---

### 题 32：Java 创建对象有哪几种方式？

**答案：**

常见 4 种：

1. `new` 关键字（最常用）。
2. 反射创建。
3. `clone()` 创建。
4. 反序列化创建。

补充：创建子类对象时执行顺序通常是父类静态块 -> 子类静态块 -> 父类构造 -> 子类构造。

---

## 4. String（8 题）

### 题 33：String 是基本数据类型吗？可以被继承吗？

**答案：**

- `String` 不是基本类型，是引用类型（类）。
- `String` 不能被继承，因为它被 `final` 修饰。

---

### 题 34：String、StringBuilder、StringBuffer 的区别？

**答案：**

- `String`：不可变。
- `StringBuilder`：可变，非线程安全，单线程拼接性能好。
- `StringBuffer`：可变，线程安全（方法多带同步），性能一般低于 `StringBuilder`。

使用建议：

- 普通场景用 `String`。
- 频繁拼接用 `StringBuilder`。
- 特殊并发修改场景才考虑 `StringBuffer`。

---

### 题 35：`new String("abc")` 和 `"abc"` 的区别？

**答案：**

- `"abc"`：优先从字符串常量池复用。
- `new String("abc")`：在堆上新建对象（常量池字面量可能已存在）。

因此 `==` 比较时通常不相等（比较的是引用地址）。

---

### 题 36：String 是不可变类吗？字符串拼接如何实现？

**答案：**

是不可变类。主要原因：

- 内部字符存储不可变语义。
- 类被 `final` 修饰，防止继承破坏不变性。
- 不提供直接修改内部状态的公开方法。

字符串 `+` 拼接在编译期通常会优化为 `StringBuilder.append(...)` 链式调用，再 `toString()`。

---

### 题 37：intern 方法有什么作用？

**答案：**

- 若常量池已有等值字符串，返回池中引用。
- 否则将当前字符串放入池并返回其引用。

用途是统一字符串引用、减少重复对象。

---

## 5. 包装类与 Object

### 题 38：Integer 127 与 128 的 `==` 比较结果？

**答案：**

```java
Integer a = 127, b = 127; // a == b -> true
Integer c = 128, d = 128; // c == d -> false
```

因为 `Integer` 默认缓存区间是 `-128~127`，区间内复用缓存对象，区间外新建对象。

---

### 题 39：String 怎么转 Integer？原理？

**答案：**

常见方法：

- `Integer.parseInt(String)`
- `Integer.valueOf(String)`

核心都依赖解析逻辑将字符串逐位转为数值；`valueOf` 额外返回包装类型并可能复用缓存。

---

### 题 40：Object 类的常见方法？

**答案：**

高频：

- `equals`：对象相等判断。
- `hashCode`：哈希码。
- `toString`：字符串表示。
- `clone`：对象拷贝（受限于接口与实现）。
- `getClass`：运行时类信息。
- `wait/notify/notifyAll`：线程协作。

其中 `equals/hashCode` 需要成对考虑。

## 6. 异常与 I/O

### 题 41：Java 中异常处理体系？

**答案：**

根类是 `Throwable`，分为两大分支：

- `Error`：JVM 级严重错误，通常程序难以恢复。
- `Exception`：程序可处理异常。

`Exception` 又分：

- 编译时异常（Checked）：必须捕获或声明抛出。
- 运行时异常（Runtime）：一般由代码逻辑问题导致。

---

### 题 42：异常的处理方式？

**答案：**

两种主流方式：

1. `throw/throws` 向上抛出，由上层处理。
2. `try-catch-finally` 本地捕获处理。

补充：当 `catch` 和 `finally` 都抛异常时，最终通常以 `finally` 抛出的异常为准。

---

### 题 43：三道经典异常处理代码题（结论）

**答案：**

高频结论：

1. `finally` 一定会在方法真正返回前执行。
2. `finally` 里若有 `return`，会覆盖 `try/catch` 的返回值。
3. `try` 返回基本类型变量时，返回的是当时的值快照，`finally` 再改变量不影响已确定返回值。

---

### 题 44：Java 中 IO 流分为几种？

**答案：**

可按多个维度划分：

- 方向：输入流/输出流。
- 单位：字节流/字符流。
- 功能：节点流/处理流/管道流。

常见增强点是缓冲处理，底层思想可类比装饰器模式。

---

### 题 45：既然有字节流，为什么还要字符流？

**答案：**

字符流面向文本处理更直接，能减少手工编码转换带来的复杂性与乱码风险。

本质上文件底层仍是字节；字符流是“按字符语义”在读写字节。

---

### 题 46：BIO、NIO、AIO 的区别？

**答案：**

- BIO：阻塞模型，一个连接通常对应一个处理线程。
- NIO：非阻塞 + 多路复用（Selector），适合高并发短连接。
- AIO：异步回调模型，发起后可立即返回，完成时通知。

---

### 题 47：什么是序列化和反序列化？

**答案：**

- 序列化：对象 -> 字节流（便于存储与传输）。
- 反序列化：字节流 -> 对象。

补充要点：

- `Serializable` 用于声明可序列化。
- `serialVersionUID` 用于版本兼容控制。
- `transient` 字段不会被序列化。

---

### 题 48：常见序列化方式有哪些？

**答案：**

1. Java 原生序列化（`ObjectOutputStream/ObjectInputStream`）。
2. JSON 序列化（如 Jackson）。
3. Protobuf 序列化（更紧凑、性能更好）。

---

### 题 49：了解 Socket 网络套接字吗？

**答案：**

`Socket` 是网络通信端点，常用于 TCP/UDP 连接。

- 客户端常用 `Socket`。
- 服务端常用 `ServerSocket`。

很多 RPC 框架底层都依赖类似套接字通信机制，并在其上封装协议、序列化与服务治理。

---

### 题 50：Java 泛型了解吗？

**答案：**

泛型核心价值：类型安全 + 减少强制类型转换。

常见形态：泛型类、泛型接口、泛型方法。

关键概念：类型擦除。Java 泛型主要在编译期生效，运行期类型参数信息被擦除（向下兼容历史版本）。

---

## 7. 注解、反射与 Java 8

### 题 51：说一下你对注解的理解？

**答案：**

注解本质是“元数据标记”，用于在编译期或运行期驱动行为。

常见生命周期：

- `SOURCE`：仅编译期。
- `CLASS`：保留到字节码，运行期默认不可见。
- `RUNTIME`：运行时可通过反射读取。

典型场景：AOP、依赖注入、代码生成（如 Lombok）。

---

### 题 52：什么是反射？应用？原理？

**答案：**

反射允许在运行时检查并操作类信息（字段、方法、构造器）。

典型应用：

- Spring 容器与依赖注入。
- 动态代理/AOP。
- 测试框架自动发现并执行测试方法。

原理要点：类加载后，类型元数据进入 JVM 运行时区域；反射 API 基于这些元数据执行动态访问。

### 题 54：Lambda 表达式了解多少？

**答案：**

Lambda 是函数式接口实例的简写形式，核心作用是让行为参数化，减少样板代码。

常与 Stream 一起使用，如 `filter/map/reduce`。

---

### 题 55：Optional 了解吗？

**答案：**

`Optional` 用于表达“值可能为空”，降低空指针风险。

常用方法：

- `isPresent()`
- `orElse(...)`
- `ifPresent(...)`
- `map(...)`

---

### 题 56：Stream 流用过吗？

**答案：**

`Stream` 是面向集合处理的声明式 API。

- 中间操作：`filter/sorted/map/distinct` 等。
- 终端操作：`forEach/count/reduce/collect` 等。

优势是链式表达清晰、可读性强，并可结合并行流优化部分计算场景。
