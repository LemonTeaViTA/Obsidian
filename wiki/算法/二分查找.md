## 一、二分查找核心思想与终极模板

二分查找的核心思想不是在数组中“找元素”，而是**“在一个单调（或具备某种排他性两段结构）的区间内，通过检验中间条件，每次能稳定排除一半的不可能域”**。

### 🌟 三大经典模板框架
许多人写二分永远在纠结 `left <= right` 还是 `left < right`，返回 `left` 还是 `right`。其实这从不取决于个人偏好，而是**完全被你的“区间收缩逻辑（代码里有没有 `right = mid` 或 `left = mid`）”所绑架的！**
只要死记以下三大基础和变型框架，即可通吃：

**1. 模板一：寻找精确值（果断跨越派）**
*   **适用场景**：找一个确切的数，找到就结束。
*   **循环条件**：**必须用 `while (left <= right)`**（因为你必须查验到最后一个点，搜遍 `[left, right]` 闭区间）。
*   **区间缩减**：`left = mid + 1;` 且 `right = mid - 1;`（每次判断完都果断跳过 `mid`，在一直在硬性强缩范围，即便最后重合，也必将借由这行代码交错并安全跳出循环）。
*   **退出时机**：找到直接 `return mid;`；找不到时区间为空退出，返回 `-1`。

**2. 模板二：寻找边界（果断跨越派·延迟判定）**
*   **适用场景**：找某个数最早或最晚出现的位置。
*   **循环条件**：依然是 **`while (left <= right)`**，因为依然每次都在坚定不移地跨越。
*   **区间缩减与寻找时机**：
    *   找左边界：遇到相等的 `nums[mid] == target` 时**绝不提前 return**，继续往左缩逼：`right = mid - 1;`。
    *   找右边界：遇到相等时继续往右缩逼：`left = mid + 1;`。
*   **退出时机**：循环结束后，最后验证 `left` 或 `right` 所指的位置是否等于 target。

**3. 模板三：寻找极值 / 局部有序（怂了包留派）**
*   **适用场景**：寻找旋转数组的最小值、寻峰等，当你无法确定 `mid` 究竟是不是唯一答案，而不敢直接跨越跳过它。
*   **区间缩减**：保留原位，必定出现如 `right = mid` 或 `left = mid` 这种因害怕漏掉答案而“不敢加减的保留操作”。
*   **循环条件**：**必须降阶使用 `while (left < right)`**！

> [!danger] **【灵魂死穴拷问：模板三为什么绝对不能加等号？】**
> 如果在有 `right = mid` 语句存在的情况下，你还敢头铁用 `while left <= right`，请在脑海中想一下这个地狱画卷：
> 假设范围已经极限收缩到了最后一个格子 `left=2, right=2`。
> 此时计算 `mid=2`。如果根据条件不幸跑进了 `right = mid` 这一分支，这意味着什么？
> 意味着新一轮的 `right` 居然还是 2！区间大小根本没有发生缩减！由于大前提 `2 <= 2` 依然成立，程序会再次进入循环重新算 `mid=2`，接着又一次 `right=2`……
> **恭喜你，万劫不复的无尽死循环诞生了。**
> 所以，只要出现了原位保留，你的总开关就只能降级为 `<`！这样当最后 `left` 与 `right` 相撞（相等）的那一弹指间就能惊险避免进循环。相撞所指的那个独苗 `left`，就是那个幸存你最后要查验的答案。

*(注：计算 `mid` 时统一使用 `mid = left + (right - left) // 2`，以防止大数溢出，且在 Python 中保持整除)*

## 二、基础入门：标准二分与搜边界
*利用最基础的大小关系来排除一半边界。*

### 二分查找（难度：简单）

**题目：**
给定一个 n 个元素有序的（升序）整型数组 nums 和一个目标值 target，写一个函数搜索 nums 中的 target，如果目标值存在返回下标，否则返回 -1。

**【模板解析与思路归纳】：**
- 这是最标准的**【寻找精确值模板】**。
- 只要判断 `nums[mid] == target` 就立刻返回。
- 循环条件：`while (left <= right)`。
- 转移跳过：左侧偏小就去右侧找 `left = mid + 1`，偏大就去左侧找 `right = mid - 1`。

**答案代码（Python版）：**
```python
class Solution:
    def search(self, nums: List[int], target: int) -> int:
        left, right = 0, len(nums) - 1
        
        while left <= right:
            mid = left + (right - left) // 2
            if nums[mid] == target:
                return mid
            elif nums[mid] < target:
                left = mid + 1
            else:
                right = mid - 1
                
        return -1
```

### 搜索插入位置（难度：简单）

**题目：**
给定一个排序数组和一个目标值，在数组中找到目标值，并返回其索引。如果目标值不存在于数组中，返回它将会被按顺序插入的位置。
请必须使用时间复杂度为 `O(log n)` 的算法。

**【模板解析与思路归纳】：**
- 这题本质是**寻找第一个大于等于 target 的位置（左边界思想的雏形）**。
- 用基础精确值模板，但是当循环因为找不到而结束时（`left > right` 触发跳出），`left` 指针刚好会停在这个插入的地方（恰好越过 `right` 的那个位置，也就是插入的新位置）。

**答案代码（Python版）：**
```python
class Solution:
    def searchInsert(self, nums: List[int], target: int) -> int:
        left, right = 0, len(nums) - 1
        
        while left <= right:
            mid = left + (right - left) // 2
            if nums[mid] == target:
                # 找到了位置，如果是自己就插在自己这
                return mid
            elif nums[mid] < target:
                left = mid + 1
            else:
                right = mid - 1
                
        # 跳出循环时 left 的位置正好就是第一个比 target 大的数字的位置
        return left
```

### 在排序数组中查找元素的第一个和最后一个位置（难度：中等）

给你一个按照非递减顺序排列的整数数组 `nums`，和一个目标值 `target`。请你找出给定目标值在数组中的开始位置和结束位置。
如果数组中不存在目标值 `target`，返回 `[-1, -1]`。必须设计并实现时间复杂度为 `O(log n)` 的算法。

#### 解题思路
- 这是**最正统的【寻找左右边界】双模板齐放题**。
- **关于返回 `[-1, -1]` 的兜底设计**：我们在每个子函数中初始都预设了 `res = -1`。只要二分搜寻全程都没有命中过 `target`（没有触发赋值），最终自然就会返回这个默认的 `-1`，两个函数合抱刚好就是 `[-1, -1]`，极其优雅。
- **找第一位 (左边界)**：即使 `nums[mid] == target` 也别返回！而是收缩右逼，让 `right = mid - 1`，企图继续往左寻找更早出现的。
- **找最后一位 (右边界)**：同理，即使命中，也收缩左逼，让 `left = mid + 1`，企图继续往右逼近！

#### 代码实现（Python版）
```python
class Solution:
    def searchRange(self, nums: List[int], target: int) -> List[int]:
        def findLeft():
            left, right = 0, len(nums) - 1
            res = -1
            while left <= right:
                mid = left + (right - left) // 2
                if nums[mid] == target:
                    res = mid
                    right = mid - 1  # 核心：遇到相等也把右眼盖住，往左逼近
                elif nums[mid] < target:
                    left = mid + 1
                else:
                    right = mid - 1
            return res

        def findRight():
            left, right = 0, len(nums) - 1
            res = -1
            while left <= right:
                mid = left + (right - left) // 2
                if nums[mid] == target:
                    res = mid
                    left = mid + 1   # 核心：遇到相等也把左眼盖住，往右逼近
                elif nums[mid] < target:
                    left = mid + 1
                else:
                    right = mid - 1
            return res

        return [findLeft(), findRight()]
```

### 搜索二维矩阵（难度：中等）

给你一个满足下述两条属性的 `m x n` 整数矩阵：
1. 每行中的整数从左到右按非严格递增顺序排列。
2. 每行的第一个整数大于前一行的最后一个整数。

给你一个整数 `target` ，如果 `target` 在矩阵中，返回 `true` ；否则，返回 `false` 。

#### 解题思路

这道题是一个非常经典的“障眼法”题目。虽然数据结构被包装成了一个二维矩阵，但当我们仔细阅读题目的特有条件时：
**“每行的第一个整数大于前一行的最后一个整数，且每行内部递增。”**

> [!tip] **【物理意义的核心防坑：降维打击】**
> 这句话的实质物理意义就是：如果你把这个二维矩阵从上到下、每一行像贪吃蛇一样首尾相连，拼接成一根长长的面条，那么这根面条将是一个**完美的、严格单调递增的一维数组**！

既然本质上是一个完美递增的一维数组，我们完全可以直接套用最基础的**寻找精确值**二分查找模板。全题唯一需要解决的难点只剩下一个**坐标映射转换**：
假设在这个虚拟的、拉长的一维数组中，我们二分出来的中间索引是 `mid`，原矩阵有 `n` 列，对应原二维矩阵的坐标 `(row, col)` 是什么？
- **行号 `row = mid // n`**：前面排满了多少个完整的行，我就被挤到了第几行。
- **列号 `col = mid % n`**：刨去所有能站满的整行后，剩下的零头就是我在当前行的位置。

解决了这一层空间坐标转换，剩下的全是最简单的标准一维二分查找体系。

#### 代码实现（Python版）
```python
class Solution:
    def searchMatrix(self, matrix: List[List[int]], target: int) -> bool:
        if not matrix or not matrix[0]:
            return False
            
        m, n = len(matrix), len(matrix[0])
        # 虚拟一维数组的左右边界
        left, right = 0, m * n - 1
        
        while left <= right:
            mid = left + (right - left) // 2
            
            # 核心：将一维的 mid 索引降维转换回二维的实际矩阵坐标
            row = mid // n
            col = mid % n
            mid_value = matrix[row][col]
            
            if mid_value == target:
                return True
            elif mid_value < target:
                left = mid + 1
            else:
                right = mid - 1
                
        return False
```

## 三、局部有序/旋转数组模型

### 寻找旋转排序数组中的最小值（难度：中等）

已知一个长度为 `n` 的整数数组 `nums` ，按升序排列，数组中的值 **互不相同** 。
在传递给函数之前，`nums` 在预先未知的某个下标 `k`（`0 <= k < nums.length`）上进行了 **向左旋转**，使数组变为 `[nums[k], nums[k+1], ..., nums[n-1], nums[0], nums[1], ..., nums[k-1]]`（下标 **从 0 开始** 计数）。例如， `[0,1,2,4,5,6,7]` 在下标 `3` 上向左旋转后可能变为 `[4,5,6,7,0,1,2]` 。
给你 **旋转后** 的数组 `nums` ，请你找出并返回数组中的 **最小元素** 。你必须设计一个时间复杂度为 `O(log n)` 的算法解决此问题。

#### 解题思路

> [!info] **【大白话拆解：到底什么是数组“旋转”？】**
> 很多初学者会被所谓的下标 `k` 和“向左/向右旋转”绕晕，其实你的直觉超级准确！
> **旋转的实质，根本不是什么循环位移，而是“一刀切下去把数组分两半，然后两段对调位置”**。
> 比如原数组（升序）被切成了 `[A段, B段]`。由于本身是升序，所以 `A段` 的最高点，肯定也老老实实比 `B段` 的最低点要小。
> 所谓的旋转，拿刀切完然后位置互换，将其变成了 **`[B段, A段]`**（把你说的“后面的数组移动到前面去了”）。
> 此时原本一路向上的单调线，变成了一个**“两段上坡，但中间有一个悬崖跌落点”的折线图**。而题目让你找的最底下的值，就是跌落到谷底的那一下！

- 这道题是旋转数组类一切题目的基础。核心在于如何二分？答案是：**永远拿 `mid` 的值和右边界 `right` 的值做比较**。
- **情况1：`nums[mid] < nums[right]`**
  在任何情况下，只要中间这个数比最右边的数还要小，这说明什么？这说明从 `mid` 一直走到 `right`，这整整半边是“顺滑且完美递增的”。既然它是顺滑的，那右半边肯定不会有悬崖式的最低点（谷底），最低点要不就是在 `mid` 本身，要不就藏在刚才那半丢掉的左边混沌区里！所以丢弃右边的最高半侧：令 `right = mid`（**注意：绝对不能写成 `mid - 1` 跳过自身**，因为 `mid` 极有可能就是我们要找的那个最低点）。
- **情况2：`nums[mid] > nums[right]`**
  中间那个数竟然比最末尾的数还要大！说明从中间往右走的过程中，绝对经历了一个“悬崖跌落”的断层点，而那个深渊的最底部（最小值），必定就藏在右边这半边混沌里！左边这一半全是在悬崖上面的高地上，直接抛弃：令 `left = mid + 1`。

> *(注：此题代码中出现了 `right = mid` 保留 `mid` 点的操作，为了防止 `left==right` 时的死循环悲剧，循环条件必须遵从文章开篇的铁律，降阶使用 `while left < right`)*

#### 代码实现（Python版）
```python
class Solution:
    def findMin(self, nums: List[int]) -> int:
        left, right = 0, len(nums) - 1
        
        while left < right:
            mid = left + (right - left) // 2
            if nums[mid] > nums[right]:
                left = mid + 1
            else:
                right = mid
                
        return nums[left]
```

### 搜索旋转排序数组（难度：中等）

整数数组 `nums` 按升序排列，数组中的值 **互不相同**。
在传递给函数之前，`nums` 在预先未知的某个下标 `k` 上进行了向左旋转，例如， `[0,1,2,4,5,6,7]` 在下标 3 上向左旋转后可能变为 `[4,5,6,7,0,1,2]`。
给你旋转后的数组 `nums` 和一个整数 `target` ，如果 `nums` 中存在这个目标值 `target` ，则返回它的下标，否则返回 `-1` 。必须设计一个时间复杂度为 `O(log n)` 的算法解决此问题。

#### 解题思路

这也是局部有序模型中极其经典的一道必考题。它的物理意义非常直观：**“一根被中间掰断，然后把两截换了位置的冰棍”**。

这根冰棍虽然不是全局有序的，但它有一个极其重要的物理性质：**不管你拿刀从哪里（`mid`）劈下去，劈出来的两半里，绝对有一半是完好无损（纯单调递增）的！**

> [!tip] **【逻辑推演：如何利用这“完好的一半”？】**
> 我们二分查找的核心是“排除一半不可能的空间”。在这道题里，我们可以用“肯定有序的那一半”来当探测器：
> 1. **第一步：拿刀切（算 `mid`）并判断哪半边完美。**
>    只要对比 `nums[left]` 和 `nums[mid]` 的大小：如果 `nums[left] <= nums[mid]`，说明从左切口到中间这一段是完美递增的（也就是左半边完美）；否则，完美的肯定在右半边！
> 2. **第二步：用“完美的半边”框定嫌疑人（判定丢弃哪一半）。**
>    - 假设左半边完美：我们就看 `target` 是否在 `[nums[left], nums[mid])` 这个完美区间内。如果在这个范围内，那凶手一定在左边，直接砍掉右边（`right = mid - 1`）；如果不在这个范围，那凶手肯定藏在右半截那个混沌区间里，我们砍掉左边（`left = mid + 1`）。
>    - 假设右半边完美：同理，看 `target` 是否在 `(nums[mid], nums[right]]` 这个完美区间内。如果在，收缩到右半边；不在，去左半边找。

通过这种“优先寻找顺滑区间，借此排除混沌区间”的方式，每次稳定收缩一半，稳稳的 $O(\log N)$。

#### 代码实现（Python版）
```python
class Solution:
    def search(self, nums: List[int], target: int) -> int:
        left, right = 0, len(nums) - 1
        
        while left <= right:
            mid = left + (right - left) // 2
            if nums[mid] == target:
                return mid
            
            # 步骤一：判断哪半边是纯纯的有序段（完美的半边）
            if nums[left] <= nums[mid]:
                # 左半部分有序
                # 步骤二：用完美的一半来框定 target
                if nums[left] <= target < nums[mid]:
                    # 确定在那半边之内，大胆舍弃另一半
                    right = mid - 1
                else:
                    # 确定不在这边，去另一半找
                    left = mid + 1
            else:
                # 右半部分一定是有序的
                if nums[mid] < target <= nums[right]:
                    # 确定在右边之内
                    left = mid + 1
                else:
                    right = mid - 1
                    
        return -1
```

## 四、二分答案模型（终极天花板）

### 寻找两个正序数组的中位数（难度：困难）

给定两个大小分别为 `m` 和 `n` 的正序（从小到大）数组 `nums1` 和 `nums2`。请你找出并返回这两个正序数组的 **中位数** 。
算法的时间复杂度应该为 `O(log(m + n))` 。

#### 解题思路

这是 LeetCode 第4题，一道令无数人头皮发麻的 Hard 题。关键在于题目定死的时间复杂度：`O(log(m + n))`。
如果没有这个限制，我们完全可以开一个指针，像合并两个有序链表一样，谁小就挑谁，走到中间就是中位数，但那样复杂度是 `O(m + n)`，会超时/不达标。只要看到 `O(log)`，唯一的出路就是**二分**。但在两个不同的数组中找单一的中位数，到底该在这个还是那个里面二分呢？

> [!tip] **【维度升级：将中位数转化为“求第 $K$ 小的数”】**
> 我们不要只盯着“中位数”看，中位数在物理意义上其实就是一个**特定的排名**：
> - 如果两个数组总长度是奇数，中位数就是合并后的第 `(total + 1) // 2` 小的数。
> - 如果两个数组总长度是偶数，中位数就是合并后的第 `total // 2` 小 和 `total // 2 + 1` 小 这两个数的平均值。
> 所以，只要我们写出了一个通用的法宝函数：`getKthElement(k)` —— **能够以 $O(\log K)$ 的速度在两个有序数组里找到排名第 $K$ 小的数字**，全剧终！

**【以武林高手对决（Kth 淘汰赛）理解二分逻辑】**
假设现在要找第 $K=10$ 名高手，目前 `nums1` 和 `nums2` 两大门派都按武功从低到高排好了队。
1. **各出奇兵对决**：既然要找两队合并后的第 `K` 名，那我们不妨分别去看看两队里的第 `K/2`（比如第 5 名）高手的实力。
2. **实力检验淘汰**：
   - 假设 `nums1` 的第 5 名名叫小阿，战斗力是 50。
   - 假设 `nums2` 的第 5 名名叫小波，战斗力是 90。
   - 因为小阿（50）打不过小波（90），说明什么？说明在这个宇宙里，不管别的门派有多弱，**小阿及排在小阿前面的那 4 个人，加起来最多也就是全宇宙第 9 弱（即使 `nums2` 里的前 4 个人也比小阿穷，总共只有 4+5=9 个人比小阿弱），他们四个永远、永远不可能成为全宇宙第 10 弱（K=10）的人！**
3. **批量淘汰**：既然绝对不可能是我们要找的目标，我们就可以**直接把 `nums1` 里小阿包括小阿前面的这 5 个人全部淘汰当炮灰**，连看都不用看。
4. **排名顺推**：由于一口气淘汰了 5 个人，那我们接下来要找的目标，就从“在所有大军中找第 10 名”，变成了“**在剩下的两路大军中找第 $10 - 5 = 5$ 名**”。目标从 $K$ 断崖式缩小成了 $K - K/2$。

**每一次淘汰，目标名次 $K$ 都会折半！** 这就是完美的 $O(\log(m+n))$ 复杂度的来源！当 $K$ 活活被扣减到 1 时，两队排头里稍微弱一点的那个，就是最后那个幸存者。

#### 代码实现（Python版）
```python
class Solution:
    def findMedianSortedArrays(self, nums1: List[int], nums2: List[int]) -> float:
        # 这个内嵌函数就是我们的“淘汰赛大印”：寻找门派并集里的第 K 小元素
        def getKthElement(k: int) -> int:
            index1, index2 = 0, 0
            while True:
                # 边界1：当 nums1 这个门派的人已经被淘汰成了空壳，直接从 nums2 剩下的里面选第 k 个即可
                if index1 == len(nums1):
                    return nums2[index2 + k - 1]
                # 同理：若 nums2 被掏空，从 nums1 拿
                if index2 == len(nums2):
                    return nums1[index1 + k - 1]
                # 边界2：当 k 已经被疯狂折半扣减到了 1 的时候
                # 这说明前面该死的炮灰全填上了，现在这就是决战轮！选两队目前排头里更弱的那一个就行。
                if k == 1:
                    return min(nums1[index1], nums2[index2])
                
                # 正常情况：两个门派各自往前探视 k//2 的距离
                half = k // 2
                # 防止这个小门派总共都没剩下 k/2 个人，最多探到他们的队尾即可
                newIndex1 = min(index1 + half - 1, len(nums1) - 1)
                newIndex2 = min(index2 + half - 1, len(nums2) - 1)
                
                pivot1, pivot2 = nums1[newIndex1], nums2[newIndex2]
                if pivot1 <= pivot2:
                    # nums1 那个出列的人比较弱，那么他及他前面的人就全员淘汰！
                    # 动态计算真正淘汰的人数，并从要找的排名 k 中扣除
                    k -= (newIndex1 - index1 + 1)
                    # 幸存者的队头往后推（丢弃炮灰）
                    index1 = newIndex1 + 1
                else:
                    k -= (newIndex2 - index2 + 1)
                    index2 = newIndex2 + 1

        totalLength = len(nums1) + len(nums2)
        if totalLength % 2 == 1:
            # 总人数是奇数，直接找第 (N+M)/2 + 1 小
            return float(getKthElement(totalLength // 2 + 1))
        else:
            # 总人数是偶数，找到正中间偏左和偏右的两名，求平均值
            return (getKthElement(totalLength // 2) + getKthElement(totalLength // 2 + 1)) / 2.0
```
