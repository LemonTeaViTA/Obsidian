# 动态规划 · 背包问题 (Knapsack)

> [[算法_MOC|返回算法导航]] ｜ 主篇见 [[动态规划]]

> [!info] 本文速览
> 本文从主篇 [[动态规划]] 拆出，专讲背包模型。背包问题最常见的有两大宗门：**0-1 背包**（每种物品只能用一次）和**完全背包**（每种物品可以无限用）。它们在一维滚动 DP 里的核心区别只有一处——**内层容量的循环方向**：
> - **0-1 背包**：内层容量**逆序**遍历（`range(target, num-1, -1)`），保证每个物品只用一次。代表题：分割等和子集。
> - **完全背包**：内层容量**正序**遍历，允许同一物品反复使用。代表题：零钱兑换、单词拆分、完全平方数。

## 一、分割等和子集（0-1 背包·难度：中等）

给你一个只包含正整数的非空数组 `nums` 。请你判断是否可以将这个数组分割成两个子集，使得两个子集的元素和相等。

### 解题思路

这是最经典的 **0-1 背包问题** 变种。
我们把问题翻译一下：“将数组分割成和相等的两份”其实就是问：**能不能从数组里挑选出一堆数字，让它们的和恰好等于整个数组总和的一半（`sum(nums) / 2`）？** 
这就彻底变成了一个典型的 0-1 背包：我们的背包总容量是 `sum / 2`，物品就是数组里的每一个数字，每个物品的“体积”和“价值”就是这个数字本身。我们要在给定的物品中看看能不能刚好把包塞满。

- **状态定义**：`dp[j]` 表示背包容量为 `j` 时，能否用前部分数字恰好把它装满。
- **状态转移方程**：对于当前正在考察的新物品数字 `num`，我们可以选择“装它”：如果决定装它，目标就变成考察以前我们的背包能不能装满 `j - num` 这点容量。也就是 `dp[j] = dp[j] or dp[j - num]`。只要这两种方案有一条能走通，`dp[j]` 就大功告成了。
- **初始化**：`dp[0] = True`。装满容量为 0 的背包的方法就是永远“什么都不装”。作为所有后续推理唯一的根基点。
- **【直击灵魂：为什么要有两层循环？分别在干什么？】**
  这个问题对于没写过二维 DP 直接跳一维滚动数组的人来说极其困惑，本质上，这两层循环的分工是极其明确的：
  - **外层循环（遍历物品 `for num in nums`）**：你可以把整个过程想象成站在流水线旁边，数组 `nums` 里的数字就是一个个送过来的“快递盲盒”。每新送过来一个数字 `num`，你都要拿着它，基于它**去刷新一遍你手里的那个“全村希望的账本”**（即 `dp` 数组）。
  - **内层循环（遍历容量 `for j in range(...)`）**：你的总账本上密密麻麻记录了容量从 `1` 斤到 `target` 斤的各种各样的空背包“目前能不能刚好装满”。当你手里攥着传送带新送来的、重量为 `num` 的新物品去问容量为 `j` 的背包时，你会满怀期待地想：“兄弟，我手里拿着 `num` 这块砖，如果以前（就是上一轮的账本）那些老数字们能成功填满 `j - num` 那个稍小点儿的坑位（即 `dp[j-num] == True`），那我今天顺手把这块重量刚好是 `num` 的砖“咔嗒”一声塞进去，你不就完美缝合到了正好容量是 `j` 的大圆满了吗！”因此，这就是用新积木去挨个容量去试填的过程。
- **【核心易错坑：内层循环为什么要逆序？】**：本题是 0-1 背包：**数组里的数每个人仅仅只能拿一次！**
  - 如果你**正序遍历**（从 `num` 一路推高到 `target`），当你在算大容量需要用到小容量的旧数据 `dp[j-num]` 时，那个 `dp[j-num]` 可能已经在这一轮刚刚被当前这个 `num` 给填满改变过了！这就等同于你把当前数字 `num` 放进包里了**两次甚至很多次**。它就变成了后面题目的完全背包。
  - 要解决这个问题，防止串台的究极办法是：**我们必须从大到小去逆向刷新数组（从 `target` 降序滑落到 `num`）**。当你计算高位的 `dp[j]` 需要依赖左边小位的 `dp[j-num]` 的时候，因为你是从右往左抹过来的，所以左边那些小的值还保留在上一轮比赛（即还没受到当前新数字洗礼）原封未动的原样！这样，当前数字就绝对安全地只会被装进去一次。这就是 0-1 背包滚动的精髓。
- **【逻辑细节：内层循环的终点为什么停在 `num - 1`？】**
  - 在 Python 中的 `range(start, stop, step)` 函数是“前闭后开”的，意味着它会在到达 `stop` 的前一个时刻停止。因此 `range(target, num - 1, -1)` 的实际效果是：从 `target` 容量开始往下降，**降到代表当前物品重量的 `num` 为止**。
  - **物理意义上的原因**：如果当前背包的容量 $j$ 比新来的这块砖的重量 `num` 还要小（即 $j < num$），那你这容量可怜的小背包根本**塞不进去**手里这块大砖头！既然塞不进，那背包的状态 `dp[j]` 就无法借助这块新砖来更新，只能乖乖继承上一轮的老样子。因为一维数组本身原位不动就是继承老样子，所以我们干脆连算都不去算它了，直接把 $j < num$ 的情况剪枝掉。这就是为什么内循环卡在 `num`（即 Python 语法里的 `num - 1` 作为终点）就不再往下算的原因。

### 代码实现（Python版 - 0-1背包模板）

```python
class Solution:
    def canPartition(self, nums: List[int]) -> bool:
        total_sum = sum(nums)
        # 奇数总和，绝大可能不能分成等和两份整数
        if total_sum % 2 != 0:
            return False
            
        # 要拼凑的目标容量是总和的一半
        target = total_sum // 2
        
        # dp[j] 代表能否刚好凑出容量为 j 的和
        dp = [False] * (target + 1)
        dp[0] = True
        
        # 外层遍历每个人/物品（仅使用一次）
        for num in nums:
            # 内层循环遍历容量：为了保证 0-1 背包特性，必须【逆序遍历】防串台！
            for j in range(target, num - 1, -1):
                dp[j] = dp[j] or dp[j - num]
                
        return dp[target]
```

## 二、零钱兑换（完全背包·难度：中等）

给你一个整数数组 `coins`，表示不同面额的硬币；以及一个整数 `amount`，表示总金额。计算并返回可以凑成总金额所需的 **最少的硬币个数**。如果没法凑出，返回 `-1`。可以认为每种硬币的数量是无限的。

### 解题思路

这道题是经典的**完全背包求极值**问题。可以将总金额视作背包容量，各种面额的硬币视作重量不等的物品，由于硬币有无限个，便构成了完全背包。

- **状态定义**：`dp[i]` 表示凑成金额 `i` 所需的最小硬币个数。
- **状态转移方程**：对目标金额 `i`，尝试拿面值为 `coin` 的硬币。如果拿了这一枚，就需要再加上凑出 `i - coin` 的最小个数。因此状态转移方程为：`dp[i] = min(dp[i], dp[i - coin] + 1)`，仅当 `i >= coin` 时能进行。
- **初始化**：既然求“最小”，就把所有元素初始化为正无穷大 `float('inf')`，防止被 0 干扰。唯一的基石是 `dp[0] = 0`，即凑齐 0 元需要 0 枚硬币。

### 代码实现（Python版）

```python
class Solution:
    def coinChange(self, coins: List[int], amount: int) -> int:
        # dp 数组初始化为无穷大，表示初期认为凑不出
        dp = [float('inf')] * (amount + 1)
        dp[0] = 0
        
        # 外层遍金额，内层遍历硬币（或者反过来也可以，因为求的是全组合的极值而不是排列数）
        for i in range(1, amount + 1):
            for coin in coins:
                if i >= coin:
                    dp[i] = min(dp[i], dp[i - coin] + 1)
                    
        return dp[amount] if dp[amount] != float('inf') else -1
```

## 三、单词拆分（完全背包·难度：中等）

给你一个字符串 `s` 和一个字符串列表 `wordDict` 作为字典。请你判断是否可以利用字典中出现的单词拼接出 `s`。注意：不要求字典中出现的单词全部都使用，并且字典中的单词可以重复使用。

### 解题思路

这也是**完全背包问题**的变种，字典里的各个“单词”就像是待挑选的“物品”，我们要用它们去装满字符串 `s` 这个背包！不同之处在于字符串组合要求**排列顺序必须严格一致**。

- **状态定义**：`dp[i]` 代表字符串前 `i` 个字符组成的子串（即 `s[0:i]`）能否被字典里的单词拼出来。
- **状态转移方程**：利用切断思维，在 `s[0:i]` 内部找一个断点 `j`。如果前半部分 `s[0:j]` 已证实合法（即 `dp[j] == True`），并且后半段切出来的 `s[j:i]` 刚好存在于字典中，那整个长为 `i` 的字符串自然被拼接成功：`dp[i] = True`。
  - **【思维误区一：为什么是判断 `dp[j]`，而不是直接去写 `s[:j] in word_set`？】**
    如果你写成 `s[:j] in word_set`，你的意思是要求 `s[:j]` 这个全长子串必须是一个**又长又完整的、独立出没在字典里的巨型单词**。比如你要拼 `"applepen"`，你的字典里只有 `"apple"` 和 `"pen"`。你此时切出了前半部分 `"applepen"` 去判断 `in word_set`，字典肯定说“没这词儿啊，退回”。
    但如果我们写成 `dp[j]`，那就完全不一样了！**它的含义是：不管你这前半部分 `s[:j]` 有多长、是由多少个碎单词拼合而成的，只要以前历史运算过它“属于合法词汇拼合物”，那么在动态规划的世界里，它就是被点亮的**。这就是在“利用历史记录避免重复验证全串”的动态规划精髓。
  - **【思维误区二：如果循环跑完都没遇到 `break`，我能直接 `return False` 结束程序吗？】**
    绝对不能。这里的内侧循环（也就是寻找断点 `j` 的循环），只是为了给**当下的这短得可怜的 `i` 长度**的前缀尝试寻找一种拼凑方法。找到了，就说明长度为 `i` 的前缀合法，我们 `break` 掉当前 `i` 的找拼合工作，急匆匆地去继续拼下个更长的 `i` 了。
    如果针对这个特定的 `i`，所有的 `j` 切法都试了，都拼不成功，那不 `break` 也就意味着这句长度为 `i` 的阶段性子串拼不出来而已——这对应的底层客观事实是 `dp[i]` 保持为初始化的 `False` 就足够了。你千万不能因为人家半路上有一个小的阶段性残缺词没拼出来，就在循环体里直接判了全盘代码的死刑 `return False`，因为后面说不定更长的 `i` （比如包含了整个字典完整单词汇的长段）就能成功拼上历史合法的积木！这就像人生，阶段性的拼合失败不代表全局崩盘，最后的结局由 `dp[n]` 统一宣判。
- **初始化**：`dp[0] = True`。空字符串默认不需要任何单词就能拼凑出来，这是后续所有转移逻辑能够连通启动的一块重要“基石”。

### 代码实现（Python版 - 切分挂接法）

```python
class Solution:
    def wordBreak(self, s: str, wordDict: List[str]) -> bool:
        word_set = set(wordDict)  # 换成哈希集合，提速百倍
        n = len(s)
        
        # dp[i] 代表 s 的前 i 个字符组成的子串，能否被合法拼出
        dp = [False] * (n + 1)
        dp[0] = True  # 空串默认是可以拼出
        
        for i in range(1, n + 1):
            # 将前 i 个字符的串一分为二，找一个能接住的断点 j
            for j in range(i):
                if dp[j] and s[j:i] in word_set:
                    dp[i] = True
                    break  # 只要有一种切法能成功就够了！
                    
        return dp[n]
```

## 四、完全平方数（完全背包·难度：中等）

给你一个整数 `n` ，返回和为 `n` 的完全平方数的最少数量。完全平方数是一个整数，其值等于另一个整数的平方；换句话说，其值等于一个整数自乘的积。例如，1、4、9 和 16 都是完全平方数，而 3 和 11 不是。

### 解题思路

这道题在本质上和前面的《零钱兑换》**完全是一模一样**的完全背包问题！只是将无限可选的“硬币面额”换成了无限可选的“完全平方数（$1, 4, 9, 16 \dots$）”，要求从里面挑选最少数量的“物品”来凑满足够的容量 $n$。

- **状态定义**：`dp[i]` 表示凑出目标和为 `i` 所需的最少完全平方数的数量。
- **状态转移方程**：当前面临的目标和为 `i` 时，尝试从中减去一个完全平方数 $j^2$。如果拿掉这个 $j^2$，我们要找的就是剩下的容量 `i - j*j` 所需的最少数量，再加上刚刚拿起的这 $1$ 个。所以递推式为：`dp[i] = min(dp[i], dp[i - j*j] + 1)`。我们需要让 $j$ 从 $1$ 开始逐渐变大，只要 $j^2 \le i$ 就进行尝试截取，从中选出最好的组合方式。
- **初始化**：依然是因为求极小值，把所有元素用无穷大 `float('inf')` 填充满，只设 `dp[0] = 0` 作为没有目标余额不需要物品的基石。

### 代码实现（Python版）

```python
class Solution:
    def numSquares(self, n: int) -> int:
        dp = [float('inf')] * (n + 1)
        dp[0] = 0
        
        # 遍历所有的容量数字
        for i in range(1, n + 1):
            # j 从 1 开始，只要它的完全平方不超过 i 就试着放入这个"新硬币"
            j = 1
            while j * j <= i:
                dp[i] = min(dp[i], dp[i - j * j] + 1)
                j += 1
                
        return dp[n]
```
