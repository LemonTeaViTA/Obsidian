## 一、贪心算法核心思想

贪心算法（Greedy Strategy）其实不用背模板，它的本质是：**在每一步选择中都采取在当前状态下最好或最优的选择，从而希望能导致结果是全局最好或最优的算法。**

能用贪心做的题是有特征的（拥有**贪心选择性质**和**最优子结构**）。如果贪心策略举不出反例，那就是对的。常见的有：“跳跃游戏”、“分发饼干”、“加油站”等。

## 二、不断刷新极限

### 跳跃游戏 II（难度：中等）

**题目：**
给定一个长度为 `n` 的非负整数数组 `nums`。初始位置为 `nums[0]`。每个元素 `nums[i]` 表示从索引 `i` 向前跳转的最大长度。返回到达 `nums[n - 1]` 的最小跳跃次数。（假设你总是可以到达数组的最后一个位置）。

**【模板解析与思路归纳】：**
- 为什么可以用贪心而不是 BFS 最短路径？因为跳的范围是连续的！它能跳到 3 的位置，就必定能跳到 1、2。
- 维护变量：
  - `farthest`：我当前能展望到的最远地平线。
  - `current_end`：在**本次这发跳跃**的范围内，最远能波及到哪。
  - `jumps`：我已经起跳了多少次。
- i 不断往前走，一直刷新最远地平线 `farthest`。当我走到了当前跳跃的尽头（`i == current_end`），就不得不强行结算一次跳跃了：起跳一次，并将当前的最远地平线作为下一次跳跃的尽头。

**答案代码（Python版）：**
```python
class Solution:
    def jump(self, nums: List[int]) -> int:
        jumps = 0
        current_end = 0
        farthest = 0
        
        # 注意终止条件是 len(nums) - 1，因为如果在最后一格发生起跳是没意义的，我们到了就不跳了
        for i in range(len(nums) - 1):
            # 探索前方最远能跳到哪里
            farthest = max(farthest, i + nums[i])
            
            # 当遍历到了当前这一步所能波及的极限边界，意味着之前那一步的潜力被榨干了，必须触发下一次起跳
            if i == current_end:
                jumps += 1
                current_end = farthest
                
        return jumps
```
