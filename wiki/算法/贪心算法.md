## 一、贪心算法核心思想

贪心算法（Greedy Strategy）其实不用背模板，它的本质是：**在每一步选择中都采取在当前状态下最好或最优的选择，从而希望能导致结果是全局最好或最优的算法。**

能用贪心做的题是有特征的（拥有**贪心选择性质**和**最优子结构**）。如果贪心策略举不出反例，那就是对的。常见的有：“跳跃游戏”、“分发饼干”、“加油站”等。

## 二、不断刷新极限

### 跳跃游戏（难度：中等）

给你一个非负整数数组 `nums` ，你最初位于数组的 **第一个下标** 。数组中的每个元素代表你在该位置可以跳跃的最大长度。
判断你是否能够到达最后一个下标，如果可以，返回 `true` ；否则，返回 `false` 。

#### 解题思路

这道题是贪心算法“不断刷新极限”模型的最入门级基础题。在这道题里，我们完全不需要关心“具体哪一步跳了几格”、“总共跳了多少步”，我们的目标极度单纯：**我从出发点开始，最远到底能覆盖到哪一个格子？**

- **变量维护**：我们只需要维护一个核心变量 `farthest`，代表当前所能到达的“最远安全区边界”。
- **遍历策略（沿途搜刮潜力）**：
  1. **困死检验**：如果我们当前所遍历到的格子下标 `i` 已经大于了我们历史上创造的 `farthest`，也就是说我们连现在的 `i` 这个格子都够不着，说明我们死在了前面的某个断崖里，直接 `return False`。
  2. **极限刷新**：既然我们活着走到了当前的格子 `i`，那我们一定要榨干这格的所有潜力。只要踩在这格上，我最远能跳到哪？那就是 `i + nums[i]` 嘛！我们拿它去挑战全局的纪录簿：`farthest = max(farthest, i + nums[i])`。
  3. **提前通关**：如果你发现你刷新后的 `farthest` 已经大于等于了整个数组最后一个格子的下标 `len(nums) - 1`，也就没必要再往后走了，后面的路必然都通了，直接大喊 `return True`！

#### 代码实现（Python版）

```python
class Solution:
    def canJump(self, nums: List[int]) -> bool:
        farthest = 0
        n = len(nums)
        
        for i in range(n):
            # 第一重门：先看当前这个格子我们有没有资格踩上来
            if i > farthest:
                return False
            
            # 第二重门：踩上来后，借力打力，看能不能把人类的跳远极限推向新高
            farthest = max(farthest, i + nums[i])
            
            # 只要极限能够覆盖终点，就可以提前放鞭炮庆祝了
            if farthest >= n - 1:
                return True
                
        return False
```

### 跳跃游戏 II（难度：中等）

给定一个长度为 `n` 的非负整数数组 `nums`。初始位置为 `nums[0]`。每个元素 `nums[i]` 表示从索引 `i` 向前跳转的最大长度。返回到达 `nums[n - 1]` 的最小跳跃次数。（假设你总是可以到达数组的最后一个位置）。

#### 解题思路
- 为什么可以用贪心而不是 BFS 最短路径？因为跳的范围是连续的！它能跳到 3 的位置，就必定能跳到 1、2。

> [!warning] **【思维误区】为什么不直接跳到当前步数所能达到的最远位置（`farthest`）？**
> 很多人觉得：“既然我当前在 `nums[i]` 能跳 5 步，选最远的落脚点不就是最短次数吗，直接跨 5 步不就行了？”
> **大错特错！** 最远落脚点的“下一步”可能是个死胡同，而如果你稍微往回落一点，那一格的弹簧可能超级强（比如是 100），能把你直接弹到终点。
> 比如数组 `[2, 3, 1, 1, 4]`：
> - 站在索引 0（值 2），当前极限最远能跳到索引 2（值 1）。如果你无脑跳最远，后面的路径就是 `2 -> 1 -> 1 -> 4`，总共要跳 3 跳。
> - 如果你退一步跳到索引 1（值 3），它虽然当前不是最远，但由于数字 3 够大，它下一步能直接弹到终点 4。`2 -> 3 -> 4` 只要 2 跳！
> 
> **这就是这段代码为什么要引入 `current_end` 并且一步一步遍历的原因。这是一个“侦察兵模型”。**

- **物理意义与代码逻辑（侦察兵模型）**：
  - `i` 不是你真正在起跳的动作，`i` 是你派出去的**侦察兵**。
  - `current_end`：是你**当前这一跳所能覆盖的最大“辖区”（侦察区）**。
  - `farthest`：这一整个辖区内所有的落脚点中，**下一步的最强起跳点能把你送到的极限位置**。
  - 侦察兵 `i` 在辖区内一步步走，每扫过一块新地砖，就掏出计算器算算这块地砖的弹力（`farthest = max(farthest, i + nums[i])`）。
  - **关键代码 `if i == current_end`**：当侦察兵终于走到了当前辖区的尽头，说明这一跳的“潜力”你已经全部看完评估完了。不管你刚才到底相中哪块地砖，你**无论如何都必须结算并且完成下一次真正的起跳了**（`jumps += 1`）。同时，既然决定起跳，那么下一跳辖区的尽头，就是你刚才侦察到的最强弹力尽头（`current_end = farthest`）。

#### 代码实现（Python版）
```python
class Solution:
    def jump(self, nums: List[int]) -> int:
        jumps = 0
        current_end = 0
        farthest = 0
        
        # 注意终止条件是 len(nums) - 1，因为如果在最后一格发生起跳是没意义的，我们到了就不跳了
        for i in range(len(nums) - 1):
            # 侦察兵 i 探索前方最远能跳到哪里（持续记录辖区内的最强弹簧）
            farthest = max(farthest, i + nums[i])
            
            # 当侦察兵走到了当前跳跃辖区的尽头，强行结算起跳次数，并把下一跳的波及范围更新为刚才记下的最高记录
            if i == current_end:
                jumps += 1
                current_end = farthest
                
        return jumps
```

### 买卖股票的最佳时机（难度：简单）

给定一个数组 `prices` ，它的第 `i` 个元素 `prices[i]` 表示一支给定股票第 `i` 天的价格。

你只能选择 **某一天** 买入这只股票，并选择在 **未来的某一个不同的日子** 卖出该股票。设计一个算法来计算你所能获取的最大利润。

返回你可以从这笔交易中获取的最大利润。如果你不能获取任何利润，返回 0 。

#### 解题思路

这也是一道非常典型的归在这最前面章节（“不断刷新极限”）的贪心题（或者叫动态规划空间优化模型）。
如果想暴力枚举“所有的买入和卖出组合”，我们需要双重循环 $O(N^2)$ 的时间复杂度，这在遇到庞大数据时必然超时。

其实贪心的核心思想就是换个角度切入：假设我非要在**第 `i` 天强行卖出股票**，那怎么才能赚得最多？
答案显而易见：我一定希望当初它是在**第 `i` 天之前的历史最低点**买进来的。
所以，我们只需要站在流水线边上遍历这个数组，并同时做两件事情：
1. **贪心地记录历史最低点**：随时随地用一个变量 `min_price` 记录到目前为止遇到的最低价格。
2. **贪心地计算全局最高利润**：在每一天，如果用当天的最新价格去减那个你背熟的 `min_price`，就会有一笔当天的潜在利润。只要这个潜在利润比过往算的 `max_profit` 还要大，我们就把全村最大的利润更新掉它。

通过这样“双向打擂台”，我们走完一遍数组就得到了终极答案。时间复杂度稳稳降到 $O(N)$！

#### 代码实现（Python版）

```python
class Solution:
    def maxProfit(self, prices: List[int]) -> int:
        # 初始状态
        min_price = float('inf') # 第一天买入前，假设过去的底价是无限大
        max_profit = 0           # 兜底保障：不管怎样亏不能亏本，只要没利润就强制返回0
        
        for price in prices:
            # 步骤一：遇到更低的价格，就把它当作后面继续走时的新的最划算买入点
            min_price = min(min_price, price)
            
            # 步骤二：尝试今天把它结账卖掉，看看能不能刷新记录簿里的最大利润
            max_profit = max(max_profit, price - min_price)
            
        return max_profit
```

### 划分字母区间（难度：中等）

给你一个字符串 `s` 。我们要把这个字符串划分为尽可能多的片段，同一字母最多出现在一个片段中。例如，字符串 `"ababcc"` 能够被分为 `["abab", "cc"]`，但类似 `["aba", "bcc"]` 或 `["ab", "ab", "cc"]` 的划分是非法的。
注意，划分结果需要满足：将所有划分结果按顺序连接，得到的字符串仍然是 `s` 。返回一个表示每个字符串片段的长度的列表。

#### 解题思路

这也是一道非常典型的“不断刷新势力范围（极限）”的题目，它的物理意义很像上面提到的【跳跃游戏】，都是在维护一个“辖区”，不同的是：**跳跃游戏是越跳越远刷出新势力范围，而这道题是被牵扯进来的字母强行把我们拖向更远的势力范围。**

**【核心突破口：最后一次出现的位置】**
题目要求“同一个字母最多只能出现在**一个**片段中”。这意味着什么？
这意味着如果你在这个片段里装进了一个字母 `'a'`，那么为了满足条件，哪怕你在后面看到了天涯海角，你也**必须**把宇宙尽头的最后一个 `'a'` 也包进这个同一个片段里！
一旦你包了首尾所有的 `'a'`，这个片段又被迫把中间夹杂的 `'b'`、`'c'` 也包裹进去了。那不好意思，宇宙尽头的最后一个 `'b'` 和 `'c'` 你也得乖乖包进去。片段的右边界就这样被一次次无情地向后拖拽（拉扯大）。

**【步骤拆解】**
1. **建立通讯录（上帝视角）**：不管三七二十一，我们先遍历一次整个字符串，用一个哈希表或者是数组把每一个字母真正在历史上**最后一次出现的位置下标**全记下来。
2. **侦察兵巡发（贪心划分）**：
   - 用一个变量 `end` 表示当前正在画的这条圈的“死线边界”。再用一个变量 `start` 记录这圈起笔的位置。
   - 侦察兵 `i` 开始走：每遇到一个字母，就去翻“上帝通讯录”，看看这个字母的最后一个在哪里。如果那个位置比我们当前的圈 `end` 还要远，我们就把圈强行拉长到那个位置（`end = max(end, 最后一舞的位置)`）。
   - **闭环收网（关键点）**：什么时候说明这个圈画圆满了呢？当侦察兵 `i` 拼死拼活走到了 `end` 的位置！说明这个死线范围内**所有的**字母，它们的“最后一舞”都乖乖待在圈里了，没人有本事把圈再往外扯了。这个时候，干净利落地一刀切下（记录长度：`end - start + 1`），重置 `start = i + 1` 准备砍下一刀。

#### 代码实现（Python版）

```python
class Solution:
    def partitionLabels(self, s: str) -> List[int]:
        # 建立通讯录：统计每个字母最后出现的位置
        last_occurrence = {}
        for i, char in enumerate(s):
            last_occurrence[char] = i
            
        result = []
        start = 0
        end = 0
        
        # 侦察兵开始走
        for i, char in enumerate(s):
            # 看看眼下这个字母最后死在了哪里，逼着边界向外拉扯
            end = max(end, last_occurrence[char])
            
            # 当侦察兵苟到了当前的死线边界处，说明前面所有的字母都没有能力再生拉硬拽了
            # 斩断情丝！
            if i == end:
                result.append(end - start + 1)
                start = i + 1 # 重入轮回
                
        return result
```
