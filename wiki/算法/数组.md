## 一、数组核心思想与归类技巧

数组是最基础的数据结构，题目种类繁多，但万变不离其宗。遇到数组题目，绝大多数情况下可以从以下三大套路切入：
1. **双指针法（同向/相向/快慢）**：解决绝大部分寻找元组（两数之和、三数之和）、区间覆盖（滑动窗口、接雨水）问题。
2. **前缀和与哈希表**：解决“连续子数组”求和、求积之类的问题，用 O(1) 的时间快速求出某个区间的状态。
3. **贪心与动态规划状态压缩**：处理股票买卖、跳跃游戏等求最值的问题。

## 二、对撞双指针：查找与夹逼
*左右端点各放一个指针，向中间靠拢。核心在于“每次移动哪一边”的贪心排他性逻辑。*

### 三数之和（难度：中等）

**题目：**
给你一个包含 `n` 个整数的数组 `nums`，判断 `nums` 中是否存在三个元素 `a, b, c `，使得 `a + b + c = 0` ？请你找出所有和为 0 且不重复的三元组。

**【模板解析与思路归纳】：**
- 既然只要求返回数值，不要求返回原下标，起手式绝对是 `排序`！
- 排序后，固定第一个元素 `nums[i]`（注意去重），然后在其右侧区间使用 **双指针相向而行** 寻找剩下的两个数。
- 只有排好序，双指针的 `left += 1`（让数字变大）和 `right -= 1`（让数字变小）才有逻辑支撑。
- **注意点**：`== 0` 时需要同时移动左右指针（并去重），不能和 `< 0` 或 `> 0` 的逻辑混在一起。`< 0` 只移左，`> 0` 只移右，`== 0` 两边都移——三种情况必须分开处理。

**答案代码（Python版）：**
```python
class Solution:
    def threeSum(self, nums: List[int]) -> List[List[int]]:
        nums.sort()
        res = []
        n = len(nums)
        
        for i in range(n):
            # 剪枝：如果排过序后，第一个固定的数都 > 0，那后面再怎么加也是 > 0 的
            if nums[i] > 0:
                break
            # 去重：如果选过这个基准数了，跳过
            if i > 0 and nums[i] == nums[i - 1]:
                continue
                
            # 在 i 之后的区间内找两数之和
            left, right = i + 1, n - 1
            while left < right:
                total = nums[i] + nums[left] + nums[right]
                if total == 0:
                    res.append([nums[i], nums[left], nums[right]])
                    # 左指针去重推进
                    while left < right and nums[left] == nums[left + 1]:
                        left += 1
                    # 右指针去重推进
                    while left < right and nums[right] == nums[right - 1]:
                        right -= 1
                    left += 1
                    right -= 1
                elif total < 0:
                    left += 1   # 因为还需要更大的数字来填平亏空
                else:
                    right -= 1  # 溢出了，需要调小一点
                    
        return res
```

### 盛最多水的容器（难度：中等）

**题目：**
给定一个长度为 `n` 的整数数组 `height`。有 `n` 条垂线，第 `i` 条线的两个端点是 `(i, 0)` 和 `(i, height[i])`。找出其中的两条线，使得它们与 x 轴共同构成的容器可以容纳最多的水。

**【模板解析与思路归纳】：**
- `面积 = min(左边高, 右边高) * (右下标 - 左下标)`
- 采用最极端的两端向中央逼近的思路：一开始宽度最大，要使得面积有可能被增加，**绝对只能去移动两根柱子中较矮的那一根**！如果移动高的那根，宽度不仅变小了，短板效应决定了高度根本不可能变高。

**答案代码（Python版）：**
```python
class Solution:
    def maxArea(self, height: List[int]) -> int:
        left, right = 0, len(height) - 1
        max_area = 0
        
        while left < right:
            # 计算当前面积
            area = min(height[left], height[right]) * (right - left)
            max_area = max(max_area, area)
            
            # 木桶效应，谁丑（小）谁移动
            if height[left] < height[right]:
                left += 1
            else:
                right -= 1
                
        return max_area
```

### 接雨水（难度：困难）

**题目：**
给定 `n` 个非负整数表示每个宽度为1的柱子的高度图，计算按此排列的柱子，下雨之后能接多少雨水。

**【模板解析与思路归纳】：**
- 接雨水的最直观理解：某个位置能接上雨水的高度，是 **它左边最高的柱子 和 它右边最高的柱子 这两者的较小值 - 它本身的高度**。
- 双指针法就是维护 `left_max` 和 `right_max`：在 `height[left] < height[right]` 的情况下，左边肯定拦不住右边这个大汉，所以左侧 `left` 处的蓄水量完全由刚才记住的 `left_max` 决定！由此可边更新最值边计算。

**答案代码（Python版）：**
```python
class Solution:
    def trap(self, height: List[int]) -> int:
        if not height:
            return 0
            
        left, right = 0, len(height) - 1
        left_max, right_max = 0, 0
        res = 0
        
        while left < right:
            # 如果左边比较矮，雨水一定是从左边漏走，所以以左右两侧的短板 left_max 为准
            if height[left] < height[right]:
                if height[left] >= left_max:
                    left_max = height[left]  # 更新最大身高
                else:
                    res += left_max - height[left]  # 洼地积水
                left += 1
            else:
                if height[right] >= right_max:
                    right_max = height[right]
                else:
                    res += right_max - height[right]
                right -= 1
                
        return res
```

## 三、前缀和技巧
*子数组极速求和的法宝。连续子数组的和 `sum(i...j) = prefix[j] - prefix[i-1]`。*

### 和为 K 的子数组（难度：中等）

**题目：**
给你一个整数数组 `nums` 和一个整数 `k` ，请你统计并返回该数组中和为 `k` 的连续子数组的个数。注意数组里面可能含有负数！

**【模板解析与思路归纳】：**
- 为什么不能用滑动窗口？因为有**负数**！滑动窗口放大和缩小不具备单调性！
- **前缀和 + 哈希表** 是正解：当前遍历到了 `prefix_sum`，如果想知道之前有没有一段加上刚好是 `k`，就去哈希表里查从头到尾有没有出现过和为 `prefix_sum - k` 的前缀！（有的话，从那段到现在的这段子数组的和恰好就是 `k`）。
- **注意点**：`sum_map = {0: 1}` 不能省。它代表"空前缀"出现了 1 次，用于处理 `prefix_sum` 本身恰好等于 `k` 的情况。此时 `target_prefix = prefix_sum - k = 0`，如果哈希表里没有 `0: 1`，就会漏掉"从数组开头到当前位置"这整段子数组。

**答案代码（Python版）：**
```python
class Solution:
    def subarraySum(self, nums: List[int], k: int) -> int:
        count = 0
        prefix_sum = 0
        # sum_map 记录各个前缀和出现的次数。不要忘了初始化 0:1 （前缀和刚好等于 k 时使用）
        sum_map = {0: 1}
        
        for num in nums:
            prefix_sum += num
            
            # 要找的目标前缀！
            target_prefix = prefix_sum - k
            if target_prefix in sum_map:
                count += sum_map[target_prefix]
                
            # 记录当前的前缀和次数
            sum_map[prefix_sum] = sum_map.get(prefix_sum, 0) + 1
            
        return count
```

### 除自身以外数组的乘积（难度：中等）

**题目：**
返回数组 answer，其中 `answer[i]` 等于 `nums` 中除 `nums[i]` 之外其余各元素的乘积。要求时间复杂度为 `O(n)`，并且**不能使用除法**。

**【模板解析与思路归纳】：**
- 不能用除法怎么求？自己以外的乘积 = **它左边所有数的前缀乘积** × **它右边所有数的后缀乘积**。
- 建一个 `answer` 数组，先从左往右遍历一路乘起来，再从右往左一路倒推乘进去。空间只需 `O(1)`！

**答案代码（Python版）：**
```python
class Solution:
    def productExceptSelf(self, nums: List[int]) -> List[int]:
        n = len(nums)
        ans = [0] * n
        ans[0] = 1
        # 1. 正向：ans[i] 存的是 nums[0] * nums[1] * ... * nums[i-1]
        for i in range(1, n):
            ans[i] = ans[i - 1] * nums[i - 1]
        # 2. 反向：R 累积右侧乘积，直接乘进 ans
        R = 1
        for i in range(n - 1, -1, -1):
            ans[i] = ans[i] * R
            R = R * nums[i]
        return ans
```

## 四、同向双指针：移动与操作

### 移动零（难度：简单）

**题目：**
给定一个数组 `nums`，编写一个函数将所有 0 移动到数组的末尾，同时保持非零元素的相对顺序。

**【模板解析与思路归纳】：**
- 快慢指针。快指针在前面探路找“非0”，慢指针在后面负责“安顿队伍”。
- 遇到不是 0 的元素，就把快指针指向的值直接赋给（或者交换给）慢指针的位置，慢指针前进一格。全是零的就留下来被交换或者被覆盖。

**答案代码（Python版）：**
```python
class Solution:
    def moveZeroes(self, nums: List[int]) -> None:
        """
        Do not return anything, modify nums in-place instead.
        """
        left = 0  # 指示下一个应当安顿非零数字的位置
        
        for right in range(len(nums)):
            if nums[right] != 0:
                nums[left], nums[right] = nums[right], nums[left]
                left += 1
```

### 轮转数组（难度：中等）

**题目：**
给定一个整数数组 `nums`，将数组中的元素向右轮转 `k` 个位置，其中 `k` 是非负数。要求原地修改。

#### 解题思路

三次翻转法：先把整个数组翻转，再分别翻转前 `k` 个和后 `n - k` 个元素。

举个例子，`nums = [1,2,3,4,5,6,7]`，`k = 3`：
1. 整体翻转 → `[7,6,5,4,3,2,1]`
2. 翻转前 3 个 → `[5,6,7,4,3,2,1]`
3. 翻转后 4 个 → `[5,6,7,1,2,3,4]` ✅

关键坑点：`k` 可能大于数组长度，所以必须先 `k = k % n`。

#### 代码实现

```python
class Solution:
    def rotate(self, nums: List[int], k: int) -> None:
        def reverse(i: int, j: int):
            while i < j:
                nums[i], nums[j] = nums[j], nums[i]
                i += 1
                j -= 1
        n = len(nums)
        k = k % n
        reverse(0, n - 1)
        reverse(0, k - 1)
        reverse(k, n - 1)
```

#### 复杂度分析

- **时间复杂度**：O(n)，每个元素被翻转两次。
- **空间复杂度**：O(1)，纯原地操作。

### 缺失的第一个正数（难度：困难）

**题目：**
给你一个未排序的整数数组 `nums`，请你找出其中没有出现的最小的正整数。要求 O(n) 时间、O(1) 空间。

#### 解题思路

排座位法（原地哈希）：把每个数字送回它"该坐的位子"上——数字 `1` 应该坐在下标 `0`，数字 `2` 坐在下标 `1`，以此类推，数字 `x` 的座位就是 `nums[x - 1]`。

用 `while` 循环不断交换，直到当前位置的数字要么不在 `[1, n]` 范围内（没有合法座位），要么已经坐对了位置。全部归位后，再扫一遍，第一个 `nums[i] != i + 1` 的位置，`i + 1` 就是缺失的最小正整数。

坑点在于 while 的终止条件为什么是 `nums[i] != nums[nums[i] - 1]`，而不是看起来更直觉的 `nums[i] != i + 1`。

`nums[i] != i + 1` 的含义是"当前位置坐的人不对"，看似合理，但遇到重复元素就死循环了。比如 `nums = [1, 1]`，处理下标 1 时，`nums[1] = 1`，`1 != 2` 永远成立，于是不停地把下标 1 和下标 0 的两个 `1` 互换，永远停不下来。

`nums[i] != nums[nums[i] - 1]` 的含义是"我手里这个数字的目标座位上，坐的人和我不一样"。换句话说，它检查的不是"我坐对了没"，而是"我要去的那个座位上是不是已经有一个和我一样的人了"。如果目标座位上已经是相同的数字，再换也没意义，直接放弃。同样 `nums = [1, 1]`，处理下标 1 时，`nums[1] = 1`，目标座位是 `nums[0] = 1`，`1 == 1`，条件不满足，循环终止。

#### 代码实现

```python
class Solution:
    def firstMissingPositive(self, nums: List[int]) -> int:
        n = len(nums)
        for i in range(n):
            while 1 <= nums[i] <= n and nums[i] != nums[nums[i] - 1]:
                j = nums[i] - 1
                nums[i], nums[j] = nums[j], nums[i]
        for i in range(n):
            if nums[i] != i + 1:
                return i + 1
        return n + 1
```

#### 复杂度分析

- **时间复杂度**：O(n)，虽然有 while 嵌套在 for 里，但每个元素最多被交换一次到正确位置，总交换次数不超过 n。
- **空间复杂度**：O(1)，原地交换，不需要额外数组。

### 矩阵置零（难度：中等）

**题目：**
给定一个 `m x n` 的矩阵，如果一个元素为 `0`，则将其所在行和列的所有元素都设为 `0`。请使用原地算法。

#### 解题思路

两次扫描法：第一遍记录哪些行、哪些列含有 `0`（用两个 set），第二遍遍历矩阵，凡是行号或列号在 set 里的，直接置零。

关键点：不能边扫描边置零，否则新产生的 `0` 会污染后续判断，导致整行整列被误清空。必须先把所有原始 `0` 的位置记下来，再统一处理。

#### 代码实现

```python
class Solution:
    def setZeroes(self, matrix: List[List[int]]) -> None:
        m, n = len(matrix), len(matrix[0])
        row, col = set(), set()
        for i in range(m):
            for j in range(n):
                if matrix[i][j] == 0:
                    row.add(i)
                    col.add(j)
        for i in range(m):
            for j in range(n):
                if i in row or j in col:
                    matrix[i][j] = 0
```

#### 复杂度分析

- **时间复杂度**：O(m × n)，两次全量扫描。
- **空间复杂度**：O(m + n)，两个 set 分别存行号和列号。

### 螺旋矩阵（难度：中等）

**题目：**
给你一个 `m` 行 `n` 列的矩阵 `matrix`，请按照顺时针螺旋顺序，返回矩阵中的所有元素。

#### 解题思路

四边界收缩法，像剥洋葱一样一圈一圈往里走。维护四条边界 `top/bottom/left/right`，每一圈按固定顺序走四条边：

1. **→ 从左到右**：沿 `top` 行，走完后 `top += 1`（顶层剥掉了）
2. **↓ 从上到下**：沿 `right` 列，走完后 `right -= 1`（右侧剥掉了）
3. **← 从右到左**：沿 `bottom` 行，走完后 `bottom -= 1`（底层剥掉了）
4. **↑ 从下到上**：沿 `left` 列，走完后 `left += 1`（左侧剥掉了）

坑点一：走第 3、4 条边之前必须判断 `top <= bottom` 和 `left <= right`。前两条边不需要判断，因为 `while` 的入口条件已经保证了安全；但走完前两条边后边界已经收缩了（`top += 1`、`right -= 1`），如果矩阵只剩一行或一列，不重新检查就会重复遍历。以单行矩阵 `[[1, 2, 3]]` 为例：第 1 条边 → 走完 `[1,2,3]` 后 `top` 变成 1，此时 `top > bottom`（0），这行已经走完了；如果不加 `if`，第 3 条边 ← 会把 `[3,2,1]` 再走一遍。

坑点二：为什么代码里有时候是 `+1` 有时候是 `-1`？这完全是 `range()` **不包含终点**的特性决定的。正向遍历想走到 `right`，就得写 `range(left, right + 1)`；反向遍历想走到 `left`，就得写 `range(right, left - 1, -1)`。本质上 `+1` 和 `-1` 做的是同一件事——补偿开区间，只是方向不同。

#### 代码实现

```python
class Solution:
    def spiralOrder(self, matrix: List[List[int]]) -> List[int]:
        res = []
        top, bottom = 0, len(matrix) - 1
        left, right = 0, len(matrix[0]) - 1

        while top <= bottom and left <= right:
            # → 从左到右
            for j in range(left, right + 1):
                res.append(matrix[top][j])
            top += 1
            # ↓ 从上到下
            for i in range(top, bottom + 1):
                res.append(matrix[i][right])
            right -= 1
            # ← 从右到左（注意判断）
            if top <= bottom:
                for j in range(right, left - 1, -1):
                    res.append(matrix[bottom][j])
                bottom -= 1
            # ↑ 从下到上（注意判断）
            if left <= right:
                for i in range(bottom, top - 1, -1):
                    res.append(matrix[i][left])
                left += 1

        return res
```

#### 复杂度分析

- **时间复杂度**：O(m × n)，每个元素恰好访问一次。
- **空间复杂度**：O(1)，不算返回数组的话，只用了四个边界变量。

### 旋转图像（难度：中等）

**题目：**
给定一个 `n × n` 的二维矩阵 `matrix` 表示一个图像。请你将图像顺时针旋转 90 度。必须原地旋转，不能使用另一个矩阵。

#### 解题思路

两步拆解法：**先沿主对角线转置，再左右翻转每一行**。

为什么这样做能实现顺时针 90°？观察坐标变化：原位置 `(i, j)` 旋转 90° 后应该到 `(j, n-1-i)`。转置操作把 `(i, j)` 送到 `(j, i)`，左右翻转再把 `(j, i)` 送到 `(j, n-1-i)`，两步合起来刚好完成目标。

举个例子，`3 × 3` 矩阵：
```
原矩阵        转置后        左右翻转后（最终结果）
1 2 3        1 4 7        7 4 1
4 5 6   →    2 5 8   →    8 5 2
7 8 9        3 6 9        9 6 3
```

坑点一：转置时内层循环必须从 `j = i + 1` 开始，而不是 `j = 0`。如果从 0 开始，每对元素会被交换两次，等于没换。对角线上的元素 `(i, i)` 自己和自己交换也是多余操作，所以直接跳过。

坑点二：还有一种等价的两步法是**先上下翻转，再沿主对角线转置**，效果相同，只是操作顺序不同。但"转置 + 左右翻转"更直观，面试时推荐用这个。

#### 代码实现

```python
class Solution:
    def rotate(self, matrix: List[List[int]]) -> None:
        n = len(matrix)
        # 第一步：沿主对角线转置（行列互换）
        for i in range(n):
            for j in range(i + 1, n):
                matrix[i][j], matrix[j][i] = matrix[j][i], matrix[i][j]
        # 第二步：左右翻转每一行
        for i in range(n):
            matrix[i].reverse()
```

#### 复杂度分析

- **时间复杂度**：O(n²)，转置遍历上三角 n(n-1)/2 次，翻转遍历 n²/2 次，总共 O(n²)。
- **空间复杂度**：O(1)，纯原地交换，不需要额外矩阵。

#### 延伸：旋转 180° 和 270°

本质都是两步组合操作，差异只在第二步的翻转方向：

| 旋转角度 | 第一步 | 第二步 | 坐标变换 |
|---------|--------|--------|---------|
| **90° 顺时针** | 转置 | 左右翻转 | `(i,j) → (j, n-1-i)` |
| **180°** | 左右翻转 | 上下翻转 | `(i,j) → (n-1-i, n-1-j)` |
| **270° 顺时针**（= 90° 逆时针） | 转置 | 上下翻转 | `(i,j) → (n-1-j, i)` |

用 `3 × 3` 矩阵直观感受：
```
原矩阵       90° CW      180°       270° CW
1 2 3       7 4 1       9 8 7       3 6 9
4 5 6  →    8 5 2  →    6 5 4  →    2 5 8
7 8 9       9 6 3       3 2 1       1 4 7
```

为什么 180° 不需要转置？因为 180° 本质上就是把矩阵"倒过来"——左右翻一次、上下翻一次，两次镜像操作顺序可交换，不涉及行列互换。

```python
# 180°：左右翻 + 上下翻
def rotate_180(matrix):
    for row in matrix:
        row.reverse()
    matrix.reverse()

# 270° 顺时针：转置 + 上下翻
def rotate_270(matrix):
    n = len(matrix)
    for i in range(n):
        for j in range(i + 1, n):
            matrix[i][j], matrix[j][i] = matrix[j][i], matrix[i][j]
    matrix.reverse()
```

## 五、贪心与状态记录
*不需要深层递归，只需要维护好手里的几个极限变量。*

### 买卖股票的最佳时机（难度：简单）

**题目：**
只能选择某一天买入股票，并选择某一天（不同的一天）卖出。求最大利润。

**答案代码（Python版）：**
```python
class Solution:
    def maxProfit(self, prices: List[int]) -> int:
        min_price = float('inf')  # 记录当前遇到的史低价格
        max_profit = 0
        
        for price in prices:
            min_price = min(min_price, price)  # 贪心：如果今天以前有更低的，就记那个更低的成本
            max_profit = max(max_profit, price - min_price) # 今天卖出去的话能挣多少？和历史最佳对比
            
        return max_profit
```

### 买卖股票的最佳时机 II（难度：中等）

**题目：**
可以无限次买卖，但在任何时候最多只能持有一股股票。求最大利润。

**【模板解析与思路归纳】：**
- 既然可以无限次买卖，那么每天的微小上涨波动我都把它吃了！只要今天比昨天高，我就假装昨天买了今天卖了，累积盈利。

**答案代码（Python版）：**
```python
class Solution:
    def maxProfit(self, prices: List[int]) -> int:
        profit = 0
        for i in range(1, len(prices)):
            if prices[i] > prices[i - 1]:
                profit += prices[i] - prices[i - 1]
        return profit
```

### 跳跃游戏（难度：中等）

**题目：**
给定一个非负整数数组 `nums`，你最初位于数组的 第一个下标。元素代表在该位置可以跳跃的 最大长度。判断你是否能够到达最后一个下标。

**【模板解析与思路归纳】：**
- 只需维护一个变量：`max_reach` 表示我**所能跨及到的最远地平线**。
- 如果当前遍历到了第 `i` 步，而 `i` 竟然在我的 `max_reach` 之外，对不起，你困死在前面了，直接 `False`。否则不断贪婪地根据 `i + nums[i]` 刷新最远地平线。

**答案代码（Python版）：**
```python
class Solution:
    def canJump(self, nums: List[int]) -> bool:
        max_reach = 0
        
        for i in range(len(nums)):
            # 步子垮不到当前这点，直接暴毙
            if i > max_reach:
                return False
            # 在该位置上向远方跳，看看能不能比我以往到过的地带还要远？
            max_reach = max(max_reach, i + nums[i])
            # 如果尽头已经在覆盖范围内，提前胜利
            if max_reach >= len(nums) - 1:
                return True
                
        return False
```

## 六、哈希表：以空间换时间

### 两数之和（难度：简单）

**题目：**
给定一个整数数组 `nums` 和一个整数目标值 `target`，找出和为 `target` 的那两个整数，并返回它们的数组下标。

**【模板解析与思路归纳】：**
- 一边遍历，一边回头看：`目标值 - 当前值` 是否已经在我的哈希表（备忘录）里了。
- **注意点**：必须先检查再添加到哈希表。若先添加再检查，会导致同一个元素匹配自身（如 `target=6, nums=[3,...]`，3 会和自己配对）。

**答案代码（Python版）：**
```python
class Solution:
    def twoSum(self, nums: List[int], target: int) -> List[int]:
        val_to_index = {}
        for i, num in enumerate(nums):
            complement = target - num
            if complement in val_to_index:
                return [val_to_index[complement], i]
            val_to_index[num] = i
        return []
```

### 最长连续序列（难度：中等）

**题目：**
给定一个未排序的整数数组 `nums`，找出数字连续的最长序列（不要求序列元素在原数组中连续）的长度。要求时间复杂度 O(n)。

**【模板解析与思路归纳】：**
- 先把所有数放入 `set`，然后只从序列的**起点**开始往后数。
- 判断起点：`num - 1` 不在 set 里，说明 `num` 就是某段连续序列的最小值，从这里出发。
- 若 `num - 1` 在 set 里，直接 `continue` 跳过——它不是起点，从它出发会重复计算。
- **注意点**：往上找最大值时不能用 `for + continue`，必须用 `while y in num_set: y += 1`。因为 `continue` 会跳到下一个外层循环元素，而不是继续往上找，导致逻辑错误。
- **长度计算**：`length = y - num`，不需要 `-1`。因为循环结束时 `y` 已经是序列外的第一个值，`y - num` 恰好等于序列长度。

**答案代码（Python版）：**
```python
class Solution:
    def longestConsecutive(self, nums: List[int]) -> int:
        num_set = set(nums)
        max_len = 0

        for num in num_set:
            # 只从序列起点出发，跳过非起点
            if num - 1 in num_set:
                continue
            y = num + 1
            while y in num_set:
                y += 1
            max_len = max(max_len, y - num)

        return max_len
```
