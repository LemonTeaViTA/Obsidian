# 回溯算法 (Backtracking)

> [[算法_MOC|返回算法导航]]

## 一、回溯算法核心思想与终极模板

回溯算法的本质是**“多叉树的遍历过程”**。只要遇到需要求“所有可能结果”或者“在所有方案中寻找一个最优解”，都可以将其抽象为一棵决策树。
核心思想：**站在一个节点上，考虑“下一步有哪些选择”，尝试一个选择，往下走，走不通或者得出结果后，再退回来，撤销刚才的选择，去尝试下一个。**

### 🌟 终极模板框架
所有回溯问题都可以套用这个基础框架：

```python
def backtrack(路径, 选择列表, 其他状态):
    if 满足结束条件:
        res.append(路径[:]) # 需要深度拷贝/快照，否则后续操作会把结果改乱
        return

    for 选择 in 选择列表:
        if 不满足合法条件:
            continue
            
        做选择 (更新路径、状态等)
        backtrack(路径, 选择列表, 其他状态)
        撤销选择 (将路径、状态恢复到进入前的样子)
```

## 二、基础入门：组合、子集、排列模型
*最经典的“回溯三连”。重点体会 `start` 变量如何避免重复，以及 `used` 数组如何管理排列。*

### 子集（难度：简单）

**题目：**
给你一个整数数组 `nums` ，数组中的元素互不相同。返回该数组所有可能的子集（幂集），不能包含重复的子集。

**【模板解析与思路归纳】：**
- 子集问题是收集多叉树的**所有节点**（而不是只收集叶子节点）。
- 因为是组合，`[1, 2]` 和 `[2, 1]` 等价，为了避免重复，我们要加一个 `start` 参数，每次只向后选择，不往前看。

**答案代码（Python版）：**
```python
class Solution:
    def subsets(self, nums: List[int]) -> List[List[int]]:
        res = []
        path = []
        
        def backtrack(start):
            # 将当前路径加入结果
            res.append(path[:])
            
            # 遍历从 start 开始的选择列表
            for i in range(start, len(nums)):
                path.append(nums[i])      # 做选择
                backtrack(i + 1)          # 递归：因为不能重复用，下一层从 i+1 开始
                path.pop()                # 撤销选择：回溯
                
        backtrack(0)
        return res
```

### 组合总和（难度：中等）

**题目：**
给你一个无重复元素的整数数组 `candidates` 和一个目标整数 `target` ，找出 `candidates` 中可以使数字和为目标数 `target` 的所有不同组合。同一个数字可以无限制重复被选取。

**【模板解析与思路归纳】：**
- 与子集不同：元素可以**无限重复选取**。
- 处理技巧：递归进入下一层时，`start` 参数不需要加1，依然传当前元素的索引 `i` 即可（说明同一层还可以继续选它）。
- 剪枝技巧：如果累加的 `total > target`，直接返回，不再继续深搜。
- **为什么写 `current_sum + candidates[i]` 而不是 `current_sum += candidates[i]`**：数字是不可变类型，`current_sum + candidates[i]` 只是临时算出一个新值传进去，`current_sum` 本身不变，递归返回后自动"撤销"，不需要手动减回来。如果写成 `current_sum += candidates[i]` 再传，就必须在 `path.pop()` 后面再加一行 `current_sum -= candidates[i]` 才能恢复，容易漏写。

**答案代码（Python版）：**
```python
class Solution:
    def combinationSum(self, candidates: List[int], target: int) -> List[List[int]]:
        res = []
        path = []
        
        def backtrack(start, current_sum):
            if current_sum == target:
                res.append(path[:])
                return
            if current_sum > target:
                return
                
            for i in range(start, len(candidates)):
                path.append(candidates[i])
                # 可以重复使用这个数字，所以下次递归还是从 i 开始
                backtrack(i, current_sum + candidates[i])
                path.pop()
                
        backtrack(0, 0)
        return res
```

### 全排列（难度：中等）

**题目：**
给定一个不含重复数字的数组 `nums` ，返回其所有可能的全排列。

**【模板解析与思路归纳】：**
- 排列问题，`[1, 2]` 和 `[2, 1]` 是不同的，所以每一层都要**从 0 开始**看整个数组。
- 但是同一个数字不能重复使用，因此需要借助一个 `used` 数组（或 `if nums[i] in path:`，由于 `in` 是 O(N) 操作，推荐用 `used` 数组或 `set`）。

**答案代码（Python版）：**
```python
class Solution:
    def permute(self, nums: List[int]) -> List[List[int]]:
        res = []
        path = []
        used = [False] * len(nums)
        
        def backtrack():
            if len(path) == len(nums):
                res.append(path[:])
                return
                
            for i in range(len(nums)):
                if used[i]:
                    continue
                    
                path.append(nums[i])
                used[i] = True
                backtrack()
                # 撤销选择，恢复原始状态
                path.pop()
                used[i] = False
                
        backtrack()
        return res
```

### 电话号码的字母组合（难度：中等）

**题目：**
给定一个仅包含数字 2-9 的字符串，返回所有它能表示的字母组合（与电话按键相同）。

#### 解题思路

**用抽屉柜来理解**：输入 `"23"`，就是两层抽屉柜。第一层（数字2）有 a、b、c 三个球，第二层（数字3）有 d、e、f 三个球。你要做的是：从第一层取一个球，再从第二层取一个球，记录这个组合，然后放回去换一个继续取。

回溯的三步口诀：**取一个 → 往下走 → 放回来**

- 取一个：从当前数字对应的字母里选一个加入路径
- 往下走：处理下一个数字（递归）
- 放回来：撤销刚才的选择，换下一个字母试

**关键点**：这道题不需要 `used` 数组，因为每一层处理的是不同的数字，天然不会重复。`index` 参数告诉你现在处理到第几个数字了。

#### 代码实现

```python
class Solution:
    def letterCombinations(self, digits: str) -> List[str]:
        if not digits:
            return []
        
        phone_map = {
            '2': 'abc', '3': 'def', '4': 'ghi', '5': 'jkl',
            '6': 'mno', '7': 'pqrs', '8': 'tuv', '9': 'wxyz'
        }
        res = []
        path = []
        
        def backtrack(index):
            # 路径长度等于数字个数，说明每个数字都选了一个字母，记录结果
            if index == len(digits):
                res.append("".join(path))
                return
            
            # 当前数字对应的所有字母就是"选择列表"
            for ch in phone_map[digits[index]]:
                path.append(ch)       # 取一个
                backtrack(index + 1)  # 往下走（处理下一个数字）
                path.pop()            # 放回来（换下一个字母）
        
        backtrack(0)
        return res
```

#### 复杂度分析

- **时间复杂度**：O(4^n × n)，n 为数字个数，每个数字最多对应 4 个字母（7、9），最终每个组合长度为 n
- **空间复杂度**：O(n)，递归深度为 n

**和全排列的区别**：
- 全排列：每层从整个数组选，用 `used` 防止重复用同一个元素
- 这道题：每层从当前数字对应的字母选，`index` 自动推进到下一个数字，天然不重复

### 括号生成（难度：中等）

**题目：**
数字 `n` 代表生成括号的对数，请你设计一个函数，用于能够生成所有可能的并且有效的括号组合。

**【模板解析与思路归纳】：**
- 这题的特别之处在于，决策树的“选择列表”常态下只有两个：`(` 和 `)`。
- **合法性剪枝**极其经典：
  1. 左括号数量 `< n` 时，可以随意放左括号。
  2. 右括号数量必须**永远小于或者等于**当前的左括号数量，如果不满足这个条件，立刻剪枝。

**答案代码（Python版）：**
```python
class Solution:
    def generateParenthesis(self, n: int) -> List[str]:
        res = []
        
        def backtrack(path_str, left_count, right_count):
            if len(path_str) == 2 * n:
                res.append(path_str)
                return
                
            if left_count < n:
                backtrack(path_str + "(", left_count + 1, right_count)
                
            if right_count < left_count:
                backtrack(path_str + ")", left_count, right_count + 1)
                
        backtrack("", 0, 0)
        return res
```

### 分割回文串（难度：中等）

**题目：**
给你一个字符串 `s`，请你将 `s` 分割成一些子串，使每个子串都是回文串。返回 `s` 所有可能的分割方案。

**【模板解析与思路归纳】：**
- 如何映射成多叉树？可以理解为在一根线上的不同位置“切一刀”。
- "选择列表"：是从当前位置 `start` 开始，终点 `i` 不断往后移。如果切出来的子串是回文串，就算作一条有效路径，继续在切剩下的剩余部分继续往下递归切。

**答案代码（Python版）：**
```python
class Solution:
    def partition(self, s: str) -> List[List[str]]:
        res = []
        path = []
        
        def isPalindrome(sub: str) -> bool:
            return sub == sub[::-1]
            
        def backtrack(start):
            # 走到字符串尽头，说明找到了一种完全合法的分割方案
            if start == len(s):
                res.append(path[:])
                return
                
            for i in range(start, len(s)):
                # 如果 s[start:i+1] 是回文串，则切下这一刀
                substring = s[start:i + 1]
                if isPalindrome(substring):
                    path.append(substring)     # 做选择
                    backtrack(i + 1)           # 从下一个位置继续切
                    path.pop()                 # 撤销选择，准备换个地方切
                    
        backtrack(0)
        return res
```

### 单词搜索（难度：中等）

**题目：**
给定一个 m x n 二维字符网格 `board` 和一个字符串 `word`，如果 word 存在于网格中返回 true，否则返回 false。单词必须按字母顺序通过水平或垂直相邻的单元格构成，同一个单元格不允许重复使用。

#### 解题思路

**用迷宫来理解**：你站在网格里，手里拿着一张单词纸条，要在地图上找到一条路，路上经过的格子按顺序拼出这个单词。每走一步，就把当前格子"踩脏"（标记已访问），走不通就退回来把格子"擦干净"，换个方向继续走。

**核心步骤**：
1. 遍历整个网格，找到和 `word[0]` 匹配的格子作为起点
2. 从起点出发，向上下左右四个方向尝试
3. 每次进入一个格子，先把它标记为"已访问"（防止重复走）
4. 递归检查下一个字符，走不通就退回来，把标记清除

**防重复的技巧**：临时把 `board[r][c]` 改成 `'#'`（一个不可能出现的字符），递归返回后再改回来。这样不需要额外的 `visited` 数组。

#### 代码实现

```python
class Solution:
    def exist(self, board: List[List[str]], word: str) -> bool:
        rows, cols = len(board), len(board[0])
        
        def backtrack(r, c, index):
            # 找完了所有字符，成功
            if index == len(word):
                return True
            # 越界或字符不匹配，失败
            if r < 0 or r >= rows or c < 0 or c >= cols or board[r][c] != word[index]:
                return False
            
            # 标记当前格子已访问（踩脏）
            temp = board[r][c]
            board[r][c] = '#'
            
            # 向四个方向尝试下一个字符
            found = (backtrack(r+1, c, index+1) or
                     backtrack(r-1, c, index+1) or
                     backtrack(r, c+1, index+1) or
                     backtrack(r, c-1, index+1))
            
            # 恢复格子（擦干净）
            board[r][c] = temp
            return found
        
        # 遍历所有格子，找到第一个字符匹配的就开始搜索
        for r in range(rows):
            for c in range(cols):
                if backtrack(r, c, 0):
                    return True
        return False
```

#### 复杂度分析

- **时间复杂度**：O(m × n × 4^L)，L 为 word 长度，每个起点最多向 4 个方向递归 L 层
- **空间复杂度**：O(L)，递归深度为 word 长度

**和普通回溯的区别**：
- 普通回溯用 `path.append/pop` 撤销选择
- 网格回溯用 `board[r][c] = '#'` 和 `board[r][c] = temp` 撤销，原理完全一样

## 四、终极棋盘模型
*回溯巅峰：N皇后问题与数独。在二维平面上的暴力尝试。*

### N 皇后（难度：困难）

**题目：**
将 `n` 个皇后放置在 `n x n` 的棋盘上，并且使皇后彼此之间不能相互攻击。（同一行、同一列、同一斜线均不能出现两个皇后）。返回所有不同的解决方案。

**【模板解析与思路归纳】：**
- 以“行”为阶段进行遍历，即递归函数推进的是 `row`。
- 每来到一行，它的“选择列表”是该行的所有 `col`。我们要过滤掉那些会受到攻击的位置。
- 剪枝技巧：维护三个 `set`，分别记录被占用的列、正斜线 `(row - col 恒定)`、副斜线 `(row + col 恒定)`。

**答案代码（Python版）：**
```python
class Solution:
    def solveNQueens(self, n: int) -> List[List[str]]:
        res = []
        board = [['.'] * n for _ in range(n)]
        
        cols = set()      # 记录禁用的列
        diag1 = set()     # 正斜线: row - col的值相同
        diag2 = set()     # 反斜线: row + col的值相同
        
        def backtrack(row):
            if row == n:
                # 记录答案，把每一行的列表转成字符串
                res.append(["".join(r) for r in board])
                return
                
            for col in range(n):
                if col in cols or (row - col) in diag1 or (row + col) in diag2:
                    continue  # 此位置受到攻击，不能放
                    
                # 做选择，放上皇后，并标记攻击范围
                board[row][col] = 'Q'
                cols.add(col)
                diag1.add(row - col)
                diag2.add(row + col)
                
                # 递归放入下一行
                backtrack(row + 1)
                
                # 撤销选择，将棋盘还原
                board[row][col] = '.'
                cols.remove(col)
                diag1.remove(row - col)
                diag2.remove(row + col)
                
        backtrack(0)
        return res
```
