# 算法题解通用模板

每种算法通常有其固定的思考框架和代码模板。在整理具体题目时，建议在每个分类文件的开头，或者题目的解答中，融入以下结构，以便不仅知道“怎么写代码”，更知道“为什么这么写”。

## 题目解析框架

### 1. 核心思想（Why）
- **适用场景**：这个算法通常用来解决什么类型的问题？（例如：二分查找适用于“有序数组中寻找特定值”或“存在单调性的查找问题”）
- **算法本质**：用一句话概括算法的核心机制。（例如：二分查找是通过不断缩小一半的搜索范围来逼近目标值）
- **时间/空间复杂度**：这种模板的预期复杂度是多少？为什么要用它（时空带来的收益）？

### 2. 通用代码模板（How）
提供该类算法的标准代码骨架。即使题目发生变体，往往只需要在这个模板上做增删。

\`\`\`python
# [语言定义] 模板代码
\`\`\`

### 3. 常见题型与变体细节
- **基础题型**：最直接应用模板的题目。
- **常见变体**：
  - 细节 1：如边界条件的处理（如二分查找中的 `left <= right` 还是 `left < right`）
  - 细节 2：如变体问题（如寻找左侧边界、寻找右侧边界）

---

## 示例：二分查找类模板

**适用场景**：有序数组、部分有序数组（如旋转数组）、或者可以抽象为单调性的查找问题。

**通用代码模板**：
\`\`\`python
def binary_search(nums, target):
    left, right = 0, len(nums) - 1
    
    # 注意点1：这里的 = 决定了退出循环时 left == right + 1 
    # 也就是区间为空 [right+1, right]
    while left <= right:
        # 注意点2：防止溢出（如果是C++/Java等语言），Python中无所谓
        mid = left + (right - left) // 2 
        
        if nums[mid] == target:
            # 找到目标时的处理。如果是找边界，这里不直接return，而是收缩范围
            return mid 
        elif nums[mid] < target:
            # 目标在右半边
            left = mid + 1
        else:
            # 目标在左半边
            right = mid - 1
            
    return -1
\`\`\`

**细节与变体关注点**：
1. **边界条件 `left <= right` or `left < right`**：一般用 `<=` 寻找精确值；用 `<` 寻找边界。
2. **中点计算**：使用 `left + (right - left) // 2` 避免大数溢出。
3. **寻找左右侧边界**：当 `nums[mid] == target` 时，不立即返回，而是执行 `left = mid + 1` (找右边界) 或 `right = mid - 1` (找左边界)。

通过这种结构整理，可以把每类题目从“孤立的题目”变成“一个模板的实例”，这样刷题效率会成倍提升。你希望我们从哪个分类（比如二分查找、动态规划、回溯算法等）开始，把现有的题目改成这种带有通用代码和解析结构的格式？
