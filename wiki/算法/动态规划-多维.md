# 动态规划 · 多维 (网格 / 区间 / 双串)

> [[算法_MOC|返回算法导航]] ｜ 主篇见 [[动态规划]]

> [!info] 本文速览
> 本文从主篇 [[动态规划]] 拆出，专讲二维及以上的 DP 模型，共三类：
> - **网格 DP**（`dp[i][j]` = 走到 `(i,j)` 的最优）：不同路径、最小路径和。核心是只能从上/左转移，可压缩成一维滚动数组。
> - **区间 DP**（`dp[i][j]` = 区间 `[i,j]` 的性质）：最长回文子串。核心是**按子串长度从小到大**外层遍历。
> - **双字符串 DP**（`dp[i][j]` = 串1前 i 个 vs 串2前 j 个）：最长公共子序列、编辑距离。核心是 `+1` 留空字符基石。

## 一、网格 DP

### 不同路径（难度：中等）

一个机器人位于一个 `m x n` 网格的左上角 （起始点在下图中标记为 “Start” ）。机器人每次只能向下或者向右移动一步。机器人试图达到网格的右下角（在下图中标记为 “Finish” ）。问总共有多少条不同的路径？

#### 解题思路

这是最标准的基础二维网格动态规划题，也是所有矩阵/迷宫类 DP 问题（如“最小路径和”、“带障碍物的不同路径”）的基础母题。由于机器人的移动方向被严格限制为“只能向下”或“只能向右”，这意味着**到达任何一个格子的路径来源，只可能从它上方的格子，或者左方的格子走过来**。这天然就构成了动态规划推导前置状态的前提。

- **状态定义**：`dp[i][j]` 表示机器人走到第 `i` 行、第 `j` 列的格子，总共有多少条不同的路径。
- **状态转移方程**：因为只能从上方或左方过来，因此走到 `(i, j)` 的所有可能路径，必定等于走到 `(i-1, j)`（上方格子）的路径数加上走到 `(i, j-1)`（左方格子）的路径数之和。
  即：`dp[i][j] = dp[i-1][j] + dp[i][j-1]`。
- **初始化**：
  - 起始位置 `dp[0][0] = 1`。
  - 对于第 0 行的所有格子 `dp[0][j]`，由于机器人不能向上走或者从上面走下来，只能“硬着头皮从左边一路往右走”，所以这一路过去所有格子都只有 1 条路径。
  - 对于第 0 列的所有格子 `dp[i][0]`，同理因为不能向左走或从左边走过来，只能一路往下走，一直到底全是 1 条路径。
- **返回值要点**：最终返回右下角的计算结果，即下标为 `dp[m-1][n-1]` 的位置。

这道题在熟练掌握基本的二维打表后，最精华的反而是其**空间优化降维打击**。在每次计算第 `i` 行的数据时，由于用到的信息只来自于“正上方”（即上一行同一个下标处的值）和“左方”（刚刚刷新出的左小一格的值），我们完全可以把二维地图压缩成一维。

#### 代码实现（标准二维 DP 数组）

```python
class Solution:
    def uniquePaths(self, m: int, n: int) -> int:
        # 定义一个 m 行 n 列的二维数组，初始值全为 1（这一步巧妙地把第 0 行和第 0 列的打底情况一并解决了！）
        dp = [[1] * n for _ in range(m)]
        
        # 因为最上边一行和最左边一列都已经是 1 了，我们直接从内腹地带的 (1,1) 开始递推
        for i in range(1, m):
            for j in range(1, n):
                # 当前格子的到达方式 = 从上面下来的 + 从左面过来的
                dp[i][j] = dp[i-1][j] + dp[i][j-1]
                
        return dp[m-1][n-1]
```

#### 代码实现（空间优化：一维滚动数组）

在降维到一维数组时，我们反复覆盖、反复使用长度仅为 `n` 的一行数组：
- 等号右半边的 `dp[j]` 还是老旧的历史记录（即**上方这一列格子的路径数**）。
- 等式右半边的 `dp[j-1]` 因为在我们从左往右的 for 循环里已经被刚才刷新过了，所以它天然代表了**左边这一个格子的路径数**。

```python
class Solution:
    def uniquePaths(self, m: int, n: int) -> int:
        # 只需要一个长度为 n 的一维数组，装满第 0 行的路径步数（全是1）
        dp = [1] * n
        
        for i in range(1, m):
            for j in range(1, n):
                # 更新当前列的值：旧的自己(相当于上面) + 左边刚更新过的兄弟(相当于左面)
                dp[j] = dp[j] + dp[j-1]
                
        return dp[-1]
```

#### 复杂度分析
- **时间复杂度**：$O(m \times n)$，两层嵌套循环，我们需要遍历并计算每一个格子的解。
- **空间复杂度**：二维推演版本为 $O(m \times n)$；但经过一维滚动数组空间优化后，复杂度可以骤降至 $O(n)$。

### 最小路径和（难度：中等）

给定一个包含非负整数的 `m x n` 网格 `grid` ，请找出一条从左上角到右下角的路径，使得路径上的数字总和为最小。
说明：每次只能向下或者向右移动一步。

#### 解题思路

这是《不同路径》的亲生兄弟题。在《不同路径》中，我们是求“走到这里有多少**种**可能”，也就是把上面的路径数和左边的路径数相加；而在本题中，我们是求“走到这里要交最少多少**过路费**”。因为依然被死死限制了“只能向右向下”，所以到达任意一个格子 `(i, j)`，它唯一可能的前置出发点还是它的相对上方或左方。

- **状态定义**：`dp[i][j]` 表示从左上角出发，走到 `(i, j)` 这个格子所能消耗的**最小路径和**。
- **状态转移方程**：我们要在上方和左方这两条可走的历史路线中，挑选一条积累目前为止过路费**最小**的来走，然后再交上当底这块格子的过路费。
  即：`dp[i][j] = min(dp[i-1][j], dp[i][j-1]) + grid[i][j]`。
- **初始化（寻找最前置的基石）**：
  - 起始位置 `(0, 0)` 的代价就是它起点的数字本身：`grid[0][0]`。
  - **第一行**：走在第一行的机器人无路可选，它上面根本没格子，只能一路从左边挨个走过来。所以它的代价是一直往右做加法累加：`dp[0][j] = dp[0][j-1] + grid[0][j]`。
  - **第一列**：同理，走在第一列的机器人也无路可选，左边没格子，只能一路往下砸。所以它的代价是一路往下累加：`dp[i][0] = dp[i-1][0] + grid[i][0]`。
- **空间优化（降维甚至榨干）**：和《不同路径》可以压缩到一维数组一样，本题因为我们每次只需要看“左面”和“上面”，原数组 `grid` 在被读过后就彻底失去了原有的计算价值。因此，我们为了极简最优化，甚至可以直接把网格 `grid` 当作原地的 `dp` 数组来使用，每走一步就把原格子里的数字直接覆盖成抵达它的最小路径和，从而榨干空间复杂度将其降为 $O(1)$！

#### 代码实现（标准二维 DP 数组）

作为初学者建立动态规划直觉的第一步，我们必须先写出严格使用辅助矩阵 `dp` 的常规解法，以体会状态演替流转的完整物理过程。

```python
class Solution:
    def minPathSum(self, grid: List[List[int]]) -> int:
        if not grid or not grid[0]:
            return 0
            
        m, n = len(grid), len(grid[0])
        
        # 1. 明确定义并分配 dp 数组空间
        dp = [[0] * n for _ in range(m)]
        
        # 2. 初始化基点（最左上角）
        dp[0][0] = grid[0][0]
        
        # 单独处理第一行（无脑向右累加过路费）
        for j in range(1, n):
            dp[0][j] = dp[0][j-1] + grid[0][j]
            
        # 单独处理第一列（无脑向下累加过路费）
        for i in range(1, m):
            dp[i][0] = dp[i-1][0] + grid[i][0]
            
        # 3. 状态转移：开始处理剩余被包围在核心圈的其他格子
        for i in range(1, m):
            for j in range(1, n):
                # 上面过来的代价和左边过来的代价，挑个最便宜的方案走，再加上站在这里必须交的费
                dp[i][j] = min(dp[i-1][j], dp[i][j-1]) + grid[i][j]
                
        return dp[m-1][n-1]
```

#### 代码实现（空间榨干优化：原地覆盖法 $O(1)$）

当你完全熟练掌握并理解了上面的转移关系后，再去使用这种原地黑科技。

```python
class Solution:
    def minPathSum(self, grid: List[List[int]]) -> int:
        if not grid or not grid[0]:
            return 0
            
        m, n = len(grid), len(grid[0])
        
        # 抛弃 dp 数组，直接霸占原有的 grid 画板
        for j in range(1, n):
            grid[0][j] = grid[0][j-1] + grid[0][j]
            
        for i in range(1, m):
            grid[i][0] = grid[i-1][0] + grid[i][0]
            
        for i in range(1, m):
            for j in range(1, n):
                grid[i][j] = min(grid[i-1][j], grid[i][j-1]) + grid[i][j]
                
        return grid[m-1][n-1]
```

#### 复杂度分析
- **时间复杂度**：$O(m \times n)$。我们要老老实实遍历网格中每一个元素算一边。
- **空间复杂度**：
  - 标准二维 DP 方法为 $O(m \times n)$，因为开辟了等同于地图大小的独立 `dp` 矩阵。
  - 由于我们利用了原本输入的 `grid` 进行原地覆盖，因此极致优化版不需要产生任何多余的空间分配，空间复杂度降为 $O(1)$。

## 二、区间 DP (字符串二维模型)

### 最长回文子串（难度：中等）

给你一个字符串 `s`，找到 `s` 中最长的回文子串。

#### 解题思路

本题除了最符合人类直觉的“中心扩散法”外，利用判定性质传递的**二维区间 DP** 也是极为正统的模板题。掌握它能为你解答一切回文字符串动态规划问题（例如“回文子序列数量”、“分割回文串”）打下坚实的框架基础。

- **状态定义（区间锁死）**：对于字符串子串问题，最标准的状态设定是利用明确的左右两个坐标将区间框死。
  我们定义 `dp[i][j]` 表示：字符串 `s` 从索引 `i` 到 `j` 的这一段闭区间子串，是否是一个回文串（布尔值 `True` / `False`）。
- **状态转移方程（由内向外推导）**：
  判断一个由 `...xyz...` 构成的大区间是不是回文串，本质上只取决于两点：
  1. 它的左右两个门神（头尾字符 `s[i]` 和 `s[j]`）是不是长得一样；
  2. 它“掐头去尾”后，肚子里面包着的那个小一圈的核心子串 `dp[i+1][j-1]` 是不是回文串。
  因此，如果 `s[i] == s[j]`，则 `dp[i][j] = dp[i+1][j-1]`。如果不等，那直接死刑，`dp[i][j] = False`。
- **触发条件与边界拦截**：
  我们不能永远指望剥洋葱，当洋葱剥到芯子时，有以下基础基石：
  - **长度为 1 的子串**：自己跟自己肯定是回文。即 `dp[i][i] = True`。
  - **长度为 2 或 3 的子串，且首尾相同**：比如 `"aa"` 或 `"aba"`。当你掐头去尾时，里面要么没了（长度2），要么只剩一个字符（长度3）。这时候根本不需要再查前置小洋葱的状态，只要首尾相同，它**立刻、当场、绝对**就是回文！
    所以当 `s[i] == s[j]` 时，如果 `j - i <= 2`（代表这两个指针收缩到了临界距离），直接设为 `True`。
- **【核心易错点：神奇的遍历顺序】**
  这是本题乃至所有区间 DP 最具杀伤力的地方！如果你还是像往常一样，外层 `for i 从 0 到 n`，内层 `for j 从 0 到 n`，你的代码会彻头彻尾地算出错误答案的！
  - **原因拆解**：注意你的转移依赖 `dp[i+1][j-1]`！这意味着你在算大区间 `(i, j)` 时，你需要先知道**左下方的记录、比它晚一拍出发、早一拍结束的那个短区间的历史数据**！如果用传统的正向双重循环，由于 `i+1` 还没发生，那个较短芯子的历史记录根本就还是空白没算呢！！
  - **破局办法（从中心长度往外长）**：既然大的依赖小的，我们就**把外层循环定为“子串的长度 L”！**只要我们保证是先挨个结算长度为 2 的段、再结算长度为 3 的段... 那么等我们计算长度为 10 的洋葱外皮时，它要求读取的那个长度为 8 的内芯数据肯定就在之前的回合中早早被存好了！这就是区间 DP 最常用的**“长度扩展型遍历”**法则。

#### 代码实现（标准二维 DP）

```python
class Solution:
    def longestPalindrome(self, s: str) -> str:
        n = len(s)
        # 单个或空字符直接返回自己
        if n < 2:
            return s
            
        # dp[i][j] 表示 s[i:j] 是否是回文串
        dp = [[False] * n for _ in range(n)]
        
        # 记录史上遇到的最长记录
        max_len = 1
        start = 0
        
        # 基地石：所有长度为 1 的子串都是回文
        for i in range(n):
            dp[i][i] = True
            
        # 外循环：从长度 L 横向扩展洋葱的尺寸（从 L=2 到 L=n）
        for L in range(2, n + 1):
            # 内循环：锚定洋葱左边的门神 i 的位置
            for i in range(n):
                # 既然知道了左门神 i 和手头的长度 L，右门神 j 的坐标自然固定了
                j = i + L - 1
                
                # 如果这个尺寸的大洋葱右边界已经撑破了整个字符串，那以这个 i 发起的大探索可以直接弃了
                if j >= n:
                    break
                    
                # 验证逻辑：门神对不上就绝对不是
                if s[i] != s[j]:
                    dp[i][j] = False
                else:
                    # 门神对上了！如果是 "aa" (L=2) 或者 "aba" (L=3)，当场验证成功
                    if j - i <= 2:
                        dp[i][j] = True
                    else:
                        # 对于更大的洋葱，完全查阅刚刚上一寸剥剩的内层芯子历史记录
                        dp[i][j] = dp[i + 1][j - 1]
                        
                # 凡是验证成功的结论，只要打破了历史最大长度，立刻上位
                if dp[i][j] and L > max_len:
                    max_len = L
                    start = i
                    
        # 利用最高记录做最后的切片返回
        return s[start:start + max_len]
```

> **💡 多解法联动指路：**
> 本题除了上面正统的“动态规划模板法”外，在面试和真实应用场景下十分常考其**不耗费任何额外辅助数组空间**的解法：基于双指针的“中心扩展法”（可达成极致的空间 $O(1)$ 压缩）。
> 采用该方法时，通过对每个字符以及每两个字符中间的空隙开动双指针往两侧波纹散开对比即可。由于该思路属于“双指针 / 字符串遍历”体系，详细的解析与完整打磨好的代码请移步查阅 [wiki/算法/字符串.md](wiki/算法/字符串.md#L55) 里的对应收录。

#### 复杂度分析
- **时间复杂度**：$O(N^2)$，嵌套了两层对于长度以及起点遍历枚举的双重循环，一共进行了 $\frac{N(N-1)}{2}$ 次状态推导。
- **空间复杂度**：$O(N^2)$，因为严格维护了一个 $N \times N$ 的打卡二维布尔数组 `dp[i][j]`。

## 三、双字符串二维 DP (LCS 模型)

### 最长公共子序列（难度：中等）

给定两个字符串 `text1` 和 `text2`，返回这两个字符串的最长**公共子序列**的长度。如果不存在公共子序列，返回 `0`。

一个字符串的**子序列**是指这样一个新的字符串：它是由原字符串在不改变字符的相对顺序的情况下删除某些字符（也可以不删除任何字符）后组成的新字符串。
例如，`"ace"` 是 `"abcde"` 的子序列，但 `"aec"` 不是 `"abcde"` 的子序列。
两个字符串的**公共子序列**是这两个字符串所共同拥有的子序列。

#### 解题思路

这是面试中考核频率极高的双字符对比模板题（同时也是衍生出更高难度的“编辑距离”一类应用的前置母题）。
对于两个字符串的对比找规律，标准的动态规划做法是画一个二维矩阵框，将第一个字符串映射到行，第二个字符串映射到列，用类似查户口的方式一点点扩散。

- **状态定义**：`dp[i][j]` 表示 `text1` 的前 `i` 个字符和 `text2` 的前 `j` 个字符的**最长公共子序列的长度**。注意因为涉及到个数统计，这就是我们在主篇第一节中讲过的“**必须多留出一格 `+1` 的位置给 `dp[0]` 做基石底层**”的经典代表。
- **状态转移方程**：
  我们拿出目前手电筒照着的 `text1` 的第 `i` 个字符（代码中是 `text1[i-1]`）和 `text2` 的第 `j` 个字符（即 `text2[j-1]`）：
  1. **如果命中（字符相等）**：大声欢呼！说明这两个字符可以双双牵手，直接成功并入到目前发现的子序列队伍末尾。这多出来的 1 个长度，必须毫无争议地加在“**它俩都没有上场前**”的那个独立历史时空（也就是各退一步的 `dp[i-1][j-1]`）身上！
     即：`dp[i][j] = dp[i-1][j-1] + 1`
  2. **如果错过（字符不相等）**：说明此时手电筒照着的这两个字符，**绝对不可能**携手结对出现在最长公共子序列的末尾。那路怎么走？我们要么大度地让 `text1` 退一步（试试不带这个字符去对比行不行），要么让 `text2` 退一步，去之前那两个被斩断的平行时空里，挑一个收获最大的结果继承给现在的时空。
     即：`dp[i][j] = max(dp[i-1][j], dp[i][j-1])`
- **初始化**：
  只要 `text1` 或 `text2` 中有一个长度为 0（即碰上了第 0 行或者第 0 列），那它俩的公共子集长度绝对是 0 没跑了。因为我们在构建 `(m+1) x (n+1)` 的矩阵时直接全填充为 `0`，这部分基石天生就初始化好了。

#### 代码实现（标准二维 DP 矩阵）

```python
class Solution:
    def longestCommonSubsequence(self, text1: str, text2: str) -> int:
        m, n = len(text1), len(text2)
        
        # 1. 明确定义 dp 数组，为了给"前面0个字符"的基位留空，双向加一扩展矩阵
        dp = [[0] * (n + 1) for _ in range(m + 1)]
        
        # 2. 状态转移计算
        for i in range(1, m + 1):
            for j in range(1, n + 1):
                # 检查这两位兄弟对不对得上眼（注意 dp 中的索引 i 对应字符串的下标即 i-1）
                if text1[i - 1] == text2[j - 1]:
                    # 相等，各退一格的辉煌历史再加 1
                    dp[i][j] = dp[i - 1][j - 1] + 1
                else:
                    # 没缘分，要么上面的兄弟退步，要么左边兄弟退步，继承里面记录最强的那个
                    dp[i][j] = max(dp[i - 1][j], dp[i][j - 1])
                    
        # 两人较量到最后一刻的结局，自然躺在了矩阵的最右下角
        return dp[m][n]
```

#### 复杂度分析
- **时间复杂度**：$O(M \times N)$，其中 $M$ 和 $N$ 是 `text1` 和 `text2` 的长度。因为我们稳扎稳打算满了一整个二维表。
- **空间复杂度**：$O(M \times N)$，开辟了完整的二维辅助矩阵用于打表。*(注：与《不同路径》一致，其实也能使用上下两行滚动的一维数组将空间彻底压缩至 $O(\min(M, N))$，但在面试实战与理解中，这种原汁原味的二维修建架构依然是最具展现力的第一手底牌。)*

### 编辑距离（难度：困难）

给你两个单词 `word1` 和 `word2`， 请返回将 `word1` 转换成 `word2` 所使用的最少操作数。
你可以对一个单词进行如下三种操作：
- 插入一个字符
- 删除一个字符
- 替换一个字符

#### 解题思路

这是与上面《最长公共子序列》同宗同源，但更为宏大和经典的“双字符串二维 DP”究极母题！只要搞懂了这道题的增删改状态转移，所有的字符串模糊匹配、拼写检查问题都会不攻自破。

- **状态定义**：`dp[i][j]` 表示将 `word1` 的前 `i` 个字符转换成 `word2` 的前 `j` 个字符所需要的最少操作数。同样，因为包含空字符串的状态，数组大小需要是 `(m+1) x (n+1)`。
- **状态转移方程**：
  我们同样审视 `word1[i-1]` 和 `word2[j-1]` 这两位当前考察的末尾字符：
  1. **如果相等（`word1[i-1] == word2[j-1]`）**：
     谢天谢地！既然这俩最后一个字母本来就一样，那咱们在这个位置根本**不需要做任何操作**。把这个免费的相同字母剥离，需要的总操作数其实就完全等于它俩各自少一个字母时候的操作数。
     即：`dp[i][j] = dp[i-1][j-1]`
  2. **如果不等（需要搞操作了！）**：
     不相等也没关系，题目给了我们三种魔法。我们试试哪种魔法代价最小（基础代价是 1，在这个基础上加上各自平行时空的历史代价）：
     - **替换操作**：既然最后两个不一样，那直接一发替换魔法，强行把 `word1[i-1]` 变成 `word2[j-1]`！那么前面的字符只要各自匹配就行了。历史代价是 `dp[i-1][j-1]`。总代价：`dp[i-1][j-1] + 1`
     - **删除操作**：假设我把 `word1` 当前这个烂字母 `word1[i-1]` 给强行删了！既然删了，那 `word1` 的前 `i-1` 个字母就得负责去拼凑出整个 `word2` 的前 `j` 个字母。历史代价是 `dp[i-1][j]`。总代价：`dp[i-1][j] + 1`
     - **插入操作**：假设我在 `word1` 后面白嫖插入一个和 `word2[j-1]` 一模一样的字母，这就把 `word2` 最后一个搞定了。那么 `word1` 原本的前 `i` 个字母就得去负责搞定 `word2` 剩下的前 `j-1` 个字母。历史代价是 `dp[i][j-1]`。总代价：`dp[i][j-1] + 1`
     最终，在这三条平行世界里挑个代价最小的：
     即：`dp[i][j] = min(dp[i-1][j-1], dp[i-1][j], dp[i][j-1]) + 1`
- **初始化（极其关键的三面墙）**：
  - `dp[0][0] = 0`：空变空，不用操作。
  - `dp[i][0] = i`：当你想把长度为 `i` 的字符串变成一个空字符串（`j=0`），唯一的做法就是**一路删除**到底，操作数为 `i`。
  - `dp[0][j] = j`：当你想把空字符串变成长度为 `j` 的字符串，唯一的做法就是**疯狂插入** `j` 次，操作数为 `j`。

#### 代码实现（标准二维 DP 矩阵）

```python
class Solution:
    def minDistance(self, word1: str, word2: str) -> int:
        m, n = len(word1), len(word2)
        
        # 1. 明确定义 dp 数组，为了给空字符基石留出位置，扩大一维
        dp = [[0] * (n + 1) for _ in range(m + 1)]
        
        # 2. 疯狂建立叹息之墙（把 word1 全删完，或者往空 word1 里全插满）
        for i in range(1, m + 1):
            dp[i][0] = i
        for j in range(1, n + 1):
            dp[0][j] = j
            
        # 3. 开始状态转移，填满大坑
        for i in range(1, m + 1):
            for j in range(1, n + 1):
                if word1[i - 1] == word2[j - 1]:
                    # 相等，不需要任何额外操作，继承砍掉这俩字符的历史大盘
                    dp[i][j] = dp[i - 1][j - 1]
                else:
                    # 不等，在替换(左上)、删除(上)、插入(左)这三条路里选个花钱最少的，再交上本次的1块钱操作费
                    dp[i][j] = min(dp[i - 1][j - 1], dp[i - 1][j], dp[i][j - 1]) + 1
                    
        return dp[m][n]
```

#### 复杂度分析
- **时间复杂度**：$O(m \times n)$，我们需要填满整个二维表格的数据。
- **空间复杂度**：$O(m \times n)$，同样建立了一个略大于原始长度的整张状态矩阵（包含边界初始化）。
