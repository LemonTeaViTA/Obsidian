## 一、设计题核心思想

面试中最常考的系统设计，并非让你写几万行工程代码，而是设计一个精巧的底层组件。比如 **LRU 缓存** 或者是 **LFU 缓存**，又或者是 **支持 O(1) 插入删除的随机集合**。

核心套路往往是：**哈希表 + 其他链式结构**。
- 哈希表提供 O(1) 的查询能力。
- 链表（或双向链表）提供 O(1) 的插入、删除及**顺序维护**能力。

## 二、经典面试神器

### LRU 缓存（难度：中等）

**题目：**
请你设计并实现一个满足 LRU (最近最少使用) 缓存约束的数据结构。
- `LRUCache(int capacity)` 以正整数作为容量初始化 LRU 缓存。
- `get(int key)` 如果 key 存在，返回其值，否则返回 -1。
- `put(int key, int value)` 判断 key 存在则更新，不存在则插入。若插入导致容量超载，淘汰最久未使用的关键字。
要求 O(1) 时间复杂度。

**【模板解析与思路归纳】：**
- 在 Python 中除了手写 **哈希表 + 双向链表**，还可以非常优雅地使用内置神器：`collections.OrderedDict`。
- 它既有 O(1) 查询键值的作用，又在内部维护了键的插入和淘汰顺序！它的 `move_to_end(key)` 和 `popitem(last=False)` 天然就是给 LRU 准备的。
- 如果面试官要求必须手撕底层，那就自己建一个 `Node` 类带 `prev` 和 `next`，然后在 `LRUCache` 里放一个头节点和尾节点的哨兵（dummy head/tail）。

**答案代码（Python版 - 官方外挂作弊法）：**
```python
import collections

class LRUCache:
    def __init__(self, capacity: int):
        self.capacity = capacity
        # 内部使用双向链表维护顺序的字典
        self.cache = collections.OrderedDict()

    def get(self, key: int) -> int:
        if key not in self.cache:
            return -1
        # 只要被查询了，就是近期活跃了，把它挪到链表的尾部（表示最新）
        self.cache.move_to_end(key)
        return self.cache[key]

    def put(self, key: int, value: int) -> None:
        if key in self.cache:
            # 如果原本就在，更新它的值并把它挪到末尾设为活跃
            self.cache[key] = value
            self.cache.move_to_end(key)
        else:
            self.cache[key] = value
            # 如果装不下了，弹出最久未访问的（位于 OrderedDict 头部的元素）
            if len(self.cache) > self.capacity:
                self.cache.popitem(last=False)
```

**答案代码（Python版 - 面试手撕双向链表法）：**
```python
class DLinkedNode:
    def __init__(self, key=0, value=0):
        self.key = key
        self.value = value
        self.prev = None
        self.next = None

class LRUCache:
    def __init__(self, capacity: int):
        self.capacity = capacity
        self.cache = {}
        # 建立两个虚节点，夹着中间的真实节点，免掉繁琐的空指针判断
        self.head = DLinkedNode()
        self.tail = DLinkedNode()
        self.head.next = self.tail
        self.tail.prev = self.head

    # --- 辅助方法：抽出某个节点 ---
    def remove_node(self, node):
        node.prev.next = node.next
        node.next.prev = node.prev
        
    # --- 辅助方法：将节点插到头节点后面（代表最新鲜） ---
    def add_node_to_head(self, node):
        node.prev = self.head
        node.next = self.head.next
        self.head.next.prev = node
        self.head.next = node
        
    # --- 辅助方法：移动老朋友到最新的地方 ---
    def move_to_head(self, node):
        self.remove_node(node)
        self.add_node_to_head(node)

    def get(self, key: int) -> int:
        if key not in self.cache:
            return -1
        node = self.cache[key]
        self.move_to_head(node)
        return node.value

    def put(self, key: int, value: int) -> None:
        if key in self.cache:
            node = self.cache[key]
            node.value = value
            self.move_to_head(node)
        else:
            node = DLinkedNode(key, value)
            self.cache[key] = node
            self.add_node_to_head(node)
            if len(self.cache) > self.capacity:
                # 满载了剥夺尾部哨兵的前一个（最旧）
                tail_node = self.tail.prev
                self.remove_node(tail_node)
                del self.cache[tail_node.key]
```

### 实现 Trie（前缀树）（难度：中等）

**题目：**
实现一个 Trie（前缀树），包含 `insert`、`search` 和 `startsWith` 三个操作。
- `Trie()` 初始化前缀树对象。
- `void insert(String word)` 向前缀树中插入字符串 word。
- `boolean search(String word)` 如果字符串 word 在前缀树中，返回 true；否则返回 false。
- `boolean startsWith(String prefix)` 如果之前已经插入的字符串 word 的前缀之一为 prefix，返回 true；否则返回 false。

#### 解题思路

前缀树本质上是一棵 **26 叉树**（假设只有小写字母）。你可以把它想象成一本字典的目录系统：

- 每个节点不存完整单词，而是存一个字符（或者说，边代表字符）
- 从根节点到某个节点的路径，拼起来就是一个前缀
- 节点上打个 `is_end` 标记，表示"到这里为止是一个完整单词"

**物理隐喻**：想象一棵倒挂的树，根在上面。每条树枝上刻着一个字母。你从树根出发，沿着树枝往下走，走过的字母拼起来就是一个单词。如果某个分叉点挂了个小旗子（`is_end = True`），说明走到这里已经是一个完整的词了。

**核心操作**：
- `insert`：沿着 word 的每个字符往下走，没有路就开辟新节点，最后打上 `is_end` 标记
- `search`：沿着 word 走到底，检查最后一个节点是否有 `is_end` 标记
- `startsWith`：沿着 prefix 走，只要每一步都有路就返回 True（不需要 `is_end`）

#### 代码实现

```python
class TrieNode:
    def __init__(self):
        self.children = {}  # 字符 -> TrieNode
        self.is_end = False  # 标记是否为完整单词的结尾

class Trie:
    def __init__(self):
        self.root = TrieNode()

    def insert(self, word: str) -> None:
        node = self.root
        for ch in word:
            if ch not in node.children:
                node.children[ch] = TrieNode()
            node = node.children[ch]
        node.is_end = True  # 插旗：这里是一个完整单词

    def search(self, word: str) -> bool:
        node = self._find_node(word)
        # 必须走到底，且最后一个节点有"完整单词"标记
        return node is not None and node.is_end

    def startsWith(self, prefix: str) -> bool:
        # 只要能沿着 prefix 走完就行，不需要 is_end
        return self._find_node(prefix) is not None

    def _find_node(self, prefix: str):
        """沿着 prefix 的字符路径往下走，返回最后到达的节点"""
        node = self.root
        for ch in prefix:
            if ch not in node.children:
                return None
            node = node.children[ch]
        return node
```

#### 复杂度分析

- **时间复杂度**：
  - `insert`：O(m)，m 为 word 长度
  - `search`：O(m)
  - `startsWith`：O(m)，m 为 prefix 长度
- **空间复杂度**：O(T)，T 为所有插入字符串的字符总数（最坏情况下没有公共前缀）

**为什么不直接用 HashSet？**
- HashSet 的 `search` 是 O(1)，但 `startsWith` 需要遍历所有 key，是 O(n)
- Trie 的 `startsWith` 天然 O(m)，而且还能做自动补全（遍历某前缀下的所有子树）
- 面试中 Trie 考的就是这个"前缀共享"的结构优势

### 实现 Trie（前缀树）（难度：中等）

#### 解题思路

前缀树（Trie）本质上是一棵 **26 叉树**（假设只有小写字母）。你可以把它想象成一本字典的目录系统：

- 每个节点不存完整单词，只存一个字符（或者说，通往该节点的那条边代表一个字符）
- 从根节点到某个标记节点的路径，拼起来就是一个完整的单词
- `is_end` 标记告诉你"到这里为止是一个合法单词"，而不仅仅是某个单词的前缀

为什么不用哈希表？哈希表能 O(1) 查整个单词，但做不到高效的**前缀匹配**。Trie 天然支持 `startsWith`，沿着路径走到哪就匹配到哪。

核心操作都是"沿着字符一层层往下走"：
- **insert**：逐字符建路径，最后打 `is_end` 标记
- **search**：逐字符走路径，走完且 `is_end=True` 才算找到
- **startsWith**：逐字符走路径，走完就行，不管 `is_end`

#### 代码实现

```python
class TrieNode:
    def __init__(self):
        self.children = {}  # char -> TrieNode
        self.is_end = False  # 标记：到这里是否构成一个完整单词

class Trie:
    def __init__(self):
        self.root = TrieNode()

    def insert(self, word: str) -> None:
        node = self.root
        for ch in word:
            if ch not in node.children:
                node.children[ch] = TrieNode()
            node = node.children[ch]
        node.is_end = True  # 整个单词插完，打上终止标记

    def search(self, word: str) -> bool:
        node = self._search_prefix(word)
        # 必须走完整条路径，且终点有 is_end 标记
        return node is not None and node.is_end

    def startsWith(self, prefix: str) -> bool:
        # 只要路径存在就行，不要求 is_end
        return self._search_prefix(prefix) is not None

    def _search_prefix(self, prefix: str):
        """公共辅助方法：沿着前缀走，返回终点节点（走不通返回 None）"""
        node = self.root
        for ch in prefix:
            if ch not in node.children:
                return None
            node = node.children[ch]
        return node
```

#### 复杂度分析

- **时间复杂度**：insert / search / startsWith 均为 O(m)，m 为字符串长度。每个操作最多遍历字符串的每个字符一次。
- **空间复杂度**：最坏 O(n × m)，n 为插入的单词数，m 为平均长度。如果单词之间共享前缀多，实际空间远小于此（这正是 Trie 的优势）。
