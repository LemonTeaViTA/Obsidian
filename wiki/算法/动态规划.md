## 一、动态规划核心思想与打表套路

动态规划（DP）本质是一种**利用历史记录，避免重复计算**的优化思想。面试中百分之九十的 DP 题，只要能写出**状态转移方程**，代码就迎刃而解。

**核心做题骨架（DP五部曲）**：
1. **定义 dp 数组的物理含义**：`dp[i]` 到底代表什么？是“到第 i 阶楼梯的方法数”还是“以 nums[i] 结尾的最长子序列”？
    - **【易混辨析：DP 数组初始化长度是 `n` 还是 `n + 1`？】**
        - **定义为“索引位置”时，取 `n`**：当 `dp[i]` 需要与给定数组 `nums[i]` **严格对齐绑定属性**时（例如：“打家劫舍”考虑前 `i` 个房间，“最长递增子序列”强调以 `nums[i]` 结尾），原数组最后一个下标必定为 `n-1`，所以你只需要长度为 `n` 就可以完全覆盖，最终答案常常取决于所有求出的局部最大值（即 `max(dp)` 或 `dp[-1]`）。
        - **定义为“数量/容量/总量”时，必取 `n + 1`**：当遇到“经过前 `n` 个字符拆串”、“凑齐总金额 `amount`”、“爬到第 `n` 阶楼梯”这种衡量数字大小、算数量的题目时，**必须多留出一格 `+1` 的位置给 `dp[0]`**。因为动态规划需要一个合法的“基石”，如“凑齐金额 0”、“0个字符”去启动整个状态机。这样构建出来的数组，你想查容量为 `n` 的解，就刚好直接索引 `dp[n]`（否则如果是只建长为 `n` 的数组，是不存在下标为 `n` 可供提取的）。
2. **写出状态转移方程**：今天的状态由昨天哪些状态推导而来？（如 `dp[i] = dp[i-1] + dp[i-2]`）
3. **初始化基点**：起跑线在哪里？（如 `dp[0] = 0, dp[1] = 1`）
4. **确定遍历顺序**：是从左到右，还是从右到左？对于背包问题，内外层循环顺序极为讲究！
5. **举例打印 dp 数组验证**。

### 🎯 模式识别信号（看到这些特征要联想到 DP）

| 题目特征 / 关键词 | 优先考虑的 DP 模型 |
|-----------------|------------------|
| "**连续子数组**的最大和 / 最大积" | 以 `nums[i]` **结尾**的一维挂载 DP |
| "**不连续子序列**的最长长度" | 以 `nums[i]` 结尾 + 双重循环 `O(N²)` |
| "能否凑出目标和 / 目标金额" | **背包问题**（0-1 或完全） |
| "网格 / 矩阵 走到右下角的路径数 / 最小代价" | **二维网格 DP** |
| "两个字符串的最长公共 / 编辑距离" | **双字符串二维 DP** |
| "回文子串 / 区间内某种性质" | **区间 DP**（按长度枚举） |
| "选或不选" / "偷或不偷" | 经典**选择型 DP**（打家劫舍模板） |
| "恰好分成两半 / 凑出某个数" | **0-1 背包** |

### 🧬 DP 六大分类速查

| 类型 | 状态定义关键 | 代表题 |
|------|------------|--------|
| 一维序列 DP | `dp[i]` = 到第 i 个的最优 | 爬楼梯、打家劫舍 |
| 一维挂载 DP | `dp[i]` = 以 `nums[i]` **结尾** | LIS、最大子数组和、乘积最大 |
| 背包 DP | `dp[j]` = 容量 j 时的最优 | 分割等和、零钱兑换、完全平方数 |
| 网格 DP | `dp[i][j]` = 走到 `(i,j)` 的最优 | 不同路径、最小路径和 |
| 区间 DP | `dp[i][j]` = 区间 `[i,j]` 的性质 | 最长回文子串 |
| 双字符串 DP | `dp[i][j]` = 前 i 个 vs 前 j 个 | LCS、编辑距离 |

> [!tip] DP 和贪心怎么区分？
> 能用贪心的题一定能用 DP，但反过来不行。如果"局部最优不能推出全局最优"（如背包问题），就只能用 DP。贪心的典型信号是"每一步只有一个最优方向，不需要回头"。详见 [[贪心算法]]。

## 二、基础一维序列 DP

### 爬楼梯（难度：简单）

假设你正在爬楼梯。需要 `n` 阶你才能到达楼顶。每次你可以爬 1 或 2 个台阶。你有多少种不同的方法可以爬到楼顶呢？

#### 解题思路

- **状态定义**：`dp[i]` 表示爬到第 `i` 阶楼梯的方法数。
- **状态转移方程**：到达第 `i` 阶只能从 `i-1` 阶跨 1 步上来，或者从 `i-2` 阶跨 2 步上来。因此总方法数是这两种途径之和：`dp[i] = dp[i-1] + dp[i-2]`。
- **初始化**：`dp[0] = 1`（原地不动也算一种方法），`dp[1] = 1`。

作为动态规划的第一题，我们首先应当用标准的 DP 数组模板来严格定义每一维度的状态。掌握了完整的模板推导后，我们再考虑空间优化。由于只依赖前两项，可以用两个变量滚动前进。

#### 代码实现（标准一维 DP 数组）

```python
class Solution:
    def climbStairs(self, n: int) -> int:
        if n <= 1:
            return 1
        # 1. 明确定义 dp 数组，dp[i] 爬到 i 阶的方法数
        dp = [0] * (n + 1)
        
        # 2. 初始状态
        dp[0] = 1
        dp[1] = 1
        
        # 3. 状态转移计算
        for i in range(2, n + 1):
            dp[i] = dp[i - 1] + dp[i - 2]
            
        return dp[n]
```

#### 代码实现（空间优化：滚动变量）

```python
class Solution:
    def climbStairs(self, n: int) -> int:
        if n <= 2:
            return n
            
        # dp1 代表 dp[i-2], dp2 代表 dp[i-1]
        dp1, dp2 = 1, 2
        for _ in range(3, n + 1):
            dp1, dp2 = dp2, dp1 + dp2
            
        return dp2
```

### 打家劫舍（难度：中等）

小偷计划偷窃沿街的房屋，相邻的房屋装有互连的防盗系统（如果两间相邻的房屋在同一晚上被小偷闯入，系统会自动报警）。给定一个代表每个房屋存放金额的非负整数数组，计算在不触动警报的情况下，一夜之内能够偷窃到的最高金额。

#### 解题思路

- **状态定义**：`dp[i]` 代表考虑前 `i` 间房子，所能偷到的最高金额。
- **状态转移方程**：面对第 `i` 间房，小偷只有两个选择：
  1. **偷这间房**：那么第 `i-1` 间房绝不能偷。此时金额为前 `i-2` 间房能偷到的最高金额加上当前这间房的金额：`dp[i-2] + nums[i]`
  2. **不偷这间房**：那么就可以带走前 `i-1` 间房偷来的最高金额：`dp[i-1]`
  所以，状态转移方程为：`dp[i] = max(dp[i-2] + nums[i], dp[i-1])`
- **初始化**：
  - `dp[0] = nums[0]`：如果街上只有第 0 家（第一家）房子，你能偷到的最高金额自然就是这家房子的金额。
  - `dp[1] = max(nums[0], nums[1])`：当街上有 0 和 1 两家房子时，由于会触发警报，你只能选择金额最大的那家偷（所以不能简单写成 `dp[1] = nums[1]`，因为可能第一家比第二家更有钱）。

同样地，我们应当先写出完整的状态转移数组 `dp` 建立框架思维。理解递推关系后，我们发现 `dp[i]` 只依赖它前面的两个状态 `dp[i-1]` 和 `dp[i-2]`，因此可以采用双变量进行空间滚动优化。

#### 代码实现（标准一维 DP 数组）

```python
class Solution:
    def rob(self, nums: List[int]) -> int:
        if not nums:
            return 0
        if len(nums) == 1:
            return nums[0]
            
        # 1. 明确定义 dp 数组
        dp = [0] * len(nums)
        
        # 2. 初始状态
        dp[0] = nums[0]
        dp[1] = max(nums[0], nums[1])
        
        # 3. 状态转移计算
        for i in range(2, len(nums)):
            dp[i] = max(dp[i - 2] + nums[i], dp[i - 1])
            
        return dp[-1]
```

#### 代码实现（空间优化：滚动变量）

```python
class Solution:
    def rob(self, nums: List[int]) -> int:
        # prev2 = dp[i-2], prev1 = dp[i-1]
        prev2, prev1 = 0, 0
        
        for num in nums:
            # 这里的 prev2 + num 就对应 dp[i-2] + nums[i]
            # 这里的 prev1 对应 dp[i-1]
            prev2, prev1 = prev1, max(prev2 + num, prev1)
            
        return prev1
```

## 三、一维状态挂载（以自身结尾）

### 最长递增子序列 (LIS)（难度：中等）

给你一个整数数组 `nums`，找到其中最长严格递增子序列的长度。

#### 解题思路

- **状态定义**：`dp[i]` 的定义并非“前 i 个元素中最长子序列的长度”，而是“**强制以 nums[i] 结尾的**最长递增子序列的长度”。这是序列 DP 最经典的强制挂载定义法。
- **状态转移方程**：让 `j` 从 `0` 遍历到 `i-1`。只要发现 `nums[i] > nums[j]`，说明 `nums[i]` 可以无缝接在 `nums[j]` 的屁股后面，形成一个更长的递增序列！则 `dp[i] = max(dp[i], dp[j] + 1)`。
- **返回值要点（易错坑）**：**千万不能返回 `dp[-1]`**！因为最长的递增子序列不一定是以整个数组的最后一个数字结尾的。例如 `nums = [1, 2, 3, 4, 5, 0]`，以最后一个特征值 `0` 结尾的最长子序列只能是 `[0]` 本身（长度为 1），而这数组真正的最长递增序列是前五个数（长度为 5）。因此，必须返回各个节点中打出的最大值 `max(dp)`。

#### 代码实现（Python版 - DP O(N^2)）
```python
class Solution:
    def lengthOfLIS(self, nums: List[int]) -> int:
        if not nums:
            return 0
            
        # 所有元素各自本身就是个长度为 1 的子序列
        dp = [1] * len(nums)
        
        for i in range(len(nums)):
            for j in range(i):
                if nums[i] > nums[j]:
                    dp[i] = max(dp[i], dp[j] + 1)
                    
        return max(dp)
```
*(注：本题还存在 `贪心 + 二分查找` 达成 `O(N log N)` 的高级解法，建议在二分查找篇复习)*

### 最大子数组和（难度：中等）

给你一个整数数组 `nums`，请你找出一个具有最大和的连续子数组（子数组最少包含一个元素），返回其最大和。

#### 解题思路

**DP 解法**：`dp[i]` 表示以 `nums[i]` 结尾的子数组的最大和。状态转移：如果前面的累积和 `dp[i-1]` 是负数，带上它只会拖累当前元素，不如从 `nums[i]` 重新开始。

```
dp[i] = max(dp[i-1], 0) + nums[i]
```

注意：答案是 `max(dp)` 而不是 `dp[-1]`，因为最大和不一定在末尾。

**贪心解法**：维护一个当前累积和 `cur_sum`，一旦它变为负数就归零重新开始，同时用 `ans` 记录历史最大值。本质上与 DP 等价，但空间 O(1)。

#### 代码实现

```python
# DP 版本（O(N) 时间，O(N) 空间）
def maxSubArray(nums):
    dp = [0] * len(nums)
    dp[0] = nums[0]
    for i in range(1, len(nums)):
        dp[i] = max(dp[i - 1], 0) + nums[i]
    return max(dp)

# 贪心版本（O(N) 时间，O(1) 空间）
def maxSubArray(nums):
    ans = float('-inf')
    cur_sum = 0
    for num in nums:
        if cur_sum < 0:
            cur_sum = 0
        cur_sum += num
        ans = max(ans, cur_sum)
    return ans
```

#### 复杂度分析

- DP 版：时间 O(N)，空间 O(N)
- 贪心版：时间 O(N)，空间 O(1)——两者逻辑等价，贪心是 DP 的空间优化版

### 乘积最大子数组（难度：中等）

给你一个整数数组 `nums` ，请你找出数组中乘积最大的非空连续子数组（该子数组中至少包含一个数字），并返回该子数组所对应的乘积。测试用例的答案是一个 32-位 整数。
*请注意，一个只包含一个元素的数组的乘积是这个元素的值。*

#### 解题思路

这道题同样是经典的“**强制挂载（以自身结尾）**”模型！如果要求“连续子数组”，那么状态定义就必须是“以 `nums[i]` 结尾的子数组”。
但它的一大难点在于**乘法的负负得正**特性：当前位置的最大乘积，不一定是由前一个位置的“最大乘积”推导来的。如果当前数字是负数，它乘以前面一个**极负的最小乘积**，就会瞬间翻盘变成一个极大的正数！

- **状态定义**：我们需要同时维护两个 DP 状态：
  - `max_dp[i]`：**强制以 `nums[i]` 结尾**的连续子数组的最大乘积。
  - `min_dp[i]`：**强制以 `nums[i]` 结尾**的连续子数组的最小乘积。
- **状态转移方程**：
  面对当前的 `nums[i]`，你有三种选择拼成当前的连续子数组：
  1. 顺延接着前面的最大乘积走：`max_dp[i-1] * nums[i]`
  2. 顺延接着前面的最小乘积走（期望负负得正）：`min_dp[i-1] * nums[i]`
  3. 嫌弃前面的序列（比如前面乘了个 0），自己单干：`nums[i]` 本身
  
  所以：
  `max_dp[i] = max(max_dp[i-1] * nums[i], min_dp[i-1] * nums[i], nums[i])`
  `min_dp[i] = min(max_dp[i-1] * nums[i], min_dp[i-1] * nums[i], nums[i])`
- **初始化**：`max_dp[0] = nums[0]`，`min_dp[0] = nums[0]`。
- **返回值要点**：依然是所有“以各个元素结尾的最强时刻”里的最大值（跟 LIS 大坑一样，不能直接取 `[-1]`）。

和爬楼梯、打家劫舍一样，我们先写出保留了完整状态数组的标准 DP 写法，直观感受两个 DP 数组相爱相杀的全过程。

#### 代码实现（标准一维 DP 数组）

```python
class Solution:
    def maxProduct(self, nums: List[int]) -> int:
        if not nums:
            return 0
        
        n = len(nums)
        # 1. 明确定义 dp 数组
        # max_dp[i] 表示以 i 结尾的最大乘积
        # min_dp[i] 表示以 i 结尾的最小乘积
        max_dp = [0] * n
        min_dp = [0] * n
        
        # 2. 初始状态
        max_dp[0] = nums[0]
        min_dp[0] = nums[0]
        
        # 记录全局最大值
        ans = nums[0]
        
        # 3. 状态转移计算
        for i in range(1, n):
            num = nums[i]
            max_dp[i] = max(max_dp[i-1] * num, min_dp[i-1] * num, num)
            min_dp[i] = min(max_dp[i-1] * num, min_dp[i-1] * num, num)
            
            # 记录历史上出现过的最高点
            ans = max(ans, max_dp[i])
            
        return ans
```

**空间优化**：仔细观察上面的循环，每次计算第 `i` 个状态只依赖于第 `i-1` 个状态的情报。那些 `i-2` 及以前的格子的数据完全废弃了。因此我们可以去掉长长的数组，直接用两个变量 `cur_max` 和 `cur_min` 来回滚动覆盖，达成 `O(1)` 的空间复杂度。

#### 代码实现（空间优化：滚动变量）

```python
class Solution:
    def maxProduct(self, nums: List[int]) -> int:
        if not nums:
            return 0
            
        # 初始的局部最大、局部最小、全局最大都是第一个元素
        cur_max = nums[0]
        cur_min = nums[0]
        ans = nums[0]
        
        for i in range(1, len(nums)):
            num = nums[i]
            # 因为 cur_max 会被覆盖，所以在计算 cur_min 之前先存一下
            temp_max = cur_max
            
            cur_max = max(temp_max * num, cur_min * num, num)
            cur_min = min(temp_max * num, cur_min * num, num)
            
            # 不断打擂台记录历史上出现过的最高点
            ans = max(ans, cur_max)
            
        return ans
```

### 最长有效括号（难度：困难）

给你一个只包含 `'('` 和 `')'` 的字符串，找出最长有效（格式正确且连续）括号子串的长度。
左右括号匹配，即每个左括号都有对应的右括号将其闭合的字符串是格式正确的，比如 `"(()())"`。

#### 解题思路

这道题是把“一维强制挂载”玩到极致的一道困难题。因为题目要求“**连续子串**”，所以我们的 DP 状态必须严格延续前文一维挂载体系的定海神针——**“强制以当前字符结尾”**。

- **状态定义**：`dp[i]` 表示**以 `s[i]` 为结尾**的最长有效括号子串的长度。
- **初筛（极简剪枝）**：以左括号 `'('` 结尾的字符串绝对不可能是有效的括号结构闭环！所以所有 `s[i] == '('` 的位置直接断定 `dp[i] = 0`，我们只关心 `')'`。
- **状态转移方程**：当 `s[i] == ')'` 时，需要回头看它左边紧挨着的兄弟是谁：
  1. **情况一层层包（左边是 `'('`）**：形如 `......()`。
     这说明这两个符号刚好组成了一对最基础的 `()`，这对括号给自己贡献了 2 的长度。但不要忘了，这一对括号的前面可能还有被连接完好的其他大块工程（比如之前的 `((()))` 连上现在的 `()`），所以还得大喊一声：“前面的兄弟带带我！”将 `dp[i-2]` 的成果自然衔接：
     `dp[i] = dp[i-2] + 2`（前提是不越界）。
  2. **情况二大乱炖（左边依然是 `')'`）**：形如 `......))`。
     此时你前面那兄弟 `)` 可能是一个庞大有效阵法（比如 `(...)`）的结尾。我们要去寻找：在跨过那座庞大有效阵法**更前面一点的位置**，是否有属于我这个外层右括号所痴痴等待的那个左半身 `'('`！
     由于前面的阵法跨度长度由 `dp[i-1]` 记录，所以我们要跳过去找的位置下标就是：`pre = i - 1 - dp[i-1]`。
     如果这个 `pre` 没有越界，且恰好 `s[pre] == '('`，说明我结尾的 `)` 成功穿上了一件包裹里面大结构的外衣！此时我新增的长度是：`里面的阵法长(dp[i-1]) + 我俩的首尾外衣厚度(2)`。同样别忘了如果这件外衣前面还紧接着其他有效的挂载块，还得全额带上（`+ dp[pre - 1]`）。
     经过合并同类项，总长度递推方程为：`dp[i] = dp[i-1] + dp[pre - 1] + 2`
- **返回值要点**：这题再次深刻印证了我们前文讲过的**挂载模型大坑**：由于最大括号子串可以终结在这个长长括号序列的任何半中腰截断，你依然且必定要在循环中用变量搜集所有的浪潮最高点（`max_len = max(max_len, dp[i])`），这是绝不可返回 `dp[-1]` 的连中三元实战局！

#### 代码实现（Python版 - DP）

```python
class Solution:
    def longestValidParentheses(self, s: str) -> int:
        n = len(s)
        if n < 2:
            return 0
            
        # dp[i] 代表以 s[i] 结尾的最长有效括号子串长度
        dp = [0] * n
        max_len = 0
        
        for i in range(1, n):
            # 只有遇到右括号，才有可能形成有效闭环
            if s[i] == ')':
                if s[i - 1] == '(':
                    # 形式像： "....()"，直接和前一个凑成对
                    dp[i] = (dp[i - 2] if i >= 2 else 0) + 2
                else:
                    # 形式像： "....))"，这里前面多半是个完整的有效系统，被我这个新的 ')' 整个包进去了
                    # pre 是尝试跳过那个完整大系统，去找属于我真正的另一半 '(' 的坐标
                    pre = i - 1 - dp[i - 1]
                    if pre >= 0 and s[pre] == '(':
                        # 既然越过了大系统且对上了，那就组装成了超级包壳大系统
                        dp[i] = dp[i - 1] + 2
                        # 别忘了这个大系统外头如果还顶着其他并列的有效块，还得全量相连！
                        if pre - 1 >= 0:
                            dp[i] += dp[pre - 1]
                            
                # 不断打擂台记录历史上出现过的最长连击局部
                max_len = max(max_len, dp[i])
                
        return max_len
```
