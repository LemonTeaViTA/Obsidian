> [[算法_MOC|返回算法导航]]

## 一、动态规划核心思想与打表套路

动态规划（DP）本质是一种**利用历史记录，避免重复计算**的优化思想。面试中百分之九十的 DP 题，只要能写出**状态转移方程**，代码就迎刃而解。

**核心做题骨架（DP五部曲）**：
1. **定义 dp 数组的物理含义**：`dp[i]` 到底代表什么？是“到第 i 阶楼梯的方法数”还是“以 nums[i] 结尾的最长子序列”？
    - **【易混辨析：DP 数组初始化长度是 `n` 还是 `n + 1`？】**
        - **定义为“索引位置”时，取 `n`**：当 `dp[i]` 需要与给定数组 `nums[i]` **严格对齐绑定属性**时（例如：“打家劫舍”考虑前 `i` 个房间，“最长递增子序列”强调以 `nums[i]` 结尾），原数组最后一个下标必定为 `n-1`，所以你只需要长度为 `n` 就可以完全覆盖，最终答案常常取决于所有求出的局部最大值（即 `max(dp)` 或 `dp[-1]`）。
        - **定义为“数量/容量/总量”时，必取 `n + 1`**：当遇到“经过前 `n` 个字符拆串”、“凑齐总金额 `amount`”、“爬到第 `n` 阶楼梯”这种衡量数字大小、算数量的题目时，**必须多留出一格 `+1` 的位置给 `dp[0]`**。因为动态规划需要一个合法的“基石”，如“凑齐金额 0”、“0个字符”去启动整个状态机。这样构建出来的数组，你想查容量为 `n` 的解，就刚好直接索引 `dp[n]`（否则如果是只建长为 `n` 的数组，是不存在下标为 `n` 可供提取的）。
2. **写出状态转移方程**：今天的状态由昨天哪些状态推导而来？（如 `dp[i] = dp[i-1] + dp[i-2]`）
3. **初始化基点**：起跑线在哪里？（如 `dp[0] = 0, dp[1] = 1`）
4. **确定遍历顺序**：是从左到右，还是从右到左？对于背包问题，内外层循环顺序极为讲究！
5. **举例打印 dp 数组验证**。

## 二、基础一维序列 DP

### 爬楼梯（难度：简单）

假设你正在爬楼梯。需要 `n` 阶你才能到达楼顶。每次你可以爬 1 或 2 个台阶。你有多少种不同的方法可以爬到楼顶呢？

#### 解题思路

- **状态定义**：`dp[i]` 表示爬到第 `i` 阶楼梯的方法数。
- **状态转移方程**：到达第 `i` 阶只能从 `i-1` 阶跨 1 步上来，或者从 `i-2` 阶跨 2 步上来。因此总方法数是这两种途径之和：`dp[i] = dp[i-1] + dp[i-2]`。
- **初始化**：`dp[0] = 1`（原地不动也算一种方法），`dp[1] = 1`。

作为动态规划的第一题，我们首先应当用标准的 DP 数组模板来严格定义每一维度的状态。掌握了完整的模板推导后，我们再考虑空间优化。由于只依赖前两项，可以用两个变量滚动前进。

#### 代码实现（标准一维 DP 数组）

```python
class Solution:
    def climbStairs(self, n: int) -> int:
        if n <= 1:
            return 1
        # 1. 明确定义 dp 数组，dp[i] 爬到 i 阶的方法数
        dp = [0] * (n + 1)
        
        # 2. 初始状态
        dp[0] = 1
        dp[1] = 1
        
        # 3. 状态转移计算
        for i in range(2, n + 1):
            dp[i] = dp[i - 1] + dp[i - 2]
            
        return dp[n]
```

#### 代码实现（空间优化：滚动变量）

```python
class Solution:
    def climbStairs(self, n: int) -> int:
        if n <= 2:
            return n
            
        # dp1 代表 dp[i-2], dp2 代表 dp[i-1]
        dp1, dp2 = 1, 2
        for _ in range(3, n + 1):
            dp1, dp2 = dp2, dp1 + dp2
            
        return dp2
```

### 打家劫舍（难度：中等）

小偷计划偷窃沿街的房屋，相邻的房屋装有互连的防盗系统（如果两间相邻的房屋在同一晚上被小偷闯入，系统会自动报警）。给定一个代表每个房屋存放金额的非负整数数组，计算在不触动警报的情况下，一夜之内能够偷窃到的最高金额。

#### 解题思路

- **状态定义**：`dp[i]` 代表考虑前 `i` 间房子，所能偷到的最高金额。
- **状态转移方程**：面对第 `i` 间房，小偷只有两个选择：
  1. **偷这间房**：那么第 `i-1` 间房绝不能偷。此时金额为前 `i-2` 间房能偷到的最高金额加上当前这间房的金额：`dp[i-2] + nums[i]`
  2. **不偷这间房**：那么就可以带走前 `i-1` 间房偷来的最高金额：`dp[i-1]`
  所以，状态转移方程为：`dp[i] = max(dp[i-2] + nums[i], dp[i-1])`
- **初始化**：
  - `dp[0] = nums[0]`：如果街上只有第 0 家（第一家）房子，你能偷到的最高金额自然就是这家房子的金额。
  - `dp[1] = max(nums[0], nums[1])`：当街上有 0 和 1 两家房子时，由于会触发警报，你只能选择金额最大的那家偷（所以不能简单写成 `dp[1] = nums[1]`，因为可能第一家比第二家更有钱）。

同样地，我们应当先写出完整的状态转移数组 `dp` 建立框架思维。理解递推关系后，我们发现 `dp[i]` 只依赖它前面的两个状态 `dp[i-1]` 和 `dp[i-2]`，因此可以采用双变量进行空间滚动优化。

#### 代码实现（标准一维 DP 数组）

```python
class Solution:
    def rob(self, nums: List[int]) -> int:
        if not nums:
            return 0
        if len(nums) == 1:
            return nums[0]
            
        # 1. 明确定义 dp 数组
        dp = [0] * len(nums)
        
        # 2. 初始状态
        dp[0] = nums[0]
        dp[1] = max(nums[0], nums[1])
        
        # 3. 状态转移计算
        for i in range(2, len(nums)):
            dp[i] = max(dp[i - 2] + nums[i], dp[i - 1])
            
        return dp[-1]
```

#### 代码实现（空间优化：滚动变量）

```python
class Solution:
    def rob(self, nums: List[int]) -> int:
        # prev2 = dp[i-2], prev1 = dp[i-1]
        prev2, prev1 = 0, 0
        
        for num in nums:
            # 这里的 prev2 + num 就对应 dp[i-2] + nums[i]
            # 这里的 prev1 对应 dp[i-1]
            prev2, prev1 = prev1, max(prev2 + num, prev1)
            
        return prev1
```

## 三、一维状态挂载（以自身结尾）

### 最长递增子序列 (LIS)（难度：中等）

给你一个整数数组 `nums`，找到其中最长严格递增子序列的长度。

#### 解题思路

- **状态定义**：`dp[i]` 的定义并非“前 i 个元素中最长子序列的长度”，而是“**强制以 nums[i] 结尾的**最长递增子序列的长度”。这是序列 DP 最经典的强制挂载定义法。
- **状态转移方程**：让 `j` 从 `0` 遍历到 `i-1`。只要发现 `nums[i] > nums[j]`，说明 `nums[i]` 可以无缝接在 `nums[j]` 的屁股后面，形成一个更长的递增序列！则 `dp[i] = max(dp[i], dp[j] + 1)`。
- **返回值要点（易错坑）**：**千万不能返回 `dp[-1]`**！因为最长的递增子序列不一定是以整个数组的最后一个数字结尾的。例如 `nums = [1, 2, 3, 4, 5, 0]`，以最后一个特征值 `0` 结尾的最长子序列只能是 `[0]` 本身（长度为 1），而这数组真正的最长递增序列是前五个数（长度为 5）。因此，必须返回各个节点中打出的最大值 `max(dp)`。

#### 代码实现（Python版 - DP O(N^2)）
```python
class Solution:
    def lengthOfLIS(self, nums: List[int]) -> int:
        if not nums:
            return 0
            
        # 所有元素各自本身就是个长度为 1 的子序列
        dp = [1] * len(nums)
        
        for i in range(len(nums)):
            for j in range(i):
                if nums[i] > nums[j]:
                    dp[i] = max(dp[i], dp[j] + 1)
                    
        return max(dp)
```
*(注：本题还存在 `贪心 + 二分查找` 达成 `O(N log N)` 的高级解法，建议在二分查找篇复习)*

### 乘积最大子数组（难度：中等）

给你一个整数数组 `nums` ，请你找出数组中乘积最大的非空连续子数组（该子数组中至少包含一个数字），并返回该子数组所对应的乘积。测试用例的答案是一个 32-位 整数。
*请注意，一个只包含一个元素的数组的乘积是这个元素的值。*

#### 解题思路

这道题同样是经典的“**强制挂载（以自身结尾）**”模型！如果要求“连续子数组”，那么状态定义就必须是“以 `nums[i]` 结尾的子数组”。
但它的一大难点在于**乘法的负负得正**特性：当前位置的最大乘积，不一定是由前一个位置的“最大乘积”推导来的。如果当前数字是负数，它乘以前面一个**极负的最小乘积**，就会瞬间翻盘变成一个极大的正数！

- **状态定义**：我们需要同时维护两个 DP 状态：
  - `max_dp[i]`：**强制以 `nums[i]` 结尾**的连续子数组的最大乘积。
  - `min_dp[i]`：**强制以 `nums[i]` 结尾**的连续子数组的最小乘积。
- **状态转移方程**：
  面对当前的 `nums[i]`，你有三种选择拼成当前的连续子数组：
  1. 顺延接着前面的最大乘积走：`max_dp[i-1] * nums[i]`
  2. 顺延接着前面的最小乘积走（期望负负得正）：`min_dp[i-1] * nums[i]`
  3. 嫌弃前面的序列（比如前面乘了个 0），自己单干：`nums[i]` 本身
  
  所以：
  `max_dp[i] = max(max_dp[i-1] * nums[i], min_dp[i-1] * nums[i], nums[i])`
  `min_dp[i] = min(max_dp[i-1] * nums[i], min_dp[i-1] * nums[i], nums[i])`
- **初始化**：`max_dp[0] = nums[0]`，`min_dp[0] = nums[0]`。
- **返回值要点**：依然是所有“以各个元素结尾的最强时刻”里的最大值（跟 LIS 大坑一样，不能直接取 `[-1]`）。

和爬楼梯、打家劫舍一样，我们先写出保留了完整状态数组的标准 DP 写法，直观感受两个 DP 数组相爱相杀的全过程。

#### 代码实现（标准一维 DP 数组）

```python
class Solution:
    def maxProduct(self, nums: List[int]) -> int:
        if not nums:
            return 0
        
        n = len(nums)
        # 1. 明确定义 dp 数组
        # max_dp[i] 表示以 i 结尾的最大乘积
        # min_dp[i] 表示以 i 结尾的最小乘积
        max_dp = [0] * n
        min_dp = [0] * n
        
        # 2. 初始状态
        max_dp[0] = nums[0]
        min_dp[0] = nums[0]
        
        # 记录全局最大值
        ans = nums[0]
        
        # 3. 状态转移计算
        for i in range(1, n):
            num = nums[i]
            max_dp[i] = max(max_dp[i-1] * num, min_dp[i-1] * num, num)
            min_dp[i] = min(max_dp[i-1] * num, min_dp[i-1] * num, num)
            
            # 记录历史上出现过的最高点
            ans = max(ans, max_dp[i])
            
        return ans
```

**空间优化**：仔细观察上面的循环，每次计算第 `i` 个状态只依赖于第 `i-1` 个状态的情报。那些 `i-2` 及以前的格子的数据完全废弃了。因此我们可以去掉长长的数组，直接用两个变量 `cur_max` 和 `cur_min` 来回滚动覆盖，达成 `O(1)` 的空间复杂度。

#### 代码实现（空间优化：滚动变量）

```python
class Solution:
    def maxProduct(self, nums: List[int]) -> int:
        if not nums:
            return 0
            
        # 初始的局部最大、局部最小、全局最大都是第一个元素
        cur_max = nums[0]
        cur_min = nums[0]
        ans = nums[0]
        
        for i in range(1, len(nums)):
            num = nums[i]
            # 因为 cur_max 会被覆盖，所以在计算 cur_min 之前先存一下
            temp_max = cur_max
            
            cur_max = max(temp_max * num, cur_min * num, num)
            cur_min = min(temp_max * num, cur_min * num, num)
            
            # 不断打擂台记录历史上出现过的最高点
            ans = max(ans, cur_max)
            
        return ans
```

### 最长有效括号（难度：困难）

给你一个只包含 `'('` 和 `')'` 的字符串，找出最长有效（格式正确且连续）括号子串的长度。
左右括号匹配，即每个左括号都有对应的右括号将其闭合的字符串是格式正确的，比如 `"(()())"`。

#### 解题思路

这道题是把“一维强制挂载”玩到极致的一道困难题。因为题目要求“**连续子串**”，所以我们的 DP 状态必须严格延续前文一维挂载体系的定海神针——**“强制以当前字符结尾”**。

- **状态定义**：`dp[i]` 表示**以 `s[i]` 为结尾**的最长有效括号子串的长度。
- **初筛（极简剪枝）**：以左括号 `'('` 结尾的字符串绝对不可能是有效的括号结构闭环！所以所有 `s[i] == '('` 的位置直接断定 `dp[i] = 0`，我们只关心 `')'`。
- **状态转移方程**：当 `s[i] == ')'` 时，需要回头看它左边紧挨着的兄弟是谁：
  1. **情况一层层包（左边是 `'('`）**：形如 `......()`。
     这说明这两个符号刚好组成了一对最基础的 `()`，这对括号给自己贡献了 2 的长度。但不要忘了，这一对括号的前面可能还有被连接完好的其他大块工程（比如之前的 `((()))` 连上现在的 `()`），所以还得大喊一声：“前面的兄弟带带我！”将 `dp[i-2]` 的成果自然衔接：
     `dp[i] = dp[i-2] + 2`（前提是不越界）。
  2. **情况二大乱炖（左边依然是 `')'`）**：形如 `......))`。
     此时你前面那兄弟 `)` 可能是一个庞大有效阵法（比如 `(...)`）的结尾。我们要去寻找：在跨过那座庞大有效阵法**更前面一点的位置**，是否有属于我这个外层右括号所痴痴等待的那个左半身 `'('`！
     由于前面的阵法跨度长度由 `dp[i-1]` 记录，所以我们要跳过去找的位置下标就是：`pre = i - 1 - dp[i-1]`。
     如果这个 `pre` 没有越界，且恰好 `s[pre] == '('`，说明我结尾的 `)` 成功穿上了一件包裹里面大结构的外衣！此时我新增的长度是：`里面的阵法长(dp[i-1]) + 我俩的首尾外衣厚度(2)`。同样别忘了如果这件外衣前面还紧接着其他有效的挂载块，还得全额带上（`+ dp[pre - 1]`）。
     经过合并同类项，总长度递推方程为：`dp[i] = dp[i-1] + dp[pre - 1] + 2`
- **返回值要点**：这题再次深刻印证了我们前文讲过的**挂载模型大坑**：由于最大括号子串可以终结在这个长长括号序列的任何半中腰截断，你依然且必定要在循环中用变量搜集所有的浪潮最高点（`max_len = max(max_len, dp[i])`），这是绝不可返回 `dp[-1]` 的连中三元实战局！

#### 代码实现（Python版 - DP）

```python
class Solution:
    def longestValidParentheses(self, s: str) -> int:
        n = len(s)
        if n < 2:
            return 0
            
        # dp[i] 代表以 s[i] 结尾的最长有效括号子串长度
        dp = [0] * n
        max_len = 0
        
        for i in range(1, n):
            # 只有遇到右括号，才有可能形成有效闭环
            if s[i] == ')':
                if s[i - 1] == '(':
                    # 形式像： "....()"，直接和前一个凑成对
                    dp[i] = (dp[i - 2] if i >= 2 else 0) + 2
                else:
                    # 形式像： "....))"，这里前面多半是个完整的有效系统，被我这个新的 ')' 整个包进去了
                    # pre 是尝试跳过那个完整大系统，去找属于我真正的另一半 '(' 的坐标
                    pre = i - 1 - dp[i - 1]
                    if pre >= 0 and s[pre] == '(':
                        # 既然越过了大系统且对上了，那就组装成了超级包壳大系统
                        dp[i] = dp[i - 1] + 2
                        # 别忘了这个大系统外头如果还顶着其他并列的有效块，还得全量相连！
                        if pre - 1 >= 0:
                            dp[i] += dp[pre - 1]
                            
                # 不断打擂台记录历史上出现过的最长连击局部
                max_len = max(max_len, dp[i])
                
        return max_len
```

## 四、背包问题 (Knapsack)

背包问题分类中，最最常见的有两大宗门：**0-1背包**（每种物品只能用一次）和**完全背包**（每种物品可以无限用）。它们在一维 DP 里的核心区别非常简单：就是内层容量的循环方向。

### 分割等和子集（难度：中等）

给你一个只包含正整数的非空数组 `nums` 。请你判断是否可以将这个数组分割成两个子集，使得两个子集的元素和相等。

#### 解题思路

这是最经典的 **0-1 背包问题** 变种。
我们把问题翻译一下：“将数组分割成和相等的两份”其实就是问：**能不能从数组里挑选出一堆数字，让它们的和恰好等于整个数组总和的一半（`sum(nums) / 2`）？** 
这就彻底变成了一个典型的 0-1 背包：我们的背包总容量是 `sum / 2`，物品就是数组里的每一个数字，每个物品的“体积”和“价值”就是这个数字本身。我们要在给定的物品中看看能不能刚好把包塞满。

- **状态定义**：`dp[j]` 表示背包容量为 `j` 时，能否用前部分数字恰好把它装满。
- **状态转移方程**：对于当前正在考察的新物品数字 `num`，我们可以选择“装它”：如果决定装它，目标就变成考察以前我们的背包能不能装满 `j - num` 这点容量。也就是 `dp[j] = dp[j] or dp[j - num]`。只要这两种方案有一条能走通，`dp[j]` 就大功告成了。
- **初始化**：`dp[0] = True`。装满容量为 0 的背包的方法就是永远“什么都不装”。作为所有后续推理唯一的根基点。
- **【直击灵魂：为什么要有两层循环？分别在干什么？】**
  这个问题对于没写过二维 DP 直接跳一维滚动数组的人来说极其困惑，本质上，这两层循环的分工是极其明确的：
  - **外层循环（遍历物品 `for num in nums`）**：你可以把整个过程想象成站在流水线旁边，数组 `nums` 里的数字就是一个个送过来的“快递盲盒”。每新送过来一个数字 `num`，你都要拿着它，基于它**去刷新一遍你手里的那个“全村希望的账本”**（即 `dp` 数组）。
  - **内层循环（遍历容量 `for j in range(...)`）**：你的总账本上密密麻麻记录了容量从 `1` 斤到 `target` 斤的各种各样的空背包“目前能不能刚好装满”。当你手里攥着传送带新送来的、重量为 `num` 的新物品去问容量为 `j` 的背包时，你会满怀期待地想：“兄弟，我手里拿着 `num` 这块砖，如果以前（就是上一轮的账本）那些老数字们能成功填满 `j - num` 那个稍小点儿的坑位（即 `dp[j-num] == True`），那我今天顺手把这块重量刚好是 `num` 的砖“咔嗒”一声塞进去，你不就完美缝合到了正好容量是 `j` 的大圆满了吗！”因此，这就是用新积木去挨个容量去试填的过程。
- **【核心易错坑：内层循环为什么要逆序？】**：本题是 0-1 背包：**数组里的数每个人仅仅只能拿一次！**
  - 如果你**正序遍历**（从 `num` 一路推高到 `target`），当你在算大容量需要用到小容量的旧数据 `dp[j-num]` 时，那个 `dp[j-num]` 可能已经在这一轮刚刚被当前这个 `num` 给填满改变过了！这就等同于你把当前数字 `num` 放进包里了**两次甚至很多次**。它就变成了后面题目的完全背包。
  - 要解决这个问题，防止串台的究极办法是：**我们必须从大到小去逆向刷新数组（从 `target` 降序滑落到 `num`）**。当你计算高位的 `dp[j]` 需要依赖左边小位的 `dp[j-num]` 的时候，因为你是从右往左抹过来的，所以左边那些小的值还保留在上一轮比赛（即还没受到当前新数字洗礼）原封未动的原样！这样，当前数字就绝对安全地只会被装进去一次。这就是 0-1 背包滚动的精髓。
- **【逻辑细节：内层循环的终点为什么停在 `num - 1`？】**
  - 在 Python 中的 `range(start, stop, step)` 函数是“前闭后开”的，意味着它会在到达 `stop` 的前一个时刻停止。因此 `range(target, num - 1, -1)` 的实际效果是：从 `target` 容量开始往下降，**降到代表当前物品重量的 `num` 为止**。
  - **物理意义上的原因**：如果当前背包的容量 $j$ 比新来的这块砖的重量 `num` 还要小（即 $j < num$），那你这容量可怜的小背包根本**塞不进去**手里这块大砖头！既然塞不进，那背包的状态 `dp[j]` 就无法借助这块新砖来更新，只能乖乖继承上一轮的老样子。因为一维数组本身原位不动就是继承老样子，所以我们干脆连算都不去算它了，直接把 $j < num$ 的情况剪枝掉。这就是为什么内循环卡在 `num`（即 Python 语法里的 `num - 1` 作为终点）就不再往下算的原因。

#### 代码实现（Python版 - 0-1背包模板）

```python
class Solution:
    def canPartition(self, nums: List[int]) -> bool:
        total_sum = sum(nums)
        # 奇数总和，绝大可能不能分成等和两份整数
        if total_sum % 2 != 0:
            return False
            
        # 要拼凑的目标容量是总和的一半
        target = total_sum // 2
        
        # dp[j] 代表能否刚好凑出容量为 j 的和
        dp = [False] * (target + 1)
        dp[0] = True
        
        # 外层遍历每个人/物品（仅使用一次）
        for num in nums:
            # 内层循环遍历容量：为了保证 0-1 背包特性，必须【逆序遍历】防串台！
            for j in range(target, num - 1, -1):
                dp[j] = dp[j] or dp[j - num]
                
        return dp[target]
```

### 零钱兑换（难度：中等）

给你一个整数数组 `coins`，表示不同面额的硬币；以及一个整数 `amount`，表示总金额。计算并返回可以凑成总金额所需的 **最少的硬币个数**。如果没法凑出，返回 `-1`。可以认为每种硬币的数量是无限的。

#### 解题思路

这道题是经典的**完全背包求极值**问题。可以将总金额视作背包容量，各种面额的硬币视作重量不等的物品，由于硬币有无限个，便构成了完全背包。

- **状态定义**：`dp[i]` 表示凑成金额 `i` 所需的最小硬币个数。
- **状态转移方程**：对目标金额 `i`，尝试拿面值为 `coin` 的硬币。如果拿了这一枚，就需要再加上凑出 `i - coin` 的最小个数。因此状态转移方程为：`dp[i] = min(dp[i], dp[i - coin] + 1)`，仅当 `i >= coin` 时能进行。
- **初始化**：既然求“最小”，就把所有元素初始化为正无穷大 `float('inf')`，防止被 0 干扰。唯一的基石是 `dp[0] = 0`，即凑齐 0 元需要 0 枚硬币。

#### 代码实现（Python版）

```python
class Solution:
    def coinChange(self, coins: List[int], amount: int) -> int:
        # dp 数组初始化为无穷大，表示初期认为凑不出
        dp = [float('inf')] * (amount + 1)
        dp[0] = 0
        
        # 外层遍金额，内层遍历硬币（或者反过来也可以，因为求的是全组合的极值而不是排列数）
        for i in range(1, amount + 1):
            for coin in coins:
                if i >= coin:
                    dp[i] = min(dp[i], dp[i - coin] + 1)
                    
        return dp[amount] if dp[amount] != float('inf') else -1
```

### 单词拆分（难度：中等）

给你一个字符串 `s` 和一个字符串列表 `wordDict` 作为字典。请你判断是否可以利用字典中出现的单词拼接出 `s`。注意：不要求字典中出现的单词全部都使用，并且字典中的单词可以重复使用。

#### 解题思路

这也是**完全背包问题**的变种，字典里的各个“单词”就像是待挑选的“物品”，我们要用它们去装满字符串 `s` 这个背包！不同之处在于字符串组合要求**排列顺序必须严格一致**。

- **状态定义**：`dp[i]` 代表字符串前 `i` 个字符组成的子串（即 `s[0:i]`）能否被字典里的单词拼出来。
- **状态转移方程**：利用切断思维，在 `s[0:i]` 内部找一个断点 `j`。如果前半部分 `s[0:j]` 已证实合法（即 `dp[j] == True`），并且后半段切出来的 `s[j:i]` 刚好存在于字典中，那整个长为 `i` 的字符串自然被拼接成功：`dp[i] = True`。
  - **【思维误区一：为什么是判断 `dp[j]`，而不是直接去写 `s[:j] in word_set`？】**
    如果你写成 `s[:j] in word_set`，你的意思是要求 `s[:j]` 这个全长子串必须是一个**又长又完整的、独立出没在字典里的巨型单词**。比如你要拼 `"applepen"`，你的字典里只有 `"apple"` 和 `"pen"`。你此时切出了前半部分 `"applepen"` 去判断 `in word_set`，字典肯定说“没这词儿啊，退回”。
    但如果我们写成 `dp[j]`，那就完全不一样了！**它的含义是：不管你这前半部分 `s[:j]` 有多长、是由多少个碎单词拼合而成的，只要以前历史运算过它“属于合法词汇拼合物”，那么在动态规划的世界里，它就是被点亮的**。这就是在“利用历史记录避免重复验证全串”的动态规划精髓。
  - **【思维误区二：如果循环跑完都没遇到 `break`，我能直接 `return False` 结束程序吗？】**
    绝对不能。这里的内侧循环（也就是寻找断点 `j` 的循环），只是为了给**当下的这短得可怜的 `i` 长度**的前缀尝试寻找一种拼凑方法。找到了，就说明长度为 `i` 的前缀合法，我们 `break` 掉当前 `i` 的找拼合工作，急匆匆地去继续拼下个更长的 `i` 了。
    如果针对这个特定的 `i`，所有的 `j` 切法都试了，都拼不成功，那不 `break` 也就意味着这句长度为 `i` 的阶段性子串拼不出来而已——这对应的底层客观事实是 `dp[i]` 保持为初始化的 `False` 就足够了。你千万不能因为人家半路上有一个小的阶段性残缺词没拼出来，就在循环体里直接判了全盘代码的死刑 `return False`，因为后面说不定更长的 `i` （比如包含了整个字典完整单词汇的长段）就能成功拼上历史合法的积木！这就像人生，阶段性的拼合失败不代表全局崩盘，最后的结局由 `dp[n]` 统一宣判。
- **初始化**：`dp[0] = True`。空字符串默认不需要任何单词就能拼凑出来，这是后续所有转移逻辑能够连通启动的一块重要“基石”。

#### 代码实现（Python版 - 切分挂接法）

```python
class Solution:
    def wordBreak(self, s: str, wordDict: List[str]) -> bool:
        word_set = set(wordDict)  # 换成哈希集合，提速百倍
        n = len(s)
        
        # dp[i] 代表 s 的前 i 个字符组成的子串，能否被合法拼出
        dp = [False] * (n + 1)
        dp[0] = True  # 空串默认是可以拼出
        
        for i in range(1, n + 1):
            # 将前 i 个字符的串一分为二，找一个能接住的断点 j
            for j in range(i):
                if dp[j] and s[j:i] in word_set:
                    dp[i] = True
                    break  # 只要有一种切法能成功就够了！
                    
        return dp[n]
```

### 完全平方数（难度：中等）

给你一个整数 `n` ，返回和为 `n` 的完全平方数的最少数量。完全平方数是一个整数，其值等于另一个整数的平方；换句话说，其值等于一个整数自乘的积。例如，1、4、9 和 16 都是完全平方数，而 3 和 11 不是。

#### 解题思路

这道题在本质上和前面的《零钱兑换》**完全是一模一样**的完全背包问题！只是将无限可选的“硬币面额”换成了无限可选的“完全平方数（$1, 4, 9, 16 \dots$）”，要求从里面挑选最少数量的“物品”来凑满足够的容量 $n$。

- **状态定义**：`dp[i]` 表示凑出目标和为 `i` 所需的最少完全平方数的数量。
- **状态转移方程**：当前面临的目标和为 `i` 时，尝试从中减去一个完全平方数 $j^2$。如果拿掉这个 $j^2$，我们要找的就是剩下的容量 `i - j*j` 所需的最少数量，再加上刚刚拿起的这 $1$ 个。所以递推式为：`dp[i] = min(dp[i], dp[i - j*j] + 1)`。我们需要让 $j$ 从 $1$ 开始逐渐变大，只要 $j^2 \le i$ 就进行尝试截取，从中选出最好的组合方式。
- **初始化**：依然是因为求极小值，把所有元素用无穷大 `float('inf')` 填充满，只设 `dp[0] = 0` 作为没有目标余额不需要物品的基石。

#### 代码实现（Python版）

```python
class Solution:
    def numSquares(self, n: int) -> int:
        dp = [float('inf')] * (n + 1)
        dp[0] = 0
        
        # 遍历所有的容量数字
        for i in range(1, n + 1):
            # j 从 1 开始，只要它的完全平方不超过 i 就试着放入这个“新硬币”
            j = 1
            while j * j <= i:
                dp[i] = min(dp[i], dp[i - j * j] + 1)
                j += 1
                
        return dp[n]
```

## 五、多维动态规划 (网格 DP)

### 不同路径（难度：中等）

一个机器人位于一个 `m x n` 网格的左上角 （起始点在下图中标记为 “Start” ）。机器人每次只能向下或者向右移动一步。机器人试图达到网格的右下角（在下图中标记为 “Finish” ）。问总共有多少条不同的路径？

#### 解题思路

这是最标准的基础二维网格动态规划题，也是所有矩阵/迷宫类 DP 问题（如“最小路径和”、“带障碍物的不同路径”）的基础母题。由于机器人的移动方向被严格限制为“只能向下”或“只能向右”，这意味着**到达任何一个格子的路径来源，只可能从它上方的格子，或者左方的格子走过来**。这天然就构成了动态规划推导前置状态的前提。

- **状态定义**：`dp[i][j]` 表示机器人走到第 `i` 行、第 `j` 列的格子，总共有多少条不同的路径。
- **状态转移方程**：因为只能从上方或左方过来，因此走到 `(i, j)` 的所有可能路径，必定等于走到 `(i-1, j)`（上方格子）的路径数加上走到 `(i, j-1)`（左方格子）的路径数之和。
  即：`dp[i][j] = dp[i-1][j] + dp[i][j-1]`。
- **初始化**：
  - 起始位置 `dp[0][0] = 1`。
  - 对于第 0 行的所有格子 `dp[0][j]`，由于机器人不能向上走或者从上面走下来，只能“硬着头皮从左边一路往右走”，所以这一路过去所有格子都只有 1 条路径。
  - 对于第 0 列的所有格子 `dp[i][0]`，同理因为不能向左走或从左边走过来，只能一路往下走，一直到底全是 1 条路径。
- **返回值要点**：最终返回右下角的计算结果，即下标为 `dp[m-1][n-1]` 的位置。

这道题在熟练掌握基本的二维打表后，最精华的反而是其**空间优化降维打击**。在每次计算第 `i` 行的数据时，由于用到的信息只来自于“正上方”（即上一行同一个下标处的值）和“左方”（刚刚刷新出的左小一格的值），我们完全可以把二维地图压缩成一维。

#### 代码实现（标准二维 DP 数组）

```python
class Solution:
    def uniquePaths(self, m: int, n: int) -> int:
        # 定义一个 m 行 n 列的二维数组，初始值全为 1（这一步巧妙地把第 0 行和第 0 列的打底情况一并解决了！）
        dp = [[1] * n for _ in range(m)]
        
        # 因为最上边一行和最左边一列都已经是 1 了，我们直接从内腹地带的 (1,1) 开始递推
        for i in range(1, m):
            for j in range(1, n):
                # 当前格子的到达方式 = 从上面下来的 + 从左面过来的
                dp[i][j] = dp[i-1][j] + dp[i][j-1]
                
        return dp[m-1][n-1]
```

#### 代码实现（空间优化：一维滚动数组）

在降维到一维数组时，我们反复覆盖、反复使用长度仅为 `n` 的一行数组：
- 等号右半边的 `dp[j]` 还是老旧的历史记录（即**上方这一列格子的路径数**）。
- 等式右半边的 `dp[j-1]` 因为在我们从左往右的 for 循环里已经被刚才刷新过了，所以它天然代表了**左边这一个格子的路径数**。

```python
class Solution:
    def uniquePaths(self, m: int, n: int) -> int:
        # 只需要一个长度为 n 的一维数组，装满第 0 行的路径步数（全是1）
        dp = [1] * n
        
        for i in range(1, m):
            for j in range(1, n):
                # 更新当前列的值：旧的自己(相当于上面) + 左边刚更新过的兄弟(相当于左面)
                dp[j] = dp[j] + dp[j-1]
                
        return dp[-1]
```

#### 复杂度分析
- **时间复杂度**：$O(m \times n)$，两层嵌套循环，我们需要遍历并计算每一个格子的解。
- **空间复杂度**：二维推演版本为 $O(m \times n)$；但经过一维滚动数组空间优化后，复杂度可以骤降至 $O(n)$。

### 最小路径和（难度：中等）

给定一个包含非负整数的 `m x n` 网格 `grid` ，请找出一条从左上角到右下角的路径，使得路径上的数字总和为最小。
说明：每次只能向下或者向右移动一步。

#### 解题思路

这是《不同路径》的亲生兄弟题。在《不同路径》中，我们是求“走到这里有多少**种**可能”，也就是把上面的路径数和左边的路径数相加；而在本题中，我们是求“走到这里要交最少多少**过路费**”。因为依然被死死限制了“只能向右向下”，所以到达任意一个格子 `(i, j)`，它唯一可能的前置出发点还是它的相对上方或左方。

- **状态定义**：`dp[i][j]` 表示从左上角出发，走到 `(i, j)` 这个格子所能消耗的**最小路径和**。
- **状态转移方程**：我们要在上方和左方这两条可走的历史路线中，挑选一条积累目前为止过路费**最小**的来走，然后再交上当底这块格子的过路费。
  即：`dp[i][j] = min(dp[i-1][j], dp[i][j-1]) + grid[i][j]`。
- **初始化（寻找最前置的基石）**：
  - 起始位置 `(0, 0)` 的代价就是它起点的数字本身：`grid[0][0]`。
  - **第一行**：走在第一行的机器人无路可选，它上面根本没格子，只能一路从左边挨个走过来。所以它的代价是一直往右做加法累加：`dp[0][j] = dp[0][j-1] + grid[0][j]`。
  - **第一列**：同理，走在第一列的机器人也无路可选，左边没格子，只能一路往下砸。所以它的代价是一路往下累加：`dp[i][0] = dp[i-1][0] + grid[i][0]`。
- **空间优化（降维甚至榨干）**：和《不同路径》可以压缩到一维数组一样，本题因为我们每次只需要看“左面”和“上面”，原数组 `grid` 在被读过后就彻底失去了原有的计算价值。因此，我们为了极简最优化，甚至可以直接把网格 `grid` 当作原地的 `dp` 数组来使用，每走一步就把原格子里的数字直接覆盖成抵达它的最小路径和，从而榨干空间复杂度将其降为 $O(1)$！

#### 代码实现（标准二维 DP 数组）

作为初学者建立动态规划直觉的第一步，我们必须先写出严格使用辅助矩阵 `dp` 的常规解法，以体会状态演替流转的完整物理过程。

```python
class Solution:
    def minPathSum(self, grid: List[List[int]]) -> int:
        if not grid or not grid[0]:
            return 0
            
        m, n = len(grid), len(grid[0])
        
        # 1. 明确定义并分配 dp 数组空间
        dp = [[0] * n for _ in range(m)]
        
        # 2. 初始化基点（最左上角）
        dp[0][0] = grid[0][0]
        
        # 单独处理第一行（无脑向右累加过路费）
        for j in range(1, n):
            dp[0][j] = dp[0][j-1] + grid[0][j]
            
        # 单独处理第一列（无脑向下累加过路费）
        for i in range(1, m):
            dp[i][0] = dp[i-1][0] + grid[i][0]
            
        # 3. 状态转移：开始处理剩余被包围在核心圈的其他格子
        for i in range(1, m):
            for j in range(1, n):
                # 上面过来的代价和左边过来的代价，挑个最便宜的方案走，再加上站在这里必须交的费
                dp[i][j] = min(dp[i-1][j], dp[i][j-1]) + grid[i][j]
                
        return dp[m-1][n-1]
```

#### 代码实现（空间榨干优化：原地覆盖法 $O(1)$）

当你完全熟练掌握并理解了上面的转移关系后，再去使用这种原地黑科技。

```python
class Solution:
    def minPathSum(self, grid: List[List[int]]) -> int:
        if not grid or not grid[0]:
            return 0
            
        m, n = len(grid), len(grid[0])
        
        # 抛弃 dp 数组，直接霸占原有的 grid 画板
        for j in range(1, n):
            grid[0][j] = grid[0][j-1] + grid[0][j]
            
        for i in range(1, m):
            grid[i][0] = grid[i-1][0] + grid[i][0]
            
        for i in range(1, m):
            for j in range(1, n):
                grid[i][j] = min(grid[i-1][j], grid[i][j-1]) + grid[i][j]
                
        return grid[m-1][n-1]
```

#### 复杂度分析
- **时间复杂度**：$O(m \times n)$。我们要老老实实遍历网格中每一个元素算一边。
- **空间复杂度**：
  - 标准二维 DP 方法为 $O(m \times n)$，因为开辟了等同于地图大小的独立 `dp` 矩阵。
  - 由于我们利用了原本输入的 `grid` 进行原地覆盖，因此极致优化版不需要产生任何多余的空间分配，空间复杂度降为 $O(1)$。

## 六、区间 DP (字符串二维模型)

### 最长回文子串（难度：中等）

给你一个字符串 `s`，找到 `s` 中最长的回文子串。

#### 解题思路

本题除了最符合人类直觉的“中心扩散法”外，利用判定性质传递的**二维区间 DP** 也是极为正统的模板题。掌握它能为你解答一切回文字符串动态规划问题（例如“回文子序列数量”、“分割回文串”）打下坚实的框架基础。

- **状态定义（区间锁死）**：对于字符串子串问题，最标准的状态设定是利用明确的左右两个坐标将区间框死。
  我们定义 `dp[i][j]` 表示：字符串 `s` 从索引 `i` 到 `j` 的这一段闭区间子串，是否是一个回文串（布尔值 `True` / `False`）。
- **状态转移方程（由内向外推导）**：
  判断一个由 `...xyz...` 构成的大区间是不是回文串，本质上只取决于两点：
  1. 它的左右两个门神（头尾字符 `s[i]` 和 `s[j]`）是不是长得一样；
  2. 它“掐头去尾”后，肚子里面包着的那个小一圈的核心子串 `dp[i+1][j-1]` 是不是回文串。
  因此，如果 `s[i] == s[j]`，则 `dp[i][j] = dp[i+1][j-1]`。如果不等，那直接死刑，`dp[i][j] = False`。
- **触发条件与边界拦截**：
  我们不能永远指望剥洋葱，当洋葱剥到芯子时，有以下基础基石：
  - **长度为 1 的子串**：自己跟自己肯定是回文。即 `dp[i][i] = True`。
  - **长度为 2 或 3 的子串，且首尾相同**：比如 `"aa"` 或 `"aba"`。当你掐头去尾时，里面要么没了（长度2），要么只剩一个字符（长度3）。这时候根本不需要再查前置小洋葱的状态，只要首尾相同，它**立刻、当场、绝对**就是回文！
    所以当 `s[i] == s[j]` 时，如果 `j - i <= 2`（代表这两个指针收缩到了临界距离），直接设为 `True`。
- **【核心易错点：神奇的遍历顺序】**
  这是本题乃至所有区间 DP 最具杀伤力的地方！如果你还是像往常一样，外层 `for i 从 0 到 n`，内层 `for j 从 0 到 n`，你的代码会彻头彻尾地算出错误答案的！
  - **原因拆解**：注意你的转移依赖 `dp[i+1][j-1]`！这意味着你在算大区间 `(i, j)` 时，你需要先知道**左下方的记录、比它晚一拍出发、早一拍结束的那个短区间的历史数据**！如果用传统的正向双重循环，由于 `i+1` 还没发生，那个较短芯子的历史记录根本就还是空白没算呢！！
  - **破局办法（从中心长度往外长）**：既然大的依赖小的，我们就**把外层循环定为“子串的长度 L”！**只要我们保证是先挨个结算长度为 2 的段、再结算长度为 3 的段... 那么等我们计算长度为 10 的洋葱外皮时，它要求读取的那个长度为 8 的内芯数据肯定就在之前的回合中早早被存好了！这就是区间 DP 最常用的**“长度扩展型遍历”**法则。

#### 代码实现（标准二维 DP）

```python
class Solution:
    def longestPalindrome(self, s: str) -> str:
        n = len(s)
        # 单个或空字符直接返回自己
        if n < 2:
            return s
            
        # dp[i][j] 表示 s[i:j] 是否是回文串
        dp = [[False] * n for _ in range(n)]
        
        # 记录史上遇到的最长记录
        max_len = 1
        start = 0
        
        # 基地石：所有长度为 1 的子串都是回文
        for i in range(n):
            dp[i][i] = True
            
        # 外循环：从长度 L 横向扩展洋葱的尺寸（从 L=2 到 L=n）
        for L in range(2, n + 1):
            # 内循环：锚定洋葱左边的门神 i 的位置
            for i in range(n):
                # 既然知道了左门神 i 和手头的长度 L，右门神 j 的坐标自然固定了
                j = i + L - 1
                
                # 如果这个尺寸的大洋葱右边界已经撑破了整个字符串，那以这个 i 发起的大探索可以直接弃了
                if j >= n:
                    break
                    
                # 验证逻辑：门神对不上就绝对不是
                if s[i] != s[j]:
                    dp[i][j] = False
                else:
                    # 门神对上了！如果是 "aa" (L=2) 或者 "aba" (L=3)，当场验证成功
                    if j - i <= 2:
                        dp[i][j] = True
                    else:
                        # 对于更大的洋葱，完全查阅刚刚上一寸剥剩的内层芯子历史记录
                        dp[i][j] = dp[i + 1][j - 1]
                        
                # 凡是验证成功的结论，只要打破了历史最大长度，立刻上位
                if dp[i][j] and L > max_len:
                    max_len = L
                    start = i
                    
        # 利用最高记录做最后的切片返回
        return s[start:start + max_len]
```

> **💡 多解法联动指路：**
> 本题除了上面正统的“动态规划模板法”外，在面试和真实应用场景下十分常考其**不耗费任何额外辅助数组空间**的解法：基于双指针的“中心扩展法”（可达成极致的空间 $O(1)$ 压缩）。
> 采用该方法时，通过对每个字符以及每两个字符中间的空隙开动双指针往两侧波纹散开对比即可。由于该思路属于“双指针 / 字符串遍历”体系，详细的解析与完整打磨好的代码请移步查阅 [wiki/算法/字符串.md](wiki/算法/字符串.md#L55) 里的对应收录。

#### 复杂度分析
- **时间复杂度**：$O(N^2)$，嵌套了两层对于长度以及起点遍历枚举的双重循环，一共进行了 $\frac{N(N-1)}{2}$ 次状态推导。
- **空间复杂度**：$O(N^2)$，因为严格维护了一个 $N \times N$ 的打卡二维布尔数组 `dp[i][j]`。

## 七、双字符串二维 DP (LCS 模型)

### 最长公共子序列（难度：中等）

给定两个字符串 `text1` 和 `text2`，返回这两个字符串的最长**公共子序列**的长度。如果不存在公共子序列，返回 `0`。

一个字符串的**子序列**是指这样一个新的字符串：它是由原字符串在不改变字符的相对顺序的情况下删除某些字符（也可以不删除任何字符）后组成的新字符串。
例如，`"ace"` 是 `"abcde"` 的子序列，但 `"aec"` 不是 `"abcde"` 的子序列。
两个字符串的**公共子序列**是这两个字符串所共同拥有的子序列。

#### 解题思路

这是面试中考核频率极高的双字符对比模板题（同时也是衍生出更高难度的“编辑距离”一类应用的前置母题）。
对于两个字符串的对比找规律，标准的动态规划做法是画一个二维矩阵框，将第一个字符串映射到行，第二个字符串映射到列，用类似查户口的方式一点点扩散。

- **状态定义**：`dp[i][j]` 表示 `text1` 的前 `i` 个字符和 `text2` 的前 `j` 个字符的**最长公共子序列的长度**。注意因为涉及到个数统计，这就是我们在第一节中讲过的“**必须多留出一格 `+1` 的位置给 `dp[0]` 做基石底层**”的经典代表。
- **状态转移方程**：
  我们拿出目前手电筒照着的 `text1` 的第 `i` 个字符（代码中是 `text1[i-1]`）和 `text2` 的第 `j` 个字符（即 `text2[j-1]`）：
  1. **如果命中（字符相等）**：大声欢呼！说明这两个字符可以双双牵手，直接成功并入到目前发现的子序列队伍末尾。这多出来的 1 个长度，必须毫无争议地加在“**它俩都没有上场前**”的那个独立历史时空（也就是各退一步的 `dp[i-1][j-1]`）身上！
     即：`dp[i][j] = dp[i-1][j-1] + 1`
  2. **如果错过（字符不相等）**：说明此时手电筒照着的这两个字符，**绝对不可能**携手结对出现在最长公共子序列的末尾。那路怎么走？我们要么大度地让 `text1` 退一步（试试不带这个字符去对比行不行），要么让 `text2` 退一步，去之前那两个被斩断的平行时空里，挑一个收获最大的结果继承给现在的时空。
     即：`dp[i][j] = max(dp[i-1][j], dp[i][j-1])`
- **初始化**：
  只要 `text1` 或 `text2` 中有一个长度为 0（即碰上了第 0 行或者第 0 列），那它俩的公共子集长度绝对是 0 没跑了。因为我们在构建 `(m+1) x (n+1)` 的矩阵时直接全填充为 `0`，这部分基石天生就初始化好了。

#### 代码实现（标准二维 DP 矩阵）

```python
class Solution:
    def longestCommonSubsequence(self, text1: str, text2: str) -> int:
        m, n = len(text1), len(text2)
        
        # 1. 明确定义 dp 数组，为了给“前面0个字符”的基位留空，双向加一扩展矩阵
        dp = [[0] * (n + 1) for _ in range(m + 1)]
        
        # 2. 状态转移计算
        for i in range(1, m + 1):
            for j in range(1, n + 1):
                # 检查这两位兄弟对不对得上眼（注意 dp 中的索引 i 对应字符串的下标即 i-1）
                if text1[i - 1] == text2[j - 1]:
                    # 相等，各退一格的辉煌历史再加 1
                    dp[i][j] = dp[i - 1][j - 1] + 1
                else:
                    # 没缘分，要么上面的兄弟退步，要么左边兄弟退步，继承里面记录最强的那个
                    dp[i][j] = max(dp[i - 1][j], dp[i][j - 1])
                    
        # 两人较量到最后一刻的结局，自然躺在了矩阵的最右下角
        return dp[m][n]
```

#### 复杂度分析
- **时间复杂度**：$O(M \times N)$，其中 $M$ 和 $N$ 是 `text1` 和 `text2` 的长度。因为我们稳扎稳打算满了一整个二维表。
- **空间复杂度**：$O(M \times N)$，开辟了完整的二维辅助矩阵用于打表。*(注：与《不同路径》一致，其实也能使用上下两行滚动的一维数组将空间彻底压缩至 $O(\min(M, N))$，但在面试实战与理解中，这种原汁原味的二维修建架构依然是最具展现力的第一手底牌。)*

### 编辑距离（难度：困难）

给你两个单词 `word1` 和 `word2`， 请返回将 `word1` 转换成 `word2` 所使用的最少操作数。
你可以对一个单词进行如下三种操作：
- 插入一个字符
- 删除一个字符
- 替换一个字符

#### 解题思路

这是与上面《最长公共子序列》同宗同源，但更为宏大和经典的“双字符串二维 DP”究极母题！只要搞懂了这道题的增删改状态转移，所有的字符串模糊匹配、拼写检查问题都会不攻自破。

- **状态定义**：`dp[i][j]` 表示将 `word1` 的前 `i` 个字符转换成 `word2` 的前 `j` 个字符所需要的最少操作数。同样，因为包含空字符串的状态，数组大小需要是 `(m+1) x (n+1)`。
- **状态转移方程**：
  我们同样审视 `word1[i-1]` 和 `word2[j-1]` 这两位当前考察的末尾字符：
  1. **如果相等（`word1[i-1] == word2[j-1]`）**：
     谢天谢地！既然这俩最后一个字母本来就一样，那咱们在这个位置根本**不需要做任何操作**。把这个免费的相同字母剥离，需要的总操作数其实就完全等于它俩各自少一个字母时候的操作数。
     即：`dp[i][j] = dp[i-1][j-1]`
  2. **如果不等（需要搞操作了！）**：
     不相等也没关系，题目给了我们三种魔法。我们试试哪种魔法代价最小（基础代价是 1，在这个基础上加上各自平行时空的历史代价）：
     - **替换操作**：既然最后两个不一样，那直接一发替换魔法，强行把 `word1[i-1]` 变成 `word2[j-1]`！那么前面的字符只要各自匹配就行了。历史代价是 `dp[i-1][j-1]`。总代价：`dp[i-1][j-1] + 1`
     - **删除操作**：假设我把 `word1` 当前这个烂字母 `word1[i-1]` 给强行删了！既然删了，那 `word1` 的前 `i-1` 个字母就得负责去拼凑出整个 `word2` 的前 `j` 个字母。历史代价是 `dp[i-1][j]`。总代价：`dp[i-1][j] + 1`
     - **插入操作**：假设我在 `word1` 后面白嫖插入一个和 `word2[j-1]` 一模一样的字母，这就把 `word2` 最后一个搞定了。那么 `word1` 原本的前 `i` 个字母就得去负责搞定 `word2` 剩下的前 `j-1` 个字母。历史代价是 `dp[i][j-1]`。总代价：`dp[i][j-1] + 1`
     最终，在这三条平行世界里挑个代价最小的：
     即：`dp[i][j] = min(dp[i-1][j-1], dp[i-1][j], dp[i][j-1]) + 1`
- **初始化（极其关键的三面墙）**：
  - `dp[0][0] = 0`：空变空，不用操作。
  - `dp[i][0] = i`：当你想把长度为 `i` 的字符串变成一个空字符串（`j=0`），唯一的做法就是**一路删除**到底，操作数为 `i`。
  - `dp[0][j] = j`：当你想把空字符串变成长度为 `j` 的字符串，唯一的做法就是**疯狂插入** `j` 次，操作数为 `j`。

#### 代码实现（标准二维 DP 矩阵）

```python
class Solution:
    def minDistance(self, word1: str, word2: str) -> int:
        m, n = len(word1), len(word2)
        
        # 1. 明确定义 dp 数组，为了给空字符基石留出位置，扩大一维
        dp = [[0] * (n + 1) for _ in range(m + 1)]
        
        # 2. 疯狂建立叹息之墙（把 word1 全删完，或者往空 word1 里全插满）
        for i in range(1, m + 1):
            dp[i][0] = i
        for j in range(1, n + 1):
            dp[0][j] = j
            
        # 3. 开始状态转移，填满大坑
        for i in range(1, m + 1):
            for j in range(1, n + 1):
                if word1[i - 1] == word2[j - 1]:
                    # 相等，不需要任何额外操作，继承砍掉这俩字符的历史大盘
                    dp[i][j] = dp[i - 1][j - 1]
                else:
                    # 不等，在替换(左上)、删除(上)、插入(左)这三条路里选个花钱最少的，再交上本次的1块钱操作费
                    dp[i][j] = min(dp[i - 1][j - 1], dp[i - 1][j], dp[i][j - 1]) + 1
                    
        return dp[m][n]
```

#### 复杂度分析
- **时间复杂度**：$O(m \times n)$，我们需要填满整个二维表格的数据。
- **空间复杂度**：$O(m \times n)$，同样建立了一个略大于原始长度的整张状态矩阵（包含边界初始化）。
