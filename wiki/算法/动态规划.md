> [[算法_MOC|返回算法导航]]

## 一、动态规划核心思想与打表套路

动态规划（DP）本质是一种**利用历史记录，避免重复计算**的优化思想。面试中百分之九十的 DP 题，只要能写出**状态转移方程**，代码就迎刃而解。

**核心做题骨架（DP五部曲）**：
1. **定义 dp 数组的物理含义**：`dp[i]` 到底代表什么？是“到第 i 阶楼梯的方法数”还是“以 nums[i] 结尾的最长子序列”？
    - **【易混辨析：DP 数组初始化长度是 `n` 还是 `n + 1`？】**
        - **定义为“索引位置”时，取 `n`**：当 `dp[i]` 需要与给定数组 `nums[i]` **严格对齐绑定属性**时（例如：“打家劫舍”考虑前 `i` 个房间，“最长递增子序列”强调以 `nums[i]` 结尾），原数组最后一个下标必定为 `n-1`，所以你只需要长度为 `n` 就可以完全覆盖，最终答案常常取决于所有求出的局部最大值（即 `max(dp)` 或 `dp[-1]`）。
        - **定义为“数量/容量/总量”时，必取 `n + 1`**：当遇到“经过前 `n` 个字符拆串”、“凑齐总金额 `amount`”、“爬到第 `n` 阶楼梯”这种衡量数字大小、算数量的题目时，**必须多留出一格 `+1` 的位置给 `dp[0]`**。因为动态规划需要一个合法的“基石”，如“凑齐金额 0”、“0个字符”去启动整个状态机。这样构建出来的数组，你想查容量为 `n` 的解，就刚好直接索引 `dp[n]`（否则如果是只建长为 `n` 的数组，是不存在下标为 `n` 可供提取的）。
2. **写出状态转移方程**：今天的状态由昨天哪些状态推导而来？（如 `dp[i] = dp[i-1] + dp[i-2]`）
3. **初始化基点**：起跑线在哪里？（如 `dp[0] = 0, dp[1] = 1`）
4. **确定遍历顺序**：是从左到右，还是从右到左？对于背包问题，内外层循环顺序极为讲究！
5. **举例打印 dp 数组验证**。

## 二、基础一维序列 DP

### 爬楼梯（难度：简单）

假设你正在爬楼梯。需要 `n` 阶你才能到达楼顶。每次你可以爬 1 或 2 个台阶。你有多少种不同的方法可以爬到楼顶呢？

#### 解题思路

- **状态定义**：`dp[i]` 表示爬到第 `i` 阶楼梯的方法数。
- **状态转移方程**：到达第 `i` 阶只能从 `i-1` 阶跨 1 步上来，或者从 `i-2` 阶跨 2 步上来。因此总方法数是这两种途径之和：`dp[i] = dp[i-1] + dp[i-2]`。
- **初始化**：`dp[0] = 1`（原地不动也算一种方法），`dp[1] = 1`。

作为动态规划的第一题，我们首先应当用标准的 DP 数组模板来严格定义每一维度的状态。掌握了完整的模板推导后，我们再考虑空间优化。由于只依赖前两项，可以用两个变量滚动前进。

#### 代码实现（标准一维 DP 数组）

```python
class Solution:
    def climbStairs(self, n: int) -> int:
        if n <= 1:
            return 1
        # 1. 明确定义 dp 数组，dp[i] 爬到 i 阶的方法数
        dp = [0] * (n + 1)
        
        # 2. 初始状态
        dp[0] = 1
        dp[1] = 1
        
        # 3. 状态转移计算
        for i in range(2, n + 1):
            dp[i] = dp[i - 1] + dp[i - 2]
            
        return dp[n]
```

#### 代码实现（空间优化：滚动变量）

```python
class Solution:
    def climbStairs(self, n: int) -> int:
        if n <= 2:
            return n
            
        # dp1 代表 dp[i-2], dp2 代表 dp[i-1]
        dp1, dp2 = 1, 2
        for _ in range(3, n + 1):
            dp1, dp2 = dp2, dp1 + dp2
            
        return dp2
```

### 打家劫舍（难度：中等）

小偷计划偷窃沿街的房屋，相邻的房屋装有互连的防盗系统（如果两间相邻的房屋在同一晚上被小偷闯入，系统会自动报警）。给定一个代表每个房屋存放金额的非负整数数组，计算在不触动警报的情况下，一夜之内能够偷窃到的最高金额。

#### 解题思路

- **状态定义**：`dp[i]` 代表考虑前 `i` 间房子，所能偷到的最高金额。
- **状态转移方程**：面对第 `i` 间房，小偷只有两个选择：
  1. **偷这间房**：那么第 `i-1` 间房绝不能偷。此时金额为前 `i-2` 间房能偷到的最高金额加上当前这间房的金额：`dp[i-2] + nums[i]`
  2. **不偷这间房**：那么就可以带走前 `i-1` 间房偷来的最高金额：`dp[i-1]`
  所以，状态转移方程为：`dp[i] = max(dp[i-2] + nums[i], dp[i-1])`
- **初始化**：
  - `dp[0] = nums[0]`：如果街上只有第 0 家（第一家）房子，你能偷到的最高金额自然就是这家房子的金额。
  - `dp[1] = max(nums[0], nums[1])`：当街上有 0 和 1 两家房子时，由于会触发警报，你只能选择金额最大的那家偷（所以不能简单写成 `dp[1] = nums[1]`，因为可能第一家比第二家更有钱）。

同样地，我们应当先写出完整的状态转移数组 `dp` 建立框架思维。理解递推关系后，我们发现 `dp[i]` 只依赖它前面的两个状态 `dp[i-1]` 和 `dp[i-2]`，因此可以采用双变量进行空间滚动优化。

#### 代码实现（标准一维 DP 数组）

```python
class Solution:
    def rob(self, nums: List[int]) -> int:
        if not nums:
            return 0
        if len(nums) == 1:
            return nums[0]
            
        # 1. 明确定义 dp 数组
        dp = [0] * len(nums)
        
        # 2. 初始状态
        dp[0] = nums[0]
        dp[1] = max(nums[0], nums[1])
        
        # 3. 状态转移计算
        for i in range(2, len(nums)):
            dp[i] = max(dp[i - 2] + nums[i], dp[i - 1])
            
        return dp[-1]
```

#### 代码实现（空间优化：滚动变量）

```python
class Solution:
    def rob(self, nums: List[int]) -> int:
        # prev2 = dp[i-2], prev1 = dp[i-1]
        prev2, prev1 = 0, 0
        
        for num in nums:
            # 这里的 prev2 + num 就对应 dp[i-2] + nums[i]
            # 这里的 prev1 对应 dp[i-1]
            prev2, prev1 = prev1, max(prev2 + num, prev1)
            
        return prev1
```

## 三、一维状态挂载（以自身结尾）

### 最长递增子序列 (LIS)（难度：中等）

给你一个整数数组 `nums`，找到其中最长严格递增子序列的长度。

#### 解题思路

- **状态定义**：`dp[i]` 的定义并非“前 i 个元素中最长子序列的长度”，而是“**强制以 nums[i] 结尾的**最长递增子序列的长度”。这是序列 DP 最经典的强制挂载定义法。
- **状态转移方程**：让 `j` 从 `0` 遍历到 `i-1`。只要发现 `nums[i] > nums[j]`，说明 `nums[i]` 可以无缝接在 `nums[j]` 的屁股后面，形成一个更长的递增序列！则 `dp[i] = max(dp[i], dp[j] + 1)`。
- **返回值要点（易错坑）**：**千万不能返回 `dp[-1]`**！因为最长的递增子序列不一定是以整个数组的最后一个数字结尾的。例如 `nums = [1, 2, 3, 4, 5, 0]`，以最后一个特征值 `0` 结尾的最长子序列只能是 `[0]` 本身（长度为 1），而这数组真正的最长递增序列是前五个数（长度为 5）。因此，必须返回各个节点中打出的最大值 `max(dp)`。

#### 代码实现（Python版 - DP O(N^2)）
```python
class Solution:
    def lengthOfLIS(self, nums: List[int]) -> int:
        if not nums:
            return 0
            
        # 所有元素各自本身就是个长度为 1 的子序列
        dp = [1] * len(nums)
        
        for i in range(len(nums)):
            for j in range(i):
                if nums[i] > nums[j]:
                    dp[i] = max(dp[i], dp[j] + 1)
                    
        return max(dp)
```
*(注：本题还存在 `贪心 + 二分查找` 达成 `O(N log N)` 的高级解法，建议在二分查找篇复习)*

### 乘积最大子数组（难度：中等）

给你一个整数数组 `nums` ，请你找出数组中乘积最大的非空连续子数组（该子数组中至少包含一个数字），并返回该子数组所对应的乘积。测试用例的答案是一个 32-位 整数。
*请注意，一个只包含一个元素的数组的乘积是这个元素的值。*

#### 解题思路

这道题同样是经典的“**强制挂载（以自身结尾）**”模型！如果要求“连续子数组”，那么状态定义就必须是“以 `nums[i]` 结尾的子数组”。
但它的一大难点在于**乘法的负负得正**特性：当前位置的最大乘积，不一定是由前一个位置的“最大乘积”推导来的。如果当前数字是负数，它乘以前面一个**极负的最小乘积**，就会瞬间翻盘变成一个极大的正数！

- **状态定义**：我们需要同时维护两个 DP 状态：
  - `max_dp[i]`：**强制以 `nums[i]` 结尾**的连续子数组的最大乘积。
  - `min_dp[i]`：**强制以 `nums[i]` 结尾**的连续子数组的最小乘积。
- **状态转移方程**：
  面对当前的 `nums[i]`，你有三种选择拼成当前的连续子数组：
  1. 顺延接着前面的最大乘积走：`max_dp[i-1] * nums[i]`
  2. 顺延接着前面的最小乘积走（期望负负得正）：`min_dp[i-1] * nums[i]`
  3. 嫌弃前面的序列（比如前面乘了个 0），自己单干：`nums[i]` 本身
  
  所以：
  `max_dp[i] = max(max_dp[i-1] * nums[i], min_dp[i-1] * nums[i], nums[i])`
  `min_dp[i] = min(max_dp[i-1] * nums[i], min_dp[i-1] * nums[i], nums[i])`
- **初始化**：`max_dp[0] = nums[0]`，`min_dp[0] = nums[0]`。
- **返回值要点**：依然是所有“以各个元素结尾的最强时刻”里的最大值（跟 LIS 大坑一样，不能直接取 `[-1]`）。

和爬楼梯、打家劫舍一样，我们先写出保留了完整状态数组的标准 DP 写法，直观感受两个 DP 数组相爱相杀的全过程。

#### 代码实现（标准一维 DP 数组）

```python
class Solution:
    def maxProduct(self, nums: List[int]) -> int:
        if not nums:
            return 0
        
        n = len(nums)
        # 1. 明确定义 dp 数组
        # max_dp[i] 表示以 i 结尾的最大乘积
        # min_dp[i] 表示以 i 结尾的最小乘积
        max_dp = [0] * n
        min_dp = [0] * n
        
        # 2. 初始状态
        max_dp[0] = nums[0]
        min_dp[0] = nums[0]
        
        # 记录全局最大值
        ans = nums[0]
        
        # 3. 状态转移计算
        for i in range(1, n):
            num = nums[i]
            max_dp[i] = max(max_dp[i-1] * num, min_dp[i-1] * num, num)
            min_dp[i] = min(max_dp[i-1] * num, min_dp[i-1] * num, num)
            
            # 记录历史上出现过的最高点
            ans = max(ans, max_dp[i])
            
        return ans
```

**空间优化**：仔细观察上面的循环，每次计算第 `i` 个状态只依赖于第 `i-1` 个状态的情报。那些 `i-2` 及以前的格子的数据完全废弃了。因此我们可以去掉长长的数组，直接用两个变量 `cur_max` 和 `cur_min` 来回滚动覆盖，达成 `O(1)` 的空间复杂度。

#### 代码实现（空间优化：滚动变量）

```python
class Solution:
    def maxProduct(self, nums: List[int]) -> int:
        if not nums:
            return 0
            
        # 初始的局部最大、局部最小、全局最大都是第一个元素
        cur_max = nums[0]
        cur_min = nums[0]
        ans = nums[0]
        
        for i in range(1, len(nums)):
            num = nums[i]
            # 因为 cur_max 会被覆盖，所以在计算 cur_min 之前先存一下
            temp_max = cur_max
            
            cur_max = max(temp_max * num, cur_min * num, num)
            cur_min = min(temp_max * num, cur_min * num, num)
            
            # 不断打擂台记录历史上出现过的最高点
            ans = max(ans, cur_max)
            
        return ans
```

### 最长有效括号（难度：困难）

给你一个只包含 `'('` 和 `')'` 的字符串，找出最长有效（格式正确且连续）括号子串的长度。
左右括号匹配，即每个左括号都有对应的右括号将其闭合的字符串是格式正确的，比如 `"(()())"`。

#### 解题思路

这道题是把“一维强制挂载”玩到极致的一道困难题。因为题目要求“**连续子串**”，所以我们的 DP 状态必须严格延续前文一维挂载体系的定海神针——**“强制以当前字符结尾”**。

- **状态定义**：`dp[i]` 表示**以 `s[i]` 为结尾**的最长有效括号子串的长度。
- **初筛（极简剪枝）**：以左括号 `'('` 结尾的字符串绝对不可能是有效的括号结构闭环！所以所有 `s[i] == '('` 的位置直接断定 `dp[i] = 0`，我们只关心 `')'`。
- **状态转移方程**：当 `s[i] == ')'` 时，需要回头看它左边紧挨着的兄弟是谁：
  1. **情况一层层包（左边是 `'('`）**：形如 `......()`。
     这说明这两个符号刚好组成了一对最基础的 `()`，这对括号给自己贡献了 2 的长度。但不要忘了，这一对括号的前面可能还有被连接完好的其他大块工程（比如之前的 `((()))` 连上现在的 `()`），所以还得大喊一声：“前面的兄弟带带我！”将 `dp[i-2]` 的成果自然衔接：
     `dp[i] = dp[i-2] + 2`（前提是不越界）。
  2. **情况二大乱炖（左边依然是 `')'`）**：形如 `......))`。
     此时你前面那兄弟 `)` 可能是一个庞大有效阵法（比如 `(...)`）的结尾。我们要去寻找：在跨过那座庞大有效阵法**更前面一点的位置**，是否有属于我这个外层右括号所痴痴等待的那个左半身 `'('`！
     由于前面的阵法跨度长度由 `dp[i-1]` 记录，所以我们要跳过去找的位置下标就是：`pre = i - 1 - dp[i-1]`。
     如果这个 `pre` 没有越界，且恰好 `s[pre] == '('`，说明我结尾的 `)` 成功穿上了一件包裹里面大结构的外衣！此时我新增的长度是：`里面的阵法长(dp[i-1]) + 我俩的首尾外衣厚度(2)`。同样别忘了如果这件外衣前面还紧接着其他有效的挂载块，还得全额带上（`+ dp[pre - 1]`）。
     经过合并同类项，总长度递推方程为：`dp[i] = dp[i-1] + dp[pre - 1] + 2`
- **返回值要点**：这题再次深刻印证了我们前文讲过的**挂载模型大坑**：由于最大括号子串可以终结在这个长长括号序列的任何半中腰截断，你依然且必定要在循环中用变量搜集所有的浪潮最高点（`max_len = max(max_len, dp[i])`），这是绝不可返回 `dp[-1]` 的连中三元实战局！

#### 代码实现（Python版 - DP）

```python
class Solution:
    def longestValidParentheses(self, s: str) -> int:
        n = len(s)
        if n < 2:
            return 0
            
        # dp[i] 代表以 s[i] 结尾的最长有效括号子串长度
        dp = [0] * n
        max_len = 0
        
        for i in range(1, n):
            # 只有遇到右括号，才有可能形成有效闭环
            if s[i] == ')':
                if s[i - 1] == '(':
                    # 形式像： "....()"，直接和前一个凑成对
                    dp[i] = (dp[i - 2] if i >= 2 else 0) + 2
                else:
                    # 形式像： "....))"，这里前面多半是个完整的有效系统，被我这个新的 ')' 整个包进去了
                    # pre 是尝试跳过那个完整大系统，去找属于我真正的另一半 '(' 的坐标
                    pre = i - 1 - dp[i - 1]
                    if pre >= 0 and s[pre] == '(':
                        # 既然越过了大系统且对上了，那就组装成了超级包壳大系统
                        dp[i] = dp[i - 1] + 2
                        # 别忘了这个大系统外头如果还顶着其他并列的有效块，还得全量相连！
                        if pre - 1 >= 0:
                            dp[i] += dp[pre - 1]
                            
                # 不断打擂台记录历史上出现过的最长连击局部
                max_len = max(max_len, dp[i])
                
        return max_len
```

## 四、背包问题 (Knapsack)

背包问题分类中，最最常见的有两大宗门：**0-1背包**（每种物品只能用一次）和**完全背包**（每种物品可以无限用）。它们在一维 DP 里的核心区别非常简单：就是内层容量的循环方向。

### 分割等和子集（难度：中等）

给你一个只包含正整数的非空数组 `nums` 。请你判断是否可以将这个数组分割成两个子集，使得两个子集的元素和相等。

#### 解题思路

这是最经典的 **0-1 背包问题** 变种。
我们把问题翻译一下：“将数组分割成和相等的两份”其实就是问：**能不能从数组里挑选出一堆数字，让它们的和恰好等于整个数组总和的一半（`sum(nums) / 2`）？** 
这就彻底变成了一个典型的 0-1 背包：我们的背包总容量是 `sum / 2`，物品就是数组里的每一个数字，每个物品的“体积”和“价值”就是这个数字本身。我们要在给定的物品中看看能不能刚好把包塞满。

- **状态定义**：`dp[j]` 表示背包容量为 `j` 时，能否用前部分数字恰好把它装满。
- **状态转移方程**：对于当前正在考察的新物品数字 `num`，我们可以选择“装它”：如果决定装它，目标就变成考察以前我们的背包能不能装满 `j - num` 这点容量。也就是 `dp[j] = dp[j] or dp[j - num]`。只要这两种方案有一条能走通，`dp[j]` 就大功告成了。
- **初始化**：`dp[0] = True`。装满容量为 0 的背包的方法就是永远“什么都不装”。作为所有后续推理唯一的根基点。
- **【直击灵魂：为什么要有两层循环？分别在干什么？】**
  这个问题对于没写过二维 DP 直接跳一维滚动数组的人来说极其困惑，本质上，这两层循环的分工是极其明确的：
  - **外层循环（遍历物品 `for num in nums`）**：你可以把整个过程想象成站在流水线旁边，数组 `nums` 里的数字就是一个个送过来的“快递盲盒”。每新送过来一个数字 `num`，你都要拿着它，基于它**去刷新一遍你手里的那个“全村希望的账本”**（即 `dp` 数组）。
  - **内层循环（遍历容量 `for j in range(...)`）**：你的总账本上密密麻麻记录了容量从 `1` 斤到 `target` 斤的各种各样的空背包“目前能不能刚好装满”。当你手里攥着传送带新送来的、重量为 `num` 的新物品去问容量为 `j` 的背包时，你会满怀期待地想：“兄弟，我手里拿着 `num` 这块砖，如果以前（就是上一轮的账本）那些老数字们能成功填满 `j - num` 那个稍小点儿的坑位（即 `dp[j-num] == True`），那我今天顺手把这块重量刚好是 `num` 的砖“咔嗒”一声塞进去，你不就完美缝合到了正好容量是 `j` 的大圆满了吗！”因此，这就是用新积木去挨个容量去试填的过程。
- **【核心易错坑：内层循环为什么要逆序？】**：本题是 0-1 背包：**数组里的数每个人仅仅只能拿一次！**
  - 如果你**正序遍历**（从 `num` 一路推高到 `target`），当你在算大容量需要用到小容量的旧数据 `dp[j-num]` 时，那个 `dp[j-num]` 可能已经在这一轮刚刚被当前这个 `num` 给填满改变过了！这就等同于你把当前数字 `num` 放进包里了**两次甚至很多次**。它就变成了后面题目的完全背包。
  - 要解决这个问题，防止串台的究极办法是：**我们必须从大到小去逆向刷新数组（从 `target` 降序滑落到 `num`）**。当你计算高位的 `dp[j]` 需要依赖左边小位的 `dp[j-num]` 的时候，因为你是从右往左抹过来的，所以左边那些小的值还保留在上一轮比赛（即还没受到当前新数字洗礼）原封未动的原样！这样，当前数字就绝对安全地只会被装进去一次。这就是 0-1 背包滚动的精髓。
- **【逻辑细节：内层循环的终点为什么停在 `num - 1`？】**
  - 在 Python 中的 `range(start, stop, step)` 函数是“前闭后开”的，意味着它会在到达 `stop` 的前一个时刻停止。因此 `range(target, num - 1, -1)` 的实际效果是：从 `target` 容量开始往下降，**降到代表当前物品重量的 `num` 为止**。
  - **物理意义上的原因**：如果当前背包的容量 $j$ 比新来的这块砖的重量 `num` 还要小（即 $j < num$），那你这容量可怜的小背包根本**塞不进去**手里这块大砖头！既然塞不进，那背包的状态 `dp[j]` 就无法借助这块新砖来更新，只能乖乖继承上一轮的老样子。因为一维数组本身原位不动就是继承老样子，所以我们干脆连算都不去算它了，直接把 $j < num$ 的情况剪枝掉。这就是为什么内循环卡在 `num`（即 Python 语法里的 `num - 1` 作为终点）就不再往下算的原因。

#### 代码实现（Python版 - 0-1背包模板）

```python
class Solution:
    def canPartition(self, nums: List[int]) -> bool:
        total_sum = sum(nums)
        # 奇数总和，绝大可能不能分成等和两份整数
        if total_sum % 2 != 0:
            return False
            
        # 要拼凑的目标容量是总和的一半
        target = total_sum // 2
        
        # dp[j] 代表能否刚好凑出容量为 j 的和
        dp = [False] * (target + 1)
        dp[0] = True
        
        # 外层遍历每个人/物品（仅使用一次）
        for num in nums:
            # 内层循环遍历容量：为了保证 0-1 背包特性，必须【逆序遍历】防串台！
            for j in range(target, num - 1, -1):
                dp[j] = dp[j] or dp[j - num]
                
        return dp[target]
```

### 零钱兑换（难度：中等）

给你一个整数数组 `coins`，表示不同面额的硬币；以及一个整数 `amount`，表示总金额。计算并返回可以凑成总金额所需的 **最少的硬币个数**。如果没法凑出，返回 `-1`。可以认为每种硬币的数量是无限的。

#### 解题思路

这道题是经典的**完全背包求极值**问题。可以将总金额视作背包容量，各种面额的硬币视作重量不等的物品，由于硬币有无限个，便构成了完全背包。

- **状态定义**：`dp[i]` 表示凑成金额 `i` 所需的最小硬币个数。
- **状态转移方程**：对目标金额 `i`，尝试拿面值为 `coin` 的硬币。如果拿了这一枚，就需要再加上凑出 `i - coin` 的最小个数。因此状态转移方程为：`dp[i] = min(dp[i], dp[i - coin] + 1)`，仅当 `i >= coin` 时能进行。
- **初始化**：既然求“最小”，就把所有元素初始化为正无穷大 `float('inf')`，防止被 0 干扰。唯一的基石是 `dp[0] = 0`，即凑齐 0 元需要 0 枚硬币。

#### 代码实现（Python版）

```python
class Solution:
    def coinChange(self, coins: List[int], amount: int) -> int:
        # dp 数组初始化为无穷大，表示初期认为凑不出
        dp = [float('inf')] * (amount + 1)
        dp[0] = 0
        
        # 外层遍金额，内层遍历硬币（或者反过来也可以，因为求的是全组合的极值而不是排列数）
        for i in range(1, amount + 1):
            for coin in coins:
                if i >= coin:
                    dp[i] = min(dp[i], dp[i - coin] + 1)
                    
        return dp[amount] if dp[amount] != float('inf') else -1
```

### 单词拆分（难度：中等）

给你一个字符串 `s` 和一个字符串列表 `wordDict` 作为字典。请你判断是否可以利用字典中出现的单词拼接出 `s`。注意：不要求字典中出现的单词全部都使用，并且字典中的单词可以重复使用。

#### 解题思路

这也是**完全背包问题**的变种，字典里的各个“单词”就像是待挑选的“物品”，我们要用它们去装满字符串 `s` 这个背包！不同之处在于字符串组合要求**排列顺序必须严格一致**。

- **状态定义**：`dp[i]` 代表字符串前 `i` 个字符组成的子串（即 `s[0:i]`）能否被字典里的单词拼出来。
- **状态转移方程**：利用切断思维，在 `s[0:i]` 内部找一个断点 `j`。如果前半部分 `s[0:j]` 已证实合法（即 `dp[j] == True`），并且后半段切出来的 `s[j:i]` 刚好存在于字典中，那整个长为 `i` 的字符串自然被拼接成功：`dp[i] = True`。
  - **【思维误区一：为什么是判断 `dp[j]`，而不是直接去写 `s[:j] in word_set`？】**
    如果你写成 `s[:j] in word_set`，你的意思是要求 `s[:j]` 这个全长子串必须是一个**又长又完整的、独立出没在字典里的巨型单词**。比如你要拼 `"applepen"`，你的字典里只有 `"apple"` 和 `"pen"`。你此时切出了前半部分 `"applepen"` 去判断 `in word_set`，字典肯定说“没这词儿啊，退回”。
    但如果我们写成 `dp[j]`，那就完全不一样了！**它的含义是：不管你这前半部分 `s[:j]` 有多长、是由多少个碎单词拼合而成的，只要以前历史运算过它“属于合法词汇拼合物”，那么在动态规划的世界里，它就是被点亮的**。这就是在“利用历史记录避免重复验证全串”的动态规划精髓。
  - **【思维误区二：如果循环跑完都没遇到 `break`，我能直接 `return False` 结束程序吗？】**
    绝对不能。这里的内侧循环（也就是寻找断点 `j` 的循环），只是为了给**当下的这短得可怜的 `i` 长度**的前缀尝试寻找一种拼凑方法。找到了，就说明长度为 `i` 的前缀合法，我们 `break` 掉当前 `i` 的找拼合工作，急匆匆地去继续拼下个更长的 `i` 了。
    如果针对这个特定的 `i`，所有的 `j` 切法都试了，都拼不成功，那不 `break` 也就意味着这句长度为 `i` 的阶段性子串拼不出来而已——这对应的底层客观事实是 `dp[i]` 保持为初始化的 `False` 就足够了。你千万不能因为人家半路上有一个小的阶段性残缺词没拼出来，就在循环体里直接判了全盘代码的死刑 `return False`，因为后面说不定更长的 `i` （比如包含了整个字典完整单词汇的长段）就能成功拼上历史合法的积木！这就像人生，阶段性的拼合失败不代表全局崩盘，最后的结局由 `dp[n]` 统一宣判。
- **初始化**：`dp[0] = True`。空字符串默认不需要任何单词就能拼凑出来，这是后续所有转移逻辑能够连通启动的一块重要“基石”。

#### 代码实现（Python版 - 切分挂接法）

```python
class Solution:
    def wordBreak(self, s: str, wordDict: List[str]) -> bool:
        word_set = set(wordDict)  # 换成哈希集合，提速百倍
        n = len(s)
        
        # dp[i] 代表 s 的前 i 个字符组成的子串，能否被合法拼出
        dp = [False] * (n + 1)
        dp[0] = True  # 空串默认是可以拼出
        
        for i in range(1, n + 1):
            # 将前 i 个字符的串一分为二，找一个能接住的断点 j
            for j in range(i):
                if dp[j] and s[j:i] in word_set:
                    dp[i] = True
                    break  # 只要有一种切法能成功就够了！
                    
        return dp[n]
```

### 完全平方数（难度：中等）

给你一个整数 `n` ，返回和为 `n` 的完全平方数的最少数量。完全平方数是一个整数，其值等于另一个整数的平方；换句话说，其值等于一个整数自乘的积。例如，1、4、9 和 16 都是完全平方数，而 3 和 11 不是。

#### 解题思路

这道题在本质上和前面的《零钱兑换》**完全是一模一样**的完全背包问题！只是将无限可选的“硬币面额”换成了无限可选的“完全平方数（$1, 4, 9, 16 \dots$）”，要求从里面挑选最少数量的“物品”来凑满足够的容量 $n$。

- **状态定义**：`dp[i]` 表示凑出目标和为 `i` 所需的最少完全平方数的数量。
- **状态转移方程**：当前面临的目标和为 `i` 时，尝试从中减去一个完全平方数 $j^2$。如果拿掉这个 $j^2$，我们要找的就是剩下的容量 `i - j*j` 所需的最少数量，再加上刚刚拿起的这 $1$ 个。所以递推式为：`dp[i] = min(dp[i], dp[i - j*j] + 1)`。我们需要让 $j$ 从 $1$ 开始逐渐变大，只要 $j^2 \le i$ 就进行尝试截取，从中选出最好的组合方式。
- **初始化**：依然是因为求极小值，把所有元素用无穷大 `float('inf')` 填充满，只设 `dp[0] = 0` 作为没有目标余额不需要物品的基石。

#### 代码实现（Python版）

```python
class Solution:
    def numSquares(self, n: int) -> int:
        dp = [float('inf')] * (n + 1)
        dp[0] = 0
        
        # 遍历所有的容量数字
        for i in range(1, n + 1):
            # j 从 1 开始，只要它的完全平方不超过 i 就试着放入这个“新硬币”
            j = 1
            while j * j <= i:
                dp[i] = min(dp[i], dp[i - j * j] + 1)
                j += 1
                
        return dp[n]
```
