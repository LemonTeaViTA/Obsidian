# 字符串与双指针/哈希表 (String & Pointers)

> [[算法_MOC|返回算法导航]]

## 一、字符串核心思想与归类技巧

字符串问题本质上是**元素受限的数组**问题（往往只包含字母、数字或符号）。但由于其往往用来表示一段“连续语义”，常考察以下三大核心思路：

1. **滑动窗口**：专门处理“最长”、“最短”的连续子串问题。一前一后两个指针相向或同向滑动。
2. **中心扩展与多指针**：处理回文串问题时的经典思想（向两边扩展），或者处理匹配和解析问题。
3. **哈希映射与特定数据结构**：借助哈希表给异位词分组、或使用栈（Stack）来匹配括号。
4. **字符串匹配（KMP）**：在主串中高效定位模式串。利用模式串自身的「最长相等前后缀」信息避免指针回溯，把暴力的 `O(n·m)` 优化到 `O(n+m)`。

## 二、滑动窗口与双指针搜索

### 无重复字符的最长子串（难度：中等）

**题目：**
给定一个字符串 `s`，请你找出其中不含有重复字符的最长子串的长度。
例如：输入 `s = "abcabcbb"`，输出 `3` (最长的是 "abc")。

**【模板解析与思路归纳】：**
- **滑动窗口法**：用 `left` 和 `right` 划定一个窗口，一直向右扩张。
- 当右侧新加入的字符已经在窗口内有重复时，左边界（`left`）直接跳到**引起重复字符的上一次出现位置的下一个位置**。
- 用一个哈希表 `char_map` 记录字符最后一次出现的位置，以支持 O(1) 的左边界跳转。

**答案代码（Python版）：**
```python
class Solution:
    def lengthOfLongestSubstring(self, s: str) -> int:
        char_map = {}
        left = 0
        max_len = 0
        
        for right in range(len(s)):
            # 如果当前字符出现过，且其在哈希表中记录的位置在当前窗口 left 及其右侧，则触发冲突
            if s[right] in char_map and char_map[s[right]] >= left:
                # 左边界直接越过冲突字符
                left = char_map[s[right]] + 1
                
            # 记录当前字符的最新的下标
            char_map[s[right]] = right
            # 更新最大长度
            max_len = max(max_len, right - left + 1)
            
        return max_len
```

### 找到字符串中所有字母异位词（难度：中等）

**题目：**
给定两个字符串 `s` 和 `p`，找到 `s` 中所有 `p` 的异位词的子串，返回这些子串的起始索引。

**【模板解析与思路归纳】：**
- 固定窗口大小为 `len(p)`，用两个 `defaultdict` 分别记录 `p` 的字符频次和当前窗口的字符频次。
- **注意点**：窗口滑动时用 `while` 控制长度，当窗口超过 `len(p)` 时移除最左字符（频次减 1，若为 0 则删除 key）；比较两个 `defaultdict` 是否相等即可判断是否为异位词。长度判断用 `right - left + 1 > len(p)`，不是 `>=`。

**答案代码（Python版）：**
```python
from collections import defaultdict

class Solution:
    def findAnagrams(self, s: str, p: str) -> List[int]:
        p_count = defaultdict(int)
        for c in p:
            p_count[c] += 1

        window = defaultdict(int)
        left = 0
        res = []

        for right in range(len(s)):
            window[s[right]] += 1
            while right - left + 1 > len(p):
                window[s[left]] -= 1
                if window[s[left]] == 0:
                    del window[s[left]]
                left += 1
            if window == p_count:
                res.append(left)

        return res
```

### 最小覆盖子串（难度：困难）

**题目：**
给你一个字符串 `s`、一个字符串 `t`。返回 `s` 中涵盖 `t` 所有字符的最小子串。如果不存在，返回空字符串。

**【模板解析与思路归纳】：**
- **滑动窗口 + 哈希表**：用 `need` 记录 `t` 中各字符所需频次，用 `window` 记录当前窗口中满足需求的部分。
- **关键点**：`formed` 计数表示"已经凑齐的字符种类数"。当 `formed == len(need)` 时说明窗口已覆盖 `t`，可以尝试收缩左边界。
- **注意点**：收缩左边界时要判断 `s[left]` 是否在 `need` 里，只有影响覆盖条件的字符才操作 `formed` 计数；窗口右移后要及时更新答案。

**答案代码（Python 版）：**
```python
from collections import defaultdict

class Solution:
    def minWindow(self, s: str, t: str) -> str:
        if not s or not t:
            return ""
        
        need = defaultdict(int)
        for c in t:
            need[c] += 1
        
        window = defaultdict(int)
        left = 0
        formed = 0
        required = len(need)
        
        min_len = float('inf')
        min_left = 0
        min_right = 0

        for right, char in enumerate(s):
            window[char] += 1
            # 如果当前字符达到需求数量，已满足的种类数 +1
            if char in need and window[char] == need[char]:
                formed += 1
            
            # 当窗口满足条件时尝试收缩
            while formed == required:
                # 更新最优解
                if right - left + 1 < min_len:
                    min_len = right - left + 1
                    min_left = left
                    min_right = right
                
                # 移除左边元素
                left_char = s[left]
                if left_char in need:
                    if window[left_char] == need[left_char]:
                        formed -= 1
                    window[left_char] -= 1
                
                left += 1
        
        if min_len == float('inf'):
            return ""
        
        return s[min_left:min_right + 1]
```

## 三、中心扩展与回文属性

### 最长回文子串（难度：中等）

给你一个字符串 `s`，找到 `s` 中最长的回文子串。

#### 解题思路
- 回文串有两个情况：长度是奇数（有一个独立的中心点），或是偶数（中心在两个字符中间）。
- 我们可以遍历字符串中的每个位置，把它当作“中心”，同时向两边扩展（`l -= 1`，`r += 1`）看两边字等不等，记录下扩得最远的。
- **【核心数学降维：为什么循环 `2*n - 1` 次？】**
  常规写法会把“奇数中心”和“偶数中心”拆成两遍函数调用。但基于数学映射，我们可以用一个统一的循环搞定！
  - 长度为 `n` 的字符串，有 `n` 个实体字符（可作为奇数回文中心），和 `n - 1` 个字符与字符之间的夹缝（可作为偶数回文中心）。加起来正好是 $n + (n-1) = 2 \times n - 1$ 个潜在的回文中心点。
  - 我们让中心点索引 `i` 从 `0` 跑到 `2*n - 2`。通过向下取整的公式：左指针 `l = i // 2`，右指针 `r = (i + 1) // 2`。
    - **当 `i` 是偶数**（比如 0, 2）：`i // 2` 和 `(i+1) // 2` 的结果是一样的（比如都是 1）。左右指针重合在实体字符上，完美模拟了**奇数中心**。
    - **当 `i` 是奇数**（比如 1, 3）：`(i+1) // 2` 会比 `i // 2` 大 1（比如 1求出 l=0, r=1）。左右指针巧妙地一左一右骑在了字符夹缝之间，完美模拟了**偶数中心**！
- 时间复杂度是 $O(N^2)$，空间是 $O(1)$，相比动态规划省去了一大块存储。

#### 代码实现（Python版 - 统一化数学映射代码）
```python
class Solution:
    def longestPalindrome(self, s: str) -> str:
        n = len(s)
        ans_left = ans_right = 0
        
        # 遍历所有 2*n - 1 个奇偶中心点
        for i in range(2 * n - 1):
            # 神奇的映射公式，统一奇数/偶数中心起跳点
            l, r = i // 2, (i + 1) // 2
            
            # 当左右指针未越界且字符相等时，向两边像水波纹一样扩散
            while l >= 0 and r < n and s[l] == s[r]:
                l -= 1
                r += 1
                
            # 退出循环时，说明 s[l] != s[r] 或越界了。
            # 真实回文串的边界是 (l+1) 到 (r-1)，其长度即为 (r-1) - (l+1) + 1 = r - l - 1
            if r - l - 1 > ans_right - ans_left:
                ans_left = l + 1
                ans_right = r
                
        return s[ans_left : ans_right]
```

> **💡 多解法联动指路：**
> 本题除了上述直观的且空间占用最优的“中心扩展法”外，利用状态转移特性的**二维区间 DP**也是极为正统的模板题。掌握它能解答一切回文字符串相关的扩展难题（如分割回文串数量等）。
> 关于如何建立打表矩阵、以及其“依据子串长度横向遍历”这种防坑的遍历顺序，请双向参阅：[wiki/算法/动态规划.md](wiki/算法/动态规划.md#六区间-dp-字符串二维模型) 中的详尽推演。

## 四、字符串编码与哈希运用

### 字母异位词分组（难度：中等）

**题目：**
给你一个字符串数组，请你将字母异位词组合在一起。可以按任意顺序返回结果列表。（字母异位词指由相同字母重新排列构成的所有字符串，也就是组成的字符和每个字符的频次完全相同）。

**【模板解析与思路归纳】：**
- 互为异位词的话，**若将它们进行内部排序，排序后得到的字符串必定是全宇宙唯一的同一个**。
- 因此，把排序后的字符串作为字典的 `key`，把原字符串 `append` 到对应 key 的列表中。
- **注意点**：`defaultdict(list)` 的类型必须是 `list`，因为每个 key 对应多个原字符串需要列表收集；最终 `return list(anagram_map.values())` 将 dict_values 转为 `List[List[str]]` 符合题目返回要求。

**答案代码（Python版）：**
```python
import collections

class Solution:
    def groupAnagrams(self, strs: List[str]) -> List[List[str]]:
        anagram_map = collections.defaultdict(list)
        
        for s in strs:
            # 将字符串内部排序后当成 hash key
            key = "".join(sorted(s))
            anagram_map[key].append(s)
            
        return list(anagram_map.values())
```

## 五、大数运算与位操作模拟

### 字符串相乘（难度：中等）

**题目：**
给定两个以字符串形式表示的非负整数 `num1` 和 `num2`，返回它们的乘积（也表示为字符串形式）。不能使用内置的 BigInteger 库或者把它们全转成整数。

**【模板解析与思路归纳】：**
- 想想小学的“竖式乘法”是怎么列的！
- 长度为 `m` 和 `n` 的两个数字相乘，结果的最大长度不会超过 `m + n`。我们可以分配一个长度为 `m+n` 的全 0 数组来存每一位。
- 规律：`num1[i]` 和 `num2[j]` 的乘积，它最终会落在数组结果的 `[i+j]`（进位） 和 `[i+j+1]`（个位） 处！

**答案代码（Python版）：**
```python
class Solution:
    def multiply(self, num1: str, num2: str) -> str:
        if num1 == "0" or num2 == "0":
            return "0"
            
        m, n = len(num1), len(num2)
        res = [0] * (m + n)
        
        # 从最后一位开始往前乘
        for i in range(m - 1, -1, -1):
            for j in range(n - 1, -1, -1):
                # 两项小乘积
                mul = int(num1[i]) * int(num2[j])
                
                # 它们对应的 res 数组里的位置
                p1 = i + j      # 高位（进位的地方）
                p2 = i + j + 1  # 低位（本身数字的地方）
                
                # 叠加在这之前的进位记录
                total = mul + res[p2]
                
                # 把最新的进位留在 p1, 余数留在 p2
                res[p2] = total % 10
                res[p1] += total // 10
                
        # 结果数组里前面可能会有前导0，把它剔除并转成字符
        result_str = "".join(map(str, res)).lstrip('0')
        return result_str
```

## 六、特殊结构：栈的扫描匹配

### 最长有效括号（难度：困难）

**题目：**
给你一个只包含 `'('` 和 `')'` 的字符串，找出最长有效（格式正确且连续）括号子串的长度。

**【模板解析与思路归纳】：**
- 用栈存**下标**，而不是存括号本身！这就类似一把“记录错误边界的尺子”。
- 一开始先把 `-1` 压入栈中，表示“我们在 -1 这个位置，之前啥都没有”。
- 遇到 `(`，无脑将下标压栈。
- 遇到 `)`，先弹出一个栈顶元素（因为这一对匹配掉了）。如果这时候栈空了（说明刚才弹走的甚至不是左括号，而是用来做边界的地基），说明当前的 `)` 是个非法的边界，把它的下标作为新的地基地基压入。如果不空，当前下标减去新的栈顶元素，就是你刚刚完成匹配的这一整段合法区间的长度！

**答案代码（Python版）：**
```python
class Solution:
    def longestValidParentheses(self, s: str) -> int:
        # 放入 -1 作为边界地基保证可以计算长度
        stack = [-1]
        max_len = 0
        
        for i, char in enumerate(s):
            if char == '(':
                # 左括号记录坐标
                stack.append(i)
            else:
                # 遇到右括号，匹配掉上一个左括号
                stack.pop()
                
                # 如果栈空了，说明刚才弹掉的是栈底的守门员，表示目前的右括号是多余的，成为新的合法区间的边界
                if not stack:
                    stack.append(i)
                else:
                    # 如果栈没空，用当前索引减去这时候暴漏出来的栈顶（上一个边界或者没匹配完的左括号）就是有效长度
                    max_len = max(max_len, i - stack[-1])
                    
        return max_len
```

## 七、字符串匹配：KMP

### 找出字符串中第一个匹配项的下标（难度：简单）

**题目：**
给你两个字符串 `haystack` 和 `needle`，请你在 `haystack` 字符串中找出 `needle` 字符串的第一个匹配项的下标（下标从 0 开始）。如果 `needle` 不是 `haystack` 的一部分，则返回 `-1`。

#### 解题思路

暴力匹配每次失配都要把主串指针退回去重新比，最坏 `O(n·m)`。KMP 的核心洞察是：**模式串自己最清楚自己长什么样**——失配时不必从头再来，而是利用已经匹配过的那段信息，让模式串“滑动”到下一个有希望的位置，主串指针永不回退，做到 `O(n+m)`。

关键是预处理出模式串的 **`next` 数组（也叫前缀函数）**：`next[i]` 表示子串 `needle[0..i]` 中，**最长的「既是真前缀又是真后缀」的长度**。

- 「前缀」指不含最后一个字符的所有头部子串，「后缀」指不含第一个字符的所有尾部子串。
- 例如 `"ABAB"` 的最长相等前后缀是 `"AB"`，长度 2。
- 一旦在主串某位失配，说明模式串已匹配的那段前缀里，长度为 `next[j-1]` 的头部和刚扫过的尾部完全相同，可以直接把模式串指针跳到 `next[j-1]`，省去重复比较。

求 `next` 数组本身就是一次「模式串和自己做匹配」的过程：用 `j` 指向当前最长相等前后缀的末尾，遇到 `needle[i] == needle[j]` 就一起前进，失配则让 `j` 回退到 `next[j-1]`。

#### 代码实现

```python
class Solution:
    def strStr(self, haystack: str, needle: str) -> int:
        if not needle:
            return 0

        # 1. 构建 next 数组（前缀函数）：next[i] = needle[0..i] 的最长相等前后缀长度
        n = len(needle)
        nxt = [0] * n
        j = 0  # j 指向当前候选前缀的末尾，也等于已匹配的前缀长度
        for i in range(1, n):
            # 失配则回退 j，直到匹配上或回到开头
            while j > 0 and needle[i] != needle[j]:
                j = nxt[j - 1]
            if needle[i] == needle[j]:
                j += 1
            nxt[i] = j

        # 2. 主串匹配：i 永不回退，失配时只回退模式串指针 j
        j = 0
        for i in range(len(haystack)):
            while j > 0 and haystack[i] != needle[j]:
                j = nxt[j - 1]
            if haystack[i] == needle[j]:
                j += 1
            # 模式串走到头，匹配成功
            if j == n:
                return i - n + 1
        return -1
```

#### 复杂度分析

- **时间复杂度**：`O(n + m)`，`n`、`m` 分别为主串与模式串长度。构建 `next` 与主串扫描中，`j` 的总回退次数不超过其总增长次数，因此各自摊还 `O(m)` 与 `O(n)`。
- **空间复杂度**：`O(m)`，存储 `next` 数组。

### 重复的子字符串（难度：简单）

**题目：**
给定一个非空字符串 `s`，检查它是否可以由它的一个子串重复多次构成。

#### 解题思路

KMP `next` 数组的经典应用。设字符串长度为 `n`，最长相等前后缀长度记为 `k = next[n-1]`。

- 若 `s` 由某个子串重复构成，则**最小重复单元的长度为 `n - k`**。
- 当且仅当 `k > 0` 且 `n` 能被 `n - k` 整除时，`s` 可由其子串重复多次构成。

直觉：最长相等前后缀重叠的部分越长，说明字符串自我「错位对齐」得越严整，错位的步长 `n - k` 正好就是那个周期。

#### 代码实现

```python
class Solution:
    def repeatedSubstringPattern(self, s: str) -> bool:
        n = len(s)
        nxt = [0] * n
        j = 0
        for i in range(1, n):
            while j > 0 and s[i] != s[j]:
                j = nxt[j - 1]
            if s[i] == s[j]:
                j += 1
            nxt[i] = j

        k = nxt[n - 1]
        # 存在非空相等前后缀，且总长能被「周期 n-k」整除
        return k > 0 and n % (n - k) == 0
```

#### 复杂度分析

- **时间复杂度**：`O(n)`，一次 `next` 数组构建。
- **空间复杂度**：`O(n)`，存储 `next` 数组。
