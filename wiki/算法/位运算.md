# 位运算 (Bit Manipulation)

> [[算法_MOC|返回算法导航]]

## 一、位运算核心思想与归类技巧

位运算虽然在平时开发中用得不多，但在算法和底层源码中却极为常见。它的核心优势是**极致的性能**和**利用计算机底层的加解密特性**。

### 🎯 模式识别信号（看到这些特征就该想到位运算）

| 题目特征 / 关键词 | 优先用的位运算 Trick |
|-----------------|---------------------|
| "只出现一次" / "多数出现两次，一个出现奇数次" | **异或 `^`**（自消除特性） |
| "不使用加减乘除做加法" / 数位相关 | **异或 + 与 + 左移** 模拟加法 |
| "判断是否为 2 的幂" / "统计二进制中 1 的个数" | **`n & (n-1)`** 消除最低位 1 |
| "交换两个数不用临时变量" | **异或** 三连 |
| "找缺失的数字 / 找重复的数字" | 异或 或 **`n & (-n)`** 分组 |
| 状态压缩 DP（如 TSP 旅行商） | **位掩码** 表示集合 |
| 子集枚举 | `for mask in range(1 << n)` |

### 🧠 必须形成肌肉记忆的 8 个核心 Trick

```python
# 1. 判断奇偶（比 % 2 快）
n & 1 == 1   # 奇数
n & 1 == 0   # 偶数

# 2. 乘除 2 的幂
n << k       # n * 2^k
n >> k       # n / 2^k

# 3. 获取最低位的 1（树状数组基石，lowbit）
n & (-n)     # 比如 n = 0b1100，结果是 0b0100

# 4. 消除最低位的 1（判断 2 的幂、数 1 的个数）
n & (n - 1)  # 比如 n = 0b1100 → 0b1000

# 5. 判断某一位是否为 1
(n >> i) & 1

# 6. 把第 i 位置 1
n | (1 << i)

# 7. 把第 i 位清零
n & ~(1 << i)

# 8. 翻转第 i 位
n ^ (1 << i)
```

### 🧬 异或 `^` 的三大天性（面试最高频）

- 自消除：`a ^ a = 0`
- 保留身份：`a ^ 0 = a`
- 满足交换律与结合律：`a ^ b ^ a = b`

> [!tip] 异或为什么叫"无进位加法"？
> `1 ^ 1 = 0`（不进位）、`1 ^ 0 = 1`、`0 ^ 0 = 0`，正好是加法忽略进位的结果。这就是为什么后面的"不用加减做加法"题可以用异或+与来实现。

## 二、异或全家桶

### 只出现一次的数字（难度：简单）

**题目：**
给你一个非空整数数组 `nums`，除了某个元素只出现一次以外，其余每个元素均出现两次。找出那个只出现了一次的元素。要求 `O(n)` 时间、`O(1)` 空间。

#### 🔨 思维推导链

**1. 暴力解**：用哈希表记录每个数的出现次数，最后找出次数为 1 的 → 时间 `O(n)`，空间 `O(n)`。❌ 空间不达标。

**2. 瓶颈在哪**：哈希表占用的空间是因为我们需要"记住"每个数字出现过几次。有没有一种"免记忆"的自消除机制？

**3. 关键洞察**：题目的"其余每个元素均出现两次"，配合**异或的自消除特性 `a ^ a = 0`**——成对的数字一异或就消失了，留下的就是那个唯一的！

**4. 最优解**：一次遍历，把所有数字异或起来，最终结果就是答案。

```python
class Solution:
    def singleNumber(self, nums: List[int]) -> int:
        res = 0
        for num in nums:
            res ^= num
        return res
```

**复杂度**：时间 `O(n)`，空间 `O(1)`。

### 只出现一次的数字 II（难度：中等）

**题目：**
给你一个整数数组 `nums`，其中恰好有一个元素只出现一次，其余每个元素均出现**三次**。找出那个出现一次的数字。

#### 🔨 思维推导链

异或自消除只对"出现偶数次"有效，"出现三次"会异或出一次。直接异或法失效了怎么办？

**关键洞察**：虽然异或不能直接用，但可以**按位思考**——每一位上，如果那个独特的数字贡献了 1，那么这一位的总和必然是 `3k + 1`（其他数字每个贡献 3 次），对 3 取余得 1 就说明独特数在这位有 1。

```python
class Solution:
    def singleNumber(self, nums: List[int]) -> int:
        res = 0
        for i in range(32):  # 对每一位（32 位整数）分别处理
            bit_sum = sum((num >> i) & 1 for num in nums)
            if bit_sum % 3 == 1:
                res |= (1 << i)
        # Python 的负数处理特殊：如果最高位是 1，要转成补码负数
        if res >= (1 << 31):
            res -= (1 << 32)
        return res
```

> [!tip] 通用套路：出现 k 次 + 一个出现 1 次
> 只要出现次数是 `k`（非 1），按位统计每一位的和，对 `k` 取余就能还原出那个独特数字。

### 数组中的两个独特数字（难度：中等）

**题目：**
有一个数组，恰好有**两个**数字只出现一次，其他都出现两次。找出这两个数字。

#### 🔨 思维推导链

**瓶颈**：直接异或整个数组，得到的是两个独特数字的异或值 `a ^ b`，但没法拆开。

**关键洞察**：`a ^ b` 的某一位为 1，说明 `a` 和 `b` 在这一位上**不同**。那我们就可以利用这一位把数组分成两组：这一位是 1 的放一组，是 0 的放另一组。这样 `a` 和 `b` 被分到了不同组，而每组里重复的数字还是两两配对的。

```python
class Solution:
    def singleNumbers(self, nums: List[int]) -> List[int]:
        xor_all = 0
        for num in nums:
            xor_all ^= num
        # 找到 xor_all 中最低位的 1（这一位 a 和 b 不同）
        diff_bit = xor_all & (-xor_all)
        
        a, b = 0, 0
        for num in nums:
            if num & diff_bit:
                a ^= num
            else:
                b ^= num
        return [a, b]
```

## 三、`n & (n-1)` 的经典应用

### 位 1 的个数（难度：简单）

**题目：**
编写一个函数，输入一个无符号整数，返回其二进制表示中 `'1'` 的个数（汉明重量）。

#### 🔨 思维推导链

**1. 暴力解**：每次 `n & 1` 判断最低位是否为 1，然后 `n >>= 1`，循环 32 次。时间 `O(32)` = `O(1)`。

**2. 优化**：如果 `n` 的二进制大部分都是 0（例如 `0b10000000`），上面的做法依然会空转 32 次。能否每次只处理"有 1 的位"？

**3. 关键洞察**：`n - 1` 的本质是把最低位的 1 变成 0，同时把它后面的所有 0 变成 1。那么 `n & (n-1)` 就**刚好消除了最低位的 1**。这样有多少个 1 就循环多少次，再也不空转。

```python
class Solution:
    def hammingWeight(self, n: int) -> int:
        count = 0
        while n:
            n &= (n - 1)  # 每次消灭最右边的一个 1
            count += 1
        return count
```

**举例验证**：`n = 12 = 0b1100`
- 第 1 次：`1100 & 1011 = 1000`（消掉了最低位的 1）
- 第 2 次：`1000 & 0111 = 0000`（消掉了剩下那个 1）
- 循环 2 次，答案为 2 ✅

### 2 的幂（难度：简单）

**题目：** 给定一个整数，判断它是否是 2 的幂。

#### 🔨 思维推导链

2 的幂的二进制特征：**有且只有一位是 1**，例如 `1 = 0b1, 2 = 0b10, 4 = 0b100, 8 = 0b1000`。

借用上一题的 trick：`n & (n - 1)` 能消除最低位的 1。如果 `n` 只有一个 1，消完就是 0；反之消完还有剩。

```python
class Solution:
    def isPowerOfTwo(self, n: int) -> bool:
        return n > 0 and (n & (n - 1)) == 0
```

> [!warning] 必须先判 `n > 0`
> 如果 `n = 0`，`n & (n-1) = 0 & -1 = 0`，会误判为 True。而 0 不是 2 的幂（2⁰ = 1 才是）。

### 比特位计数（难度：简单）

**题目：** 给你一个整数 `n`，返回长度为 `n + 1` 的数组 `ans`，其中 `ans[i]` 表示 `i` 的二进制中 1 的个数。

#### 🔨 思维推导链

暴力：对每个数字调用 `hammingWeight`，时间 `O(n log n)`。

**关键洞察（动态规划思维）**：把 `i` 的 1 的个数，用已知的更小的数字推出来。
- `i & (i-1)` 是比 `i` 少一个 1 的数字（而且比 `i` 小）
- 所以 `dp[i] = dp[i & (i-1)] + 1`

```python
class Solution:
    def countBits(self, n: int) -> List[int]:
        dp = [0] * (n + 1)
        for i in range(1, n + 1):
            dp[i] = dp[i & (i - 1)] + 1
        return dp
```

**时间 `O(n)`，完胜暴力！**

## 四、位运算的进阶应用

### 不用加减做加法（难度：中等）

**题目：** 写一个函数，求两个整数之和，要求不得使用 `+`、`-`、`*`、`/`。

#### 🔨 思维推导链

**关键洞察**：加法可以拆成两步：
1. **不考虑进位**的相加：这就是**异或** `a ^ b`（1+1 得 0 不进位，其他正常）
2. **只算进位**：这就是**与 + 左移** `(a & b) << 1`（只有两位都为 1 才产生进位）

把这两部分加起来就是真正的和。但"加"本身是不允许的啊？—— 递归！把这两个结果再丢进去重新做一次"异或 + 与左移"，直到进位为 0。

```python
class Solution:
    def add(self, a: int, b: int) -> int:
        MASK = 0xFFFFFFFF  # Python 整数无限长，需要手动截断到 32 位
        while b & MASK:
            carry = ((a & b) << 1) & MASK  # 进位
            a = (a ^ b) & MASK              # 无进位加
            b = carry
        return a if a < (1 << 31) else a - (1 << 32)  # 处理负数
```

### 子集（难度：中等）

**题目：** 给你一个整数数组 `nums`，返回该数组所有可能的子集。

> 本题的回溯标准解法见 [[回溯算法#子集（难度：简单）|回溯模板]]。这里介绍**位运算枚举法**。

#### 🔨 思维推导链

**关键洞察**：长度为 `n` 的数组有 `2^n` 个子集。我们可以用一个 `n` 位的二进制数来表示"选 / 不选"每个元素：
- 第 `i` 位为 1 → 选 `nums[i]`
- 第 `i` 位为 0 → 不选 `nums[i]`

遍历 `0` 到 `2^n - 1`，就能枚举所有子集。

```python
class Solution:
    def subsets(self, nums: List[int]) -> List[List[int]]:
        n = len(nums)
        res = []
        for mask in range(1 << n):  # 0 到 2^n - 1
            subset = [nums[i] for i in range(n) if mask & (1 << i)]
            res.append(subset)
        return res
```

**时间 `O(n * 2^n)`，空间 `O(n * 2^n)`**。和回溯解法复杂度相同，但代码更简洁。

> [!tip] 位掩码是状态压缩 DP 的基础
> 处理"集合"类问题时（如 TSP 旅行商、访问过哪些节点），用 `int` 的每一位表示一个元素是否被访问过，比 `List[bool]` 快好几倍。
