## 一、栈与队列核心思想与归类技巧

栈（LIFO后进先出）和队列（FIFO先进先出）是两种最基础受限数据结构。
在高级算法面试中，考得最多的是它们的两大进阶形态：**单调栈 (Monotonic Stack)** 和 **单调队列 (Monotonic Queue)**。

1. **基础栈应用**：一般用于**括号匹配**、**路径解析**、**设计拥有特殊功能的栈（如最小栈）**，核心是“最后入场的最先被处理”。
2. **单调栈**：专门解决**“寻找每个元素左边/右边第一个比自己大/小的元素”**的问题。通常存的是数组的**索引（下标）**而不是值。
3. **单调队列**：专门拿来打败滑动窗口中的极限问题（求区间最值）。队列头永远是当前窗口的老大。

## 二、基础栈结构设计

### 有效的括号（难度：简单）

**题目：**
给定一个只包括 `'('，')'，'{'，'}'，'['，']'` 的字符串 `s`，判断字符串是否有效。

**【模板解析与思路归纳】：**
- 遇到左侧符号就入栈。遇到右侧符号，就看跟栈顶的左侧符号配不配对。
- 最优雅的写法是用哈希表存右括号到左括号的映射。

**答案代码（Python版）：**
```python
class Solution:
    def isValid(self, s: str) -> bool:
        stack = []
        mapping = {')': '(', '}': '{', ']': '['}
        
        for char in s:
            if char in mapping:
                # 弹出栈顶，如果栈空了用一个假符号兜底
                top_element = stack.pop() if stack else '#'
                if mapping[char] != top_element:
                    return False
            else:
                stack.append(char)
                
        # 遍历完了，如果栈里还有剩下没被消灭的左括号，也是无效的
        return not stack
```

### 最小栈（难度：中等）

**题目：**
设计一个支持 `push`、`pop`、`top` 操作，并能在常数时间 `O(1)` 内检索到最小元素的栈。

**【模板解析与思路归纳】：**
- `O(1)` 时间拿最小值，不能去遍历。
- 必须要用**两个栈**：一个普通栈正常弹入弹出；另一个**辅助包含历史最小值的栈**同步推入数据。
- 辅助栈怎么存？“只要你要压入比我当前辅助栈顶更小（或相等）的数，我就把它压入辅助栈；弹栈时，如果弹出的那家伙刚好是我们现在辅助栈记录的最小值，辅助栈也同步把它弹走。”

**答案代码（Python版）：**
```python
class MinStack:
    def __init__(self):
        self.stack = []
        self.min_stack = []
        
    def push(self, val: int) -> None:
        self.stack.append(val)
        # 如果这是第一个元素，或者这个元素比当前的最小值还要小或相等，进入最小记录栈
        if not self.min_stack or val <= self.min_stack[-1]:
            self.min_stack.append(val)
            
    def pop(self) -> None:
        # 如果弹出的元素刚好是当前的最小值，最小栈也弹走这个记录
        if self.stack.pop() == self.min_stack[-1]:
            self.min_stack.pop()
            
    def top(self) -> int:
        return self.stack[-1]
        
    def getMin(self) -> int:
        return self.min_stack[-1]
```

## 三、单调栈：寻找最近的最值

### 每日温度（难度：中等）

**题目：**
给定一个整数数组 `temperatures`，表示每天的温度，返回一个数组 `answer`，其中 `answer[i]` 是指对于第 `i` 天，下一个更高温度出现在几天后。

**【模板解析与思路归纳】：**
- 单调栈的绝对入门标准题。找**右边第一个比自己大的元素**。
- 维护一个**单调递减栈**（底层最大，栈顶最小，里面装的是数组下标！）。
- 每次来个新温度，如果比栈顶的温度高，说明栈顶那帮苦苦等待升温的人，终于等到了！依次弹出他们，新温度的下标记减去弹出的栈顶的下标就是等待的天数。

**答案代码（Python版）：**
```python
class Solution:
    def dailyTemperatures(self, temperatures: List[int]) -> List[int]:
        n = len(temperatures)
        res = [0] * n
        stack = []  # 存下标
        
        for i in range(n):
            # 只要新来的人比栈顶记录的人温度高，就可以计算栈顶那天等了多久
            while stack and temperatures[i] > temperatures[stack[-1]]:
                prev_idx = stack.pop()
                res[prev_idx] = i - prev_idx
            stack.append(i)
            
        return res
```

### 柱状图中最大的矩形（难度：困难）

**题目：**
给定 `n` 个非负整数用来表示柱状图中各个柱子的高度，求在该柱状图中，能够勾勒出来的最大矩形面积。

**【模板解析与思路归纳】：**
- 以某个数字作为高，能画出来的矩形到底有多宽？取决于它**左边第一个比它矮的柱子**和**右边第一个比它矮的柱子**！
- 所以需要维护一个**单调递增栈**。当你遇到一个比当前的栈顶“矮”的柱子时，这就意味着栈顶那个柱子的“右边界”确定了。它的高度就是 `heights[stack.pop()]`，宽度是 `现在的下标 - 新的栈顶下标 - 1`。
- 首尾加个 `0` 占位符（哨兵），可以免去结尾时栈中还有元素没弹出来的边界判定烦恼。

**答案代码（Python版）：**
```python
class Solution:
    def largestRectangleArea(self, heights: List[int]) -> int:
        # 首尾加 0 哨兵。前面的 0 保证栈底永远不会弹空，后面的 0 保证所有柱子最后都能被弹出来结算
        heights = [0] + heights + [0]
        stack = []
        max_area = 0
        
        for i in range(len(heights)):
            # 遇到比栈顶矮的了，说明栈顶柱子的右边界已然被封死，可以开始结算栈顶这根柱子的极限面积了
            while stack and heights[i] < heights[stack[-1]]:
                h = heights[stack.pop()]
                w = i - stack[-1] - 1
                max_area = max(max_area, h * w)
            stack.append(i)
            
        return max_area
```

## 四、栈与贪心结合

### 移掉 K 位数字（难度：中等）

**题目：**
给你一个以字符串表示的非负整数 `num` 和一个整数 `k`，移除这个数中的 `k` 位数字，使得剩下的数字最小。返回极小字符串。

**【模板解析与思路归纳】：**
- 贪心法则：靠前的高位数字越大，越有干掉它的价值（比如 `1432219` 删掉 `4` 肯定比删掉后面的大数字赚的多）。
- 如果当前数字比前面的小，并且我手头还有删除次数 `k`，那我就把前面的无情淘汰掉（保证越往前数字越小：这不就是**单调栈**嘛？）。
- 全遍历完了要是 `k` 还没用光（比如本身就是单调递增的数字 `12345`），那就直接把屁户末尾最大的几位切掉。

**答案代码（Python版）：**
```python
class Solution:
    def removeKdigits(self, num: str, k: int) -> str:
        stack = []
        
        for digit in num:
            # 遇到比栈顶小的元素，弹栈，废弃那个大数字
            while k > 0 and stack and stack[-1] > digit:
                stack.pop()
                k -= 1
            stack.append(digit)
            
        # 如果数组已经是升序的，k 没消耗完，从队尾把大的删掉
        while k > 0:
            stack.pop()
            k -= 1
            
        # 把残留数字拼起来，去掉前导的0
        res = "".join(stack).lstrip('0')
        return res if res else "0"
```

## 五、单调队列极限求值

### 滑动窗口最大值（难度：困难）

**题目：**
给你一个整数数组 `nums`，大小为 `k` 的滑动窗口从数组左侧移动到右侧。只能看到窗口内的 `k` 个数字。返回滑动窗口中的最大值数组。

**【模板解析与思路归纳】：**
- 想用 `O(n)` 的时间解决，那必须用到**单调双端队列**。
- 这本质是个职场潜规则：**“如果一个年轻人（新数字）比你小，但他比你晚进公司（在窗口靠后），他是可能把你熬走的。但如果一个年轻人比你强，甚至还比你晚进公司，那你就彻底没用了！”**
- 维护一个**单调递减**的双端队列（装下标）：遇到比队列里还要大的数字，把队列尾部的老家伙们全弹掉。窗口移过时，把头部的元素踢出去。每步的答案就是队列老大的位置！

**答案代码（Python版）：**
```python
import collections

class Solution:
    def maxSlidingWindow(self, nums: List[int], k: int) -> List[int]:
        if not nums:
            return []
            
        res = []
        # 双端队列，存储下标
        queue = collections.deque()
        
        for i in range(len(nums)):
            # 1. 随着窗口向右移动，如果队列头部的老大年纪太大了（越界了），勒令退休
            if queue and queue[0] < i - k + 1:
                queue.popleft()
                
            # 2. 如果新员工比队列后面的老员工能力还要强（值大），老员工全部开除（保持单调递减）
            while queue and nums[queue[-1]] <= nums[i]:
                queue.pop()
                
            queue.append(i)
            
            # 3. 如果工龄超过了实习期 k（说明窗口完全成立了），上交目前的主管老大！
            if i >= k - 1:
                res.append(nums[queue[0]])
                
        return res
```
