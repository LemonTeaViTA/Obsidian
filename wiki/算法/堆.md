## 一、堆的核心思想与归类技巧

**堆 (Heap)** 其实就是一棵满足特殊性质的完全二叉树：**根节点永远是最大（或最小）的**。
在 Python 中，内建的 `heapq` 模块只提供**最小堆**。
如果需要最大堆，可以将存入的数值取负号（如 `-val`），弹出来之后再加个负号还原即可。

堆解决的核心问题有两大类：
1. **Top K 问题**：不论是出现频率前 K 高，还是第 K 大/小的元素。只需要维护一个大小为 `K` 的堆。如果求最大的 K 个，维护“小顶堆”；如果求最小的 K 个，维护“大顶堆”。
2. **多路归并极值/动态维护极值**：比如合并 K 个链表（实时找出 K 个列头上最小的那个），或是经典的找流式中位数。

## 二、Top K 频率提取

### 前 K 个高频元素（难度：中等）

**题目：**
给你一个整数数组 `nums` 和一个整数 `k`，请你返回其中出现频率前 `k` 高的元素。

**【模板解析与思路归纳】：**
- 这是标准的 Top K 问题。
- 计算频率后，我们可以直接用 `heapq.nlargest` 库函数走捷径。如果不走捷径自己手写，应该维护一个**大小恒等于 K 的最小堆**。
- 这里我们既可以用完整把所有的键扔进最大堆然后弹 `K` 次，但在极其苛刻的空间下，更高级的做法是只把 `K` 个元素扔进去，发现堆大小超了 `K`，就把堆顶（已知频率最小的）弹走，留下的自然是最大的 `K` 个。

**答案代码（Python版 - Pythonic库函数流）：**
```python
import collections
import heapq

class Solution:
    def topKFrequent(self, nums: List[int], k: int) -> List[int]:
        # 1. 统计频率 {数字: 频率}
        count = collections.Counter(nums)
        # 2. 直接根据 count 的 values() 从大到小提取 nlargest 个 keys
        return heapq.nlargest(k, count.keys(), key=count.get)
```

### 前 K 个高频单词（难度：中等）

**题目：**
给定一个单词列表 `words` 和一个整数 `k`，返回前 `k` 个出现次数最多的单词。返回的答案应该按单词出现频率由高到低排序。如果不同的单词有相同出现频率，按字典顺序排序。

**【模板解析与思路归纳】：**
- Python 堆存 Tuple 时，比较规则是 `(第一关键字, 第二关键字...)`，也就是说如果你遇到相同的频率，堆底层依然可以智能地用字符串的字典序帮你再排一次！

**答案代码（Python版 - 纯手工排版）：**
```python
import collections
import heapq

class Solution:
    def topKFrequent(self, words: List[str], k: int) -> List[str]:
        count = collections.Counter(words)
        
        # 使用最小堆思想，因为要输出"最高频"，频率取负实现降序，但坑点来了！
        # 如果频率取负变降序，那单词本身还是字典序升序。
        # 恰好题目要求：频率相同时按字典序字母排列！这就是绝配。
        heap = [(-freq, word) for word, freq in count.items()]
        
        # 一次性 O(N) 堆化数组
        heapq.heapify(heap)
        
        # 探囊取物
        return [heapq.heappop(heap)[1] for _ in range(k)]
```

## 三、多路归并极值判定

### 合并 K 个升序链表（难度：困难）

**题目：**
给你一个链表数组，每个链表都已经按升序排列。将所有链表合并到一个升序链表中。

**【模板解析与思路归纳】：**
- 也是典型的利用最小堆找动态全局最小值的套路！
- 把那 K 条链表的第一个节点全部无情地丢进最小堆中，由最小堆帮你把“全长总冠军”挑出来当大链表的新头部。紧接着，如果拔走冠军后这根链表还有后面的人，把第二名推入堆中补充兵源！
- 注意坑：Python 的 heapq 如果存储 Tuple，第一位 `val` 如果相同会去比对第二位。但 ListNode 自身是没有定义比较大小魔术方法的（会报 TypeError）。所以我们可以插入一个没有任何用的自增 `id` 作为第二位，化解尴尬。

**答案代码（Python版）：**
```python
import heapq

class Solution:
    def mergeKLists(self, lists: List[Optional[ListNode]]) -> Optional[ListNode]:
        # dummy 防丢失找不着北的游标
        dummy = ListNode(0)
        curr = dummy
        heap = []
        
        # 1. 兵行险招：先收集各链表的头节点压入
        for i, node in enumerate(lists):
            if node:
                # 压入的三元组结构：(节点值, 序列号防冲突, 节点本身对象)
                heapq.heappush(heap, (node.val, i, node))
                
        # 2. 拔丁抽楔，吐出一个接上一个
        while heap:
            val, idx, node = heapq.heappop(heap)
            
            # 这个节点就是目前全局最小的一个
            curr.next = node
            curr = curr.next
            
            # 补兵入堆
            if node.next:
                heapq.heappush(heap, (node.next.val, idx, node.next))
                
        return dummy.next
```

## 四、高级结构组合：对顶堆

### 数据流的中位数（难度：困难）

**题目：**
设计一个支持 `addNum(num)` 和 `findMedian()` 操作的数据结构，随时算出流式数据的中位数。如果元素是偶数，返回左右两侧两个中间数的平均。

**【模板解析与思路归纳】：**
- 非常绝妙的思想：维护一个**天平**！左半边用最大堆装较小的一半，右半边用最小堆装较大的一半。
- **平衡原则**：
  1. 所有的数据进来，无脑先扔去左半区（走 `-num` 压入大顶堆）。
  2. 左半区拿到后，立刻把自己刚才吸纳后的堆顶（左半区最大者）吐出来，塞给右半区（小顶堆）。以此实现一次平滑的过渡过滤。
  3. 最后发现右半边人太多了，把右半边堆顶的人再塞回给左面，确保左边的人数 **>=** 右边。
- 这样，如果是奇数，天平向左偏了一点，中位数就是左边堆顶！刚好偶数的话，就是左右两堆顶除以 2！

**答案代码（Python版）：**
```python
import heapq

class MedianFinder:
    def __init__(self):
        self.max_heap = []  # 左半部分，存较小一半的值（最大堆，通过存放取反的值实现）
        self.min_heap = []  # 右半部分，存较大一半的值（最小堆）
        
    def addNum(self, num: int) -> None:
        # 新数字统一先混进大顶棚子（取反）
        heapq.heappush(self.max_heap, -num)
        
        # 把大顶里头最大那个转交到小顶棚子
        biggest_in_lower_half = -heapq.heappop(self.max_heap)
        heapq.heappush(self.min_heap, biggest_in_lower_half)
        
        # 若打破天平（我们人为规定大顶棚子人数要 >= 小顶棚子人数）
        if len(self.min_heap) > len(self.max_heap):
            smallest_in_upper_half = heapq.heappop(self.min_heap)
            heapq.heappush(self.max_heap, -smallest_in_upper_half)
            
    def findMedian(self) -> float:
        # 偶数情况？两房平分春色
        if len(self.max_heap) == len(self.min_heap):
            return (-self.max_heap[0] + self.min_heap[0]) / 2.0
            
        # 奇数情况，答案就是大顶堆霸占的那多出来的一个人
        return -self.max_heap[0] / 1.0
```
