# 堆与优先队列 (Heap / Priority Queue)

> [[算法_MOC|返回算法导航]]

## 一、堆的核心思想与归类技巧

**堆 (Heap)** 其实就是一棵满足特殊性质的完全二叉树：**根节点永远是最大（或最小）的**。
在 Python 中，内建的 `heapq` 模块只提供**最小堆**。
如果需要最大堆，可以将存入的数值取负号（如 `-val`），弹出来之后再加个负号还原即可。

堆解决的核心问题有两大类：
1. **Top K 问题**：不论是出现频率前 K 高，还是第 K 大/小的元素。只需要维护一个大小为 `K` 的堆。如果求最大的 K 个，维护“小顶堆”；如果求最小的 K 个，维护“大顶堆”。
2. **多路归并极值/动态维护极值**：比如合并 K 个链表（实时找出 K 个列头上最小的那个），或是经典的找流式中位数。

## 二、Top K 与极值提取

### 数组中的第K个最大元素（难度：中等）

给定整数数组 `nums` 和整数 `k`，请返回数组中第 `k` 个最大的元素。
请注意，你需要找的是数组排序后的第 `k` 个最大的元素，而不是第 `k` 个不同的元素。
你必须设计并实现时间复杂度为 $O(N)$ 的算法解决此问题。

#### 解题思路

寻找第 K 大的元素是最经典的 Top K 问题，它有两条主流的技术路线：
1. **最小堆（常规解法，$O(N \log K)$）**：维护一个容量为 K 的小顶堆。遍历数组将元素入堆，若堆容量超过 K，则弹出堆顶（最小的那个）。遍历结束后，堆顶即为第 K 大的元素。
2. **快速选择算法（题目要求的最优解法，平均 $O(N)$）**：基于**快速排序**的 `partition` 分区思想演化而来。
   - 快排每次分区，会随机选一个**基准值 (pivot)**，将数组分成“大于等于基准”和“小于基准”两部分。
   - 分区结束后，基准值所在的位置就是它在最终有序数组中的绝对正确位置。
   - 我们只需判断基准值所在位置下标 `p` 是否等于我们想要找的排名索引（我们要找第 `k` 大，即降序排列后的目标下标 `k-1`）：
     - 如果 `p == k - 1`：恭喜你，直接找到了这第 K 大的元素！
     - 如果 `p < k - 1`：说明第 `k` 大的元素在基准值的**右侧**，砍掉左边，只在右半区递归寻找。
     - 如果 `p > k - 1`：说明第 `k` 大的元素在基准值的**左侧**，砍掉右边，只在左半区递归寻找。
   - 因为每次几乎都能排除一半的数据，时间复杂度为 $N + N/2 + N/4 + ... = 2N$，所以平均时间复杂度为 $O(N)$！

> [!tip] **防杠指南与极端情况**
> 为了避免在特殊极端数据（假如数组已经有序或全是一样的数字）下退化成 $O(N^2)$，我们在选 `pivot` 的时候一定要**随机选择**。

#### 代码实现

**解法一：标准快速选择（双指针 2-Way Quickselect）**
> [!warning] **注意**
> 此为逻辑最清晰的常规模板（将数组强行分为左右两边）。但在力扣环境中，如果遇到大量重复元素如 `[3, 3, 3... 3]`，虽然加入了随机种子，但依旧会因为指针穿透无法有效切分数组，使得时间复杂度退化为 $O(N^2)$ 导致**超时**。适合面试手撕和说明基本思路。

```python
import random
from typing import List

class Solution:
    def findKthLargest(self, nums: List[int], k: int) -> int:
        def quick_select(left, right, target_index):
            pivot_index = random.randint(left, right)
            nums[left], nums[pivot_index] = nums[pivot_index], nums[left]
            pivot = nums[left]

            i, j = left, right
            while i < j:
                # 寻找比基准大的数 (注意我们这里是找第K"大"，所以把它降序排)
                while i < j and nums[j] <= pivot:
                    j -= 1
                nums[i] = nums[j]

                # 寻找比基准小的数
                while i < j and nums[i] >= pivot:
                    i += 1
                nums[j] = nums[i] 

            nums[i] = pivot

            if i == target_index:
                return nums[i]
            elif i < target_index:
                return quick_select(i + 1, right, target_index)
            else:
                return quick_select(left, i - 1, target_index)
                
        return quick_select(0, len(nums) - 1, k - 1)
```

**解法二：最优解 - 三路划分快速选择 (3-Way Quickselect，严格防超时，平均 $O(N)$)**
> [!success] **进阶：荷兰国旗问题融合**
> 把数组切分为 `< pivot`、`== pivot` 和 `> pivot` 三部分。当遇到全相同的畸形数组时，中间全等的区间会把所有相同数字全部吞噬，直接一步结束战斗。

```python
import random
from typing import List

class Solution:
    def findKthLargest(self, nums: List[int], k: int) -> int:
        def quick_select(l: int, r: int, target_index: int) -> int:
            if l == r:
                return nums[l]
                
            # 随机选择基准，防止有序数组导致的极端退化
            pivot = nums[random.randint(l, r)]
            
            # 开始三路划分 (Dutch National Flag 问题)
            left, right, i = l, r, l
            while i <= right:
                # 因为我们要找第 K 大，相当于在降序数组中找索引 k-1
                # 因此我们将 > pivot 的数统统放到左侧
                if nums[i] > pivot:
                    nums[left], nums[i] = nums[i], nums[left]
                    left += 1
                    i += 1
                # < pivot 的数统统放到右侧
                elif nums[i] < pivot:
                    nums[i], nums[right] = nums[right], nums[i]
                    right -= 1
                # == pivot 的数留在中间
                else:
                    i += 1
                    
            # 划分结束后：
            # nums[l : left-1] 皆为 > pivot 的数
            # nums[left : right] 皆为 == pivot 的数
            # nums[right+1 : r] 皆为 < pivot 的数
            
            # 判断我们苦寻的 target_index 到底落在哪一个区间？
            if target_index < left:
                # 目标在大于基准值的左区间
                return quick_select(l, left - 1, target_index)
            elif target_index > right:
                # 目标在小于基准值的右区间
                return quick_select(right + 1, r, target_index)
            else:
                # 目标正好处在全部等于基准值的区间，直接秒杀返回！
                # 遇到全部相等的极端用例时，这一步能一次吞噬所有相同元素，直接免去递归！
                return pivot
                
        # 降序数组中的第 K 大，也就是下标 k - 1
        return quick_select(0, len(nums) - 1, k - 1)
```

### 前 K 个高频元素（难度：中等）

给定一个整数数组 `nums` 和一个整数 `k`，请你返回其中出现频率前 `k` 高的元素。

#### 解题思路

作为标准的 Top K 问题，最直接的常规思路是使用**堆**。我们可以先统计每个数字出现的频率，然后维护一个大小恒等于 K 的最小堆。遍历频率字典时，将元素加入堆中；如果堆的大小超过了 K，就把堆顶（目前频率最小的元素）弹出，这样最终留在堆里的必然是频率最大的 K 个元素。如果在 Python 中，甚至可以直接调用库函数 `heapq.nlargest` 来走捷径。

然而，这道题存在一个极其巧妙的**线性复杂度 $O(N)$ 桶排序解法**。
既然我们要按频率找元素，而一个元素无论怎么出现，它的频率绝对不可能超过数组的总长度 $N$。所以，我们可以直接把频率作为“桶”的索引，把对应频率的元素扔进对应的桶里。
具体来说，我们先用哈希表统计所有元素的频率，并找出最大的频率 `max_cnt`。接着，创建一个长度为 `max_cnt + 1` 的数组（桶），遍历哈希表，把出现次数相同的元素一股脑塞进同一个对应角标的桶里（比如出现 3 次的全部放进 `buckets[3]` 的列表里）。
最后，我们只需要**倒序遍历这个桶数组**。因为越靠后的桶，代表出现的频率越高，我们把桶里的元素逐个倒出来装入答案中，一旦答案数组的长度达到了题目的要求 `k`，就立刻收手返回。

#### 代码实现

**解法一：桶排序最优解（$O(N)$ 时空复杂度）**
```python
from collections import Counter
from typing import List

class Solution:
    def topKFrequent(self, nums: List[int], k: int) -> List[int]:
        # 第一步：统计每个元素的出现次数
        cnt = Counter(nums)
        max_cnt = max(cnt.values())

        # 第二步：把出现次数相同的元素，放到同一个桶中
        # 索引代表频率，值代表具有该频率的元素列表
        buckets = [[] for _ in range(max_cnt + 1)]  
        for x, c in cnt.items():
            buckets[c].append(x)

        # 第三步：倒序遍历 buckets，把出现次数前 k 大的元素加入答案
        ans = []
        for bucket in reversed(buckets):
            ans += bucket
            # 注意题目保证答案唯一，一定会出现恰好等于 k 的情况
            if len(ans) == k:
                return ans
```

**解法二：最小堆常规解法（$O(N \log K)$）**
```python
import collections
import heapq

class Solution:
    def topKFrequent(self, nums: List[int], k: int) -> List[int]:
        # 1. 统计频率 {数字: 频率}
        count = collections.Counter(nums)
        # 2. 直接根据 count 的 values() 从大到小提取 nlargest 个 keys
        return heapq.nlargest(k, count.keys(), key=count.get)
```

### 前 K 个高频单词（难度：中等）

给定一个单词列表 `words` 和一个整数 `k`，返回前 `k` 个出现次数最多的单词。返回的答案应该按单词出现频率由高到低排序。如果不同的单词有相同出现频率，按字典顺序排序。

#### 解题思路
- Python 堆存 Tuple 时，比较规则是 `(第一关键字, 第二关键字...)`，也就是说如果你遇到相同的频率，堆底层依然可以智能地用字符串的字典序帮你再排一次！

#### 代码实现
```python
import collections
import heapq

class Solution:
    def topKFrequent(self, words: List[str], k: int) -> List[str]:
        count = collections.Counter(words)
        
        # 【核心代码深度解析】：
        # 1. count.items(): `count` 是记录频率的字典, 例如 {'i': 2, 'love': 2, 'coding': 1}。
        #    `.items()` 会把它解包成一个个键值对组合 (word, freq) 供我们用 for 循环提取出来。
        # 2. (-freq, word): Python 的堆在存储元组 (Tuple) 时，拥有严格的“顺位比较优先级”：
        #    - 第一优先级（决胜权） `(-freq)`：Python 里的 `heapq` 默认是“最小堆”（每次只弹最小的数字）。
        #      但我们要先输出“最高频”，怎么办？把频率加个负号！比如频率 5 变成 -5，反而成了最小的数，自然就能第一个从堆顶出来。
        #    - 第二优先级（加时赛） `word`：万一频率相同怎么办（比如 'i' 和 'love' 都是 2 次，取负后都是 -2，第一局打平）？
        #      堆底层会自动去比对元组的第二个元素（也就是这个单词的字符串）。字符串的默认比较规则刚好是“字典序升序”（'i' < 'l'），
        #      于是字典序靠前的 'i' 会被判定为更“小”的那个，优先从堆顶弹出。
        #      完美命中了题目最恶心要求：“频率相同时，按字典顺序升序排列”。
        # 
        # 注意：这里我们只是“打包装箱”，把数据凑成了适合去比较的 Tuple 格式，放进了一个普通的 List（列表）里。
        # 这个 List 此时并不是有序的！更不是一棵满足父节点小于子节点的树（堆）！
        heap = [(-freq, word) for word, freq in count.items()]
        
        # 所以必须执行 heapify，它能在 O(N) 的时间内把一个杂乱无章的普通 List，原地重组、建立成一棵标准的最小堆。
        # 只有变成了真正的堆，我们才能在后面的循环里，利用 heappop 每次以 O(logN) 的速度安全地拿到最上面的极值。
        heapq.heapify(heap)
        
        # 3. 探囊取物：`heapq.heappop(heap)` 会弹出堆顶的元素（即出现次数最多的那个单词组成的 Tuple）。
        # 比如弹出的是 (-2, 'i')，我们不需要频率 -2，我们只要单词，所以用 `[1]` 取元组的第二项（即 'i'）。
        # `for _ in range(k)` 意味着连取 K 次，生成一个只包含单词的列表作为最终答案。
        return [heapq.heappop(heap)[1] for _ in range(k)]
```

## 三、多路归并极值判定

### 合并 K 个升序链表（难度：困难）

给你一个链表数组，每个链表都已经按升序排列。将所有链表合并到一个升序链表中。

#### 解题思路
- 这是典型的利用最小堆找动态全局最小值的套路。
- 把那 K 条链表的第一个节点全部无情地丢进最小堆中，由最小堆帮你把“全长总冠军”挑出来当大链表的新头部。紧接着，如果拔走冠军后这根链表还有后面的人，把第二名推入堆中补充兵源。
- **神级防坑技巧**：Python 的 `heapq` 如果直接存储 `ListNode` 对象，当进行比较排序时会报错 `TypeError`，因为 `ListNode` 自身是没有定义比较大小的魔术方法的。
  - 老式的笨办法：插入一个 `(node.val, i, node)` 的格式，加一个没有任何用的自增 `id` 作为第二顺位缓冲，防止去比较 `node` 本体。
  - **最新黑科技做法（极简）**：直接在类外给全局的 `ListNode` 强行打一个猴子补丁（Monkey Patch）：`ListNode.__lt__ = lambda a, b: a.val < b.val`。也就是我们帮它定义什么是“小于”。这样堆就能直接毫无阻碍地吞吐纯纯的 `node` 对象了！

#### 代码实现
```python
from heapq import heapify, heappop, heappush
from typing import List, Optional

# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, val=0, next=None):
#         self.val = val
#         self.next = next

# 猴子补丁：教 ListNode 怎么比大小！
ListNode.__lt__ = lambda a, b: a.val < b.val

class Solution:
    def mergeKLists(self, lists: List[Optional[ListNode]]) -> Optional[ListNode]:
        cur = dummy = ListNode()
        
        # 1. 过滤掉空链表，直接把所有链表头收集起来
        h = [head for head in lists if head]
        
        # 2. 原地 O(K) O(K) 建立最小堆
        heapify(h)
        
        # 3. 拔丁抽楔，吐出一个接上一个
        while h:
            node = heappop(h)
            
            # 如果刚弹出来这个冠军，它背后还有后续节点，赶紧推入堆中补充兵源
            if node.next:
                heappush(h, node.next)
                
            # 把弹出的最小节点连到我们最终的总链表上
            cur.next = node
            cur = cur.next
            
        return dummy.next
```

## 四、高级结构组合：对顶堆

### 数据流的中位数（难度：困难）

设计一个支持 `addNum(num)` 和 `findMedian()` 操作的数据结构，随时算出流式数据的中位数。如果元素是偶数，返回左右两侧两个中间数的平均。

#### 解题思路
- 非常绝妙的思想：维护一个**天平**！左半边用最大堆装较小的一半，右半边用最小堆装较大的一半。
- **平衡原则**：
  1. 所有的数据进来，无脑先扔去左半区（走 `-num` 压入大顶堆）。
  2. 左半区拿到后，立刻把自己刚才吸纳后的堆顶（左半区最大者）吐出来，塞给右半区（小顶堆）。以此实现一次平滑的过渡过滤。
  3. 最后发现右半边人太多了，把右半边堆顶的人再塞回给左面，确保左边的人数 **>=** 右边。
- 这样，如果是奇数，天平向左偏了一点，中位数就是左边堆顶！刚好偶数的话，就是左右两堆顶除以 2！

#### 代码实现
```python
import heapq

class MedianFinder:
    def __init__(self):
        self.max_heap = []  # 左半部分，存较小一半的值（最大堆，通过存放取反的值实现）
        self.min_heap = []  # 右半部分，存较大一半的值（最小堆）
        
    def addNum(self, num: int) -> None:
        # 新数字统一先混进大顶棚子（取反）
        heapq.heappush(self.max_heap, -num)
        
        # 把大顶里头最大那个转交到小顶棚子
        biggest_in_lower_half = -heapq.heappop(self.max_heap)
        heapq.heappush(self.min_heap, biggest_in_lower_half)
        
        # 若打破天平（我们人为规定大顶棚子人数要 >= 小顶棚子人数）
        if len(self.min_heap) > len(self.max_heap):
            smallest_in_upper_half = heapq.heappop(self.min_heap)
            heapq.heappush(self.max_heap, -smallest_in_upper_half)
            
    def findMedian(self) -> float:
        # 偶数情况？两房平分春色
        if len(self.max_heap) == len(self.min_heap):
            return (-self.max_heap[0] + self.min_heap[0]) / 2.0
            
        # 奇数情况，答案就是大顶堆霸占的那多出来的一个人
        return -self.max_heap[0] / 1.0
```
