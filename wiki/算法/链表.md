## 一、链表核心思想与三大终极武器

在链表题目中，由于不能像数组一样随机访问，只能顺藤摸瓜，导致极易出现空指针异常（NullPointerException）或者链表断裂的情况。只要掌握这三个究极武器，即可闭眼秒杀 90% 的题目：

**1. 武器一：虚拟头节点（Dummy Head）**
*   **适用场景**：只要原链表的**头节点可能被移动、修改或删除**，就必须用虚拟头节点！
*   **怎么写**：`dummy = ListNode(0, head)`，最后返回 `dummy.next`。这能让你免去无穷无尽的 `if not head:` 边界判断。

**2. 武器二：快慢指针（Fast & Slow Pointers）**
*   **适用场景**：寻找链表中心节点、判断环的存在、寻找环的入口、寻找倒数第 K 个节点。
*   **怎么写**：起手式 `slow = head, fast = head`。然后 `while fast and fast.next:` 保证奇数偶数长度都不会越界。

**3. 武器三：穿针引线反转法**
*   **适用场景**：一切涉及链表结构翻转或局部的反转。
*   **背诵模板（三指针法）**：
```python
prev, curr = None, head
while curr:
    next_temp = curr.next # 1. 暂存后续，防止断链
    curr.next = prev      # 2. 箭头反指
    prev = curr           # 3. 整体后移
    curr = next_temp
```

## 二、基础入门：指针的艺术与 Dummy Head

### 反转链表（难度：简单）

**题目：**
给你单链表的头节点 `head` ，请你反转链表，并返回反转后的链表。

**【模板解析与思路归纳】：**
- 直接默写**武器三**。链表核心就是修改引用的指向，不要去修改里面的值。
- 使用三个指针：被指者 (`prev`)，当前者 (`curr`)，探路者 (`next_temp`)。

**答案代码（Python版）：**
```python
class Solution:
    def reverseList(self, head: ListNode) -> ListNode:
        prev = None
        curr = head
        while curr:
            next_temp = curr.next
            curr.next = prev
            prev = curr
            curr = next_temp
        return prev  # 出循环时 curr 必然是空，prev 是新的头节点
```

### 两两交换链表中的节点（难度：中等）

**题目：**
给你一个链表，两两交换相邻节点，返回交换后的头节点。不能修改节点内部的值。

**【模板解析与思路归纳】：**
- 虚拟头节点 + 穿针引线。每轮处理一对 `node1 → node2`，交换后变成 `node2 → node1`，然后 `prev` 移到 `node1`（交换后的尾部），继续处理下一对。
- 三步操作顺序不能乱：先断开 `node1` 和 `node2` 的连接，再重新接线，最后让 `prev` 接上新头。

**答案代码（Python版）：**
```python
class Solution:
    def swapPairs(self, head: ListNode) -> ListNode:
        dummy = ListNode(0, head)
        prev = dummy
        while prev.next and prev.next.next:
            node1 = prev.next
            node2 = prev.next.next
            node1.next = node2.next  # 1. node1 跳过 node2
            node2.next = node1       # 2. node2 指回 node1
            prev.next = node2        # 3. prev 接上新头 node2
            prev = node1             # prev 移到这对的尾部
        return dummy.next
```

### 删除链表的倒数第 N 个节点（难度：中等）

**题目：**
给你一个链表，删除链表的倒数第 `n` 个节点，并返回链表的头节点。

**【模板解析与思路归纳】：**
- 快慢指针：让 `fast` 先走 `n` 步，然后 `fast` 和 `slow` 同时走，`fast` 到尾时 `slow` 正好在倒数第 `n+1` 个节点（即要删除节点的前一个），执行 `slow.next = slow.next.next`。
- **注意点**：`ListNode(0, head)` 的第二个参数不是必须的，它只是 `dummy.next = head` 的简写。这道题必须用虚拟头节点，因为删除的可能是第一个节点（原 `head`），有了 `dummy` 就不需要特判。

**答案代码（Python版）：**
```python
class Solution:
    def removeNthFromEnd(self, head: ListNode, n: int) -> ListNode:
        dummy = ListNode(0, head)
        fast = slow = dummy
        for _ in range(n):
            fast = fast.next
        while fast.next:
            fast = fast.next
            slow = slow.next
        slow.next = slow.next.next
        return dummy.next
```

### 合并两个有序链表（难度：简单）

**题目：**
将两个升序链表合并为一个新的 升序 链表并返回。新链表是通过拼接给定的两个链表的所有节点组成的。

**【模板解析与思路归纳】：**
- 新链表的头部是由 `l1` 和 `l2` 中较小的一个决定的，头节点会变，立刻祭起**武器一（虚拟头节点）**。
- 设置一个移动指针 `curr = dummy`，不断挂载较小的值，最后把没遍历完的那长长的一截直接挂接在最后。
- **注意点一**：`ListNode(-1)` 括号里的 `-1` 只是虚拟头节点的占位值，填什么都行，因为最终返回的是 `dummy.next`，这个节点本身不会出现在结果里。
- **注意点二**：`curr.next = list1` 能一次性把剩余链表全挂上去，因为 `list1` 指向的是剩余链表的头节点，它后面的 `next` 链还完整地连着，不需要逐个添加。

**答案代码（Python版）：**
```python
class Solution:
    def mergeTwoLists(self, list1: ListNode, list2: ListNode) -> ListNode:
        dummy = ListNode(-1)
        curr = dummy
        
        while list1 and list2:
            if list1.val <= list2.val:
                curr.next = list1
                list1 = list1.next
            else:
                curr.next = list2
                list2 = list2.next
            curr = curr.next
            
        # 只要有一个走完了，就把剩下一大串直接挂在 curr 后面
        curr.next = list1 if list1 else list2
        
        return dummy.next
```

### 合并 K 个升序链表（难度：困难）

**题目：**
给你一个链表数组，每个链表都已经按升序排列，请将所有链表合并到一个升序链表中，返回合并后的链表。

**【模板解析与思路归纳】：**
- 直接复用「合并两个有序链表」，用**最小堆（优先队列）**每次从所有链表头部取最小值。
- 把所有链表的头节点放入堆，每次弹出最小节点接到结果链表，再把该节点的 `next` 压入堆。
- **注意点**：Python 的 `heapq` 是最小堆，但 `ListNode` 不支持直接比较大小，会报 `TypeError`。解决方法是往堆里存 `(val, index, node)` 三元组，`index` 是入堆顺序，当 `val` 相同时用 `index` 打破平局，避免比较 `node`。
- **为什么要 `if node.next` 才压堆**：弹出节点后，只有 `node.next` 不为 `None` 时才压入下一个节点。如果去掉这个判断，直接访问 `node.next.val`，当 `node` 是链表末尾时 `node.next` 是 `None`，访问 `None.val` 会报 `AttributeError`。
- **初始化时为什么要 `if node`**：`lists` 数组里可能有空链表（`None`），如果不判断直接访问 `node.val`，会报 `AttributeError`。`if node` 跳过空链表，只把有节点的链表头压入堆。
- **heapq 常用方法**：
  - `heapq.heappush(heap, item)`：把元素压入堆，自动维护最小值在 `heap[0]`。
  - `heapq.heappop(heap)`：弹出并返回最小元素。
  - `heapq.heapify(list)`：把普通列表原地转成堆，O(n)。
  - **只有最小堆**：要实现最大堆，压入时取负值（`-val`），弹出后再取负。
  - **`heapq` vs `heap` 的区别**：`heapq` 是 Python 内置模块（工具箱），`heap` 是你自己定义的普通列表（盒子）。`heapq` 不存数据，只提供操作函数，操作的对象是你传进去的列表。

**答案代码（Python版）：**
```python
import heapq

class Solution:
    def mergeKLists(self, lists: List[Optional[ListNode]]) -> Optional[ListNode]:
        dummy = ListNode(0)
        curr = dummy
        heap = []
        # 把所有链表头节点入堆
        for i, node in enumerate(lists):
            if node:
                heapq.heappush(heap, (node.val, i, node))
        while heap:
            val, i, node = heapq.heappop(heap)
            curr.next = node
            curr = curr.next
            if node.next:
                heapq.heappush(heap, (node.next.val, i, node.next))
        return dummy.next
```

#### 复杂度分析

- **时间复杂度**：O(n log k)，n 是所有节点总数，k 是链表数量，每次堆操作 O(log k)。
- **空间复杂度**：O(k)，堆中最多同时存 k 个节点。

### 两数相加（难度：中等）

**题目：**
两个非空链表表示两个非负整数，每位数字按逆序存储，每个节点只存一位数字。将两数相加，返回同样形式的链表。

**【模板解析与思路归纳】：**
- 虚拟头节点 + 模拟竖式加法。逐位取值相加，用 `carry` 记录进位。
- **核心坑点**：不知道怎么创建节点并赋值。写法就是 `ListNode(digit)`，把计算好的值直接传进构造函数，然后挂到链表上：
  ```python
  curr.next = ListNode(digit)
  curr = curr.next
  ```
- 循环条件是 `while l1 or l2 or carry`，三者任意一个非空/非零都要继续，防止最高位进位漏掉（如 `5 + 5 = 10`，最后还有一个进位 `1` 要单独成节点）。

**答案代码（Python版）：**
```python
class Solution:
    def addTwoNumbers(self, l1: ListNode, l2: ListNode) -> ListNode:
        dummy = ListNode(-1)
        curr = dummy
        carry = 0
        while l1 or l2 or carry:
            total = (l1.val if l1 else 0) + (l2.val if l2 else 0) + carry
            carry = total // 10
            curr.next = ListNode(total % 10)
            curr = curr.next
            if l1: l1 = l1.next
            if l2: l2 = l2.next
        return dummy.next
```

### 删除排序链表中的重复元素 II（难度：中等）

**题目：**
给定一个已排序的链表的头 `head` ， 删除原始链表中所有重复数字的节点，只留下不同的数字 。返回已排序的链表。

**【模板解析与思路归纳】：**
- 因为头节点 `head` 可能是重复的，也有被删除的可能，因此**必用 Dummy Head**。
- 因为要“删光所有重复节点”（连自己都不能留），判断重复必须是 `curr.next.val == curr.next.next.val`，提前探两步！如果发现重复，通过内部 `while` 把所有的同段值全部跳过。

**答案代码（Python版）：**
```python
class Solution:
    def deleteDuplicates(self, head: ListNode) -> ListNode:
        dummy = ListNode(-1, head)
        curr = dummy
        
        # 必须要保证后面有两个节点能凑一块比较
        while curr.next and curr.next.next:
            if curr.next.val == curr.next.next.val:
                val = curr.next.val # 记下这个罪恶的重复值
                # 全部跳过拥有这个值的节点
                while curr.next and curr.next.val == val:
                    curr.next = curr.next.next
            else:
                # 只有确认安全（不一样），curr 才敢往前挪
                curr = curr.next
                
        return dummy.next
```

## 三、进阶绝学：快慢指针 (Fast & Slow)
*解决找中点、测距和找环神器。*

### 链表的中间结点（难度：简单）

**题目：**
给你单链表的头节点 `head` ，请你找出并返回链表的中间结点。如果有两个中间结点，则返回第二个中间结点。

**【模板解析与思路归纳】：**
- 利用物理学里的追及问题：跑得快的人速度是跑得慢的两倍，当快的人跑到底或者跑出界，慢的人恰好走到正中央。

**答案代码（Python版）：**
```python
class Solution:
    def middleNode(self, head: ListNode) -> ListNode:
        slow = fast = head
        # 只要 fast 还能跨两步，就继续跑
        while fast and fast.next:
            slow = slow.next
            fast = fast.next.next
        return slow
```

### 环形链表（难度：简单）

**题目：**
给你一个链表的头节点 `head` ，判断链表中是否有环。

**【模板解析与思路归纳】：**
- 在操场跑步，如果跑道有圈，快的人迟早会套圈追上慢的人。
- 一旦 `fast == slow`，说明有环。如果是直道（遇到 `None`），那就返回 `False`。
- **注意点**：`while fast and fast.next` 两个条件缺一不可。`fast` 防止自身为 `None` 时访问 `.next` 报错；`fast.next` 防止 `fast.next.next` 在 `None` 上访问 `.next` 报错。`fast` 保护第一跳，`fast.next` 保护第二跳。

**答案代码（Python版）：**
```python
class Solution:
    def hasCycle(self, head: ListNode) -> bool:
        slow = fast = head
        while fast and fast.next:
            slow = slow.next
            fast = fast.next.next
            if slow == fast:
                return True
        return False
```

### 环形链表 II（难度：中等）

**题目：**
给定链表头节点 `head`，返回链表开始入环的第一个节点。如果无环，返回 `null`。

**【模板解析与思路归纳】：**
- 第一阶段和判断有环一样：快慢指针，`fast` 走两步，`slow` 走一步，相遇说明有环。
- 第二阶段找入环点：相遇后把 `slow` 重置回 `head`，然后 `slow` 和 `fast` **每次各走一步**，再次相遇处就是入环点。数学上可以证明：从相遇点到入环点的距离，恰好等于从 `head` 到入环点的距离。
- **注意点**：第二阶段必须**先判断再移动**（`while slow is not fast` 在循环头部）。如果先移动再判断，会跳过入环点——当入环点恰好是 `head` 时，`slow` 第一步就走过去了，永远不会在入环点相遇。

**答案代码（Python版）：**
```python
class Solution:
    def detectCycle(self, head: ListNode) -> ListNode:
        slow = fast = head
        while fast and fast.next:
            slow = slow.next
            fast = fast.next.next
            if slow == fast:
                slow = head
                while slow is not fast:
                    slow = slow.next
                    fast = fast.next
                return slow
        return None
```

### 回文链表（难度：简单）

**题目：**
给你一个单链表的头节点 `head` ，请你判断该链表是否为回文链表。

**【模板解析与思路归纳】：**
- 这题是前面技能的集大成者：**快慢指针找中点 + 穿针引线反转法**。
- 先用快慢指针找到链表的一半。然后从中间把后半部分反转，再将头和尾部一个一个往里对比。
- **注意点**：也可以用更简洁的**数组法**：遍历链表把所有值存入数组，再判断 `ans == ans[::-1]`。坑点是倒序写法是 `[::-1]`，不是 `[:-1]`（后者是去掉最后一个元素，完全不同的语义）。

**答案代码（Python版）：**

**简洁写法（数组法）：**
```python
class Solution:
    def isPalindrome(self, head: Optional[ListNode]) -> bool:
        ans = []
        cur = head
        while cur:
            ans.append(cur.val)
            cur = cur.next
        return ans == ans[::-1]
```

**进阶写法（O(1) 空间，快慢指针 + 反转后半段）：**
```python
class Solution:
    def isPalindrome(self, head: ListNode) -> bool:
        if not head or not head.next:
            return True
            
        # 1. 快慢指针找中点
        slow, fast = head, head
        while fast and fast.next:
            slow = slow.next
            fast = fast.next.next
            
        # 2. 反转后半段链表 (翻转以 slow 为头结点的部分)
        prev, curr = None, slow
        while curr:
            next_temp = curr.next
            curr.next = prev
            prev = curr
            curr = next_temp
            
        # 3. 把前半段与被翻转的后半段挨个比较
        left, right = head, prev
        while right: # 以 right 为限，因为后半部分可能短一个
            if left.val != right.val:
                return False
            left = left.next
            right = right.next
            
        return True
```

## 四、大师操作：区间操作与穿针引线
*开始处理极其复杂的指针跃迁，极其考验逻辑清晰度。*

### 反转链表 II（难度：中等）

**题目：**
给你单链表的头指针 `head` 和两个整数 `left` 和 `right` ，其中 `left <= right` 。请你反转从位置 `left` 到位置 `right` 的链表节点，返回反转后的链表。

**【模板解析与思路归纳】：**
- 万一从 `left = 1` 开始反转，就意味着换了头，所以必须借助 **虚拟头节点 (Dummy)**。
- 绝妙的**头插法**：让 `prev` 永远站在 `left - 1` 这个位置上稳住不动。然后拿他后面的兵也就是 `curr` 开始干活：每次把 `curr` 身后的一个新兵 (`next_temp`) 揪出来，直接越塔插到 `prev` 后面当作先锋。直到插了 `right - left` 次。

**答案代码（Python版）：**
```python
class Solution:
    def reverseBetween(self, head: ListNode, left: int, right: int) -> ListNode:
        # Dummy head 防止 left = 1 换头带来的麻烦
        dummy = ListNode(-1, head)
        prev = dummy
        
        # 1. 让 prev 走到进入反转区间的前一站
        for _ in range(left - 1):
            prev = prev.next
            
        # 2. curr 是当前反转小分队的尾巴（最开始它也是打头阵的节点）
        curr = prev.next
        
        # 3. 头插法：每次把 curr 后的那个人揪出来，甩回到 prev 身后
        for _ in range(right - left):
            next_temp = curr.next
            curr.next = next_temp.next
            next_temp.next = prev.next
            prev.next = next_temp
            
        return dummy.next
```

### K 个一组翻转链表（难度：困难）

**题目：**
给你链表的头节点 `head`，每 `k` 个节点一组进行翻转，返回修改后的链表。如果节点总数不是 `k` 的整数倍，最后剩余节点保持原顺序。

**【模板解析与思路归纳】：**
- 虚拟头节点 + 分组反转。用一个 `prev_group` 指针标记每组反转前的"前驱"，每次先探测后面够不够 `k` 个节点：
  - 不够 → 直接返回，剩余部分保持原顺序。
  - 够 → 反转这一组的 k 个节点，然后把 `prev_group` 移到这组反转后的尾部（也就是反转前的头节点），继续下一轮。
- **核心坑点**：反转一组后要做"四件事"：
  1. 记住这一组的原始头（反转后变尾）`group_start`
  2. 记住这一组的下一组起点 `next_group`
  3. 反转这 k 个节点
  4. 接线：`prev_group.next = 新头`、`group_start.next = next_group`
- **注意点**：探测够不够 k 个的时候要走 k 步，循环里如果 `tail` 提前变 `None` 就说明不够，直接返回 `dummy.next`。

**答案代码（Python版）：**
```python
class Solution:
    def reverseKGroup(self, head: Optional[ListNode], k: int) -> Optional[ListNode]:
        dummy = ListNode(0)
        dummy.next = head
        pre = dummy

        while True:
            # 1. 检查后面是否还有 k 个节点
            end = pre
            for _ in range(k):
                end = end.next
                if not end:
                    return dummy.next

            # 2. 记录下一组的起点
            next_group = end.next
            # 3. 当前组的起点
            start = pre.next
            # 4. 断开当前组
            end.next = None
            # 5. 翻转当前这一组
            pre.next = self.reverse(start)
            # 6. 翻转后 start 变成当前组尾节点，接回下一组
            start.next = next_group
            # 7. pre 移动到当前组尾部
            pre = start

    def reverse(self, head):
        prev = None
        cur = head
        while cur:
            nxt = cur.next
            cur.next = prev
            prev = cur
            cur = nxt
        return prev
```

### 相交链表（难度：简单）

**题目：**
给你两个单链表的头节点 `headA` 和 `headB` ，请你找出并返回两个单链表相交的起始节点。如果不存在相交节点，返回 `null` 。

**【模板解析与思路归纳】：**
- **浪漫的追针思想**："你走过我走过的路，我走过你走过的路，最终我们必将相遇"。
- 设定两个探路者，走完了 A 链表，就跳到 B 链表头部；走完了 B 链表，就跳到 A 链表头部。消除长度差。最后两者必定一起走到相交点（或者一同走到尽头的 `None`）。
- **注意点**：`while` 的终止条件是 `pA != pB`（两指针相等时停下），而不是 `pA` 或 `pB` 为 `None`。因为不相交时两者会同时走到 `None`，`None == None` 也满足相等条件，循环同样能正常终止。如果用 `while pA and pB`，不相交的情况就永远出不了循环。

**答案代码（Python版）：**
```python
class Solution:
    def getIntersectionNode(self, headA: ListNode, headB: ListNode) -> ListNode:
        if not headA or not headB:
            return None
            
        pA, pB = headA, headB
        while pA != pB:
            # A 走完切去 B；没走完就继续
            pA = pA.next if pA else headB
            # B 走完切去 A；没走完就继续
            pB = pB.next if pB else headA
            
        return pA
```

### 排序链表（难度：中等）

**题目：**
给你链表的头结点 `head` ，请将其按升序排列并返回排序后的链表。要求时间复杂度 `O(n log n)` 空间复杂度 `O(1)`。

**【模板解析与思路归纳】：**
- 链表天然适合**归并排序**，因为不用像数组一样在合并时需要大量内存移动。
- 利用 快慢指针找中点（截断链表） -> 然后进行分治递归 -> 最终利用前文学过的**合并两个有序链表**把他们拼起来。
- **为什么会想到递归（归并排序）？**
  1. **题目要求 O(n log n)**：这是归并排序/快排的标志性复杂度。链表不适合快排（随机访问慢），首选归并。
  2. **链表天然适合归并**：数组归并需要额外 O(n) 空间存临时数组，链表归并只需改指针，原地合并。
  3. **递归的思维方式**："我不知道怎么排 n 个节点，但我知道怎么合并两个已排序的链表。那我就假设左右两半已经排好了（递归黑盒），我只负责合并。"
  4. **归并排序的递归模板**：找中点分两半 → 递归排序左半 → 递归排序右半 → 合并两个有序链表。
- **注意点**：找中点时必须用 `prev` 记录 `slow` 的前一个节点，用来断开链表（`prev.next = None`），否则左半部分会包含整个链表，递归无法终止。

**答案代码（Python版）：**
```python
class Solution:
    def sortList(self, head: ListNode) -> ListNode:
        if not head or not head.next:
            return head
            
        # 1. 找中点并截断 (此处必须使用 prev 记录 slow 之前的位置用来断开)
        slow = fast = head
        prev = None
        while fast and fast.next:
            prev = slow
            slow = slow.next
            fast = fast.next.next
            
        # 从中线断开
        prev.next = None
        
        # 2. 分治归并
        left = self.sortList(head)
        right = self.sortList(slow)
        
        # 3. 经典绝学：合并两个有序链表
        dummy = ListNode(-1)
        curr = dummy
        while left and right:
            if left.val <= right.val:
                curr.next = left
                left = left.next
            else:
                curr.next = right
                right = right.next
            curr = curr.next
            
        curr.next = left if left else right
        return dummy.next
```

### 随机链表的复制（难度：中等）

**题目：**
给你一个长度为 `n` 的链表，每个节点除了 `next` 指针外，还有一个 `random` 指针，可能指向链表中任意节点或 `null`。要求返回这个链表的**深拷贝**，所有新节点都是全新的，指针也都指向新节点。

**【模板解析与思路归纳】：**
- 难点在于 `random` 指针可能指向后面还没创建的节点，不能一次遍历就搞定。
- **节点结构区别**：这道题的节点是 `Node` 类，比普通的 `ListNode` 多了一个 `random` 属性，可以指向链表中任意节点或 `None`。
- **哈希表法（最好理解）**：用 `{老节点 → 新节点}` 的哈希表建立映射。
  1. 第一次遍历：只创建所有新节点，把映射关系存进哈希表，暂时不处理指针。
  2. 第二次遍历：此时所有新节点都已存在，通过哈希表查找，给每个新节点赋 `next` 和 `random` 指针。
- **核心坑点**：不能像普通链表那样边走边 `new`，因为 `random` 指针可能指向链表后半部分，那时新节点还没创建。必须分两趟——**先建节点，再建关系**。
- **注意点**：
  - 哈希表要用 `hash_map.get(x)` 而不是 `hash_map[x]`，因为 `next` 和 `random` 可能是 `None`，直接用 `[]` 会 `KeyError`。`get(None)` 会返回 `None`，正好符合预期。
  - 也可以提前塞入 `hash_map[None] = None`，然后就能直接用 `[]` 访问。

**答案代码（Python版）：**
```python
class Solution:
    def copyRandomList(self, head: 'Node') -> 'Node':
        if not head:
            return None
        hash_map = {}
        # 第一趟：只建节点，不管指针
        cur = head
        while cur:
            hash_map[cur] = Node(cur.val)
            cur = cur.next
        # 第二趟：建 next 和 random 指针
        cur = head
        while cur:
            hash_map[cur].next = hash_map.get(cur.next)
            hash_map[cur].random = hash_map.get(cur.random)
            cur = cur.next
        return hash_map[head]
```

#### 复杂度分析

- **时间复杂度**：O(n)，两趟遍历。
- **空间复杂度**：O(n)，哈希表存所有节点映射。
