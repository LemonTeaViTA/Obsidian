## 一、二叉树核心思想与终极模板

二叉树题目的本质只有两种解法思路：**“遍历一遍树（DFS/BFS）”** 或者 **“分解问题（分治/递归）”**。
绝大多数难解的树的题目，靠的都是**后序遍历（Bottom-Up 分治）**：即先向左右子树要答案，结合当前节点的情况，再把答案往上返回给父节点。

### 🌟 递归的“退元法”三大灵魂拷问
写递归时永远不要人脑压栈，只要想清楚：
1. **这个函数是干嘛的？**（明确定义和返回值）
2. **这个函数参数中的节点要做什么？**（当前节点的单层逻辑是什么，什么时候做操作：前、中、后序）
3. **递归的出口（base case）是什么？**（通常是 `if not root: return ...`）

## 二、基础入门：树的遍历 (Traversal)
*掌握递归与迭代形式的树的遍历是所有树形 DP 的基础。*

### 二叉树的中序遍历

给定一个二叉树的根节点 `root`，返回它的中序遍历（左 -> 根 -> 右）。

#### 解题思路

中序遍历非常适合二叉搜索树（BST），因为 BST 的中序遍历就是一个**递增**序列。
这里的核心在于代码模板的记忆：
1. **递归法**：非常符合人类直觉，按“处理左子节点 -> 把当前节点加入结果 -> 处理右子节点”的顺序书写即可。
2. **迭代法**：它是非递归遍历二叉树的基础模板。核心思想是借助栈（模拟系统的函数压栈过程）：
   - 不断地往左下角钻，沿途把遇到的所有节点全都压进栈（既然要中序，那就一头扎进最左面）。
   - 钻到底之后没有节点了，开始从栈顶弹出一个节点，将它的值记录下来。
   - 既然左侧处理完了，根处理完了，现在转向刚刚那个弹出的节点的**右子树**。
   - 重复这个钻地的动作。

#### 代码实现

```python
class Solution:
    # --- 解法 1: 递归法 ---
    def inorderTraversal_recursive(self, root: TreeNode) -> List[int]:
        res = []
        def dfs(node):
            if not node:
                return
            dfs(node.left)
            res.append(node.val)
            dfs(node.right)
        dfs(root)
        return res
        
    # --- 解法 2: 迭代法（常用模板） ---
    def inorderTraversal(self, root: TreeNode) -> List[int]:
        res, stack = [], []
        curr = root
        
        while curr or stack:
            # 1. 一直向左走，把沿途节点全压入栈
            while curr:
                stack.append(curr)
                curr = curr.left
                
            # 2. 走到头了，弹出一个节点进行处理 (这是中序遍历最左侧的那个)
            curr = stack.pop()
            res.append(curr.val)
            
            # 3. 转向该节点的右子树
            curr = curr.right
            
        return res
```

### 二叉树的层序遍历

给你二叉树的根节点 `root` ，返回其节点值的层序遍历。 （即逐层地，从左到右访问所有节点）。

#### 解题思路

层序遍历考察的是**广度优先搜索（BFS）**的基本功。进行 BFS 必须要借助队列（双端队列 `deque`）来实现“先入先出”。

为了实现题目要求的“按层分组”（即返回 `List[List[int]]` 结构，一层一个数组），有一个极为经典的模板套路：
在 `while queue:`（只要队列非空）的最外层大循环内部，**第一件事就是立刻快照并记录下当前队列里元素的数量 `size = len(queue)`**。因为此时队列里所装载的所有节点，恰好就是当前“完整一层”的全部节点。
接下来，直接针对这个 `size` 使用一个 `for ... in range(size)` 的内层循环，刚好把这层节点完全摘除下来保存，同时把它们的子节点再压进队列排队。当这个 `for` 完结时，意味着这一层已经干干净净被打包好了，而队列里装的全都是下一层的节点了。

#### 代码实现

```python
from collections import deque

class Solution:
    def levelOrder(self, root: TreeNode) -> List[List[int]]:
        if not root:
            return []
            
        res = []
        queue = deque([root])
        
        while queue:
            level_nodes = []
            size = len(queue)  # 核心：记录当前层的节点数
            
            for _ in range(size):
                node = queue.popleft()
                level_nodes.append(node.val)
                # 将下一层的孩子加入队列
                if node.left:
                    queue.append(node.left)
                if node.right:
                    queue.append(node.right)
                    
            res.append(level_nodes)
            
        return res
```

### 二叉树的右视图

给定一个二叉树的 根节点 `root`，想象自己站在它的右侧，按照从顶部到底部的顺序，返回从右侧所能看到的节点值。

#### 解题思路

寻找二叉树的“右视图”，本质上就是寻找树中每一层的**最右侧节点**。理解了这一点，就顺带打通了这道题的两种经典解法：

1. **层序遍历法（BFS）**：
   既然是要找每一层的最右节点，我们完全可以直接照搬上面“层序遍历”的模板代码。在使用 `size = len(queue)` 并进行 `for` 循环剥离每一层节点时，我们只需要捕获这一层被弹出的**最后一个节点**（或者循环进行到 `i == size - 1` 时的那个节点），将它的值加入结果数组即可。这种方法无需任何特殊记忆，凭借肌肉记忆默写 BFS 队列即可完美解决。

2. **深度优先搜索（DFS）变种法**：
   除了按层搜索，如果一定要使用递归该怎么办？我们只需要稍作变通：正常的先序遍历是“根 -> 左 -> 右”，而在右视图这道题里，我们可以强行按照 **“根 -> 右 -> 左”** 的顺序去递归访问，并且每次把当前节点所处的深度 `depth` 传下去。
   当结果数组的长度正好等于当前深度时，说明这是本层深度 **第一次** 被访问到！因为我们是优先往右子树扎的，所以第一次碰到的自然就是这一层最右边的节点。

#### 代码实现

**解法一：层序遍历 BFS（最直观的标准模板）**
这是最为推荐和顺理成章的思考方式。直接套用上文刚讲过的“层序遍历”模板，每一层向右侧看的时候，视野里一定都是这一层的**最后被抛出来的节点**，将其拦截进结果数组即可：

```python
from collections import deque

class Solution:
    def rightSideView(self, root: Optional[TreeNode]) -> List[int]:
        if not root:
            return []
            
        res = []
        queue = deque([root])
        
        while queue:
            size = len(queue)
            
            for i in range(size):
                node = queue.popleft()
                
                # 当前层的最后一个节点（即最右侧节点）
                if i == size - 1:
                    res.append(node.val)
                    
                # 依然是正常的从左向右推送子节点
                if node.left:
                    queue.append(node.left)
                if node.right:
                    queue.append(node.right)
                    
        return res
```

**解法二：深度优先 DFS（打破刻板印象的奇技淫巧）**
既然正常的先序遍历是“根->左->右”，那右视图自然就是“根->右->左”。只需强制记录一个深度变量，并且让它强行优先往右侧遍历深潜。只要碰到 `res` 的长度刚刚好和当前探测层数相等，说明它就是这一层最右边的节点：

```python
class Solution:
    def rightSideView(self, root: Optional[TreeNode]) -> List[int]:
        res = []
        
        def dfs(node, depth):
            if not node:
                return
            
            # 如果当前结果列表的长度等于当前深度
            # 说明这是该深度层“破冰”首个被访问的节点
            if len(res) == depth:
                res.append(node.val)
                
            # 变异的先序遍历：根 -> 右 -> 左
            dfs(node.right, depth + 1)
            dfs(node.left, depth + 1)
            
        dfs(root, 0)
        return res
```

## 三、进阶：属性与结构变换 (Attributes & Structure)
*这类题目通常采用分解问题的思路。递归处理左半边和右半边，再结合根节点。*

### 二叉树的最大深度

给定二叉树的根节点 `root`，返回其最大深度。二叉树的深度为根节点到最远叶子节点的最长路径上的节点数。

#### 解题思路

这里面其实隐含了二叉树学习中极其重要的“**分治法（Divide and Conquer）**”思维：
如果要计算一颗苍天大树的总深度，对于一个处在树干某处的节点来说，其实这等于**它左子树的最大深度** 和 **它右子树的最大深度** 中的较大者，再加上 1（指这个节点自己本身）。

这种思路完美契合了**最经典的后序遍历**（底层往上收集结果）：
先向左孩子要深度，再向右孩子要深度，得到两个深度之后挑个大的，加一，向上返回递交给父节点，一路上抛直到 `root`。如果是空节点，代表到底了，返回 `0`。

#### 代码实现

```python
class Solution:
    def maxDepth(self, root: TreeNode) -> int:
        if not root:
            return 0
        
        # 向左、右子树分别发起深度查询的调用（分出子问题）
        left_depth = self.maxDepth(root.left)
        right_depth = self.maxDepth(root.right)
        
        # 将子问题的结果合并之后返回给更上层的调用
        return 1 + max(left_depth, right_depth)
```

### 翻转二叉树

给你一棵二叉树的根节点 `root` ，翻转这棵二叉树，并返回其根节点。*(注：著名的 Homebrew 作者因写不出这道题被大厂拒).*

#### 解题思路

翻转二叉树这道题，看起来玄乎，背后的逻辑却极其质朴无华：其实就是对于树里面的任意一个节点，直接动手**把它的左边亲儿子和右边亲儿子交换一下位置**。
交换完这个节点的两个小儿子后，再以这俩儿子为新的根节点，去递归翻转它们自己内部的子树组织即可。

因为我们要在进入节点的时候就立马交换两边的子树，“根、左、右”这样的**前序遍历**正好可以在这里大展身手。

#### 代码实现

```python
class Solution:
    def invertTree(self, root: TreeNode) -> TreeNode:
        if not root:
            return None
            
        # “前序遍历”操作时间点：首先就地交换当前节点的左右子树
        root.left, root.right = root.right, root.left
        
        # 让此时的左侧子树继续跑到内部去翻转它底下的内容
        self.invertTree(root.left)
        # 让此时的右侧子树继续跑去内部翻转
        self.invertTree(root.right)
        
        # 返回翻转好的这棵树木
        return root
```

### 二叉树的直径

给你一棵二叉树的根节点 `root`，返回该树的 直径 。
二叉树的 直径 是指树中任意两个节点之间最长路径的 长度 。这条路径可能经过也可能不经过根节点 `root` 。两节点之间路径的 长度 由它们之间边数表示。

#### 解题思路

计算二叉树的直径核心在于：任何一条最长路径一定会以某个特定的节点作为“桥接点”，将左右两边的路径连接起来。

在这个解法中，我们采用更符合“边数”直觉的定义。我们定义 `dfs(node)` 的含义为：**从当前节点 `node` 出发，不断向下延伸所能达到的最长单边路径的“边数”**。

- **空节点处理（Base Case）**：当节点为空时返回 `-1`。之所以返回 `-1`，是因为我们在计算父节点的单边长度时会加 1（即 `dfs(node.left) + 1`）。这样当真实节点为叶子节点时，其左右单边的长度就是 `-1 + 1 = 0`，完美符合“叶子节点向下没有边”的物理意义。
- **左右边数计算**：对于任意节点，其左子枝条能提供的最长边数为 `l_len = dfs(node.left) + 1`，右子枝条能提供的最长边数为 `r_len = dfs(node.right) + 1`。
- **更新全局直径**：经过当前节点作为桥接点的最长路径长度，恰好等于两边提供的单边边数之和，即 `l_len + r_len`。我们在后序遍历的过程中，用变量 `self.ans` 不断“打擂台”记录下这个最大和。
- **向父节点述职（Return）**：当前节点也需要作为其父节点枝条的一部分。因此，它只需挑选自己左右两边最长的一条返回，即 `max(l_len, r_len)`。

#### 代码实现

```python
class Solution:
    def diameterOfBinaryTree(self, root: Optional[TreeNode]) -> int:
        self.ans = 0
        def dfs(node: Optional[TreeNode]) -> int:
            if node is None:
                return -1
            l_len = dfs(node.left) + 1
            r_len = dfs(node.right) + 1
            self.ans = max(self.ans, l_len + r_len)
            return max(l_len, r_len)
        dfs(root)
        return self.ans
```

### 二叉树中的最大路径和

给你一个二叉树的根节点 `root` ，返回其 **最大路径和** 。

二叉树中的 路径 被定义为一条节点序列，序列中每对相邻节点之间都存在一条边。同一个节点在一条路径序列中 **至多出现一次** 。该路径 **至少包含一个** 节点，且不一定经过根节点。路径和 是路径中各节点值的总和。

#### 解题思路

这道题是“二叉树的直径”这道题的威力加强版。在《二叉树的直径》中，我们是求路径中最多的“边数”；而在这道题中，我们要在这个路径形态的基础上，求路径上节点值的“最大和”。
但是和求边数（一定大于等于0）不同的是，由于节点的值可能为负数，一旦某段子路径的和为负数，将它拼接到总路径中不仅没有增益，反而会拖后腿。这就是本题的核心考点！

因此，我们沿用后序遍历的思路，设计一个递归函数 `maxGain(node)`。这个函数的定义是：**计算出如果一条路径要从当前节点 `node` 开始并只顺着左、右其中一条边向下延伸，所能获得的最大收益（用于向父节点交代）**。

在递归函数的内部，在向父节点抛出收益之前，我们需要先暗中盘算一下：
1. 先递归拿到左子节点所能提供的最大向下收益 `left_gain`。**注意：如果收益是负数，我们宁愿不走左边（把它当成 0）**。
2. 同理拿到右子节点的最大收益 `right_gain`（负数也看作 0）。
3. 此时，以当前 `node` 为最高点直接串联起左右两边所能得到的**“局部最大闭环路径和”**等于 `node.val + left_gain + right_gain`。
4. 我们用这个经过当前节点的“局部最大路径和”去挑战并更新**全局的最大路径和** `max_sum`。
5. 最后，递归函数向上一层返回的内容，必须是严格遵守“一条向下分支”的规则的结果（毕竟路径不能分岔两头走再折返回去）：`node.val + max(left_gain, right_gain)`。

#### 代码实现

```python
class Solution:
        # 既然有可能全是负数节点，我们的初始全局最大值就应该被初始化为一个极小值
        self.max_sum = float('-inf')

    def maxPathSum(self, root: Optional[TreeNode]) -> int:
        def maxGain(node):
            if not node:
                return 0

            # 递归向下去拿左右子树所能提供的最大单边收益
            # 关键剪枝：只有当子树收益为正，我们才会加进来，否则哪怕断开不走把它当成 0 也比扣分强
            left_gain = max(maxGain(node.left), 0)
            right_gain = max(maxGain(node.right), 0)

            # 以当前节点作为这条局部路径最高点（即“拱桥”的顶部）时的总收益
            price_newpath = node.val + left_gain + right_gain

            # 用这条可能串联了两侧后形成的最强子路径，去挑战历史遗留的全局最大纪录
            self.max_sum = max(self.max_sum, price_newpath)

            # 为了能合乎规则地挂靠到父节点上，我们只能返回它自己加上这副骨架上“更强的一边”
            return node.val + max(left_gain, right_gain)

        maxGain(root)
        return self.max_sum
```

### 对称二叉树

给你一个二叉树的根节点 `root` ， 检查它是否轴对称。

#### 解题思路

检查一棵二叉树是否对称，实质上是要比较它的**左右两棵子树** `t1` 和 `t2`：
1. 它们的值是否相等？
2. `t1` 的左边子树 和 `t2` 的右边子树 是否对称？
3. `t1` 的右边子树 和 `t2` 的左边子树 是否对称？

这显然可以用递归求解，同时我们只需把左树的左分支和右树的右分支一起传给下一层判断；把左树的右分支和右树的左分支传给另一路判断即可。

#### 代码实现

```python
class Solution:
    def isSymmetric(self, root: TreeNode) -> bool:
        if not root:
            return True
            
        def check(left_node, right_node):
            if not left_node and not right_node:
                return True
            if not left_node or not right_node:
                return False
            if left_node.val != right_node.val:
                return False
                
            # 左边的左孩子 和 右边的右孩子 比较；左边的右孩子 和 右边的左孩子 比较
            return check(left_node.left, right_node.right) and check(left_node.right, right_node.left)
            
        return check(root.left, root.right)
```

## 四、高阶构造与深度搜索 (Advanced DFS)
*这部分开始考验对二叉树形态的掌控能力，通常需要更深度的后序遍历或者携带外部状态。*

### 二叉树的最近公共祖先 (LCA)

给定一个二叉树, 找到该树中两个指定节点 `p` 和 `q` 的最近公共祖先。

#### 解题思路

这是非常非常经典的**后序遍历**（Bottom-Up 分治）代表题！
核心思路是：遇到 `p` 或者 `q` 就赶紧向上返回，在中间相遇的那个点就是它们的共同祖先：
1. **终止条件**：如果当前节点 `root` 为空，或者它就是 `p` 或 `q` 中的一个，直接返回它自己（相当于自爆身份：“我在这里！”）。
2. **分解问题**：分别去左子树找找有没有 `p` 或 `q`，再去右子树找找。
3. **合并结果**：
   - 如果左边和右边都返回了非空节点，说明 `p` 和 `q` 一左一右散落在当前节点的两边。这说明当前 `root` 就是它们的**最近公共祖先**，立刻把它向上抛！
   - 如果只在左边找到了，那就说明 `p` 和 `q` 都在左边（另一个其实就是找到的这一个的子节点），把左边找到的结果继续往上抛。同理右边。

#### 代码实现

```python
class Solution:
    def lowestCommonAncestor(self, root: 'TreeNode', p: 'TreeNode', q: 'TreeNode') -> 'TreeNode':
        # Base case: 越界或者是直接碰到了p或q
        if not root or root == p or root == q:
            return root
            
        # 分治：去左右子树找
        left = self.lowestCommonAncestor(root.left, p, q)
        right = self.lowestCommonAncestor(root.right, p, q)
        
        # 如果左右两边都有收获，说明我(root)就是它们的结合点，LCA诞生！
        if left and right:
            return root
            
        # 如果只有一边有，那么把有的那一边继续向上托举
        return left if left else right
```

### 从前序与中序遍历序列构造二叉树

给定两个整数数组 `preorder` 和 `inorder` ，其中 `preorder` 是先序（根->左->右），`inorder` 是中序（左->根->右），构造二叉树并返回其根节点。

#### 解题思路

这类构造树题目的核心，是把不同遍历产生的天然规律组合利用：
1. **先序遍历的性质**：它弹出的第一个数字，**永远是这棵树（或子树）的根节点**！
2. **中序遍历的性质**：拿到上面那个根节点数字，去中序数组里寻找它的位置。找到后，**它左边的一大堆数字就是左子树的所有成员，右边的一大堆数字就是右子树的所有成员**。

所以我们只要每次取出 `preorder[0]` 作为根节点，用它跑到 `inorder` 中一分为二，计算出左半边节点集合的长度（设为 `len`），就可以相应地从 `preorder` 后面的数字里割出长度为 `len` 的部分，连同左半边的中序数组一起交给**递归**函数继续构造左子树；剩下的丢过去构造右子树即可。

#### 代码实现

```python
class Solution:
    def buildTree(self, preorder: List[int], inorder: List[int]) -> TreeNode:
        if not preorder or not inorder:
            return None
            
        # 1. 抓出根节点
        root_val = preorder[0]
        root = TreeNode(root_val)
        
        # 2. 找到根节点在中序数组里的位置
        idx = inorder.index(root_val)
        
        # 3. 递归构造左右子树
        # idx 恰巧也就是左子树中节点的个数
        root.left = self.buildTree(preorder[1 : idx + 1], inorder[:idx])
        root.right = self.buildTree(preorder[idx + 1:], inorder[idx + 1:])
        
        return root
```

### 二叉树展开为链表

给你二叉树的根结点 `root` ，请你将它展开为一个单链表：
- 展开后的单链表应该同样使用 `TreeNode` ，其中 `right` 子指针指向链表中下一个结点，而左子指针始终为 `null` 。
- 展开后的单链表应该与二叉树 **先序遍历** 顺序相同。

#### 解题思路

这道题如果使用正常的“先序遍历（根->左->右）”或者寻找前驱节点就地展开会有一定的思维绕弯。实际上，最标准且最不容易出错的主流写法，是利用**倒序的先序遍历**（也就是逆后序遍历：`右 -> 左 -> 根`）配合链表的**头插法（Head Insertion）**。

试想一下，如果我们要把一堆节点串成：`[1] -> [2] -> [3] -> [4]`。如果我们能够最先访问到最末尾的 `[4]`，再访问 `[3]` 把它的 next 指向 `[4]`，再访问 `[2]` 把它的 next 指向 `[3]`……依次从尾到头处理，事情就会变得极其直观。

这个从后往前的访问顺序，其实就是将标准先序遍历（根、左、右）反过来，变成**“右、左、根”**。
具体操作为：
1. 建立一个全局或实例变量 `head`，初始为 `None`，它永远指向我们**当前已经拼接好的这条单链表的头部**。
2. 进行变种后序递归：优先一头扎进右子树，然后扎进左子树。
3. 当递归在这两步都出栈，终于轮到处理**当前节点 `root`** 的时候，意味着它后面的子节点们都已经串成了一根从 `head` 开始的长长单链表了。
4. 我们需要做的只有三件事：
   - 断开当前节点向左的牵绊：`root.left = None`
   - 将当前节点指向那根已经拼好的链表头部：`root.right = self.head`
   - 既然当前节点加入了首位，那它本身就成了新的链表头：`self.head = root`

#### 代码实现

```python
class Solution:
    def __init__(self):
        # 这个指针代表当前已经拼接好的“单链表的头节点”
        self.head = None

    def flatten(self, root: Optional[TreeNode]) -> None:
        """
        Do not return anything, modify root in-place instead.
        """
        if root is None:
            return
            
        # 颠倒先序的过程：先递归右边，再递归左边
        self.flatten(root.right)
        self.flatten(root.left)
        
        # 此时到了处理“根节点”的时候（右和左都已经拼接在 head 里了）
        root.left = None
        # 当前树节点指向已建好的头节点（链表头插法）
        root.right = self.head  
        # 我们把新的头节点更新为当前节点自身
        self.head = root 
```

### 路径总和 III

给定一个二叉树的根节点 `root` ，和一个整数 `targetSum` ，求该二叉树里节点值之和等于 `targetSum` 的 **路径** 的数目。

路径不需要从根节点开始，也不需要在叶子节点结束，但是路径方向必须是向下的（只能从父节点到子节点）。

#### 解题思路

这道题最容易让人想到的是“双重 DFS（暴力搜索）”：让每一个节点都成为一次起点，再用一个 DFS 去向下暴力遍历并累加看是否能等于 `targetSum`。但这种解法会有大量的重复计算，在节点成了一条长链等极端情况下，时间复杂度会退化到 $O(N^2)$。

根据上文定下的偏好规矩，我为你直接提供**最标准、最进阶、且被广泛推崇的主流解法**——结合 **“前缀和 + 哈希表” (Prefix Sum + Hash Map)** 的深度优先搜索（DFS），它能够做到 $O(N)$ 的惊人效率。

如果把树看成多条从上到下的数组岔路，那么寻找一段和为 `target` 的子树枝，就变成了在数组中寻找一段和为 `target` 的连续子数组！
具体操作步骤如下：
1. **维护前缀和哈希表**：用一个字典 `prefix_sum_count` 记录从根节点走到当前节点这一路上，各个“前缀和”出现的次数。
   *(注意：初始时必须手动塞入一个 `{0: 1}`，代表前缀和为 0 的路径默认出现了一次。这是为了当“从根节点直接到当前节点”的这整条满路径恰好等于 `targetSum` 时，能够成功通过 `curr_sum - targetSum == 0` 匹配出 1 条有效路径。)*
2. **向下探索并匹配差值**：每向下走到一个节点，就计算当前层级的累加前缀和 `curr_sum`，看看能够补齐差值的那个 `curr_sum - targetSum` 在不在历史上的祖先哈希表中。如果在，说明曾经有一个祖先节点到当前这段子路的和恰好为目标值，我们直接把此时出现的次数累加到结果里！
3. **入栈与回溯状态（极其关键的一步）**：将当前的 `curr_sum` 频次加 1 登记进哈希表，继续递归探索左孩子和右孩子。当 `root` 结点的左右子树都已经探索完了，函数要 `return` 去另一条截然不同的岔路（即去别的并行树枝上搜索）之前，**必须将哈希表中当前计算贡献的 `curr_sum` 频次减去 1**。因为跨树枝之间的路径是相互不能连通也不能借用的，绝对不能“过界污染”兄弟路径。

#### 代码实现

```python
class Solution:
    def pathSum(self, root: Optional[TreeNode], targetSum: int) -> int:
        # prefix_sum_count: 记录所有的【前缀和: 出现次数】
        # 初始插入 0:1 保证从根节点正合的路径能匹配上
        self.prefix_sum_count = {0: 1}
        
        def dfs(node, curr_sum):
            if not node:
                return 0
                
            # 1. 走到当前节点，更新这根树枝上的累加前缀和
            curr_sum += node.val
            
            # 2. 若哈希表里有 (当前和 - 目标和)，说明它和上面某个祖先之间的路径和恰为目标和
            # 直接把那几个祖先的出现次数拿到，这就是新诞生的有效路径数目
            res = self.prefix_sum_count.get(curr_sum - targetSum, 0)
            
            # 3. 将当前的前缀和加进哈希表（入栈），为接下来左孩子和右孩子做准备
            self.prefix_sum_count[curr_sum] = self.prefix_sum_count.get(curr_sum, 0) + 1
            
            # 4. 继续沿着这条树枝探入其左边与右边深处
            res += dfs(node.left, curr_sum)
            res += dfs(node.right, curr_sum)
            
            # 5. 回溯清理：从当前的这条长树枝退出了，准备横向移动去别的树枝
            # 必须把当前节点加进去的那个前缀和频次划掉，避免跨树枝互串污染！
            self.prefix_sum_count[curr_sum] -= 1
            
            return res
            
        return dfs(root, 0)
```

## 五、特殊结构：前缀树 (Trie)

### 实现 Trie (前缀树)

Trie 是一种树形数据结构，用于高效地存储和检索字符串数据集中的键。请你实现 `insert`, `search` 和 `startsWith` 方法。

#### 解题思路

前缀树（字典树）有着一个极其反直觉的天然特质：**它的节点里面并不用来存储一个完整的字符！** 这个字符的存在，是由“指向这个节点的那条边的 Key”所代表的。所以一个完整的字符串在它身上就像顺着一系列管道（字典）溜下来。
实现上的两大要点技巧：
1. `children = {}`：我们不用传统的 `left/right` 来拉出边，而是建立一个字典。字典的键是字符名字（边），字典的值就是它连接的新的次级 `Trie` 节点对象。
2. `is_end = False`：由于单词有长短，有些单词走到一半就停了（比如 `app` 停在第二个 `p`），所以还需要一个用于“在这个节点打个卡，证明有一条完整的词停留在这个节点”的标识。

#### 代码实现

```python
class Trie:
    def __init__(self):
        # 节点内不需要强行存储 char，它的存在由上一级的 key 指向该节点的引线保证
        self.children = {}
        # 是否有任何一条词在这里打末尾标记
        self.is_end = False

    def insert(self, word: str) -> None:
        node = self
        for char in word:
            # 如果这条字符边还没造好，先开通一个新的树形节点供其流下来
            if char not in node.children:
                node.children[char] = Trie()
            # 顺着这条路滑到下面的节点里去
            node = node.children[char]
        # 到达单词末端最后这个停顿节点，给它亮起打卡标识
        node.is_end = True

    def search(self, word: str) -> bool:
        node = self
        for char in word:
            # 中断了的话说明没有这棵树的引路条
            if char not in node.children:
                return False
            node = node.children[char]
        # 跑到最后，还要看这里是不是一个词的终点（防止你只查个 `app` 却跑到了 `apple` 的一半）
        return node.is_end

    def startsWith(self, prefix: str) -> bool:
        node = self
        for char in prefix:
            if char not in node.children:
                return False
            node = node.children[char]
        # 这里是前缀树，所以只要前面几个管子通了、管这里是不是终点站打卡区，都算前缀通过
        return True
```

## 六、二叉搜索树 (BST)

> **💡 BST 的核心心法与终极规律**
> 二叉搜索树（Binary Search Tree）最基本的定义是：左子树所有节点严格小于根节点，右子树所有节点严格大于根节点。
> **最核心的解题规律：对 BST 进行“中序遍历”（左 -> 根 -> 右），得到的必定是一个严格递增的有序序列！**
> 绝大多数二叉搜索树的算法题，只要联想到“把它当中序遍历展开看作一个有序数组”，一切都能迎刃而解！

### 验证二叉搜索树

给你一个二叉树的根节点 `root` ，判断其是否是一个有效的二叉搜索树。

有效 二叉搜索树定义如下：
- 节点的左子树只包含 **严格小于** 当前节点的数。
- 节点的右子树只包含 **严格大于** 当前节点的数。
- 所有左子树和右子树自身必须也是二叉搜索树。

#### 解题思路

验证二叉搜索树（BST）的核心陷阱在于：不仅要保证当前节点的值分别大于左孩子且小于右孩子，还必须保证当前节点 **大于左子树中的所有节点，且小于右子树中的所有节点**。

为了实现这一点，我们可以采用一种自顶向下的“区间限定”策略。我们通过在递归过程中传递一个允许的值域区间 `(low, high)` 来约束每一个节点：
- 对于根节点，它的值域范围显然是不受限的，即 `(-∞, +∞)`。
- 当我们向左子树递归时，左子树中的所有节点不仅要符合原有的下界，还必须严格小于当前节点的值。因此，我们将当前节点的值作为新的上限 `high` 传递下去。
- 同理，当我们向右子树递归时，右子树中的所有节点必须严格大于当前节点的值，于是我们将当前节点的值作为新的下限 `low` 传递下去。
- 在递归过程中如果遇到任何一个节点的值越界了（`node.val <= low` 或 `node.val >= high`），则直接判定为非法 `False`；如果一路校验直到空节点，说明整条路径都合法，返回 `True`。

#### 代码实现

```python
class Solution:
    def isValidBST(self, root: Optional[TreeNode]) -> bool:
        
        # 递归函数：判断 node 是否在合法的 (low, high) 区间内
        def valid(node, low, high):
            if node is None:
                return True
                
            # 如果当前节点的值超出了允许的界限，则不是有效的二叉搜索树    
            if node.val >= high or node.val <= low:
                return False
                
            # 递归校验左子树（上限更新为当前节点的值） 且 递归校验右子树（下限更新为当前节点的值）
            return valid(node.left, low, node.val) and valid(node.right, node.val, high)
            
        # 初始时，根节点不受限制，下界为负无穷，上界为正无穷
        return valid(root, float('-inf'), float('inf')) 
```

### 二叉搜索树中第 k 小的元素

给定一个二叉搜索树的根节点 `root` ，和一个整数 `k` ，请你设计一个算法查找其中第 `k` 小的元素（`k` 从 1 开始计数）。

#### 解题思路

破解这道题的钥匙在于牢记二叉搜索树（BST）的最核心基因：**对 BST 进行“中序遍历”（即左子树 -> 根节点 -> 右子树），得到的序列天然就是一个严格递增的有序序列**。

既然我们要找“第 $k$ 小的元素”，最直截了当的思路就是模拟这个从小到大输出的过程，一边遍历一边计数，并在刚好数到第 $k$ 个元素时停下。
具体而言，我们可以按照中序遍历的模板套路，先一路递归到最左侧的最小叶子节点；当函数从最底层“出栈”第一次访问到具体的某个节点时（即此时刚处理完它的左子树），将全局或实例计数器加 1。一旦计数器累加到了 $k$，就说明当前访问的这个节点正是我们苦苦寻找的目标，我们可以立即将其值记录下来，并用条件判断让后续的递归调用直接返回（剪枝），从而避免把整棵树都多此一举地遍历完。

#### 代码实现

```python
class Solution:
    def kthSmallest(self, root: Optional[TreeNode], k: int) -> int:
        self.count = 0  # 计数器，记录当前是第几个被访问的节点
        self.res = None # 最终的结果
        
        def inorder(node):
            # 如果节点为空，或者我们已经找到了答案，则直接返回（剪枝操作）
            if not node or self.res is not None:
                return
            
            # 中序遍历第一步：处理左子树
            inorder(node.left)
            
            # 中序遍历第二步：处理当前节点（这一步代表了节点的“访问”）
            self.count += 1
            if self.count == k:
                self.res = node.val
                return
            
            # 中序遍历第三步：处理右子树
            inorder(node.right)
            
        # 启动中序遍历
        inorder(root)
        
        return self.res
```
