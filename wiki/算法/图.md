# 图与广度/深度优先遍历 (Graph & BFS/DFS)

> [[算法_MOC|返回算法导航]]

## 一、图的核心思想与归类技巧

图论比起树，多了“环”的可能性，且节点可以任意连接。
在算法面试中，90%的图论题目都是通过**广度优先搜索（BFS）** 和 **深度优先搜索（DFS）** 来解决。根据题意可以细分为以下几大类套路：

1. **网格图 / 岛屿问题 (Flood Fill)**：在二维矩阵中进行上下左右的探索。核心是**边界判断**和**打卡标记（visited）**以防死循环。
2. **拓扑排序 (Topological Sort)**：针对有向无环图（DAG），比如“课程表”、“任务依赖”。核心思想是**入度表 + 队列**。
3. **最短路径问题**：如果边权为 1，一律使用普通的 **BFS + 队列** 逐层扩展（如单词接龙）；如果有大小不一的边权（或者网格中高度不一），需引入 **优先队列（Heap）做 Dijkstra**（如接雨水 II）。

## 二、网格与岛屿模型 (Grid DFS)

### 岛屿数量（Number of Islands）

给你一个由 `'1'`（陆地）和 `'0'`（水）组成的的二维网格，请你计算网格中岛屿的数量。
岛屿总是被水包围，并且每座岛屿只能由水平方向和/或竖直方向上相邻的陆地连接形成。
此外，你可以假设该网格的四条边均被水包围。

#### 核心思路与模板解析

经典的“沉岛思想”（Flood Fill）。遇到陆地时，将其计入岛屿数量，并立刻启动搜索（DFS 或 BFS），将与该陆地相连的所有 `'1'` 统统改为 `'0'`（即“沉没”处理，或打上 visited 标记）。这就保证了后续遍历到同一片岛屿的其他地块时，会被直接忽略，避免重复计算。

优先展示结构清晰的 DFS（深度优先搜索）与 BFS（广度优先搜索）两种标准解法。

#### 解法一：深度优先搜索 (DFS)

DFS 代码最简洁，逻辑上像病毒蔓延，一条路走到黑然后回退。

```python
class Solution:
    def numIslands(self, grid: List[List[str]]) -> int:
        if not grid:
            return 0
            
        def dfs(r, c):
            # 边界判定，如果越界或者是水（'0'），就退回
            if r < 0 or r >= len(grid) or c < 0 or c >= len(grid[0]) or grid[r][c] == '0':
                return
            
            # 标记当前陆地为已访问（沉岛）
            grid[r][c] = '0'
            
            # 向上下左右四个方向继续感染
            dfs(r + 1, c)
            dfs(r - 1, c)
            dfs(r, c + 1)
            dfs(r, c - 1)
            
        count = 0
        for i in rawnge(len(grid)):
            for j in range(len(grid[0])):
                # 发现新的未访问过的陆地
                if grid[i][j] == '1':
                    count += 1
                    dfs(i, j)  # 派兵接管这片岛屿，将其全部沉没
                    
        return count
```

#### 解法二：广度优先搜索 (BFS)

BFS 使用队列逐层向外推展，更符合直觉上的“水波扩散”，并且在递归深度极大的极端网格中不会导致栈溢出。

```python
import collections

class Solution:
    def numIslands(self, grid: List[List[str]]) -> int:
        if not grid:
            return 0
            
        count = 0
        rows, cols = len(grid), len(grid[0])
        
        for i in range(rows):
            for j in range(cols):
                if grid[i][j] == '1':
                    count += 1
                    # 发现新岛屿，启动 BFS 开始沉岛
                    grid[i][j] = '0'
                    queue = collections.deque([(i, j)])
                    
                    while queue:
                        r, c = queue.popleft()
                        # 遍历相邻的四个方向
                        for dr, dc in [(1, 0), (-1, 0), (0, 1), (0, -1)]:
                            nr, nc = r + dr, c + dc
                            # 如果在界内且也是陆地，就加入队列并立即沉岛
                            if 0 <= nr < rows and 0 <= nc < cols and grid[nr][nc] == '1':
                                queue.append((nr, nc))
                                grid[nr][nc] = '0' # 入队时立刻标记，避免被重复入队
                                
        return count
```

#### 复杂度分析
- **时间复杂度**：$O(M \times N)$，其中 $M$ 是网格的行数，$N$ 是网格的列数。虽然代码中有双重 `for` 循环和内部的 DFS/BFS 递归/循环，但在整个遍历过程中，每个节点（无论是陆地还是水）最多只会被访问和处理常数次（一旦变成 `'0'` 就不再作为起点触发搜索）。因此总时间与网格的面积成正比。
- **空间复杂度**：
  - **DFS**：最坏情况下（整个网格全都是 `'1'`，没有水），深度优先搜索的递归深度会达到 $M \times N$，空间复杂度为 $O(M \times N)$。
  - **BFS**：最坏情况下（整个网格全都是 `'1'`），队列中最多同时存放 $\min(M, N)$ 个节点，空间复杂度为 $O(\min(M, N))$。

### 单词搜索（Word Search）

给定一个 `m x n` 二维字符网格 `board` 和一个字符串单词 `word`。如果 `word` 存在于网格中，返回 `true`。同一个单元格内的字母不允许被重复使用。

#### 解题思路
- 岛屿问题是不回头地踩坑，踩过就不要了（因为目标是数一整片的总数）；而搜索单词是一种**回溯算法**，走到死胡同时，要把借过的格子“还”回去，以便向其它方向探索时这个格子还能用。
- 核心标志就是在 DFS 探测结束后，要抹除当前格子的 visited 状态（或者恢复原来的字母，本题采用临时替换为 `#` 防重踩的技巧）。

#### 代码实现
```python
class Solution:
    def exist(self, board: List[List[str]], word: str) -> bool:
        rows, cols = len(board), len(board[0])
        
        def dfs(r, c, idx):
            # 单词全部匹配完毕，胜利
            if idx == len(word):
                return True
                
            # 越界或者当前方格的字母跟咱们正在寻找的不匹配
            if r < 0 or r >= rows or c < 0 or c >= cols or board[r][c] != word[idx]:
                return False
                
            # 暂存这个格子的字母，并标记为已访问（防止转圈圈）
            temp = board[r][c]
            board[r][c] = '#'
            
            # 向四个方向探寻 word 的下个字母
            res = (dfs(r + 1, c, idx + 1) or 
                   dfs(r - 1, c, idx + 1) or 
                   dfs(r, c + 1, idx + 1) or 
                   dfs(r, c - 1, idx + 1))
                   
            # 回溯：恢复格子原本的面貌！
            board[r][c] = temp
            
            return res
            
        for i in range(rows):
            for j in range(cols):
                if dfs(i, j, 0):
                    return True
        return False
```

#### 复杂度分析
- **时间复杂度**：$O(M \times N \times 3^L)$，其中 $M$ 和 $N$ 是网格的长宽，$L$ 是单词 `word` 的长度。网格中每个格子都可以作为起点，对于每个起点最长可能探索长度为 $L$ 的路径。除第一步有 4 个方向外，后续每一步由于不能往回走，只有 3 个候选方向，所以时间复杂度是指数级的。
- **空间复杂度**：$O(L)$，取决于单词长度 $L$。递归栈深度最大为 $L$，并且我们采用原地修改（`board[r][c] = '#'`），没有使用额外的 visited 数组，因而大大优化了空间。

### 腐烂的橘子（Rotting Oranges）

在给定的 `m x n` 网格 `grid` 中，每个单元格可以有以下三个值之一：
- 值 `0` 代表空单元格；
- 值 `1` 代表新鲜橘子；
- 值 `2` 代表腐烂的橘子。

每分钟，腐烂的橘子周围 4 个方向上相邻的新鲜橘子都会腐烂。
返回直到单元格中没有新鲜橘子为止所必须经过的最小分钟数。如果不可能，返回 `-1`。

#### 解题思路
- 本题是绝对经典的**多源广度优先搜索（Multi-source BFS）**模板题。
- 和《岛屿数量》中遇到一个起点就辐射一片不同，腐烂的橘子是**同时**向外扩散传染的。因此，我们不能只把某一个腐烂的橘子作为一个起点，而是在第 0 分钟时，将网格里**所有的腐烂橘子一起放入初始队列**。
- 将每一轮队列的完全遍历视作“1分钟”的流逝。像切洋葱或者水波纹一样，这一分钟内队列里所有的橘子集体向外感染一层。
- **细节注意**：要提前统计好新鲜橘子的总数（`fresh_count`）。如果在 BFS 扩散完成后，新鲜橘子数仍然大于 0，说明存在被空气（0）或者死角隔离的橘子永远无法被感染，返回 `-1`。如果初始就没有新鲜橘子，应直接返回 `0`。

#### 代码实现
```python
import collections
from typing import List

class Solution:
    def orangesRotting(self, grid: List[List[int]]) -> int:
        rows, cols = len(grid), len(grid[0])
        queue = collections.deque()
        fresh_count = 0
        
        # 1. 统计新鲜橘子的总数，并将所有初始时就已经腐烂的橘子入队
        for r in range(rows):
            for c in range(cols):
                if grid[r][c] == 1:
                    fresh_count += 1
                elif grid[r][c] == 2:
                    queue.append((r, c))
                    
        # 边界情况：一开始就没有新鲜橘子，不需要经过任何时间
        if fresh_count == 0:
            return 0
            
        minutes = 0
        directions = [(1, 0), (-1, 0), (0, 1), (0, -1)]
        
        # 2. 多源 BFS：所有腐烂橘子齐头并进向外扩散
        while queue:
            minutes += 1  # 时间流逝了 1 分钟
            # 必须提前记录当前队列的长度，确保一次只处理“同一分钟”内要爆发的橘子
            for _ in range(len(queue)):
                r, c = queue.popleft()
                
                # 感染四个方向的相邻橘子
                for dr, dc in directions:
                    nr, nc = r + dr, c + dc
                    # 只有在界内且是新鲜橘子才会被感染
                    if 0 <= nr < rows and 0 <= nc < cols and grid[nr][nc] == 1:
                        grid[nr][nc] = 2       # 标记为腐烂，避免重复遍历
                        fresh_count -= 1       # 新鲜橘子库存减 1
                        queue.append((nr, nc)) # 落网的橘子成为下一分钟的传染源
                        
            # 如果这分钟传染完后已经没有新鲜橘子了，即可提前终止
            if fresh_count == 0:
                return minutes
                
        # 队列都空了（没东西能接着传染了），还是有新鲜橘子幸存，说明感染不到
        return -1
```

#### 复杂度分析
- **时间复杂度**：$O(M \times N)$。同样地，无论初始有多少腐烂橘子，整个网格中每个格子最多被访问并加进队列一次（或者一开始是新鲜橘子，或者一开始就是腐烂橘子，或是空地根本不进队）。因此时间与矩阵的面积线性相关。
- **空间复杂度**：$O(M \times N)$。最坏的情况下（也就是大家全是腐烂橘子），初始会将所有的格子都加进队列里，队列的大小可能会达到 $M \times N$ 级别。

## 三、依赖关系与拓扑排序 (Topological Sort)

### 课程表（难度：中等）

**题目：**
你必须选修 `numCourses` 门课程，记为 0 到 `numCourses - 1`。在选修某些课程之前需要一些先修课程，按数组 `prerequisites` 给出。判断是否可能完成所有课程。

**【模板解析与思路归纳】：**
- 这是赤裸裸的有向图求“成环”问题！经典的破局之法就是**拓扑排序（入度剥离法）**。
- **模板两件套**：1. `indegrees` 记录每个节点被多少个箭头指着（欠了几个先决条件）；2. `graph`（邻接表）记录学完当前课能解锁哪些后续课。
- 所有入度为 0 的课（不要先修的课）直接入队。循环弹出，每次它弹出来，相当于这门课修完了，把它指向的所有的后续课的入度减 1，减到了 0 就说明那个课也解锁了，继续入队。
- 最后判断学过的课数等于不等于总课数。

**答案代码（Python版）：**
```python
import collections

class Solution:
    def canFinish(self, numCourses: int, prerequisites: List[List[int]]) -> bool:
        # 1. 准备邻接表和入度记录
        graph = collections.defaultdict(list)
        indegrees = [0] * numCourses
        
        for course, pre in prerequisites:
            graph[pre].append(course)
            indegrees[course] += 1
            
        # 2. 将所有最初就没有依赖的门课丢进队列
        queue = collections.deque()
        for i in range(numCourses):
            if indegrees[i] == 0:
                queue.append(i)
                
        # 3. 剥洋葱的过程
        learned_count = 0
        while queue:
            curr = queue.popleft()
            learned_count += 1
            
            # 学完了当前课，它的那些下游课们的依赖包袱减轻啦！
            for neighbor in graph[curr]:
                indegrees[neighbor] -= 1
                # 如果有个别课程的包袱全卸掉了，解锁！
                if indegrees[neighbor] == 0:
                    queue.append(neighbor)
                    
        return learned_count == numCourses
```

## 四、等权最短路径 (BFS)

### 单词接龙（难度：困难）

**题目：**
给定两个单词 `beginWord` 和 `endWord` 和一个字典 `wordList`，返回从起点到终点的最短转换序列中的单词数目。每次只能变一个字母。

**【模板解析与思路归纳】：**
- 求最短距离/最少步数，**首选且唯一的正解就是队列 BFS（波纹式展开）**！
- 把字符串看成图里的节点，只要差一个字母，就是直接相连的。一层一层向外扩，第一次碰到终点肯定就是最近的步数。
- 加速查找邻居：通过 `set` 并对单词的每个位置 26 个字母尝试替换，看是不是在 set 里。（比遍历 set 判断是不是差一个字母快得多）。

**答案代码（Python版）：**
```python
import collections

class Solution:
    def ladderLength(self, beginWord: str, endWord: str, wordList: List[str]) -> int:
        word_set = set(wordList)
        if endWord not in word_set:
            return 0
            
        # 初始入队：(当前单词，已经走过的长度)
        queue = collections.deque([(beginWord, 1)])
        
        while queue:
            curr_word, step = queue.popleft()
            
            if curr_word == endWord:
                return step
                
            # 暴力遍历改变 curr_word 的每一个字符
            for i in range(len(curr_word)):
                for c in "abcdefghijklmnopqrstuvwxyz":
                    next_word = curr_word[:i] + c + curr_word[i+1:]
                    
                    if next_word in word_set:
                        # 从未探索区域删去，相当于打上了 visited 标记
                        word_set.remove(next_word)
                        queue.append((next_word, step + 1))
                        
        return 0
```

## 五、特殊权图 (Dijkstra / Heap+BFS)

### 接雨水 II（难度：困难）

**题目：**
给你一个 `m x n` 的矩阵，代表二维高度图每个单元的高度，请计算图中形状最多能接多少体积的雨水。

**【模板解析与思路归纳】：**
- 二维接雨水，决定某个坑能接多少水的是**包围它的“木桶最矮边”**。这个短板可以不断向内传染。
- 这是极其经典的 **多源海平面上升 / Dijkstra** 变种。
- 把最外圈全部扔进最小堆（Priority Queue）。每次从堆里弹出的，必定是当前所有已知边界里最矮的一个。
- 拿这个最矮的边界去灌溉它的相邻内陆。如果内陆比它还矮，就能接下 `(边界 - 内陆) `的雨水；同时这个内陆就变成了新的边界，其水位高度为 `max(原来本身高度, 水面高度)`，推入堆中。

**答案代码（Python版）：**
```python
import heapq

class Solution:
    def trapRainWater(self, heightMap: List[List[int]]) -> int:
        if not heightMap or len(heightMap) < 3 or len(heightMap[0]) < 3:
            return 0
            
        m, n = len(heightMap), len(heightMap[0])
        visited = [[False] * n for _ in range(m)]
        heap = []
        
        # 将矩阵的四条边界塞入最小堆
        for i in range(m):
            for j in [0, n - 1]:
                heapq.heappush(heap, (heightMap[i][j], i, j))
                visited[i][j] = True
        for j in range(n):
            for i in [0, m - 1]:
                heapq.heappush(heap, (heightMap[i][j], i, j))
                visited[i][j] = True
                
        res = 0
        directions = [(0, 1), (0, -1), (1, 0), (-1, 0)]
        
        # 开始从外向内缩
        while heap:
            h, r, c = heapq.heappop(heap)
            
            for dr, dc in directions:
                nr, nc = r + dr, c + dc
                if 0 <= nr < m and 0 <= nc < n and not visited[nr][nc]:
                    visited[nr][nc] = True
                    # 如果内陆比当前的边矮，注入落差的水量
                    if heightMap[nr][nc] < h:
                        res += h - heightMap[nr][nc]
                        
                    # 水涨船高，将这个位置变成新的堤坝推入堆中
                    heapq.heappush(heap, (max(h, heightMap[nr][nc]), nr, nc))
                    
        return res
```
