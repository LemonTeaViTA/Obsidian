# 排序与区间 (Sorting & Intervals)

> [[算法_MOC|返回算法导航]]

## 一、排序核心思想与归类技巧

### 🎯 模式识别信号（看到这些特征要联想到的解法）

| 题目特征 / 关键词 | 优先用的解法 |
|-----------------|------------|
| "区间合并 / 区间覆盖" | **按左端点排序**后扫一遍 |
| "第 K 大 / 第 K 小" 且要求 `O(N)` | **快速选择**（快排 partition 思想） |
| "求逆序对" / 数组中的"严格升序对" | **归并排序**的合并阶段计数 |
| "原地排序 0、1、2" / 三色 | **荷兰国旗三指针** |
| "最大数 / 最小数 拼接" | **自定义比较器**（compareTo） |
| "K 个有序数组合并" | **小顶堆** + 多路归并（详见 [[堆]]） |
| 数据规模超大、外存排序 | 多路归并 + 外部排序 |

### 🧬 必背的三大排序复杂度对照

| 排序算法 | 平均时间 | 最坏时间 | 空间 | 稳定性 | 适用场景 |
|---------|---------|---------|------|-------|---------|
| **快速排序** | `O(N log N)` | `O(N²)` | `O(log N)` | ❌ 不稳定 | 通用最快，原地排序 |
| **归并排序** | `O(N log N)` | `O(N log N)` | `O(N)` | ✅ 稳定 | 链表排序、求逆序对 |
| **堆排序** | `O(N log N)` | `O(N log N)` | `O(1)` | ❌ 不稳定 | Top-K、动态维护极值 |

> [!tip] 面试官最爱问的"快排为什么不稳定"
> 快排在 `partition` 时会交换跨距离的元素，相同值的相对位置可能被打乱（如 `[3a, 3b, 1]` 排完可能变成 `[1, 3b, 3a]`）。而归并排序合并时是按顺序拷贝，所以稳定。

## 二、必须能手撕的两大经典排序

### 快速排序（必背手写题）

**核心思想**：分治 + partition。每轮选一个 pivot，把 `< pivot` 的丢左边，`> pivot` 的丢右边，pivot 自然落到正确位置；然后对左右两半递归。

#### 🔨 思维推导链

**1. 朴素思路**：选第一个元素当 pivot，开两个新数组，小的进左、大的进右。
- 问题：空间 `O(N)`，不是原地排序。

**2. 优化**：双指针原地交换（[Lomuto / Hoare 分区](https://en.wikipedia.org/wiki/Quicksort#Algorithm)）。
- 关键：`i` 指针找比 pivot 大的，`j` 指针找比 pivot 小的，找到后交换。

**3. 易错坑**：
- 数组本身有序时退化为 `O(N²)`。**面试要主动加随机化**。
- 大��重复元素时退化（如 `[3,3,3,3...]`）。可用**三路快排**优化。

```python
import random

class Solution:
    def sortArray(self, nums: List[int]) -> List[int]:
        def quick_sort(left, right):
            if left >= right:
                return
            # 随机化：避免最坏情况
            pivot_idx = random.randint(left, right)
            nums[left], nums[pivot_idx] = nums[pivot_idx], nums[left]
            pivot = nums[left]
            
            i, j = left, right
            while i < j:
                # 从右向左找比 pivot 小的
                while i < j and nums[j] >= pivot:
                    j -= 1
                nums[i] = nums[j]  # 把小的填到左边
                # 从左向右找比 pivot 大的
                while i < j and nums[i] <= pivot:
                    i += 1
                nums[j] = nums[i]  # 把大的填到右边
            nums[i] = pivot  # pivot 归位
            
            quick_sort(left, i - 1)
            quick_sort(i + 1, right)
        
        quick_sort(0, len(nums) - 1)
        return nums
```

> [!warning] 面试时一定要说出这两个细节
> 1. **为什么要随机选 pivot？** 防止数组有序时退化为 `O(N²)`。
> 2. **递归深度** 平均 `O(log N)`，最坏 `O(N)`，所以空间复杂度是 `O(log N)`。

### 归并排序（必背手写题）

**核心思想**：分治。把数组砍两半，分别排序，再合并两个有序数组。

#### 🔨 思维推导链

**1. 朴素思路**：合并两个有序数组本身就是双指针，时间 `O(N)`。
**2. 递归分解**：把数组对半砍，砍到只有一个元素（天然有序），再两两合并回去。
**3. 总递归深度 `O(log N)`，每层合并 `O(N)`，总时间 `O(N log N)`。**

```python
class Solution:
    def sortArray(self, nums: List[int]) -> List[int]:
        def merge_sort(arr):
            if len(arr) <= 1:
                return arr
            mid = len(arr) // 2
            left = merge_sort(arr[:mid])    # 递归排左半
            right = merge_sort(arr[mid:])   # 递归排右半
            return merge(left, right)        # 合并两个有序数组
        
        def merge(left, right):
            result = []
            i = j = 0
            while i < len(left) and j < len(right):
                if left[i] <= right[j]:  # = 保证稳定性
                    result.append(left[i])
                    i += 1
                else:
                    result.append(right[j])
                    j += 1
            result.extend(left[i:])
            result.extend(right[j:])
            return result
        
        return merge_sort(nums)
```

## 三、快速选择（Top-K 必杀技）

### 数组中的第 K 个最大元素（难度：中等）

**题目：** 给定整数数组 `nums` 和整数 `k`，请返回数组中第 `k` 个最大的元素。**要求时间复杂度 `O(N)`**。

#### 🔨 思维推导链

**1. 朴素**：排序后取 `nums[len-k]` → `O(N log N)`，不达标。

**2. 用堆**：维护一个大小为 `k` 的小顶堆 → `O(N log K)`，不达标。详见 [[堆#数组中的第K个最大元素（难度：中等）|堆解法]]。

**3. 关键洞察**：快排每次 `partition` 后，pivot 都落在它的"绝对正确位置"。我们**只需要其中一半的递归**——如果 pivot 位置正好是第 K 大，直接返回；否则只往一边继续 partition。

**4. 复杂度证明**：`N + N/2 + N/4 + ... = 2N`，平均 `O(N)`。

```python
import random

class Solution:
    def findKthLargest(self, nums: List[int], k: int) -> int:
        # 第 k 大 = 排序后下标为 len(nums) - k 的元素
        target = len(nums) - k
        
        def partition(left, right):
            # 随机选 pivot，避免最坏情况
            pivot_idx = random.randint(left, right)
            nums[pivot_idx], nums[right] = nums[right], nums[pivot_idx]
            pivot = nums[right]
            
            i = left
            for j in range(left, right):
                if nums[j] < pivot:
                    nums[i], nums[j] = nums[j], nums[i]
                    i += 1
            nums[i], nums[right] = nums[right], nums[i]
            return i
        
        left, right = 0, len(nums) - 1
        while True:
            p = partition(left, right)
            if p == target:
                return nums[p]
            elif p < target:
                left = p + 1
            else:
                right = p - 1
```

> [!tip] 快速选择 vs 堆：怎么选？
> - 数据**全在内存**、求一次性 Top-K → 快速选择 `O(N)` 更快
> - **数据流**式输入、动态维护 Top-K → 堆 `O(N log K)`（每次插入 `O(log K)`）
> - 见 [[堆#数组中的第K个最大元素（难度：中等）|堆解法对比]]

## 四、区间问题（按左端点排序套路）

### 合并区间（难度：中等）

**题目：**
以数组 `intervals` 表示若干个区间的集合，其中单个区间为 `intervals[i] = [start_i, end_i]`。请你合并所有重叠的区间，并返回一个不重叠的区间数组。

#### 🔨 思维推导链

**1. 朴素**：双重循环 `O(N²)` 比较任意两个区间是否重叠 → 太慢。

**2. 关键洞察**：如果先**按左端点排序**，那么能合并的区间一定是**相邻**的——因为左端点有序，后面的区间不会"跳到"前面去。

**3. 扫一遍即可**：维护结果数组的最后一个区间 `last`。新区间和 `last` 比较：
   - 新区间左端点 > `last` 右端点 → 无交集，追加
   - 否则更新 `last` 的右端点为两者最大值

```python
class Solution:
    def merge(self, intervals: List[List[int]]) -> List[List[int]]:
        intervals.sort(key=lambda x: x[0])  # ⭐ 按左端点排序
        merged = []
        for interval in intervals:
            if not merged or merged[-1][1] < interval[0]:
                merged.append(interval)  # 无重叠，追加
            else:
                merged[-1][1] = max(merged[-1][1], interval[1])  # 重叠，扩展右端点
        return merged
```

**复杂度**：时间 `O(N log N)`（瓶颈是排序），空间 `O(N)`。

#### 🌐 同模式串联

| 同模式题 | 区别 |
|---------|------|
| **插入区间**（LC 57） | 已有有序区间，插入新区间后合并 → 不需要重新排序 |
| **会议室 II**（LC 253） | 求最少会议室数 → 用小顶堆维护结束时间 |
| **用最少的箭引爆气球**（LC 452） | 按右端点排序贪心 → 思路类似但排序键不同 |

### 颜色分类（荷兰国旗，难度：中等）

**题目：**
给定一个包含 0、1、2 三种数字的数组，原地排序使其按 0、1、2 顺序排列。**只能扫描一遍**。

#### 🔨 思维推导链

**1. 朴素**：扫两遍 + 计数。但题目要求一遍。

**2. 关键洞察**：用三个指针**分区**。`left` 是 0 区右边界，`right` 是 2 区左边界，`i` 是探索者。
- 遇到 0：和 `left` 交换，`left` 和 `i` 都前进
- 遇到 2：和 `right` 交换，`right` 后退（注意 `i` 不前进！换过来的值还没看）
- 遇到 1：`i` 前进

**3. 易错坑**：遇到 2 时为什么 `i` 不动？因为换过来的元素是**右边的未知数**，可能是 0 也可能是 1，必须在原位再判断一次。

```python
class Solution:
    def sortColors(self, nums: List[int]) -> None:
        left, right, i = 0, len(nums) - 1, 0
        while i <= right:
            if nums[i] == 0:
                nums[left], nums[i] = nums[i], nums[left]
                left += 1
                i += 1
            elif nums[i] == 2:
                nums[right], nums[i] = nums[i], nums[right]
                right -= 1
                # ⚠️ i 不前进！换过来的值未检查
            else:  # nums[i] == 1
                i += 1
```

## 五、自定义排序规则

### 最大数（难度：中等）

**题目：** 给定一组非负整数 `nums`，重新排列每个数的顺序（每个数不可拆分），使之组成一个最大的整数。

例如：`[3, 30, 34, 5, 9]` → `"9534330"`

#### 🔨 思维推导链

**1. 朴素**：直接按数值降序 → 错。`[3, 30]` 降序得 `"303"`，但 `"330"` 才更大。

**2. 关键洞察**：比较两个字符串 `a` 和 `b` 应该排谁前面，看的不是字典序，而是 `a + b` vs `b + a` 哪个拼出来更大。
- `"3" + "30" = "330"` > `"30" + "3" = "303"` → 所以 `"3"` 排前面

**3. 自定义比较器**：

```python
from functools import cmp_to_key

class Solution:
    def largestNumber(self, nums: List[int]) -> str:
        # 自定义比较器：a+b > b+a 则 a 排前面（返回负数）
        def compare(a, b):
            if a + b > b + a:
                return -1  # a 排前面
            elif a + b < b + a:
                return 1   # b 排前面
            else:
                return 0
        
        nums_str = list(map(str, nums))
        nums_str.sort(key=cmp_to_key(compare))
        result = ''.join(nums_str)
        return '0' if result[0] == '0' else result  # 全是 0 的特例
```

> [!tip] Python 自定义比较器套路
> Python 3 取消了 `sort` 的 `cmp` 参数，改用 `key`。如果必须用比较函数（比较两个元素决定顺序），用 `from functools import cmp_to_key`。

## 六、归并排序的杀手级应用：求逆序对

### 数组中的逆序对（难度：困难）

**题目：** 在数组中的两个数字，如果前面一个数字大于后面的数字，则这两个数字组成一个逆序对。统计数组中的逆序对总数。

#### 🔨 思维推导链

**1. 朴素**：双重循环 `O(N²)`，超时。

**2. 关键洞察**：**归并排序的合并阶段天然能数出"跨左右两半的逆序对"**。
- 合并 `[3, 5]` 和 `[1, 4]` 时，左边的 5 > 右边的 4，且 5 > 右边的 1，贡献 2 个逆序对。
- 当从右半数组取出 `right[j]` 时，左半数组**剩余所有元素**都比它大，全是逆序对。

**3. 总时间 `O(N log N)`**，是归并排序的"副产品"。

```python
class Solution:
    def reversePairs(self, nums: List[int]) -> int:
        def merge_sort(left, right):
            if left >= right:
                return 0
            mid = (left + right) // 2
            count = merge_sort(left, mid) + merge_sort(mid + 1, right)
            
            # 合并阶段顺便数逆序对
            tmp = []
            i, j = left, mid + 1
            while i <= mid and j <= right:
                if nums[i] <= nums[j]:
                    tmp.append(nums[i])
                    i += 1
                else:
                    # nums[i] > nums[j]：左半 i 到 mid 的所有元素都和 nums[j] 构成逆序对
                    count += mid - i + 1
                    tmp.append(nums[j])
                    j += 1
            tmp.extend(nums[i:mid + 1])
            tmp.extend(nums[j:right + 1])
            nums[left:right + 1] = tmp
            return count
        
        return merge_sort(0, len(nums) - 1)
```

> [!tip] 归并求逆序对是面试官的最爱
> 这道题考察**对归并合并阶段的理解**——能否抓住"右边取一个就能数 mid - i + 1 个逆序对"这个关键洞察。
