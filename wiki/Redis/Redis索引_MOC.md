# Redis 篇 MOC (Map of Content)

这里是 Redis 高性能缓存与 NoSQL 的知识索引地图。现代架构无论如何演进，缓存永远是解决性能和并发的第一把尖刀。Redis 极高的性能与特有的单线程/多线程模型也是高频重点。

我们通过**主题聚类（Concept-Cluster）**将 Redis 内容做了拆解，旨在帮你理解缓存雪崩、穿透击穿这类场景的同时，搞懂其底层的设计艺术与数据结构。

---

## 🗺️ 知识体系导航

建议按照以下顺序进行系统学习和复习：

 单线程模型与基础认知
Redis 可以用来干什么？为啥它是一个基于内存但又异常凶猛的杀器？深入了解 Redis 的 I/O 多路复用（epoll）与单线程架构，以及 Redis 6.0 引入多线程的真实意图。
👉 **[[Redis 基础]]**

 核心重灾区：底层数据结构与实战
**Redis 性能强悍的根本。**
如果只懂 `get/set` 是无法通过高级面试的。彻底搞懂跳表（SkipList）、压缩列表（ziplist/listpack）、字典结构、SDS字符串。这不仅考察数据结构能力，也涉及到海量前缀 KEY 匹配和秒杀场景的实战运用。
👉 **[[底层数据结构与实战]]**

 核心重灾区：持久化机制
**让内存数据重获新生的两层防护。**
死磕 RDB 快照与 AOF 追加日志；了解两者的优劣势以及怎样保证重度写入不阻塞（`bgsave`，`fork()` 以及写时复制 - COW 技术），并掌握 4.0 之后混合持久化的破冰之举。
👉 **[[Redis 持久化]]**

 核心重灾区：缓存经典三高问题
**极高命中率的生产场景题。**
如何化解缓存穿透、击穿和雪崩？理解海量请求背后的布隆过滤器（BloomFilter）。深究为什么会有数据双写一致性难题，了解热 KEY / 大 KEY 带来的灾难及其处理方案；全面掌握 Redis 在内存打满时的过期与淘汰策略（LRU vs LFU）。
👉 **[[缓存经典问题]]**

 高可用与集群演进
从单机版走向企业级主从复制；了解哨兵机制（Sentinel）是如何实现故障自动转移与主库重新推举的，进而解析 Redis Cluster 切片集群基于“哈希槽”的分布式分区能力。
👉 **[[高可用与集群]]**

 进阶功能探索
通过 Redis 实现延时消息队列、异步并发削峰；理解管道（Pipeline）、Redis 分布式锁（SETNX 和 Redlock），以及 Lua 脚本的原子性执行。
👉 **[[Redis 进阶功能]]**
