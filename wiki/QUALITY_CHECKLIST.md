# Wiki 质量检查清单

本文档用于追踪知识库的质量状态，确保内容的完整性、一致性和可维护性。

## 📋 检查清单

### 1. 链接完整性 🔗

- [ ] 所有 `[[...]]` 内部链接都指向存在的文件
- [ ] 所有锚点链接（`[[文件#标题]]`）的标题都存在
- [ ] 没有嵌套链接（`[[xxx|[[yyy]]]]`）
- [ ] 没有格式错误的链接
- [ ] 外部链接（HTTP/HTTPS）都可访问

**检查方法**：运行 Quality Check workflow 或手动执行：
```bash
# 查找所有内部链接
grep -r '\[\[.*\]\]' wiki/

# 查找嵌套链接
grep -r '\[\[[^\]]*\[\[' wiki/
```

### 2. 内容质量 📝

- [ ] 没有明显的内容重复（相似度 > 80%）
- [ ] 重复内容已使用 `[[]]` 引用替代
- [ ] 所有数据和性能数字都标注了来源
- [ ] 项目链接（GitHub、论文）都已保留
- [ ] 没有使用教程、安装命令等无关内容

**检查方法**：运行 Quality Check workflow 的内容重复检查

### 3. 结构一致性 📐

#### 算法题解结构
- [ ] 所有算法题解遵循统一的标题结构：
  - `### 题目名称`
  - `#### 解题思路`
  - `#### 代码实现`
  - `#### 复杂度分析`
- [ ] 没有使用旧的加粗标签（`**题目**`）
- [ ] 教学循序渐进（先基础解法，再优化版）
- [ ] 包含物理隐喻和大白话解析
- [ ] 多面型题目有双向链接

**检查文件**：
- `wiki/算法/`

#### 面试题解结构
- [ ] 所有面试题解遵循 `###` 主题目 + `####` 子题目结构
- [ ] 标题中没有嵌入链接（会导致锚点匹配失败）
- [ ] 每个题解都有清晰的问题描述和答案

**检查文件**：
- `wiki/Java基础篇.md`
- `wiki/集合框架篇.md`
- `wiki/并发编程篇.md`
- `wiki/JVM篇.md`
- `wiki/MySQL篇.md`
- `wiki/Redis篇.md`
- `wiki/Spring篇.md`
- `wiki/计算机网络篇.md`
- `wiki/操作系统篇.md`

### 4. 面试题目同步 ✅

- [ ] `wiki/面试题目.md` 中的所有 checklist 条目都有对应的题解
- [ ] 所有 wiki 文件中的面试题解都在 `面试题目.md` 中有 checklist
- [ ] Checklist 中的锚点链接格式正确：`- [ ] 题目 ➡️ 👉 [[文件名#标题|查看核心解析]]`
- [ ] 锚点与实际标题完全一致（包括空格和标点）
- [ ] 没有重复题目
- [ ] 题目归入正确的模块分类

**三步同步流程**：
1. 在对应 wiki 文件中编写题解（`###` 主题目，`####` 子题目）
2. 在 `wiki/面试题目.md` 对应模块下追加 checklist 条目
3. 在根目录 `优化记录.md` 当天日期下追加变更日志

### 5. 知识图谱连接性 🕸️

- [ ] 没有孤岛页面（至少有一个入站链接）
- [ ] 相关概念之间有双向链接
- [ ] 同义词、对比关系、依赖关系都有适当的链接
- [ ] `wiki/index.md` 包含所有主要页面的入口

**例外页面**（允许无入站链接）：
- `wiki/index.md`
- `wiki/面试题目.md`
- `wiki/优化记录.md`

**检查方法**：运行 Cross-Link Builder workflow

### 6. 文档元数据 📊

- [ ] 所有 raw 文件都有完整的 frontmatter
- [ ] Frontmatter 包含必需字段：title, source, status, category
- [ ] 已整理的文件标记为 `status: processed`
- [ ] `extracted_to` 字段记录了提取到的 wiki 文件

**检查目录**：
- `raws/staged/`
- `raws/processed/`

### 7. 变更记录完整性 📝

- [ ] 最近 7 天的所有 wiki 变更都记录在 `优化记录.md`
- [ ] 变更日志格式统一：
  ```
  - `文件路径` — 一句话描述改动内容
    **原因**：一句话描述改动原因
  ```
- [ ] 同一天、同一文件的多项改动已合并为一条

**检查方法**：
```bash
# 查看最近 7 天的 git 变更
git log --since="7 days ago" --name-only --pretty=format: wiki/ | sort -u

# 对比 优化记录.md
```

### 8. 代码质量 💻

- [ ] 代码块都有语言标识（```java, ```python 等）
- [ ] 代码示例简洁、可运行
- [ ] 没有过长的代码块（> 100 行）
- [ ] 关键代码有注释说明

### 9. 格式规范 ✨

- [ ] 没有 UTF-8 损坏字符（`U+FFFD`）
- [ ] 标题层级正确（不跳级）
- [ ] 列表格式统一（`-` 或 `*`）
- [ ] 代码块前后有空行
- [ ] 中英文之间有空格（可选）

### 10. Raw 文件管理 📁

- [ ] `raws/inbox/` 为空或只包含待分类文件
- [ ] `raws/staged/` 中的文件都已分类到正确的子目录
- [ ] `raws/processed/` 中的文件都标记为 `status: processed`
- [ ] 没有重复的 raw 文件
