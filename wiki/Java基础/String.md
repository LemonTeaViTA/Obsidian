## String
### String 是基本数据类型吗？可以被继承吗？

**答案：**

- `String` 不是基本类型，是引用类型（类）。
- `String` 不能被继承，因为它被 `final` 修饰。

---

### String、StringBuilder、StringBuffer 的区别？

**答案：**

- `String`：不可变。
- `StringBuilder`：可变，非线程安全，单线程拼接性能好。
- `StringBuffer`：可变，线程安全（方法多带同步），性能一般低于 `StringBuilder`。

使用建议：

- 普通场景用 `String`。
- 频繁拼接用 `StringBuilder`。
- 特殊并发修改场景才考虑 `StringBuffer`。

---

### `new String("abc")` 和 `"abc"` 的区别？

**答案：**

- `"abc"`：优先从字符串常量池复用。
- `new String("abc")`：在堆上新建对象（常量池字面量可能已存在）。

因此 `==` 比较时通常不相等（比较的是引用地址）。

---

### String 是不可变类吗？字符串拼接如何实现？

**答案：**

是不可变类。主要原因：

- 内部字符存储不可变语义。
- 类被 `final` 修饰，防止继承破坏不变性。
- 不提供直接修改内部状态的公开方法。

字符串 `+` 拼接在编译期通常会优化为 `StringBuilder.append(...)` 链式调用，再 `toString()`。

---

### intern 方法有什么作用？

**答案：**

- 若常量池已有等值字符串，返回池中引用。
- 否则将当前字符串放入池并返回其引用。

用途是统一字符串引用、减少重复对象。
