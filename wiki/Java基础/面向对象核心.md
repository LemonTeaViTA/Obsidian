# 面向对象核心

> [!info] 速览
> 本篇聚焦面向对象的语言根基：三大特性（封装 / 继承 / 多态）、重载与重写、抽象类与接口。
> 类成员与关键字（访问控制 / static / final / 内部类）见 [[类成员与关键字]]；对象语义（equals/hashCode、值传递、深浅拷贝、对象创建）见 [[对象语义]]。

## 面向对象基础

### 面向对象和面向过程的区别？

**答案：**

- 面向过程：以步骤/函数为中心，关注“怎么做”。
- 面向对象：以对象/类为中心，关注“谁来做”。

面向对象更利于封装、复用和扩展。

---

### 面向对象编程有哪些特性？

**答案：**

三大特性：封装、继承、多态。

#### 封装

封装是把对象的**状态（字段）和行为（方法）**包装在一起，对外隐藏内部实现细节，只暴露必要的接口。

**怎么实现的：**
- 用 `private` 修饰字段，禁止外部直接访问
- 通过 `public` 的 getter/setter 方法控制读写
- 方法内部的实现逻辑对调用方不可见

```java
public class BankAccount {
    private double balance;  // 外部不能直接改余额

    public void deposit(double amount) {
        if (amount <= 0) throw new IllegalArgumentException("金额必须为正");
        this.balance += amount;
    }

    public boolean withdraw(double amount) {
        if (amount > balance) return false;  // 业务校验
        this.balance -= amount;
        return true;
    }

    public double getBalance() {
        return balance;  // 只读，不提供 setBalance
    }
}
```

这里 `balance` 被封装起来了，外部只能通过 `deposit/withdraw` 操作余额，不能直接 `account.balance = -100` 搞出非法状态。

**好处：**
- 保护数据完整性（setter 里可以加校验逻辑）
- 降低耦合（调用方不依赖内部实现，内部改了外面不用动）
- 提高可维护性（修改内部逻辑不影响外部代码）

#### 继承

子类通过 `extends` 获得父类的字段和方法，在此基础上扩展或重写。

```java
public class Animal {
    protected String name;

    public Animal(String name) {
        this.name = name;
    }

    public void eat() {
        System.out.println(name + " is eating");
    }
}

public class Dog extends Animal {
    private String breed;

    public Dog(String name, String breed) {
        super(name);          // 必须调用父类构造器
        this.breed = breed;
    }

    @Override
    public void eat() {
        System.out.println(name + " is eating dog food");  // 重写父类方法
    }

    public void fetch() {    // 子类扩展的新方法
        System.out.println(name + " is fetching the ball");
    }
}
```

**Java 的单继承限制：**
- 一个类只能 `extends` 一个父类，不支持多继承
- 但可以通过实现多个接口来弥补

**为什么不支持多继承（菱形继承问题）：**

```
      A (method())
     / \
    B   C       // B 和 C 都重写了 method()
     \ /
      D         // D.method() 该调谁的？——歧义！
```

假设类 B 和类 C 都继承了类 A，类 D 同时继承 B 和 C。如果 B 和 C 都重写了 A 的某个方法，D 调用这个方法时就不知道该用 B 的还是 C 的——这就是菱形继承（Diamond Problem）。Java 通过禁止类的多继承从根本上避免了这个歧义。

> [!note] 接口 default 方法的冲突
> 接口的 default 方法也可能产生类似冲突，此时编译器会强制要求实现类显式重写该方法来消除歧义。

**继承的缺点：**
- 破坏封装性：子类依赖父类的实现细节，父类改了子类可能跟着出问题
- 紧耦合：继承是编译期绑定的，不如组合灵活
- 层次过深时代码难以理解和维护

**组合优于继承的示例：**
```java
// 继承方式：紧耦合，如果 ArrayList 内部实现变了，子类可能出问题
public class MyStack extends ArrayList<Integer> { ... }

// 组合方式：松耦合，内部实现可以随时替换
public class MyStack {
    private List<Integer> list = new ArrayList<>();  // has-a
    public void push(int val) { list.add(val); }
    public int pop() { return list.remove(list.size() - 1); }
}
```

**设计原则**：优先使用组合（has-a）而非继承（is-a），除非确实存在“是一种”的关系。

#### 多态

同一个方法调用，作用在不同对象上产生不同的行为。

**多态的三个必要条件：**
1. 继承或实现接口
2. 方法重写（子类覆盖父类方法）
3. 父类引用指向子类对象（向上转型）

**完整示例：**
```java
public abstract class Shape {
    public abstract double area();
}

public class Circle extends Shape {
    private double radius;
    public Circle(double radius) { this.radius = radius; }

    @Override
    public double area() { return Math.PI * radius * radius; }
}

public class Rectangle extends Shape {
    private double width, height;
    public Rectangle(double w, double h) { this.width = w; this.height = h; }

    @Override
    public double area() { return width * height; }
}

// 使用多态：方法不关心具体是什么形状
public void printArea(Shape shape) {
    // 运行时根据实际类型调用对应的 area()
    System.out.println("面积: " + shape.area());
}

// 调用
printArea(new Circle(5));      // 输出圆的面积
printArea(new Rectangle(3, 4)); // 输出矩形的面积
// 新增 Triangle 只需加一个子类，printArea 方法完全不用改
```

**两种多态：**
- **编译时多态（静态绑定）**：方法重载，编译器根据参数类型决定调用哪个方法
- **运行时多态（动态绑定）**：方法重写，JVM 在运行时根据对象的实际类型决定调用哪个实现

```java
// 编译时多态（重载）
public class Printer {
    public void print(String s) { System.out.println("String: " + s); }
    public void print(int i)    { System.out.println("Int: " + i); }
    // 编译器在编译期就确定了调用哪个 print
}

// 运行时多态（重写）
Shape s = new Circle(5);  // 编译期类型是 Shape，运行期类型是 Circle
s.area();                 // 运行时才确定调用 Circle.area()
```

**向上转型与向下转型：**
```java
Animal a = new Dog("旺财", "柴犬");  // 向上转型：自动的，安全的
a.eat();     // OK，调用 Dog 重写的 eat()
// a.fetch(); // 编译错误！Animal 没有 fetch 方法

// 向下转型：需要强转，有风险
if (a instanceof Dog) {       // 先判断再转，避免 ClassCastException
    Dog d = (Dog) a;
    d.fetch();                // OK
}
```

**多态解决了什么问题：**
让业务代码依赖抽象而不是具体实现。新增一种实现只需要加一个子类，不用修改已有代码（开闭原则）。典型场景：策略模式、工厂模式、Spring 的依赖注入。

## 方法与重载重写

### 重载和重写的区别？

**答案：**

- 重载（Overload）：同一个类中，方法名相同、参数列表不同。
- 重写（Override）：子类对父类同签名方法提供新实现。

```java
// 重载：同一个类，方法名相同，参数不同
public class Calculator {
    public int add(int a, int b) { return a + b; }
    public double add(double a, double b) { return a + b; }  // 参数类型不同
    public int add(int a, int b, int c) { return a + b + c; } // 参数个数不同
    // 注意：仅返回值不同不算重载，编译报错
}

// 重写：子类覆盖父类方法
public class Animal {
    public void speak() { System.out.println("..."); }
}
public class Cat extends Animal {
    @Override  // 加注解让编译器帮你检查签名是否一致
    public void speak() { System.out.println("喵"); }
}
```

**重写的规则（两小一大）：**
- 子类方法的访问权限 ≥ 父类（不能缩小）
- 子类方法的返回类型 ≤ 父类（可以是子类型，即协变返回）
- 子类方法抛出的异常 ≤ 父类（不能抛更宽的受检异常）

补充要点：
- 重载看“参数”；重写看“继承关系 + 方法签名一致”
- 重写要遵循里氏替换原则，子类应可替代父类
- `private`、`static`、`final` 方法不能被重写

## 抽象类与接口

### 抽象类和接口有什么区别？

**答案：**

- 抽象类：描述“is-a”，可有状态、构造器、抽象/非抽象方法。
- 接口：描述能力约束，更偏“can-do / has-a”，面向规范。
- 一个类只能继承一个抽象类，但可实现多个接口。

```java
// 抽象类：有状态、有构造器、有具体方法
public abstract class AbstractPayment {
    protected String orderId;  // 可以有字段

    public AbstractPayment(String orderId) {  // 可以有构造器
        this.orderId = orderId;
    }

    public abstract boolean pay(double amount);  // 抽象方法，子类必须实现

    public void log() {  // 具体方法，子类可以直接用
        System.out.println("Processing order: " + orderId);
    }
}

// 接口：定义能力规范
public interface Serializable { }  // 标记接口
public interface Comparable<T> {
    int compareTo(T o);  // 抽象方法
}
public interface Collection<E> {
    default boolean isEmpty() {  // Java 8+ default 方法
        return size() == 0;
    }
    int size();
}

// 一个类可以继承一个抽象类 + 实现多个接口
public class WechatPayment extends AbstractPayment
        implements Serializable, Comparable<WechatPayment> {
    // ...
}
```

**对比表：**

| 维度 | 抽象类 | 接口 |
|------|--------|------|
| 关键字 | `abstract class` | `interface` |
| 继承/实现 | 单继承 | 多实现 |
| 构造器 | 可以有 | 不能有 |
| 字段 | 任意字段 | 只能 `public static final` |
| 方法 | 抽象 + 具体都行 | 抽象 + default + static（Java 8+） |
| 设计语义 | “是什么”（is-a） | “能做什么”（can-do） |

常见补充：
- Java 类不支持多继承；接口支持多继承
- Java 8 之后接口可以有 default 方法和 static 方法
- Java 9 之后接口可以有 private 方法（用于 default 方法间复用逻辑）
