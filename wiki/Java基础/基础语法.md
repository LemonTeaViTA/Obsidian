# 基础语法
## 数据类型与运算符

### Java 有哪些基本数据类型？各占用多少字节？

**答案：**

| 类型 | 关键字 | 字节 | 取值范围 |
|------|--------|------|---------|
| 字节型 | byte | 1 | -128 ~ 127 |
| 短整型 | short | 2 | -32768 ~ 32767 |
| 整型 | int | 4 | -2³¹ ~ 2³¹-1 |
| 长整型 | long | 8 | -2⁶³ ~ 2⁶³-1 |
| 单精度 | float | 4 | ±3.4×10³⁸ |
| 双精度 | double | 8 | ±1.7×10³⁰⁸ |
| 字符型 | char | 2 | \u0000 ~ \uffff |
| 布尔型 | boolean | 1/4* | true/false |

*boolean：数组中 1 字节，单独使用时 JVM 规范未定义（HotSpot 实际用 int 存储）

#### 类型转换与常见陷阱

**自动类型提升（小 → 大，安全）：**
```java
byte → short → int → long → float → double
                char ↗
```

**强制类型转换（大 → 小，可能丢失精度）：**
```java
int i = 300;
byte b = (byte) i;  // 300 超出 byte 范围，截断低 8 位 → 44

double d = 3.99;
int n = (int) d;    // 直接截断小数部分 → 3（不是四舍五入）
```

**运算时的自动提升陷阱：**
```java
byte b = 10;
b = b + 1;   // 编译错误！b + 1 结果是 int，不能直接赋给 byte
b += 1;      // OK，复合赋值运算符自带隐式强转

char c = 'A';
System.out.println(c + 1);       // 输出 66（int），不是 'B'
System.out.println((char)(c + 1)); // 输出 'B'
```

**int 溢出：**
```java
int max = Integer.MAX_VALUE;  // 2147483647
System.out.println(max + 1);  // -2147483648（溢出，不报错！）
```

#### 自动装箱/拆箱与 Integer 缓存池

```java
Integer a = 127;
Integer b = 127;
System.out.println(a == b);  // true，Integer 缓存了 -128~127

Integer c = 128;
Integer d = 128;
System.out.println(c == d);      // false，超出缓存范围
System.out.println(c.equals(d)); // true

// 拆箱时的 NPE 风险
Integer x = null;
int y = x;  // NullPointerException！
```

> [!warning] 包装类比较永远用 equals，不要用 ==
> 只有 -128~127 范围内的 Integer 才会复用缓存对象。

> [!note] 各包装类完整缓存范围
> 这里只示范 Integer 的典型陷阱，各包装类（Byte/Short/Long/Character/Boolean/Float/Double）的完整缓存范围表见 [[包装类与 Object#Integer 127 与 128 的 `==` 比较结果？各包装类缓存范围是多少？]]（域内单点）。

### & 和 && 的区别？

**答案：**

| 运算符 | 名称 | 特点 | 使用场景 |
|--------|------|------|---------|
| & | 逻辑与/按位与 | 两边都执行 | 位运算 |
| && | 短路与 | 左边 false 时右边不执行 | 常规判断（推荐） |

**示例：**
```java
// 短路保护
if (str != null && str.length() > 0) {
    // 安全：如果 str 为 null，不会执行 length()
}
```

## 关键字与修饰符

### final、finally、finalize 的区别？

**答案：**

| 关键字 | 作用 | 使用场景 |
|--------|------|---------|
| final | 修饰符，表示不可变 | 修饰类（不可继承）、方法（不可重写）、变量（不可修改） |
| finally | 异常处理的一部分 | try-catch-finally，最终会执行的代码块 |
| finalize() | Object 类的方法 | GC 回收前调用（已废弃） |

## 比较与判断

### == 和 equals 的区别？

**答案：**

| 对比项 | == | equals() |
|--------|----|---------|
| 本质 | 运算符 | Object 类的方法 |
| 基本数据类型 | 比较值 | 不能使用 |
| 引用数据类型 | 比较地址 | 默认比较地址，可重写比较值 |

**示例：**
```java
String s1 = new String("abc");
String s2 = new String("abc");
s1 == s2;      // false（地址不同）
s1.equals(s2); // true（值相同）
```

## 流程控制

### switch 语句支持哪些类型？新版 switch 有什么变化？

**答案：**

**支持的类型：**
- Java 5 之前：`byte/short/char/int`
- Java 5+：新增 `enum`
- Java 7+：新增 `String`
- `long` 始终不支持

**Java 14+ switch 表达式（重要）：**

```java
// 传统 switch：容易忘写 break，导致 fall-through
int day = 3;
String name;
switch (day) {
    case 1: name = "Mon"; break;
    case 2: name = "Tue"; break;
    default: name = "Other";
}

// Java 14+ switch 表达式：箭头语法，自动不 fall-through
String name2 = switch (day) {
    case 1 -> "Mon";
    case 2 -> "Tue";
    case 3, 4, 5 -> "Weekday";  // 多值合并
    default -> "Other";
};

// 需要多行逻辑时用 yield 返回值
String result = switch (day) {
    case 1 -> "Mon";
    default -> {
        String s = "Day " + day;
        yield s.toUpperCase();  // yield 替代 return
    }
};
```

> [!tip] 新版 switch 的优势
> 箭头语法不会 fall-through，不需要写 break，且 switch 可以作为表达式直接赋值，代码更简洁。

**enum 在 switch 里怎么用：**

`enum`（枚举）是一组固定常量的集合（如星期、状态、方向），本质是引用类型，但编译器对其 switch 做了特殊处理（底层用 `ordinal()` 序号跳转）。

```java
// 定义枚举：就这几个固定值
enum Day { MON, TUE, WED, THU, FRI, SAT, SUN }

Day today = Day.WED;

String type = switch (today) {
    case SAT, SUN -> "周末";               // 多值合并
    case MON, TUE, WED, THU, FRI -> "工作日";
};
```

> [!warning] enum switch 的两个坑
> 1. `case` 后**直接写 `WED`，不能写 `Day.WED`**——写全限定名会编译报错，这是 enum switch 的特殊规定。
> 2. enum switch 可以穷举所有值而**不写 default**（编译器知道枚举值有限）；但若漏列某个值又没 default，运行到该值会出问题，建议仍保留 default 兜底。

**为什么用 enum 而不用 int / String 当常量：**

| 写法 | 问题 |
|------|------|
| `int status = 1` | 0/1 含义不明，还能传入非法值 99 |
| `String status = "active"` | 打错字 `"actvie"` 编译不报错，运行才炸 |
| `enum Status { ACTIVE, INACTIVE }` | ✅ 类型安全，只能是固定值，打错字编译即报错，IDE 自动补全 |

> [!tip] 一句话
> enum = 把"魔法数字/字符串"变成有名字、有类型、不会传错的固定常量集合。

**break、continue、return 的区别：**

| 关键字 | 作用范围 | 效果 |
|--------|---------|------|
| `break` | 循环或 switch | 跳出整个循环/switch |
| `continue` | 循环 | 跳过本次，进入下一次循环 |
| `return` | 方法 | 结束方法并返回值 |

## 运算技巧

### 自增自减运算有什么坑？

**答案：**

- 前缀 `++a`：先自增再参与表达式
- 后缀 `a++`：先参与表达式再自增

```java
int a = 1;
int b = ++a;  // a=2, b=2
int c = a++;  // a=3, c=2

// 经典坑：i = i++ 结果是 1，不是 2
int i = 1;
i = i++;  // 后缀先返回旧值 1 参与赋值，i 最终还是 1
```

> [!tip] 位运算技巧
> `n << k` 等价于 `n × 2^k`，`n >> k` 等价于 `n / 2^k`（有符号右移）。HashMap 中大量使用位运算替代乘除法提升性能。

## 浮点数与精度

### float 是怎么表示小数的？

**答案：**

`float` 按 IEEE 754 单精度浮点数表示，32 位由三部分组成：

- 符号位（Sign）
- 指数位（Exponent）
- 尾数位（Mantissa）

通用形式是：
$$
V = (-1)^S \times M \times 2^E
$$

因为二进制浮点是近似表示，小数运算会有精度误差。

---

### 如何保证浮点数计算的精度？

**答案：**

金融计算不要用 `float/double`，常见两种方案：

1. 使用 `BigDecimal`（推荐）
2. 把金额转成最小货币单位（如分）用整数计算

**BigDecimal 的常见坑：**

```java
// 坑 1：用 double 构造，精度问题仍然存在
new BigDecimal(0.1);    // 实际是 0.10000000000000000555...
new BigDecimal("0.1");  // 正确！用字符串构造

// 坑 2：equals 比较精度位数，compareTo 才比较数值
BigDecimal a = new BigDecimal("1.0");
BigDecimal b = new BigDecimal("1.00");
a.equals(b);     // false！精度不同
a.compareTo(b);  // 0，数值相等

// 坑 3：除法必须指定精度，否则除不尽时抛 ArithmeticException
new BigDecimal("1").divide(new BigDecimal("3"));  // 抛异常！
new BigDecimal("1").divide(new BigDecimal("3"), 10, RoundingMode.HALF_UP);  // OK
```

> [!tip] 最佳实践
> - 构造用字符串，不用 double
> - 比较大小用 `compareTo`，不用 `equals`
> - 除法必须指定 `scale` 和 `RoundingMode`
