# MySQL 篇 MOC (Map of Content)

这里是 MySQL 数据库核心知识的索引地图。数据库底层原理、索引组织结构、事务并发调优永远是所有后端开发面试和工程优化的核心。

由于 MySQL 体系不仅包含基础 SQL，还涉及及其庞大的底层结构，我们基于**主题聚类（Concept-Cluster）**将其内容做了系统的拆分，帮助你在数据库设计、问题排查和面试上拔得头筹。

---

## 🗺️ 知识体系导航

建议按照以下顺序进行系统学习和复习：

 基础使用与整体架构
从 SQL 多表外连接（Left Join / Inner Join）到三大范式，再到 MySQL 服务端是如何组织一条查询语句或更新语句流转的（连接器 -> 分析器 -> 优化器 -> 执行器）。
👉 **[[MySQL 基础与架构]]**

 核心重灾区：底层引擎与日志系统
**底层设计的第一大考察点。**
死磕 InnoDB 中极为经典的 Buffer Pool，以及 MySQL 最核心的两大基石日志体系：二进制日志（Binlog）与重做日志（Redo log），并理解它为何需要“两阶段提交”来保证崩溃恢复时的数据一致性。
👉 **[[引擎与日志]]**

 👑 核心重灾区：索引机制详解
**数据库调优的灵魂与面试命中率最高的话题。**
彻底理解 B+树 为什么比普通二叉树以及 Hash 表更适合作为数据库索引；深度拆解聚簇索引与非聚簇索引带来的“回表”动作；掌握联合索引（最左前缀匹配）、覆盖索引、索引下推，以及通过 `explain` 分析 SQL 走慢的原因。
👉 **[[索引机制详解]]**

 核心重灾区：锁机制
"按索引粒度看锁"——表锁、行锁、记录锁、间隙锁、临键锁、意向锁。理解锁加在哪里，才能搞懂死锁怎么产生、怎么排查。
👉 **[[锁机制]]**

 核心重灾区：事务与 MVCC
ACID 的微观实现之道。理解四大隔离级别背后的原理；详细理解 MVCC（ReadView + Undo Log 版本链）是如何让"读"不加锁、却又能看到一致快照的。
👉 **[[事务与MVCC]]**

 高可用架构与分库分表
从单机极限到分布式架构。主从复制（读写分离）、延迟如何处理？以及当单表达到千万级数据时，垂直与水平分库分表（Sharding）的拆分策略及扩容演进方案。
👉 **[[高可用与分库分表]]**

 SQL 优化与生产实战
各种极限场景的实战应对：百万级大表热加字段？深分页问题怎么办？CPU 狂飙问题如何排查与降级定位。
👉 **[[优化与实战]]**

---

> **学习笔记提示**：
> MySQL 的学习关键是**理清各组件在 I/O 落盘与内存间的权衡**，每一个机制——从 B+ 树变粗变矮，到 Redo Log 的顺序写覆盖随机写，都是典型的空间换时间、顺序写换随机写的硬件底层博弈法则。
