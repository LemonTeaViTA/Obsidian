# JVM 篇 MOC (Map of Content)

这里是 Java 虚拟机（JVM）核心知识的索引地图。JVM 是 Java 的基石，掌控着跨平台运行、内存结构、垃圾回收和类加载等核心机制。它的调优和排查是高阶面试中最能拉开差距的难点。

由于 JVM 体系庞大复杂，我们基于**主题聚类（Concept-Cluster）**将其拆解为五大脉络，以此打破单纯背书的枯燥，帮助你构建立体化的性能优化认知。

---

## 🗺️ 知识体系导航

建议按照以下顺序进行系统学习和复习：

 整体认知与发展史
什么是 JVM？JVM 的整体架构是如何划分的？了解为什么要学习 JVM 以及 HotSpot 虚拟机的地位。
👉 **[[JVM 概述]]**

 核心重灾区：运行时数据区
**JVM 第一大面试重点。**
理解程序计数器、Java 虚拟机栈、本地方法栈的工作机制。横向对比堆（Heap）与方法区（Method Area/元空间）的存储区别；掌握对象在内存中的创建、布局及访问定位。
👉 **[[JVM 内存管理]]**

 核心重灾区：垃圾收集（GC）
**JVM 第二大面试重点，性能调优的理论基础。**
理解“对象存活的判定算法”（根可达性分析），死磕四大垃圾回收算法。进阶对比各大垃圾收集器（Serial、Parallel、CMS、G1、ZGC），掌握各类回收器应对 Stop The World (STW) 问题时的破局之道。
👉 **[[JVM 垃圾收集]]**

 进阶：JVM 调优与实战
理解 JVM 常用的高频命令行工具（jps, jstat, jmap, jstack）及可视化调优工具（JConsole, VisualVM）。针对 OOM、CPU 飙高、Full GC 频繁等生产环境痛点，给出排查思路与解决框架。
👉 **[[JVM 调优]]**

 底层解耦基石：类加载机制
彻底搞懂类的生命周期（加载、验证、准备、解析、初始化）；死磕强大的**双亲委派模型**（为什么要设计这个模型？以及 Tomcat/ JDBC 等框架为何要打破它？）。
👉 **[[JVM 类加载机制]]**

> **学习笔记提示**：
> JVM 不是靠死记硬背的理论概念！在学习过程中，必须在脑海里建立**动态内存快照**（如：一个对象分配了多少字节、它在栈还是在堆上生存、它是怎么一步步熬过 Minor GC 进入老年代的），并结合具体命令实敲分析，才是真正的进阶。
