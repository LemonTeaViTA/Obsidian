# Prompt 设计

## 一、Prompt 的核心元素

Prompt 是给大模型制定的指令，包含以下常见元素：

- **system**（系统指令）：角色设定、规则约束
- **user**（本轮问题）：用户当前提问
- **assistant**（历史回复）：对话历史
- **额外上下文**：检索结果、函数签名等

## 二、派聪明的 Prompt 构建策略

### 1. 角色设定（system）

从配置文件动态拼接系统级提示词：

```json
{ 
  "role": "system",
  "content": "你是一位严谨的图书编辑，所有回答必须使用正式中文并引用页码。" 
}
```

**实际应用**：
- 规则约束：回答格式、引用要求、语言风格
- 知识库注入：将检索到的相关文档添加到 system 中

### 2. 历史对话（assistant）

```json
{ 
  "role": "assistant", 
  "content": "您好，我可以帮助您查询产品信息。" 
}
```

派聪明会把用户在聊天助手中的历史记录动态拼接进来，维持多轮对话的连贯性。

### 3. 动态上下文整合

根据检索结果动态生成 Prompt：
- 选择合适的 Prompt 模板（根据问题类型）
- 填充检索到的知识库内容
- 添加系统级指令或限制条件
- 管理上下文长度（避免超出模型限制）

## 三、Prompt 模板管理

**配置化管理**：
- 支持多种 Prompt 模板配置与调优
- 根据场景（如技术问答、客服对话）选择不同模板
- 支持变量替换和动态内容注入

**关键设计原则**：
- 结构化：清晰的角色定义、明确的任务描述
- 可配置：模板化管理，便于调优和 A/B 测试
- 上下文感知：根据检索结果和对话历史动态调整

## 四、为什么这样设计？

1. **效果保障**：结构化 Prompt 引导模型生成高质量回答
2. **成本可控**：精确的上下文管理避免 token 浪费
3. **合规风险**：明确的规则约束减少幻觉和跑题
4. **可维护性**：配置化管理便于迭代优化

## 五、面试要点

**Q：Prompt 设计的核心是什么？**

A：三个层次：
1. **角色定位**：明确模型扮演的角色和遵循的规则
2. **上下文管理**：动态注入检索结果，控制长度
3. **模板化**：配置化管理，支持快速调优和场景切换

**Q：如何优化 Prompt 的 token 消耗？**

A：
- 精简系统指令，去除冗余描述
- 检索结果排序后只取 Top-K
- 历史对话设置轮数上限（如最近 5 轮）
- 使用 Prompt Caching（Anthropic）缓存系统指令
