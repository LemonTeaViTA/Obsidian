# 库表设计

## 一、用户表 (users)

**设计要点**：
- 基础认证信息：用户名、加密密码
- 权限管理：role 字段区分普通用户和管理员
- 组织归属：org_tags 支持多组织标签（逗号分隔），primary_org 标识主组织

**表结构**：

| 字段名 | 数据类型 | 是否主键 | 描述 |
| --- | --- | --- | --- |
| id | BIGINT | 是 | 用户唯一标识 |
| username | VARCHAR(255) | 否 | 用户名，唯一 |
| password | VARCHAR(255) | 否 | 加密后的密码 |
| role | ENUM('USER', 'ADMIN') | 否 | 用户角色 |
| org_tags | VARCHAR(255) | 否 | 用户所属组织标签，多个用逗号分隔 |
| primary_org | VARCHAR(50) | 否 | 用户主组织标签 |
| created_at | TIMESTAMP | 否 | 创建时间 |
| updated_at | TIMESTAMP | 否 | 更新时间 |

**建表语句**：

```sql
CREATE TABLE users (
    id BIGINT AUTO_INCREMENT PRIMARY KEY COMMENT '用户唯一标识',
    username VARCHAR(255) NOT NULL UNIQUE COMMENT '用户名，唯一',
    password VARCHAR(255) NOT NULL COMMENT '加密后的密码',
    role ENUM('USER', 'ADMIN') NOT NULL DEFAULT 'USER' COMMENT '用户角色',
    org_tags VARCHAR(255) DEFAULT NULL COMMENT '用户所属组织标签，多个用逗号分隔',
    primary_org VARCHAR(50) DEFAULT NULL COMMENT '用户主组织标签',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    INDEX idx_username (username) COMMENT '用户名索引'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';
```

## 二、组织标签表 (organization_tags)

**设计要点**：
- 支持层级化组织结构（通过 parent_tag 构建树形架构）
- 创建者追踪（created_by）确保可追溯性
- 与文档权限、用户访问控制紧密结合

**表结构**：

| 字段名 | 数据类型 | 是否主键 | 描述 |
| --- | --- | --- | --- |
| tag_id | VARCHAR(50) | 是 | 标签唯一标识 |
| name | VARCHAR(100) | 否 | 标签名称 |
| description | TEXT | 否 | 描述 |
| parent_tag | VARCHAR(50) | 否 | 父标签ID |
| created_by | BIGINT | 否 | 创建者ID |
| created_at | TIMESTAMP | 否 | 创建时间 |
| updated_at | TIMESTAMP | 否 | 更新时间 |

**建表语句**：

```sql
CREATE TABLE organization_tags (
    tag_id VARCHAR(50) PRIMARY KEY COMMENT '标签唯一标识',
    name VARCHAR(100) NOT NULL COMMENT '标签名称',
    description TEXT COMMENT '描述',
    parent_tag VARCHAR(50) DEFAULT NULL COMMENT '父标签ID',
    created_by BIGINT NOT NULL COMMENT '创建者ID',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    FOREIGN KEY (parent_tag) REFERENCES organization_tags(tag_id) ON DELETE SET NULL,
    FOREIGN KEY (created_by) REFERENCES users(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='组织标签表';
```

**应用场景**：
- 技术部门的文档只有技术部门用户可访问
- 公司级别的公告所有用户都能看到
- 支持"公司 → 部门 → 小组"的层级权限继承

## 三、文档表 (documents)

**设计要点**：
- 存储文档元信息（文件名、路径、大小、类型）
- 关联用户和组织标签（权限控制）
- 记录解析状态和向量化状态

**表结构**：

| 字段名 | 数据类型 | 是否主键 | 描述 |
| --- | --- | --- | --- |
| id | BIGINT | 是 | 文档唯一标识 |
| filename | VARCHAR(255) | 否 | 文件名 |
| file_path | VARCHAR(500) | 否 | 文件存储路径 |
| file_size | BIGINT | 否 | 文件大小（字节） |
| file_type | VARCHAR(50) | 否 | 文件类型（pdf/docx/txt等） |
| user_id | BIGINT | 否 | 上传用户ID |
| org_tags | VARCHAR(255) | 否 | 文档所属组织标签 |
| parse_status | ENUM('PENDING', 'PARSING', 'SUCCESS', 'FAILED') | 否 | 解析状态 |
| vector_status | ENUM('PENDING', 'VECTORIZING', 'SUCCESS', 'FAILED') | 否 | 向量化状态 |
| created_at | TIMESTAMP | 否 | 创建时间 |
| updated_at | TIMESTAMP | 否 | 更新时间 |

**建表语句**：

```sql
CREATE TABLE documents (
    id BIGINT AUTO_INCREMENT PRIMARY KEY COMMENT '文档唯一标识',
    filename VARCHAR(255) NOT NULL COMMENT '文件名',
    file_path VARCHAR(500) NOT NULL COMMENT '文件存储路径',
    file_size BIGINT NOT NULL COMMENT '文件大小（字节）',
    file_type VARCHAR(50) NOT NULL COMMENT '文件类型',
    user_id BIGINT NOT NULL COMMENT '上传用户ID',
    org_tags VARCHAR(255) DEFAULT NULL COMMENT '文档所属组织标签',
    parse_status ENUM('PENDING', 'PARSING', 'SUCCESS', 'FAILED') DEFAULT 'PENDING' COMMENT '解析状态',
    vector_status ENUM('PENDING', 'VECTORIZING', 'SUCCESS', 'FAILED') DEFAULT 'PENDING' COMMENT '向量化状态',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    FOREIGN KEY (user_id) REFERENCES users(id),
    INDEX idx_user_id (user_id),
    INDEX idx_org_tags (org_tags)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='文档表';
```

## 四、文档分块表 (document_chunks)

**设计要点**：
- 存储文档切分后的文本块
- 记录块在原文档中的位置（page_number, chunk_index）
- 关联文档 ID 和组织标签（继承文档权限）

**表结构**：

| 字段名 | 数据类型 | 是否主键 | 描述 |
| --- | --- | --- | --- |
| id | BIGINT | 是 | 分块唯一标识 |
| document_id | BIGINT | 否 | 所属文档ID |
| chunk_text | TEXT | 否 | 分块文本内容 |
| chunk_index | INT | 否 | 分块在文档中的序号 |
| page_number | INT | 否 | 所在页码（PDF） |
| org_tags | VARCHAR(255) | 否 | 继承文档的组织标签 |
| created_at | TIMESTAMP | 否 | 创建时间 |

**建表语句**：

```sql
CREATE TABLE document_chunks (
    id BIGINT AUTO_INCREMENT PRIMARY KEY COMMENT '分块唯一标识',
    document_id BIGINT NOT NULL COMMENT '所属文档ID',
    chunk_text TEXT NOT NULL COMMENT '分块文本内容',
    chunk_index INT NOT NULL COMMENT '分块在文档中的序号',
    page_number INT DEFAULT NULL COMMENT '所在页码',
    org_tags VARCHAR(255) DEFAULT NULL COMMENT '继承文档的组织标签',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    FOREIGN KEY (document_id) REFERENCES documents(id) ON DELETE CASCADE,
    INDEX idx_document_id (document_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='文档分块表';
```

## 五、会话表 (conversations)

**设计要点**：
- 存储用户对话会话元信息
- 关联用户 ID
- 支持会话过期管理（TTL）

**表结构**：

| 字段名 | 数据类型 | 是否主键 | 描述 |
| --- | --- | --- | --- |
| id | VARCHAR(50) | 是 | 会话唯一标识 |
| user_id | BIGINT | 否 | 用户ID |
| title | VARCHAR(255) | 否 | 会话标题 |
| created_at | TIMESTAMP | 否 | 创建时间 |
| updated_at | TIMESTAMP | 否 | 更新时间 |
| expires_at | TIMESTAMP | 否 | 过期时间 |

**建表语句**：

```sql
CREATE TABLE conversations (
    id VARCHAR(50) PRIMARY KEY COMMENT '会话唯一标识',
    user_id BIGINT NOT NULL COMMENT '用户ID',
    title VARCHAR(255) DEFAULT '新对话' COMMENT '会话标题',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    expires_at TIMESTAMP DEFAULT NULL COMMENT '过期时间',
    FOREIGN KEY (user_id) REFERENCES users(id),
    INDEX idx_user_id (user_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='会话表';
```

## 六、设计原则总结

1. **权限隔离**：通过 org_tags 实现多租户数据隔离
2. **可追溯性**：created_by、user_id 记录操作者
3. **状态管理**：parse_status、vector_status 跟踪异步任务
4. **层级结构**：parent_tag 支持组织树形架构
5. **外键约束**：保证数据一致性（ON DELETE CASCADE/SET NULL）
