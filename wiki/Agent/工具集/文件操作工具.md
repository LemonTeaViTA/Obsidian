# 文件操作工具

> 文件操作工具是 Coding Agent 的"四肢"——==最高频、最基础==的工具集。本文详解 Read / Write / Edit / NotebookEdit 等核心工具的设计、参数、最佳实践。

---

## 一、核心工具清单

| 工具 | 作用 | 设计要点 |
|------|------|---------|
| ==`read_file`== | 读文件内容 | 路径白名单 / 大文件分页（offset+limit） |
| ==`write_file`== | 整个文件覆盖写 | ==必须先 read 再 write==（防误删） |
| ==`edit_file`== | 局部修改（diff/replace） | 比 write 更安全——只改指定片段 |
| ==`list_directory`== | 列目录 | 默认排除 `.git` / `node_modules` |
| ==`create_directory`== | 创建目录 | 自动创建父目录 |
| ==`move`== / ==`copy`== / ==`delete`== | 文件操作 | 高危操作走 Human-in-the-Loop |

---

## 二、read_file：读文件内容

### 2.1 基础实现

==就是 Python 的 `open()` 函数==——5 行结束：

```python
def read_file(path: str, offset: int = 0, limit: int = 2000) -> str:
    with open(path, 'r', encoding='utf-8') as f:
        lines = f.readlines()
        selected = lines[offset:offset+limit]
        return ''.join(f"{i+offset+1}\t{line}" for i, line in enumerate(selected))
```

### 2.2 关键设计

**路径白名单**

==生产环境必须限制可读路径==——防止 LLM 被 Prompt Injection 诱导读取系统文件：

```python
ALLOWED_PATHS = [
    "/workspace",           # 项目目录
    "/home/user/.config",   # 配置目录
]

def read_file(path: str) -> str:
    abs_path = os.path.abspath(path)
    if not any(abs_path.startswith(ap) for ap in ALLOWED_PATHS):
        raise PermissionError(f"Path {path} not in allowed list")
    # ... 正常读取
```

**大文件分页**

==默认只读 2000 行==——防止超长文件炸 context：

```
# 读前 2000 行
read_file("large.log")

# 读第 2000-4000 行
read_file("large.log", offset=2000, limit=2000)
```

**返回格式：带行号**

==必须返回行号==——方便 LLM 后续调用 `edit_file` 时精确定位：

```
1	import os
2	import sys
3	
4	def main():
5	    print("Hello")
```

**Jupyter Notebook 特殊处理**

`.ipynb` 文件是 JSON 格式——直接读会看到原始 JSON。==Read 工具要特殊处理==：

```python
def read_file(path: str) -> str:
    if path.endswith('.ipynb'):
        return read_notebook(path)  # 解析成 cell 格式
    # ... 普通文件
```

返回格式：

```
Cell 1 (code):
import pandas as pd
df = pd.read_csv('data.csv')

Output:
   name  age
0  Alice   25
1   Bob    30

Cell 2 (markdown):
## Analysis Results
...
```

详见本文 §6 Notebook 工具。

## 三、write_file：整个文件覆盖写

### 3.1 基础实现

```python
def write_file(path: str, content: str) -> dict:
    with open(path, 'w', encoding='utf-8') as f:
        f.write(content)
    return {"status": "success", "path": path}
```

### 3.2 关键设计

**必须先 read 再 write**

==这是 Claude Code / Cursor 的核心安全机制==——防止 LLM 误删文件：

```python
# Harness 层拦截
def write_file(path: str, content: str) -> dict:
    if os.path.exists(path):
        if path not in session.read_files:
            raise PermissionError(
                f"Must read {path} before writing. "
                "This prevents accidental overwrite."
            )
    # ... 正常写入
```

==工作流==：

```
LLM: 我要改 src/app.py
   ↓
LLM: read_file("src/app.py")  ← 先读
   ↓
Harness: 记录 session.read_files.add("src/app.py")
   ↓
LLM: write_file("src/app.py", "...")  ← 再写，通过
```

==如果 LLM 直接 write 没读过的文件==：

```
LLM: write_file("src/new.py", "...")  ← 新文件，允许
LLM: write_file("src/app.py", "...")  ← 已存在但没读过，拒绝

Harness: PermissionError: Must read src/app.py before writing
   ↓
LLM 看到错误 → 自动调 read_file → 再调 write_file → 成功
```

**创建父目录**

==自动创建不存在的父目录==：

```python
def write_file(path: str, content: str) -> dict:
    os.makedirs(os.path.dirname(path), exist_ok=True)
    with open(path, 'w', encoding='utf-8') as f:
        f.write(content)
```

**编码处理**

==默认 UTF-8，失败时自动检测==：

```python
def write_file(path: str, content: str) -> dict:
    try:
        with open(path, 'w', encoding='utf-8') as f:
            f.write(content)
    except UnicodeEncodeError:
        # 尝试 latin-1 或检测编码
        with open(path, 'w', encoding='latin-1') as f:
            f.write(content)
```

## 四、edit_file：局部修改（推荐）

### 4.1 为什么 edit_file 比 write_file 更好

==现代 Coding Agent 强烈倾向 `edit_file`==：

| 维度 | write_file | edit_file |
|------|-----------|-----------|
| 风险 | 出错丢整个文件 | 出错只损失一段 |
| 上下文消耗 | 必须把整个文件发给 LLM | 只需发改动片段 |
| 并发安全 | 多处改动需要合并整个文件 | 每处独立修改 |
| 适用 | 新文件 / 整个重写 | 修改现有文件（99% 场景） |

### 4.2 实现方式

**方式一：old_string / new_string 替换**

==Claude Code 用的方式==：

```python
def edit_file(path: str, old_string: str, new_string: str) -> dict:
    with open(path, 'r') as f:
        content = f.read()
    
    if old_string not in content:
        return {"error": "old_string not found"}
    
    if content.count(old_string) > 1:
        return {"error": "old_string appears multiple times, be more specific"}
    
    new_content = content.replace(old_string, new_string)
    
    with open(path, 'w') as f:
        f.write(new_content)
    
    return {"status": "success"}
```

==LLM 调用示例==：

```json
{
  "tool": "edit_file",
  "path": "src/app.py",
  "old_string": "def authenticate(user, password):\n    return user == \"admin\"",
  "new_string": "def authenticate(user, password):\n    return check_hash(user, password)"
}
```

**方式二：unified diff 格式**

==Aider 用的方式==：

```python
def edit_file(path: str, diff: str) -> dict:
    # diff 格式：
    # @@ -10,3 +10,4 @@
    #  def foo():
    # -    return 1
    # +    return 2
    # +    # new line
    
    import patch
    patch.fromstring(diff).apply(path)
    return {"status": "success"}
```

**方式三：行号 + 替换内容**

==Cursor 用的方式==：

```python
def edit_file(path: str, start_line: int, end_line: int, new_content: str):
    with open(path, 'r') as f:
        lines = f.readlines()
    
    lines[start_line-1:end_line] = [new_content + '\n']
    
    with open(path, 'w') as f:
        f.writelines(lines)
```

### 4.3 优缺点对比

| 方式 | 优点 | 缺点 |
|------|------|------|
| old/new 替换 | LLM 最易用（自然语言） | ==old_string 必须精确匹配==（空格/换行都要对） |
| unified diff | 标准格式、支持多处改动 | ==LLM 容易生成错误 diff==（格式严格） |
| 行号替换 | 简单直接 | 文件变化后行号失效 |

==生产实践==：==old/new 替换 + 自动 fallback==：

```
LLM 调 edit_file(old_string, new_string)
   ↓
Harness: old_string 匹配失败
   ↓
Harness: 自动尝试模糊匹配（忽略空格差异）
   ↓
仍失败 → 返回错误 + 建议："Try using write_file instead"
   ↓
LLM 看到错误 → 改用 write_file
```

## 五、其他文件操作工具

### 5.1 list_directory：列目录

```python
def list_directory(path: str = ".") -> list[dict]:
    EXCLUDE = {'.git', 'node_modules', '__pycache__', '.venv', 'dist', 'build'}
    
    items = []
    for entry in os.scandir(path):
        if entry.name in EXCLUDE:
            continue
        items.append({
            "name": entry.name,
            "type": "file" if entry.is_file() else "dir",
            "size": entry.stat().st_size if entry.is_file() else None
        })
    
    return sorted(items, key=lambda x: (x["type"], x["name"]))
```

==默认排除==：`.git` / `node_modules` / `__pycache__` 等无关目录。

### 5.2 create_directory：创建目录

```python
def create_directory(path: str) -> dict:
    os.makedirs(path, exist_ok=True)
    return {"status": "success", "path": path}
```

==`exist_ok=True`==：目录已存在不报错。

### 5.3 move / copy / delete：高危操作

==这三个工具必须走 Human-in-the-Loop==：

```python
def delete_file(path: str) -> dict:
    if needs_confirmation(path):  # 项目核心文件 / 配置文件
        raise ConfirmationRequired(
            f"About to delete {path}. This cannot be undone. Confirm?"
        )
    os.remove(path)
    return {"status": "success"}
```

==Harness 层拦截==：

```
LLM: delete_file("src/important.py")
   ↓
Harness: 弹窗 "Agent wants to delete src/important.py. Allow?"
   ↓
用户点击 Allow → 执行
用户点击 Deny → 返回 PermissionDenied 给 LLM
```

详见 [[Agent 工程实践#Human-in-the-Loop]]。

## 六、Notebook 工具：NotebookEdit

### 6.1 为什么需要专门工具

==Jupyter Notebook 是 JSON 格式==——直接用 `edit_file` 改 JSON 太容易出错。==需要专门工具==：

```python
def notebook_edit(
    notebook_path: str,
    cell_id: str,
    new_source: str,
    cell_type: str = "code"
) -> dict:
    with open(notebook_path, 'r') as f:
        nb = json.load(f)
    
    # 找到对应 cell
    for cell in nb['cells']:
        if cell.get('id') == cell_id:
            cell['source'] = new_source.split('\n')
            cell['cell_type'] = cell_type
            break
    
    with open(notebook_path, 'w') as f:
        json.dump(nb, f, indent=1)
    
    return {"status": "success"}
```

### 6.2 三种编辑模式

| 模式 | 作用 | 参数 |
|------|------|------|
| ==replace== | 替换指定 cell 内容 | `cell_id` + `new_source` |
| ==insert== | 在指定位置插入新 cell | `cell_id`（插入到它后面）+ `new_source` |
| ==delete== | 删除指定 cell | `cell_id` |

### 6.3 LLM 调用示例

```json
{
  "tool": "notebook_edit",
  "notebook_path": "analysis.ipynb",
  "cell_id": "a3f2b1c4",
  "edit_mode": "replace",
  "new_source": "import pandas as pd\ndf = pd.read_csv('new_data.csv')",
  "cell_type": "code"
}
```

## 七、最佳实践

### 7.1 文件操作的安全检查清单

==上线前必须确认==：

- [ ] 路径白名单（不能读 `/etc/passwd` / `~/.ssh`）
- [ ] write_file 的"必须先 read"机制
- [ ] delete / move 的 Human-in-the-Loop
- [ ] 大文件分页（防止炸 context）
- [ ] 编码自动检测（处理非 UTF-8 文件）

### 7.2 工具选择决策树

```
需要修改文件？
  ├─ 是新文件 → write_file
  └─ 是修改现有文件
      ├─ 小改动（< 50 行） → edit_file
      ├─ 大改动（整个重构） → write_file
      └─ Jupyter Notebook → notebook_edit
```

### 7.3 错误处理

==常见错误 + LLM 自愈策略==：

| 错误 | 原因 | LLM 自愈方案 |
|------|------|------------|
| `FileNotFoundError` | 文件不存在 | 先调 `list_directory` 确认路径 |
| `PermissionError: Must read first` | 没有先 read | 自动调 `read_file` 再重试 |
| `edit_file: old_string not found` | 精确匹配失败 | fallback 到 `write_file` |
| `UnicodeDecodeError` | 编码问题 | 工具自动检测编码 |

## 八、工具实现量级

==这些工具都极简==：

| 工具 | 实现量级 |
|------|---------|
| `read_file` | 5 行（Python 内置 `open()`） |
| `write_file` | 5 行 + 10 行（路径白名单） |
| `edit_file` | 20 行（字符串替换 + 去重检查） |
| `list_directory` | 10 行（`os.scandir` + 过滤） |
| `notebook_edit` | 30 行（JSON 解析 + 修改） |

==看起来像魔法的 Coding Agent==，文件操作部分就是这么简单——==真正的智能来自 LLM 决策==，而不是工具本身有多复杂。
