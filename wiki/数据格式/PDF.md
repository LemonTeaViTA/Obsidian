# PDF：不是图片，是页面描述语言

> "PDF 就是把图片塞在一起"是常见误解。本文澄清 PDF 的本质：它是**页面描述语言**（Page Description Language），核心是==绘制指令==而不是像素。理解这一点，就能明白为什么 PDF 在格式光谱里是"低结构"而不是"无结构"。

---

## 一、PDF 不是图片

**PDF = Portable Document Format（便携式文档格式）**，由 Adobe 在 1993 年推出，2008 年成为 ISO 32000 国际标准。

==PDF 本质是一种"页面描述语言"，描述"如何在页面上画东西"，而不是"页面长什么样"的位图。==

PDF 的祖先是 Adobe **PostScript** —— 一种打印机指令语言。当年的激光打印机不是接收图片，而是接收 PostScript 指令"在 (100, 200) 位置写 'Hello'"然后自己渲染。PDF 是把 PostScript 简化、自包含化、加上结构化访问后的产物。

### 为什么这个区别重要

如果 PDF 是图片：
- 文字 = 像素 → 不能复制、不能搜索、不能选中
- 字体 = 渲染好的形状 → 改不了
- 矢量图形 = 栅格化的像素 → 放大模糊

但实际上 PDF：
- ==文字以 Unicode 存储==，可复制、可搜索、可选中
- 字体可嵌入或引用，可以是矢量
- 矢量图形是数学描述，==无限放大不模糊==

==这就是 PDF 比图片信息密度高的根本原因==。

## 二、PDF 内部到底有什么

打开一个数字版 PDF 看二进制（简化版）：

```
1 0 obj
<< /Type /Page /Resources << /Font << /F1 2 0 R >> >> >>
/Contents 3 0 R
endobj

3 0 obj
<< /Length 75 >>
stream
BT                          <-- Begin Text（开始文字块）
  /F1 12 Tf                 <-- 用字体 F1，12pt
  100 700 Td                <-- 移动到坐标 (100, 700)
  (Hello, World) Tj         <-- 写文字 "Hello, World"
ET                          <-- End Text
endstream
endobj
```

里面是一系列==指令==：
- `BT ... ET` 文字块边界
- `Tf` 设置字体
- `Td` 移动绘制位置
- `Tj` 显示文字

==文字 "Hello, World" 是字符编码，不是像素==。这就是为什么你能在 PDF 里复制粘贴文字。

矢量图形也是指令：
- `m` move to（移动笔到某点）
- `l` line to（画直线到某点）
- `c` curve to（画贝塞尔曲线）
- `S` stroke（描边）
- `f` fill（填充）

整个 PDF 像一段==绘画的剧本==，PDF 阅读器（Acrobat、Foxit、浏览器）按剧本演绎出最终的页面。

## 三、PDF 有两种：数字版 vs 扫描版

==这是关键——"PDF" 不是一种东西==。

| 类型 | 来源 | 内部 | 你能做什么 |
|------|------|------|---------|
| **数字版 PDF**（Born-digital） | Word 导出、网页"打印为 PDF"、LaTeX 编译、CAD 导出、PowerPoint 导出 | 文字指令 + 字体 + 坐标 + 矢量图形 | ✅ 复制文字、搜索、选中 |
| **扫描版 PDF**（Scanned） | 扫描仪扫纸质文档、手机拍照存 PDF、传真 | ==每页就是一张 JPEG/PNG 塞进 PDF 容器== | ❌ 不能复制文字（只能选图）、不能搜索 |

### 怎么一眼判断

打开 PDF，试着选中一段文字：
- 选中后能复制粘贴出文字 → ==数字版==（有文字层）
- 选中是矩形选区，复制粘贴拿不到文字或只是图片 → ==扫描版==（纯图片）

### 占比

==绝大多数办公场景的 PDF 都是数字版==——Word 导出、浏览器"另存为 PDF"、PowerPoint 导出、LaTeX 论文、电子合同、网银对账单。

==只有真的把纸质材料扫进去才是扫描版==——历史档案、合同复印件扫描、手写笔记、老旧资料数字化。

> [!warning] 混合 PDF
> 有些 PDF 是数字版 + 扫描版混合的——前 10 页是 Word 导出（数字版），后 5 页是合同扫描件（扫描版）。解析时要逐页判断有没有文字层。

## 四、PDF vs OOXML vs 图片：信息密度对比

PDF 在结构化程度光谱上的位置很特殊——比图片信息多得多，但比 OOXML 缺少语义。

| 格式 | 你能拿到的 | 知道"这是标题"吗？ |
|------|-----------|-----------------|
| OOXML（[[XML]]/[[OOXML]]） | 文字 + 类型 + 结构标签（`<w:pStyle>`/`<c t="n">`） | ✅ 标签直接说 |
| **数字版 PDF** | 文字 + 字体 + 坐标 + 矢量图形 | ❌ 要从字号、字重、独占一行猜 |
| 扫描版 PDF / 图片 | 纯像素 | ❌ 连写了什么都不知道，必须 OCR |

### 为什么 PDF 在格式光谱里是"低结构"

数字版 PDF 给了==基础原料==：
- 写了什么（文字）
- 在哪里写（坐标）
- 用什么字体大小（字体属性）

但==不给逻辑结构==：
- 没有"段落"概念（每个字独立坐标）
- 没有"标题"标签（要从字号大、加粗、独占一行猜）
- 没有"读序"（多栏排版时按 y 坐标会读错栏）
- 没有"表格单元格"（只是一堆文字 + 横竖线坐标）
- 没有"图表数据"（图表里的数据点要 VLM 才能理解）

==这就是为什么 PDF 解析比 OOXML 难得多——OOXML 直接告诉你"这是 Heading1 标题"，PDF 只告诉你"这里有个 24pt 加粗的字符串"，要靠规则或模型反推它是不是标题。==
