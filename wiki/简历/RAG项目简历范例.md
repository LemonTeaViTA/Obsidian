## 一、专业技能模板（AI / RAG 方向）

适合写在"专业技能"模块，主动带出 RAG/Agent 的核心考点：

```text
- 了解 RAG 流程，能完成文档解析、分段切块、向量化、混合检索和上下文拼接，并处理相似度阈值与召回数量等参数调优。
- 了解 Embedding 向量化与向量检索，使用 Elasticsearch dense_vector 或其他向量库进行相似度检索，并结合 BM25 实现混合搜索提升召回质量。
- 了解大模型调用链路和基本工程治理，能处理超时重试、限流熔断、日志追踪、成本控制和模型降级等问题。
- 了解提示词工程，能针对不同任务设计提示词结构，包括角色设定、输出格式约束、few-shot 样例、引用来源约束，减少幻觉。
- 了解 Agent / Function Calling 等机制，能通过工具调用把模型能力接入业务系统（查询数据库、检索知识库、调用内部接口），并处理工具返回的结构化结果。
- 了解 MCP 协议，能将外部工具以标准方式接入到智能体系统。
- 了解 RAG / Agent 应用的端到端工程化落地，具备接口设计、数据链路处理、异常兜底、观测治理与成本优化意识。
```

## 二、核心职责 bullet 范例（按技术方向）

### 流式对话（WebSocket / SSE）

```text
- 基于 WebSocket 实现全双工流式对话，后端通过 WebFlux 的 Reactive Stream 接入 DeepSeek 等 LLM 的 SSE 流，实现逐字输出；支持用户主动中断生成，后端同步取消 LLM 侧的流式连接，避免无效 Token 消耗；整体首字响应延迟控制在 500ms 以内。
- 实现对话内容的持久化和断线续传，每个流式内容原子写入 Redis 并通过状态机维持状态；用户断线重连后自动恢复上一次输出进度，不丢失已生成文本，保障弱网环境下的用户体验。
```

### 双引擎检索（向量 + BM25）

```text
- 利用 Elasticsearch + IK 分词器对知识库文档进行索引和向量检索，支持 Word/PDF/TXT 等多种文本类型；集成 Embedding 模型进行文本到向量的转换（支持 2048 维）；结合 ES 的 KNN 向量召回、关键词过滤和 BM25 重排序实现「关键词 + 语义」双引擎搜索，并内嵌多租户权限过滤确保数据隔离。
- 构建「向量 + BM25」双路检索能力，通过 RRF 倒数排名融合与 Rerank 模型重排结果；结合多种文档切块策略减少无关内容干扰，经测试集评估检索命中率较纯关键词检索提升约 25%。
- 当用户搜索时，利用 ES 的 KNN 算法计算查询向量与文档向量的余弦相似度，再用 BM25 对关键词的出现频率和重要性打分，最后按自定义公式综合计算置信分，方便用户判断检索结果的可靠性。
```

### 文件分片上传

```text
- 前端使用 SparkMD5 计算文件指纹，开启 4 个 worker 并发上传分片；后端通过 Redis Bitmap 追踪分片状态（即使 1000 个分片也只需 125 字节）；合并阶段按 chunkIndex 升序组装后调用 MinIO 合并，保证分片不乱序。
- 使用 Redis 缓存文件元信息与上传分片，结合 MinIO 实现大文件分片上传与断点续传。
```

### Kafka 异步处理

```text
- 利用 Kafka 进行异步文档处理，覆盖「文件解析 → 文本切片 → 向量化入库」全流程；通过事务确保生产者侧消息不丢失不重复，消费端失败消息经 4 次重试后自动路由至死信队列。
- 使用 Kafka 解耦文档解析流程，将上传、解析、向量化拆分为异步任务，并通过 Outbox 模式保证数据一致性。
```

### Agent 体系（ReAct / MCP / Function Calling）

```text
- 参考 OpenManus 开源架构设计分层智能体体系（BaseAgent / ReActAgent / ToolCallAgent），利用 Spring AI 的 FunctionCallback 机制实现本地方法自主调用与 ReAct 推理，赋予 Agent「思考-规划-行动-观察」的自主决策能力，可处理「检索 → 生成文本 → 生成总结 PDF」等跨多步骤复杂任务。
- 引入 MCP 协议对本地文件操作、PDF 生成及数据库查询等能力进行 Server 端封装，实现 Agent 与工具生态的解耦。
- 将 RAG 模块封装为 Agent Tool，构建检索能力接入 Agent Loop，支持多格式文档上传、混合检索与多轮对话。
```

### 上下文管理（多轮对话）

```text
- 设计三级上下文管理机制，基于滑动窗口控制上下文长度，对历史对话进行异步压缩，在保证对话连贯性的同时减少长上下文带来的信息丢失与 Token 消耗。
- 采用基于 Redis 的对话历史管理机制，每个用户有唯一会话 ID，对话内容按时间顺序存于 Redis 并设 7 天过期，保证多轮对话上下文完整性。
- 针对超长对话引入摘要记忆策略，触发大模型对早期上下文进行压缩总结，减少上下文长度带来的推理成本。
```

### 权限与认证

```text
- 利用 Spring Security + JWT 实现基于组织标签的 RBAC 多级权限系统，通过用户角色、组织归属和文件属性的权限过滤实现精细化文档访问控制。
- 采用 JWT + Redis 的双令牌机制，Access Token 处理业务请求，Refresh Token 实现用户无感的令牌刷新；结合 ThreadLocal 管理用户上下文，配合拦截器实现 token 校验与自动续约。
- 使用 TransmittableThreadLocal 在虚拟线程中传递用户身份信息，在数据写入与查询阶段附加用户标识，实现文档级权限控制。
```

### 多模型路由与降级

```text
- 实现可热插拔的多模型路由与 Token 配额管理，支持运行时切换 DeepSeek、通义千问等 LLM 和 Embedding 供应商，无需重启服务；通过流式 usage 解析实现精确 Token 计量，结合每日请求数 + Token 余额双维度限流控制成本。
- 在调用向量 API 失败时自动回退到纯文本搜索实现服务降级；调用时采用 Reactor 重试机制，支持固定延迟重试 3 次并设 30 秒超时保护。
```

### 测试（TDD）

```text
- 编写用户认证模块单元测试，结合 JUnit + Mockito 验证 Redis 引入前后的性能差异，为系统调优提供依据。
- 采用 Mockito 注解驱动的测试模式，践行测试驱动开发（TDD），每个业务功能都有对应测试用例，覆盖正常流程和异常流程。
```

### 部署（Docker / Linux）

```text
- 支持 Docker 容器化部署，`docker-compose up -d` 一条命令 1 分钟内一键启动整套系统，保证开发、测试、生产环境一致性。
- 编写 Shell 脚本一键启动 Kafka 的 KRaft 模式，自动处理 cluster ID 冲突（清理日志、生成集群 ID、格式化存储目录、启动服务）。
- 能在 Linux 服务器下通过 HTTPS 启动 Elasticsearch，设置 ES 的 JDK 加载版本为 17，可通过 CA 证书 + curl 获取/更新 ES 键值对。
```

## 关键认知

1. **bullet = 技术 + 问题 + 量化**——每条职责都要能回答"用什么技术、解决什么、效果如何"。
2. **量化数字必须真实**——上表是表述模板，写进简历的数字必须是自己项目实测的，否则面试一问就穿帮。
3. **技术方向归类便于改写**——按自己项目实际涉及的方向挑对应 bullet 改写，不要全堆上去。
4. **对齐 Java + Agent 方向**——这些范例聚焦后端工程能力（并发、缓存、消息队列、检索、Agent），契合求职定位。
